# Feature Specification: 유사 사진 정리 (PRD9)

**Feature Branch**: `001-similar-photo`  
**Created**: 2026-01-02  
**Status**: Draft  
**Input**: docs/prd9.md, docs/prd9algorithm.md 기반

## Clarifications

- 뷰어 notAnalyzed 진입 시, **notAnalyzed만 분석**하고 기존 analyzed 결과는 유지하며 범위 내 그룹만 재계산한다.
- 분석 요청은 **단일 큐 FIFO**로 처리한다. viewer 요청은 취소하지 않고, grid 요청은 스크롤 재개 시 pending만 취소한다. 동시 분석은 최대 5개로 제한한다.
- 분석용 이미지 해상도는 **긴 변 480px 이하**이며 `aspectFit`을 사용하고 패딩/크롭은 금지한다.
- 얼굴 크롭은 **정사각형 유지 + 경계 내 최대 크기로 축소** 정책을 사용한다.
- 얼굴 매칭 검증 임계값은 **1.0 이상 Low**로 간주한다.
- 헤더의 "인물 N (M장)"에서 M은 **해당 인물(personIndex=N)이 검출된 사진 수**다.
- +버튼 겹침 이동은 **좌 → 우 → 아래 → 위** 순서로 시도하고, 화면 밖이면 반대 방향으로 대체한다. 4회 시도 후에도 겹치면 현재 위치를 유지한다.

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 그리드 유사사진 감지와 테두리 표시 (Priority: P1)

사용자가 사진 그리드에서 스크롤을 멈추면 화면 범위 ±7장의 사진을 분석하여 유사사진썸네일그룹을 만들고, 조건을 만족하는 사진에만 테두리를 표시한다.

**Why this priority**: 유사사진정리 기능의 시작점이며, 그리드에서 유사사진이 있음을 즉시 인지할 수 있어야 한다.

**Independent Test**: 스크롤 멈춤 후 1초 이내 테두리 표시 여부 확인. 얼굴 없는 사진이나 3장 미만 그룹은 테두리 미표시 확인.

**Acceptance Scenarios**:

1. **Given** 화면에 보이는 사진이 있는 상태, **When** 스크롤이 멈추고 0.3초 디바운싱이 지나면, **Then** 화면 범위 ±7장 분석이 시작된다.
2. **Given** 유효 얼굴이 3장 이상이고 유효 인물 슬롯이 1개 이상인 유사사진썸네일그룹이 존재할 때, **When** 분석이 완료되면, **Then** 해당 사진에만 테두리가 표시된다.
3. **Given** 유사 사진이 2장뿐인 경우, **When** 분석이 완료되면, **Then** 테두리가 표시되지 않는다.
4. **Given** 스크롤 중인 상태, **When** 스크롤이 다시 시작되면, **Then** source=grid 분석은 취소되고 테두리는 해제된다.
5. **Given** 유사사진 테두리가 표시된 셀을 탭하면, **When** 뷰어로 이동하면, **Then** 해당 사진이 표시되고 +버튼 오버레이가 자동으로 표시된다.

---

### User Story 2 - 뷰어 오버레이 +버튼/토글 (Priority: P2)

사용자가 뷰어에서 유사사진썸네일그룹 사진을 볼 때, 유효 인물 슬롯 얼굴 위에 +버튼이 자동으로 나타나며, 토글 버튼으로 보임/숨김을 전환할 수 있다.

**Why this priority**: 뷰어에서 인물을 빠르게 선택해 비교 화면으로 이동하는 핵심 진입점이다.

**Independent Test**: 캐시 hit 시 +버튼 즉시 표시, notAnalyzed 시 분석 요청 후 표시, 토글로 숨김/보임 전환 확인.

**Acceptance Scenarios**:

1. **Given** analyzed 상태의 유사사진에 진입했을 때, **When** 캐시된 CachedFace가 존재하면, **Then** isValidSlot=true 얼굴 위에 +버튼이 즉시 표시된다.
2. **Given** notAnalyzed 상태의 사진에 진입했을 때, **When** 그리드 분석을 요청하면, **Then** 분석 완료 후 +버튼이 표시된다.
3. **Given** 유사사진썸네일그룹에 속하지 않는 사진에 진입하면, **When** 뷰어가 표시되면, **Then** +버튼과 토글 버튼이 표시되지 않는다.
4. **Given** 토글 버튼이 표시된 상태, **When** 토글을 탭하면, **Then** +버튼이 숨김/보임으로 전환되고 아이콘이 eye/eye.slash로 변경된다.
5. **Given** +버튼 숨김 상태에서 다른 사진으로 스와이프 후 복귀하면, **When** 유사사진에 다시 진입하면, **Then** +버튼은 자동 보임 상태로 리셋된다.

---

### User Story 3 - 얼굴 비교 화면과 삭제 (Priority: P3)

사용자가 +버튼을 탭하면 선택된 인물의 얼굴만 모아 2열 그리드로 비교하고, 선택한 사진을 앱 내 휴지통으로 이동할 수 있다.

**Why this priority**: 실제 정리 행위(좋지 않은 사진 삭제)가 이 화면에서 이루어진다.

**Independent Test**: +버튼 탭 → 얼굴 비교 화면 진입 → 다중 선택 → Delete → 뷰어로 복귀 확인.

**Acceptance Scenarios**:

1. **Given** 유효 인물 슬롯의 +버튼을 탭하면, **When** 얼굴 비교 화면으로 이동하면, **Then** 해당 인물의 얼굴 크롭이 2열 그리드로 표시된다.
2. **Given** 비교 화면에서 사진을 탭하면, **When** 선택/해제를 반복하면, **Then** 체크마크 오버레이가 토글된다.
3. **Given** Delete를 탭하면, **When** 삭제가 완료되면, **Then** 선택된 사진이 앱 내 휴지통으로 이동하고 뷰어로 복귀한다.
4. **Given** 현재 사진이 삭제된 경우, **When** 복귀하면, **Then** 전체 라이브러리 기준으로 이전 사진이 표시된다(없으면 다음 사진).
5. **Given** 인물 순환 버튼을 탭하면, **When** 인물이 변경되면, **Then** 선택 상태는 유지되며 화면은 크로스페이드된다.

---

### User Story 4 - 매칭 검증 경고 (Priority: P4)

사용자가 비교 화면에서 다른 인물이 섞였을 가능성을 즉시 인지할 수 있도록 경고 UI가 표시된다.

**Why this priority**: 위치 기반 매칭 오차를 사용자에게 알려 안전한 삭제를 돕는다.

**Independent Test**: Feature Print 검증에서 거리 1.0 이상 발생 시 경고 배지와 전체 경고가 표시되는지 확인.

**Acceptance Scenarios**:

1. **Given** 얼굴 비교 화면 진입 후 백그라운드 검증이 완료되면, **When** 거리 1.0 이상이 발견되면, **Then** 전체 경고 배너가 표시된다.
2. **Given** 거리 1.0 이상인 사진이 있을 때, **When** 검증이 완료되면, **Then** 해당 사진 셀에 경고 배지가 표시된다.
3. **Given** 전체 경고 배너의 확인 버튼을 탭하면, **When** 닫힘 처리되면, **Then** 배너만 사라지고 개별 배지는 유지된다.

---

### Edge Cases

- 사진 3장 미만: 유사사진정리 기능 자체가 표시되지 않는다.
- VoiceOver 활성화: 유사사진정리 기능 전체 비활성화(테두리/토글/+버튼 미표시).
- Select 모드: 유사사진정리 기능 비활성화.
- 휴지통 화면: 유사사진정리 기능 비활성화.
- 스크롤 중 분석 결과 도착: 테두리는 스크롤 멈춤 후에만 표시한다.
- iPad 분할 화면: 5% 얼굴 기준은 분할된 윈도우 크기를 사용한다.

## Requirements *(mandatory)*

### Functional Requirements

**유사사진 분류**
- **FR-201**: 시스템은 그리드에서 화면 범위 N~M에 대해 **max(0, N-7)~min(총개수-1, M+7)** 범위를 분석해야 한다.
- **FR-202**: 시스템은 인접 사진 간 Feature Print 거리 **10.0 이하**를 유사로 판단해 유사사진썸네일그룹을 분리해야 한다.
- **FR-203**: 유사사진썸네일그룹은 5% 이상 유효 얼굴 사진 3장 이상 + 유효 인물 슬롯 1개 이상일 때만 유효하다.
- **FR-204**: 5% 이상 유효 얼굴이 없는 사진은 유사사진썸네일그룹 멤버에서 제외해야 한다.
- **FR-205**: 유사사진정리그룹은 유사사진썸네일그룹 내에서만 선택되며 최대 8장이다(거리순 선택 후 시간순 표시).
- **FR-206**: 분석 대상은 **보관 중인 사진만** 포함하며, 앱 내 휴지통 사진은 제외한다.

**분석 이미지 규칙**
- **FR-207**: 분석용 이미지는 긴 변 480px 이하로 요청하며 `aspectFit`을 사용하고 패딩/크롭은 금지한다.
- **FR-208**: 뷰어는 그리드 분석 결과를 재사용하며 재분석하지 않는다(캐시된 CachedFace 사용).

**그리드 트리거**
- **FR-209**: 스크롤 멈춤 후 0.3초 디바운싱 뒤 분석을 시작한다.
- **FR-210**: 스크롤이 재개되면 source=grid 분석은 취소되고 테두리는 해제된다.
- **FR-211**: 테두리는 빛이 회전하는 사각형 애니메이션을 사용하며, 모션 감소 설정 시 정적 테두리로 대체한다.

**뷰어 +버튼/토글**
- **FR-212**: 유사사진썸네일그룹 소속 사진에서 isValidSlot=true인 얼굴에 +버튼을 자동 표시한다.
- **FR-213**: +버튼은 최대 5개이며, 화면 너비 5% 이상 크기의 얼굴만 표시한다.
- **FR-214**: 토글 버튼은 그룹 소속 사진에서만 표시하며 eye/eye.slash로 상태를 표현한다.
- **FR-215**: 스와이프로 다른 사진으로 이동 시 +버튼 숨김 상태로 리셋되고, 유사사진 복귀 시 자동 보임으로 전환된다.

**얼굴 비교 화면**
- **FR-216**: +버튼 탭 시 얼굴 비교 화면으로 전환하며 2열 정사각형 그리드를 표시한다.
- **FR-217**: 얼굴 크롭은 boundingBox에 30% 여백을 추가하고 정사각형 유지 + 경계 내 최대 크기로 축소한다.
- **FR-218**: 헤더는 "인물 N (M장)"을 표시하고, 순환 버튼으로 인물 1→2→3 순환한다.
- **FR-219**: Delete는 선택된 사진을 앱 내 휴지통으로 이동시키고 뷰어로 복귀한다.
- **FR-220**: 삭제 후 캐시는 그룹 멤버 수 3장 미만이면 무효화하고, 3장 이상이면 멤버만 제거한다.

**검증 경고**
- **FR-221**: 얼굴 비교 화면 진입 후 Feature Print 검증을 백그라운드로 수행한다.
- **FR-222**: 거리 1.0 이상이면 Low로 판단하고, 전체 경고 배너와 개별 배지를 표시한다.

**상태/캐시**
- **FR-223**: 분석 상태는 notAnalyzed/analyzing/analyzed(inGroup, groupID)로 관리한다.
- **FR-224**: 캐시는 LRU로 500장까지 유지하며 초과 시 eviction 규칙을 적용한다.
- **FR-225**: 뷰어 notAnalyzed 진입 시 notAnalyzed만 분석하고 기존 analyzed 결과는 유지한다.
- **FR-226**: 분석 큐는 FIFO이며, 동시 분석은 최대 5개로 제한한다.

**접근성/적용 범위**
- **FR-227**: VoiceOver 활성화 시 유사사진정리 기능을 전체 비활성화한다.
- **FR-228**: 적용 화면은 사진보관함/앨범 상세이며, 휴지통은 제외한다.
- **FR-229**: 선택 모드에서는 유사사진정리 기능을 비활성화한다.

**iOS 버전 분기**
- **FR-230**: iOS 26+에서는 시스템 네비바/툴바를 사용하고, iOS 16~25에서는 커스텀 플로팅 UI를 유지한다.

### Key Entities *(include if feature involves data)*

- **SimilarityCache**: 분석 상태, 그룹 멤버, CachedFace, 유효 인물 슬롯을 저장하는 캐시.
- **CachedFace**: boundingBox, personIndex, isValidSlot을 보관하는 얼굴 캐시 모델.
- **SimilarityGroup**: 유사사진썸네일그룹(그리드/뷰어 표시)과 유사사진정리그룹(비교 화면) 개념.
- **AnalysisRequest**: source(grid/viewer), 범위, 상태를 포함하는 분석 요청.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-201**: 스크롤 멈춤 후 1초 이내 테두리가 표시된다.
- **SC-202**: 캐시 hit 시 +버튼 오버레이가 사진 표시와 동시에 나타난다.
- **SC-203**: 캐시 miss 시 분석 완료 후 0.5초 이내 +버튼이 표시된다.
- **SC-204**: 그리드 스크롤은 네이티브 주사율을 유지한다(60Hz/120Hz).
- **SC-205**: 캐시 크기는 최근 500장 이내로 유지되고 메모리 누수가 없다.
