# Feature Specification: 유사 사진 정리 기능

**Feature Branch**: `002-similar-photo`
**Created**: 2026-01-02
**Status**: Draft
**Input**: PRD 9 (prd9.md, prd9algorithm.md) 기반 유사 사진 정리 기능

---

## 개요

연속 촬영된 유사한 사진들 중에서 눈 감은 사진이나 표정이 이상한 사진을 쉽게 발견하고 삭제할 수 있는 기능입니다. 특히 단체사진에서 특정 인물의 표정/눈 상태를 확대 비교하여 최적의 사진만 남길 수 있습니다.

### 핵심 가치

- **불필요한 사진 삭제**: 중복되거나 품질이 낮은 사진 정리
- **손쉽고 간편하게**: 직관적인 UI로 빠른 비교 및 삭제
- **좋은 사진은 확실히 남기기**: 얼굴 비교를 통한 최적 사진 선별

---

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 그리드에서 유사 사진 발견 (Priority: P1)

사용자가 사진 그리드를 스크롤하다가 멈추면, 시스템이 자동으로 화면에 보이는 사진들을 분석하여 유사한 사진 그룹을 테두리 애니메이션으로 표시합니다. 사용자는 테두리가 표시된 사진을 탭하여 뷰어로 진입할 수 있습니다.

**Why this priority**: 유사 사진을 발견하는 것이 전체 기능의 시작점이며, 이 기능 없이는 후속 기능들이 작동할 수 없음

**Independent Test**: 그리드에서 스크롤 후 멈추면 유사 사진에 테두리가 표시되는지 확인하여 테스트 가능

**Acceptance Scenarios**:

1. **Given** 사용자가 사진 그리드를 보고 있을 때, **When** 스크롤이 멈추면, **Then** 0.3초 후 화면 내 유사 사진들에 빛이 도는 테두리 애니메이션이 표시된다
2. **Given** 유사 사진에 테두리가 표시된 상태에서, **When** 다시 스크롤을 시작하면, **Then** 테두리가 사라지고 분석이 취소된다
3. **Given** 테두리가 있는 사진이 있을 때, **When** 해당 사진을 탭하면, **Then** 그 사진의 뷰어 화면으로 이동한다

---

### User Story 2 - 뷰어에서 얼굴 비교 진입 (Priority: P1)

유사 사진 그룹에 속한 사진을 뷰어에서 볼 때, 감지된 얼굴 위에 + 버튼이 자동으로 표시됩니다. 사용자가 특정 얼굴의 + 버튼을 탭하면 해당 인물의 얼굴 비교 화면으로 진입합니다.

**Why this priority**: 얼굴 비교 진입이 핵심 가치 제공의 관문이며, 그리드 발견과 함께 필수 경로

**Independent Test**: 유사 사진 뷰어에서 + 버튼 표시 및 탭으로 얼굴 비교 화면 진입 테스트 가능

**Acceptance Scenarios**:

1. **Given** 유사 사진 그룹에 속한 사진을 뷰어에서 볼 때, **When** 화면이 표시되면, **Then** 유효한 얼굴 위에 + 버튼이 자동으로 나타난다
2. **Given** + 버튼이 표시된 상태에서, **When** 특정 얼굴의 + 버튼을 탭하면, **Then** 해당 인물의 얼굴 비교 화면으로 전환된다
3. **Given** + 버튼이 표시된 상태에서, **When** 오버레이 토글 버튼(eye)을 탭하면, **Then** + 버튼이 숨겨지고 아이콘이 eye.slash로 변경된다

---

### User Story 3 - 얼굴 비교 및 삭제 (Priority: P1)

얼굴 비교 화면에서 사용자는 동일 인물의 얼굴을 크롭한 2열 그리드를 보며 표정을 비교합니다. 눈 감은 사진이나 표정이 이상한 사진을 선택하고 삭제할 수 있습니다.

**Why this priority**: 실제 가치 제공 (불필요한 사진 삭제)이 이루어지는 핵심 기능

**Independent Test**: 얼굴 비교 화면에서 사진 선택 후 삭제하여 휴지통 이동 테스트 가능

**Acceptance Scenarios**:

1. **Given** 얼굴 비교 화면이 표시되면, **When** 화면을 확인하면, **Then** 선택한 인물의 얼굴이 2열 그리드로 표시되고 헤더에 "인물 N (M장)" 형식으로 표시된다
2. **Given** 얼굴 비교 화면에서, **When** 사진을 탭하면, **Then** 해당 사진이 선택되고 체크마크가 표시된다
3. **Given** 1개 이상의 사진이 선택된 상태에서, **When** Delete 버튼을 탭하면, **Then** 선택된 사진들이 앱 내 휴지통으로 이동하고 그리드로 복귀한다
4. **Given** 얼굴 비교 화면에서, **When** 순환 버튼(↻)을 탭하면, **Then** 다음 인물로 전환되며 선택 상태는 유지된다

---

### User Story 4 - 오버레이 토글 (Priority: P3)

뷰어에서 + 버튼 오버레이를 숨기거나 보이게 전환할 수 있습니다.

**Why this priority**: 편의 기능으로, 없어도 핵심 가치에 영향 없음

**Independent Test**: 토글 버튼으로 + 버튼 숨김/보임 전환 테스트 가능

**Acceptance Scenarios**:

1. **Given** + 버튼이 보이는 상태에서, **When** eye 아이콘을 탭하면, **Then** + 버튼이 숨겨지고 아이콘이 eye.slash로 변경된다
2. **Given** + 버튼이 숨겨진 상태에서, **When** 다른 사진으로 스와이프 후 복귀하면, **Then** + 버튼이 다시 보이는 상태로 리셋된다

---

### Edge Cases

- 유사 사진이 3장 미만일 때: 기능이 활성화되지 않음 (테두리/+버튼 미표시)
- 사진에 얼굴이 없을 때: +버튼이 표시되지 않음
- 얼굴이 화면 너비의 5% 미만일 때: 해당 얼굴은 미검출로 취급
- 인물 슬롯에 2장 미만일 때: 해당 인물의 +버튼 미표시
- 사진 삭제 시: 그리드로 복귀 (뷰어와 얼굴 비교 화면 모두 닫힘)
- VoiceOver 활성화 시: 유사사진정리 기능 전체 비활성화 (시각 기반 기능이므로)
- 선택 모드일 때: 유사사진정리 기능 비활성화
- 동일 사진 중복 그룹: 범위 겹침 시 동일 사진이 여러 그룹에 속할 수 있음 (허용). 그룹 무효화 시 해당 사진이 다른 유효 그룹에도 속해있으면 테두리 유지
- 작은 해상도 사진: 원본이 480px 미만인 경우에도 원본 크기로 분석하며, 얼굴 감지 정확도 저하는 허용
- 분석 타임아웃: 단일 사진 분석이 3초를 초과하면 해당 사진은 분석 실패로 처리
- 캐시 eviction 중 분석: 현재 분석 중인 사진은 eviction 대상에서 제외

### Error Handling (분석 실패 처리)

- **Vision API 오류**: 해당 사진은 분석 실패로 처리하고, 다른 사진 분석은 계속 진행한다
- **이미지 로드 실패**: 해당 사진은 분석 실패로 처리하며, 유사 사진 그룹에 포함되지 않는다
- **분석 실패 사진**: 테두리 미표시, +버튼 미표시, 사용자에게 별도 알림 없음 (silent failure)
- **전체 분석 실패**: 화면 내 모든 사진이 분석 실패하면, 기능이 비활성화된 것처럼 동작한다
- **얼굴 감지 실패**: 얼굴 감지 API 오류 시 해당 사진은 "얼굴 없음"으로 처리한다 (폴백)

### System State Handling (시스템 상태 처리)

- **메모리 경고**: didReceiveMemoryWarning 발생 시 캐시의 50%를 LRU 기준으로 즉시 제거한다
- **디바이스 과열**: thermalState가 .serious/.critical일 때 동시 분석 수를 5개에서 2개로 제한한다
- **백그라운드 전환**: 앱이 백그라운드로 전환되면 진행 중인 분석을 취소하고, 포그라운드 복귀 시 재분석하지 않는다
- **외부 라이브러리 변경**: PHPhotoLibraryChangeObserver로 감지된 변경 시, 해당 사진의 캐시를 무효화한다
- **앱 종료/재시작**: 분석 중 앱 종료 시 캐시는 메모리에서 제거되며, 재시작 시 처음부터 분석한다 (캐시 비영속)
- **권한 거부**: 사진 라이브러리 접근 권한이 거부되면 유사사진정리 기능은 비활성화되며, 기존 앱의 권한 요청 UI를 따른다

### Gesture & Layout Handling (제스처/레이아웃 처리)

- **제스처 충돌**: 스와이프와 탭이 동시에 발생할 경우, 먼저 인식된 제스처만 처리한다 (UIGestureRecognizer 기본 동작)
- **삭제 후 재분석**: 분석 범위 내 사진 삭제 시 그룹 무효화만 수행하고, 다음 스크롤 멈춤 시 자동 재분석한다
- **화면 회전**: 화면 회전 시 +버튼 위치를 얼굴 좌표 기준으로 재계산한다 (viewWillTransition)
- **iPad 멀티윈도우**: iPad 스플릿 뷰/슬라이드 오버에서도 동일하게 동작하며, 윈도우 크기 변경 시 +버튼 위치를 재계산한다

## Requirements *(mandatory)*

### Functional Requirements

#### 유사 사진 감지 (FR-001 ~ FR-005)

- **FR-001**: 시스템은 화면에 보이는 사진 기준 앞뒤 7장 범위 내에서 유사 사진을 감지해야 한다
- **FR-002**: 유사 판정은 이미지 유사도 거리 10.0 이하를 기준으로 한다
- **FR-003**: 유사 사진 그룹으로 인정되려면 최소 3장 이상이어야 한다
- **FR-004**: 화면 너비의 5% 이상 크기로 표시되는 얼굴만 유효 얼굴로 인정한다
- **FR-005**: 유효 인물 슬롯(동일 인물이 2장 이상 감지된 슬롯)이 1개 이상 있어야 기능이 활성화된다

#### 그리드 트리거 (FR-006 ~ FR-010)

- **FR-006**: 그리드 스크롤이 멈춘 후 0.3초 디바운싱 후 분석을 시작한다
- **FR-007**: 유사 사진 그룹에 속한 셀에 빛이 도는 테두리 애니메이션을 표시한다 (흰색 그라데이션, 시계방향 회전, 1.5초 주기, 분석 완료 즉시 시작하며 모든 셀이 동일한 위상으로 동기화)
- **FR-008**: 스크롤이 재개되면 진행 중인 분석을 취소하고 테두리를 제거한다
- **FR-009**: 테두리가 있는 사진을 탭하면 해당 사진의 뷰어로 이동한다
- **FR-010**: 화면에 여러 유사 사진 그룹이 있으면 모두 동일한 스타일로 테두리를 표시한다

#### 뷰어 트리거 (FR-011 ~ FR-015)

- **FR-011**: 유사 사진 그룹에 속한 사진 진입 시 유효 슬롯 얼굴에 +버튼을 자동 표시한다
- **FR-012**: 그리드에서 분석된 결과를 캐시로 재사용하며 뷰어에서 재분석하지 않는다
- **FR-013**: 캐시에 없는 사진(notAnalyzed) 접근 시 분석을 요청하고 완료 후 +버튼을 표시한다. 분석 중에는 로딩 인디케이터를 표시한다
- **FR-014**: 오버레이 토글 버튼(eye/eye.slash)으로 +버튼 숨김/보임을 제어한다
- **FR-015**: 다른 사진으로 스와이프 후 복귀 시 +버튼은 보임 상태로 리셋된다

#### 얼굴 +버튼 (FR-016 ~ FR-020)

- **FR-016**: +버튼은 최대 5개까지 표시한다 (6개 이상 시 얼굴 크기순으로 상위 5개 선택)
- **FR-017**: +버튼은 얼굴 위 중앙에 위치하며, 겹침 시 다음 순서로 위치를 조정한다: 좌 → 우 → 아래 → 위 (각 방향 버튼 지름 × 1.2 이동, 화면 경계 초과 시 반대 방향 시도, 4회 시도 후 실패 시 현재 위치 유지)
- **FR-018**: +버튼 탭 시 해당 인물의 얼굴 비교 화면으로 전환한다
- **FR-019**: 인물 번호는 얼굴 위치 기준(좌→우, 위→아래)으로 부여한다
- **FR-020**: 유효 슬롯이 아닌 인물(2장 미만 감지)의 +버튼은 표시하지 않는다

#### 얼굴 비교 화면 (FR-021 ~ FR-028)

- **FR-021**: 얼굴 비교 화면은 2열 정사각형 그리드로 표시한다
- **FR-022**: 헤더에 "인물 N (M장)" 형식으로 현재 인물과 사진 수를 표시한다
- **FR-023**: 순환 버튼(`arrow.trianglehead.2.clockwise.rotate.90`)으로 다음 인물로 전환하며, 선택 상태는 유지된다. 인물 번호 오름차순으로 순환하며, 마지막 인물에서 첫 번째 인물로 돌아간다 (원형 순환)
- **FR-024**: 얼굴 크롭은 bounding box에 30% 여백을 추가하고 정사각형으로 조정한다
- **FR-025**: 사진 탭으로 선택/해제를 토글하며, 선택 시 체크마크를 표시한다
- **FR-026**: 하단바에 Cancel, 선택 개수, Delete 버튼을 표시한다
- **FR-027**: Delete 버튼 탭 시 선택된 사진을 앱 내 휴지통으로 이동하고 그리드로 복귀한다
- **FR-028**: 유사 사진 정리 그룹은 최대 8장으로 제한하며, 초과 시 현재 사진 기준 거리순으로 선택한다

#### 인물 매칭 (FR-029 ~ FR-030)

- **FR-029**: +버튼 탭 시 Feature Print 비교로 동일 인물만 얼굴 비교 화면에 표시한다 (~0.5초 로딩)
- **FR-030**: Feature Print 거리 1.0 이상인 사진은 다른 인물로 판정하여 비교 그리드에서 자동 제외한다

#### 삭제 후 처리 (FR-031 ~ FR-034)

- **FR-031**: 삭제 완료 시 뷰어와 얼굴 비교 화면을 닫고 그리드로 복귀한다
- **FR-032**: (FR-031에 통합됨)
- **FR-033**: 삭제 후 그룹 멤버가 3장 미만이면 그룹을 무효화하고, 테두리와 +버튼을 즉시 제거한다
- **FR-034**: 삭제된 사진은 기존 휴지통 복구와 동일하게 복구 가능하다

#### 접근성 및 제외 조건 (FR-035 ~ FR-038)

- **FR-035**: VoiceOver 활성화 시 유사사진정리 기능 전체를 비활성화한다 (시각 기반 얼굴/표정 비교 기능이므로 대체 인터페이스 제공 불가, 기존 사진 삭제 기능은 유지)
- **FR-036**: 선택 모드에서는 유사사진정리 기능을 비활성화한다
- **FR-037**: 휴지통 화면에서는 유사사진정리 기능을 적용하지 않는다
- **FR-038**: 모션 감소 설정 시 테두리 애니메이션을 정적 테두리로 대체한다

---

### Key Entities

- **유사사진썸네일그룹**: 그리드 테두리 및 뷰어 +버튼 표시를 위한 유사 사진 집합 (크기 제한 없음)
- **유사사진정리그룹**: 얼굴 비교 화면에서 비교할 사진 집합 (최대 8장)
- **CachedFace**: 얼굴 정보 캐시 (위치, 인물 번호, 유효 슬롯 여부)
- **분석 상태**: notAnalyzed(미분석), analyzing(분석중), analyzed(완료)

---

### Non-Functional Requirements

#### 개발/테스트 지원 (Feature Flag)

- **NFR-DEV-001**: 유사 사진 정리 기능은 Feature Flag(`FeatureFlags.isSimilarPhotoEnabled`)로 제어한다
- **NFR-DEV-002**: 플래그가 `false`일 때 기능이 완전히 비활성화되며, 기존 앱 동작에 영향을 주지 않는다
- **NFR-DEV-003**: 기능 관련 코드는 Extension 파일로 분리하여 기존 파일의 수정을 최소화한다

**파일 구조:**
```
PickPhoto/PickPhoto/
├── Shared/
│   └── FeatureFlags.swift                         # Feature Flag 정의
├── Features/
│   ├── Grid/
│   │   ├── GridViewController.swift               # 기존 (최소 수정)
│   │   └── GridViewController+SimilarPhoto.swift  # 신규 Extension
│   ├── Viewer/
│   │   ├── ViewerViewController.swift             # 기존 (최소 수정)
│   │   └── ViewerViewController+SimilarPhoto.swift # 신규 Extension
│   └── SimilarPhoto/                              # 신규 모듈 전체
```

**기능 끄기:** `FeatureFlags.isSimilarPhotoEnabled = false` (1줄 수정)

#### 성능 설계 근거

- **동시 분석 5개 제한**: Vision API 호출당 약 50MB 메모리 사용, 5개 동시 분석 시 최대 250MB로 저사양 기기(2GB RAM) 안정성 확보
- **분석 해상도 480px**: 얼굴 감지 정확도 95% 이상 유지하면서 분석 시간 200ms 이내 달성 (1080px 대비 4배 빠름)
- **ProMotion 지원**: CADisplayLink로 기기 네이티브 주사율 자동 감지, 별도 구현 분기 불필요

#### 배터리 및 접근성

- **배터리 소모**: 동시 분석 제한(5개)과 백그라운드 분석 취소로 배터리 영향 최소화, 별도 저전력 모드 미지원
- **정적 테두리 사양**: 모션 감소 설정 시 흰색 2pt 실선 테두리로 대체 (애니메이션 없음, cornerRadius = 0)

#### 개인정보 보호

- **캐시 데이터**: 메모리 전용 캐시로 디스크 저장 없음, 앱 종료 시 자동 삭제
- **얼굴 분석 결과**: 기기 내에서만 처리, 외부 전송 없음 (On-device processing)

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 그리드 스크롤 멈춤 후 1초 이내에 유사 사진 테두리가 표시된다
- **SC-002**: 유사 사진 뷰어 진입 시 (캐시 hit) +버튼이 100ms 이내에 표시된다
- **SC-003**: 유사 사진 뷰어 진입 시 (캐시 miss) 분석 완료 후 0.5초 이내에 +버튼이 표시된다
- **SC-004**: 10개 이상의 테두리가 동시에 표시되어도 기기 네이티브 주사율(60Hz/120Hz)이 유지된다. 검증: iPhone 12 이상에서 5만장 라이브러리, Xcode Instruments로 Frame Rate 측정
- **SC-005**: 사용자가 얼굴 비교 화면 진입 후 3탭 이내로 원하는 사진을 삭제할 수 있다 (1.사진 탭 2.Delete 탭 3.확인 없음 = 2탭)
- **SC-006**: +버튼 탭 후 0.5초 이내에 얼굴 비교 화면이 표시된다
- **SC-007**: 캐시는 최근 500장 이내로 유지되며 메모리 누수가 발생하지 않는다. 검증: Xcode Instruments Leaks 도구로 10분간 기능 사용 후 누수 0건

## Assumptions

- 사진은 시간순으로 정렬되어 있으며, 연속 촬영된 사진은 인접해 있다
- 앱 내 휴지통 기능이 이미 구현되어 있다
- 기존 선택 모드 UI (체크마크, Cancel, Delete)가 재사용 가능하다 (FloatingTabBar 컴포넌트)
- 5만 장 이상의 사진 라이브러리에서도 성능이 유지되어야 한다
- PHCachingImageManager는 기존 그리드/뷰어의 인스턴스를 공유하며, 분석용 480px 이미지는 별도 요청한다

## Scope Boundaries

### In Scope

- 사진보관함(Photos)과 앨범 상세 화면에서의 유사 사진 정리
- Feature Print 기반 인물 매칭
- 얼굴 비교를 통한 사진 삭제
- 기존 휴지통 복구와의 통합

### Out of Scope

- 휴지통 화면에서의 유사 사진 정리
- 얼굴 인식(Face Recognition) - 누구인지 식별하는 기능
- 자동 삭제 추천
- 클라우드 동기화된 사진의 실시간 분석
