# Feature Specification: 저품질 사진 자동 정리

**Feature Branch**: `001-auto-cleanup`
**Created**: 2026-01-21
**Status**: Draft
**Input**: docs/autodel 폴더의 기획 문서 (260120AutoDel.md, impl.md, test.md) 기반

## Overview

사용자가 '정리' 버튼을 클릭하면 사진 라이브러리에서 저품질 사진(블러, 극단 노출, 주머니 샷 등)을 자동으로 감지하여 최대 50장까지 휴지통으로 이동시키는 기능.

**핵심 목표**:
1. 오탐지 최소화 - 소중한 추억 보호
2. 사용자 친화적 - 검토 화면 없이 바로 휴지통 이동, 휴지통에서 복구 가능
3. 고성능 - 원본 다운로드 없이 로컬 캐시 썸네일로 분석

**UI 배치**:
- 정리 버튼: 그리드 화면 상단, 셀렉트 버튼 왼쪽

**1차 범위**:
- Precision 모드만 지원 (오탐지 최소화)
- 스크린샷 정리는 별도 기능으로 추후 개발
- Recall 모드(적극적 정리)는 추후 업데이트에서 추가

---

## User Scenarios & Testing

### User Story 1 - 최신 사진부터 정리 (Priority: P1)

사용자가 정리 버튼을 눌러 가장 최근 사진부터 저품질 사진을 찾아 휴지통으로 이동한다.

**Why this priority**: 가장 기본적인 사용 시나리오. 대부분의 사용자가 최근 사진 정리를 원함.

**Independent Test**: 정리 버튼을 누르고 "최신사진부터 정리"를 선택하면 저품질 사진이 휴지통으로 이동되는 것을 확인할 수 있다.

**Acceptance Scenarios**:

1. **Given** 휴지통이 비어있고 사진 라이브러리에 저품질 사진이 있음, **When** 사용자가 정리 버튼을 누르고 "최신사진부터 정리"를 선택함, **Then** 저품질 사진이 최대 50장까지 휴지통으로 이동하고 결과 메시지가 표시됨
2. **Given** 휴지통에 사진이 있음, **When** 사용자가 정리 버튼을 누름, **Then** "휴지통을 먼저 비워주세요" 메시지와 휴지통 이동 버튼이 표시됨
3. **Given** 탐색 중, **When** 사용자가 취소 버튼을 누름, **Then** 아무것도 휴지통으로 이동하지 않고 즉시 종료됨 (팝업 없음)

---

### User Story 2 - 연도별 정리 (Priority: P2)

사용자가 특정 연도의 사진만 선택적으로 정리한다.

**Why this priority**: 오래된 사진 정리를 원하는 사용자를 위한 기능. 범위를 좁혀 더 안전한 정리 가능.

**Independent Test**: "연도별 정리"를 선택하고 특정 연도를 선택하면 해당 연도의 저품질 사진만 정리됨.

**Acceptance Scenarios**:

1. **Given** 2024년에 저품질 사진이 있음, **When** 사용자가 "연도별 정리 > 2024"를 선택함, **Then** 2024년 사진만 분석하여 저품질 사진을 휴지통으로 이동함
2. **Given** 선택한 연도에 저품질 사진이 없음, **When** 탐색 완료, **Then** "정리할 사진을 찾지 못했습니다" 메시지가 표시됨

---

### User Story 3 - 이어서 정리 (Priority: P2)

이전에 정리를 수행한 위치부터 계속해서 정리를 진행한다.

**Why this priority**: 대용량 라이브러리 사용자가 여러 세션에 걸쳐 정리할 수 있게 함.

**Independent Test**: 첫 번째 정리 후 "이어서 정리"를 선택하면 마지막 탐색 위치부터 계속됨.

**Acceptance Scenarios**:

1. **Given** 이전에 2024년 5월까지 탐색한 이력이 있음, **When** "이어서 정리"를 선택함, **Then** 2024년 5월 이전 사진부터 탐색이 시작됨
2. **Given** 정리 이력이 없음, **When** 정리 방식 선택 화면 표시, **Then** "이어서 정리" 옵션이 비활성화됨

---

### User Story 4 - 판별 모드 전환 (Priority: P3) - 추후 추가 예정

> **1차 범위 외**: Precision 모드만 지원. Recall 모드 및 모드 전환 UI는 추후 업데이트에서 추가.

사용자가 정리 강도를 선택하여 더 신중하게 또는 더 적극적으로 정리할 수 있다.

**Why this priority**: 고급 사용자를 위한 옵션. 기본 모드(Precision)로도 충분히 사용 가능.

**Independent Test**: 모드를 변경하고 정리를 실행하면 해당 모드의 기준으로 저품질 판정이 이루어짐.

**Acceptance Scenarios**:

1. **Given** "신중한 정리" 모드(Precision), **When** 정리 실행, **Then** 확실한 저품질 사진(극단 노출, 심각 블러)만 정리됨
2. **Given** "적극적 정리" 모드(Recall), **When** 정리 실행, **Then** 더 완화된 기준으로 저품질 사진이 정리됨

---

### User Story 5 - 휴지통에서 복구 (Priority: P1)

사용자가 실수로 정리된 사진을 휴지통에서 복구한다.

**Why this priority**: 오탐지에 대한 안전망. 사용자 신뢰 확보에 필수.

**Independent Test**: 휴지통에서 정리된 사진을 선택하고 복구하면 원래 위치로 돌아감.

**Acceptance Scenarios**:

1. **Given** 정리로 인해 휴지통에 사진이 있음, **When** 사용자가 해당 사진을 복구함, **Then** 사진이 원래 위치로 복원됨
2. **Given** 휴지통에 사진이 30일 이상 보관됨, **When** 30일 경과, **Then** 시스템에 의해 영구 삭제됨

---

### Edge Cases

- **iCloud 전용 사진**: 로컬 캐시 썸네일이 없으면 해당 사진은 건너뜀 (SKIP)
- **분석 실패**: Vision/Metal API 실패 시 해당 사진은 건너뜀 (삭제 금지)
- **백그라운드 전환**: 탐색 중 앱이 백그라운드로 가면 일시정지, 복귀 시 자동 재개
- **앱 종료**: 탐색 중 앱이 종료되면 진행 상태 소실 (처음부터 다시 시작). 단, 이미 휴지통으로 이동된 사진은 유지됨.
- **즐겨찾기/편집된 사진**: Safe Guard로 보호되어 분석에서 제외
- **심도 효과/선명한 얼굴 사진**: 블러로 판정되어도 Safe Guard로 보호
- **10분 초과 비디오**: 분석에서 제외 (SKIP)
- **iCloud 전용 비디오**: 프레임 추출 불가로 건너뜀 (SKIP)

## Requirements

### Functional Requirements

**정리 진입 및 전제조건**
- **FR-001**: 시스템은 정리 버튼 클릭 시 휴지통이 비어있는지 확인해야 함
- **FR-002**: 휴지통이 비어있지 않으면 "휴지통을 먼저 비워주세요" 메시지와 휴지통 이동 버튼을 표시해야 함
- **FR-003**: 휴지통이 비어있으면 정리 방식 선택 화면을 표시해야 함

**정리 방식**
- **FR-004**: 시스템은 "최신사진부터 정리", "이어서 정리", "연도별 정리" 세 가지 방식을 제공해야 함
- **FR-005**: "이어서 정리"는 이전 정리 이력이 있을 때만 활성화되어야 함
- **FR-006**: "연도별 정리"는 연도 선택 하위 메뉴를 제공해야 함

**탐색 및 종료**
- **FR-007**: 시스템은 최대 50장의 저품질 사진을 찾으면 즉시 탐색을 중단해야 함
- **FR-008**: 시스템은 최대 1,000장을 검색하면 탐색을 종료해야 함
- **FR-009**: 연도별 정리에서는 선택한 연도만 검색하고 다른 연도로 확장하지 않아야 함
- **FR-010**: 탐색 중 진행 상황(찾은 사진 수, 현재 탐색 시점)을 사용자에게 표시해야 함
- **FR-011**: 사용자가 취소 버튼을 누르면 아무것도 휴지통으로 이동하지 않고 즉시 종료해야 함

**저품질 판정 (Precision 모드)**
- **FR-012**: 극단적으로 어두운 사진(휘도 < 0.10)을 저품질로 판정해야 함
- **FR-013**: 극단적으로 밝은 사진(휘도 > 0.90)을 저품질로 판정해야 함
- **FR-014**: 심하게 흐린 사진(Laplacian < 50)을 저품질로 판정해야 함

**저품질 판정 (Recall 모드 추가 기준) - 1차 범위 외**

> 아래 요구사항은 추후 업데이트에서 구현 예정입니다.

- **FR-015**: Recall 모드에서는 주머니 샷, 극단 단색, 렌즈 가림 조건도 저품질로 판정해야 함
- **FR-016**: Recall 모드에서는 Weak 신호(일반 블러, 일반 노출, 낮은 색상 다양성, 저해상도) 가중치 합산이 3점 이상이면 저품질로 판정해야 함

**Safe Guard**
- **FR-017**: 즐겨찾기, 편집됨, 숨김, 공유앨범 사진은 분석에서 제외해야 함
- **FR-018**: 심도 효과가 있는 사진은 블러 판정을 무효화해야 함
- **FR-019**: 얼굴 품질이 0.4 이상인 사진은 블러 판정을 무효화해야 함
- **FR-020**: 스크린샷은 저품질 로직에서 제외해야 함

**특수 미디어**
- **FR-021**: Live Photo는 정지 이미지만 분석해야 함
- **FR-022**: Burst 사진은 대표 사진만 분석해야 함
- **FR-023**: RAW+JPEG는 JPEG로 분석하고 삭제 시 함께 삭제해야 함
- **FR-024**: 비디오는 로컬 파일만 프레임 3개를 추출하여 분석해야 함
- **FR-025**: 10분 초과 비디오는 분석에서 제외해야 함

**iOS 버전별 분기**
- **FR-026**: iOS 18+에서는 AestheticsScore를 우선 사용하고, 실패 시 기존 파이프라인으로 대체해야 함
- **FR-027**: iOS 18+ AestheticsScore 사용 시에도 Safe Guard를 적용해야 함

**성능**
- **FR-028**: iCloud 사진은 로컬 캐시 썸네일만 사용하고 원본을 다운로드하지 않아야 함
- **FR-029**: 백그라운드 전환 시 탐색을 일시정지하고 포그라운드 복귀 시 자동 재개해야 함

**결과 및 복구**
- **FR-030**: 탐색 완료 후 결과 메시지를 표시해야 함 (N장 이동/0장 발견/취소됨)
- **FR-031**: 휴지통의 사진은 30일간 보관 후 영구 삭제되어야 함

### Key Entities

- **Photo**: 분석 대상 사진. 메타데이터(즐겨찾기, 편집됨, 숨김, 공유앨범, 스크린샷, 해상도, 생성일), 미디어 타입(사진, 비디오, Live Photo, Burst, RAW)을 포함
- **Quality Signal**: 품질 판정 신호. Strong(단일 조건으로 확정), Conditional(복합 조건), Weak(가중치 합산)로 분류
- **Cleanup Session**: 정리 세션. 시작 위치, 현재 위치, 찾은 사진 수, 모드(Precision/Recall) 정보를 포함
- **Trash Item**: 휴지통 항목. 원본 위치, 삭제 시점, 만료 시점 정보를 포함

## Success Criteria

### Measurable Outcomes

- **SC-001**: Precision 모드에서 오탐률 5% 이하 (정상 사진이 저품질로 판정되는 비율)
- **SC-002**: Precision 모드에서 60% 이상의 저품질 사진 탐지
- **SC-003**: Recall 모드에서 오탐률 20% 이하
- **SC-004**: Recall 모드에서 85% 이상의 저품질 사진 탐지
- **SC-005**: 1,000장 스캔 시 사용자 대기 시간 30초 이내
- **SC-006**: 정리 후 휴지통 복구율 5% 이하 (오탐지 간접 지표)
- **SC-007**: 정리 기능 재사용률 50% 이상 (사용자 만족도 간접 지표)

## Assumptions

1. 앱은 사진 라이브러리 전체 접근 권한이 있는 상태에서만 동작함
2. 휴지통은 iOS 시스템 휴지통이 아닌 앱 내부 휴지통을 사용함
3. 정리 버튼은 앱 내 적절한 위치에 배치됨 (구체적 위치는 디자인 단계에서 결정)
4. 판별 모드 전환 UI는 설정 화면 또는 정리 화면에서 접근 가능함
