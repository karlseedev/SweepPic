# Feature Specification: 자동 정리 - 저품질 사진 판별 및 삭제

**Feature Branch**: `001-auto-cleanup`
**Created**: 2026-01-19
**Status**: Draft
**Input**: 260119AutoDel.md 기획서 기반

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 기본 자동 정리 실행 (Priority: P1)

사용자가 사진 갤러리에서 '정리' 버튼을 탭하면, 시스템이 현재 스크롤 위치 기준 과거 방향으로 저품질 사진을 탐색하여 최대 50장의 후보를 찾아 검토 화면을 표시한다. 사용자는 후보 목록을 확인하고 원하지 않는 항목을 선택 해제한 뒤, 나머지를 앱 내 휴지통으로 일괄 이동할 수 있다.

**Why this priority**: 핵심 기능으로, 이 기능 없이는 자동 정리 기능 자체가 존재하지 않음. MVP의 필수 요소.

**Independent Test**: '정리' 버튼 탭 → 저품질 사진 탐색 → 검토 화면 표시 → 삭제 확인까지의 전체 흐름을 테스트 가능. 기본 Precision 모드로만 동작해도 핵심 가치 전달 가능.

**Acceptance Scenarios**:

1. **Given** 갤러리에 1,000장 이상의 사진이 있고 그 중 저품질 사진이 존재할 때, **When** 사용자가 '정리' 버튼을 탭하면, **Then** 진행률 표시와 함께 저품질 사진 탐색이 시작된다.
2. **Given** 탐색이 완료되어 30장의 저품질 후보가 발견되었을 때, **When** 검토 화면이 표시되면, **Then** "30장의 저품질 사진을 찾았습니다" 메시지와 함께 썸네일 그리드가 표시되고 모든 항목이 기본 선택 상태이다.
3. **Given** 검토 화면에서 5장을 선택 해제했을 때, **When** '삭제' 버튼을 탭하면, **Then** 25장이 앱 내 휴지통으로 이동되고 "25장이 휴지통으로 이동되었습니다" 토스트가 표시된다.
4. **Given** 삭제 완료 후 토스트가 표시될 때, **When** 3초 이내에 '실행취소' 버튼을 탭하면, **Then** 삭제된 25장이 원래 위치로 복구된다.

### User Story 2 - 저품질 판정 Safe Guard (Priority: P1)

시스템은 사용자의 소중한 사진을 보호하기 위해, 즐겨찾기로 표시되었거나 편집된 사진은 저품질 분석에서 자동으로 제외한다. 또한 블러로 판정된 사진이라도 심도 효과가 적용되었거나 선명한 얼굴이 감지되면 정상 사진으로 복구한다.

**Why this priority**: 오탐으로 인한 소중한 사진 삭제는 치명적인 사용자 경험 저하를 야기함. 핵심 기능과 동등한 우선순위.

**Independent Test**: 즐겨찾기/편집 사진 + 저품질 특성을 가진 테스트 이미지로 Safe Guard 동작 검증 가능.

**Acceptance Scenarios**:

1. **Given** 완전히 어두운 사진(휘도 < 0.10)이 즐겨찾기로 표시되어 있을 때, **When** 자동 정리가 실행되면, **Then** 해당 사진은 저품질 후보에서 제외된다.
2. **Given** 심하게 흐린 사진(Laplacian < 50)에 필터 편집이 적용되어 있을 때, **When** 자동 정리가 실행되면, **Then** 해당 사진은 저품질 후보에서 제외된다.
3. **Given** 블러 판정을 받은 인물 사진에서 선명한 얼굴(FaceQuality >= 0.4)이 감지되었을 때, **When** Safe Guard 후처리가 적용되면, **Then** 블러 판정이 무효화되어 정상 사진으로 분류된다.
4. **Given** 포트레이트 모드로 촬영된 사진(심도 효과 적용)이 배경 블러로 인해 블러 판정을 받았을 때, **When** Safe Guard 후처리가 적용되면, **Then** 블러 판정이 무효화된다.

### User Story 3 - 탐색 진행 피드백 및 취소 (Priority: P2)

사용자는 저품질 사진 탐색 중 실시간 진행률을 확인할 수 있으며, 언제든지 탐색을 취소할 수 있다. 탐색이 최대 한도(10,000장 또는 60초)에 도달하거나 조기 종료 조건을 충족하면 적절한 안내와 함께 현재까지 결과를 표시한다.

**Why this priority**: 대용량 라이브러리에서 사용자가 대기 상태를 인지하고 제어할 수 있어야 함. 핵심 기능 다음 우선순위.

**Independent Test**: 다양한 라이브러리 크기와 저품질 비율에서 진행률 표시 및 취소 동작 테스트 가능.

**Acceptance Scenarios**:

1. **Given** 탐색이 진행 중일 때, **When** 진행률 UI가 표시되면, **Then** "저품질 사진 찾는 중... N/50" 형식으로 발견된 후보 수가 실시간 업데이트된다.
2. **Given** 탐색 중 사용자가 '취소' 버튼을 탭했을 때, **When** 취소가 처리되면, **Then** 탐색이 즉시 중단되고 갤러리 화면으로 복귀한다.
3. **Given** 10,000장 스캔 또는 60초 경과로 최대 한도에 도달했을 때, **When** 탐색이 강제 중단되면, **Then** "N장을 찾았습니다 (일부만 분석됨)" 메시지와 함께 검토 화면이 표시된다.
4. **Given** 5,000장 스캔 후 저품질 사진이 0장 발견되었을 때, **When** 조기 종료가 트리거되면, **Then** "저품질 사진을 찾지 못했습니다" 메시지가 표시된다.

### User Story 4 - 판별 모드 전환 (Priority: P3)

사용자는 설정 화면 또는 정리 버튼 길게 누르기를 통해 '신중한 정리'(Precision)와 '적극적 정리'(Recall) 모드를 전환할 수 있다. 각 모드는 서로 다른 판별 기준을 적용하여 오탐률과 탐지율의 균형을 조절한다.

**Why this priority**: 고급 사용자를 위한 옵션으로, 기본 Precision 모드만으로도 핵심 가치 전달 가능.

**Independent Test**: 동일한 테스트 이미지 세트에 두 모드를 적용하여 판별 결과 차이 검증 가능.

**Acceptance Scenarios**:

1. **Given** 기본 설정 상태에서 자동 정리를 실행할 때, **When** 저품질 판별이 수행되면, **Then** Precision 모드(Strong 신호만 사용)가 적용된다.
2. **Given** 사용자가 '적극적 정리' 모드로 전환한 후 자동 정리를 실행할 때, **When** 저품질 판별이 수행되면, **Then** Recall 모드(Strong + Conditional + Weak 조합)가 적용되어 더 많은 후보가 발견된다.
3. **Given** 정리 버튼을 길게 누르면, **When** 모드 선택 UI가 표시되면, **Then** '신중한 정리 (기본)'과 '적극적 정리' 옵션이 표시되고 현재 선택이 표시된다.

### User Story 5 - iCloud 사진 처리 (Priority: P3)

로컬 저장 비율이 낮은(30% 미만) 경우 사용자에게 안내를 표시하고, 기본적으로 로컬 사진만 분석한다. 사용자 선택에 따라 iCloud 사진도 분석에 포함할 수 있다.

**Why this priority**: iCloud 의존도가 높은 사용자를 위한 기능으로, 로컬 사진만으로도 기본 기능 동작 가능.

**Independent Test**: 로컬/iCloud 혼합 환경에서 안내 표시 및 다운로드 정책 테스트 가능.

**Acceptance Scenarios**:

1. **Given** 로컬 사진 비율이 25%인 라이브러리에서, **When** '정리' 버튼을 탭하면, **Then** "대부분의 사진이 iCloud에 있습니다. 로컬에 저장된 사진만 분석됩니다" 안내와 함께 [확인] / [iCloud 사진 다운로드] 버튼이 표시된다.
2. **Given** iCloud 안내에서 [확인]을 선택했을 때, **When** 탐색이 시작되면, **Then** 로컬 사진만 분석 대상이 된다.
3. **Given** iCloud 안내에서 [iCloud 사진 다운로드]를 선택했을 때, **When** 탐색이 시작되면, **Then** iCloud 사진도 필요 시 다운로드하여 분석에 포함한다.

---

### User Story 6 - 앱 내 휴지통 관리 (Priority: P3)

삭제된 사진은 앱 내 휴지통에 30일간 보관되며, 이후 자동으로 영구 삭제된다. 사용자는 세션 내에서 삭제를 실행취소할 수 있다.

**Why this priority**: 휴지통은 기존 기능이며, 자동 정리 기능은 이를 활용함.

**Independent Test**: 휴지통 이동, 30일 보관, Undo 기능을 독립적으로 테스트 가능.

**Acceptance Scenarios**:

1. **Given** 자동 정리로 30장이 휴지통으로 이동되었을 때, **When** 휴지통을 확인하면, **Then** 해당 30장이 휴지통 목록에 표시된다.
2. **Given** 휴지통에 사진이 30일간 보관되었을 때, **When** 31일째가 되면, **Then** 해당 사진들이 자동으로 영구 삭제된다.
3. **Given** 앱을 종료하지 않은 상태에서 삭제 토스트의 '실행취소'를 탭했을 때, **When** Undo가 처리되면, **Then** 삭제된 사진들이 원래 위치로 복구된다.

---

### Edge Cases

- **저품질 사진이 0장일 때**: "저품질 사진을 찾지 못했습니다" 메시지 표시 후 갤러리로 복귀
- **라이브러리가 비어있을 때**: 정리 버튼 비활성화 또는 "분석할 사진이 없습니다" 메시지
- **탐색 중 메모리 부족 발생 시**: 즉시 중단 후 현재까지 발견된 후보로 검토 화면 표시, "분석이 중단되었습니다. N장을 찾았습니다" 메시지
- **비디오 분석 타임아웃 시**: 해당 비디오 SKIP, 후보 수에 포함하지 않음
- **10분 초과 비디오**: 분석 제외 (SKIP)
- **스크린샷**: 저품질 로직에서 제외 (별도 카테고리로 분리 예정)
- **모든 후보를 선택 해제한 경우**: 삭제 버튼이 "삭제 (0)"으로 표시되고 탭 시 동작 없음 또는 비활성화

## Requirements *(mandatory)*

### Functional Requirements

**저품질 판별**
- **FR-001**: 시스템 MUST 극단적으로 어두운 사진(평균 휘도 < 0.10)을 저품질로 판정한다 (Strong 신호)
- **FR-002**: 시스템 MUST 극단적으로 밝은 사진(평균 휘도 > 0.90)을 저품질로 판정한다 (Strong 신호)
- **FR-003**: 시스템 MUST 심각하게 흐린 사진(Laplacian Variance < 50)을 저품질로 판정한다 (Strong 신호)
- **FR-004**: 시스템 MUST Recall 모드에서 Conditional 신호(렌즈 가림, 극단 단색, 주머니 샷)를 저품질로 판정한다
- **FR-005**: 시스템 MUST Recall 모드에서 Weak 신호 2개 이상 조합 시 저품질로 판정한다

**Safe Guard**
- **FR-006**: 시스템 MUST 즐겨찾기(isFavorite) 사진을 저품질 분석에서 제외한다
- **FR-007**: 시스템 MUST 편집된(hasAdjustments) 사진을 저품질 분석에서 제외한다
- **FR-008**: 시스템 MUST 스크린샷을 저품질 로직에서 제외하고 별도 카테고리로 분리한다
- **FR-009**: 시스템 MUST 블러 판정된 사진에서 심도 효과(photoDepthEffect)가 감지되면 판정을 무효화한다
- **FR-010**: 시스템 MUST 블러 판정된 사진에서 선명한 얼굴(FaceQuality >= 0.4)이 감지되면 판정을 무효화한다

**탐색 및 실행**
- **FR-011**: 시스템 MUST 현재 스크롤 위치 기준 과거(오래된) 방향으로 사진을 스캔한다
- **FR-012**: 시스템 MUST 저품질 후보 50장 발견 시 즉시 탐색을 중단한다
- **FR-013**: 시스템 MUST 최대 10,000장 또는 60초 경과 시 탐색을 강제 중단한다
- **FR-014**: 시스템 MUST 5,000장 스캔 후 0장 발견 시 조기 종료한다
- **FR-015**: 시스템 MUST 기본적으로 로컬 사진만 분석한다 (iCloud 제외)
- **FR-016**: 사용자가 iCloud 다운로드를 허용하면 시스템 MUST iCloud 사진도 분석에 포함한다

**미디어 타입 처리**
- **FR-017**: 시스템 MUST 라이브포토의 키 프레임(대표 이미지)만 분석한다
- **FR-018**: 시스템 MUST 비디오에서 3개 대표 프레임(시작/중간/끝)을 추출하여 중앙값 기준으로 판정한다
- **FR-019**: 시스템 MUST 비디오 분석이 5초를 초과하면 해당 비디오를 SKIP한다
- **FR-020**: 시스템 MUST 10분 초과 비디오를 분석에서 제외한다

**UX 흐름**
- **FR-021**: 시스템 MUST 탐색 중 진행률(발견된 후보 수)을 실시간으로 표시한다
- **FR-022**: 시스템 MUST 탐색 중 취소 버튼을 제공하고, 탭 시 즉시 중단한다
- **FR-023**: 시스템 MUST 검토 화면에서 후보 사진 그리드를 표시하고 개별 선택 해제를 허용한다
- **FR-024**: 시스템 MUST 삭제 완료 후 3초간 토스트와 실행취소 버튼을 표시한다
- **FR-025**: 시스템 MUST 세션 내(앱 종료 전)에서만 Undo 기능을 지원한다

**판별 모드**
- **FR-026**: 시스템 MUST 기본값으로 Precision 모드(Strong 신호만)를 적용한다
- **FR-027**: 사용자 MUST 설정 또는 정리 버튼 길게 누르기로 Recall 모드로 전환할 수 있다

**휴지통**
- **FR-028**: 시스템 MUST 삭제된 사진을 앱 내 휴지통에 30일간 보관한다
- **FR-029**: 시스템 MUST 30일 경과 후 휴지통 사진을 자동으로 영구 삭제한다
- **FR-030**: 시스템 MUST 휴지통 용량에 대해 "시스템 앨범에서 삭제해야 실제 용량 확보" 안내를 제공한다

### Key Entities

- **LowQualityCandidate**: 저품질로 판정된 사진 후보. 판정 사유(Strong/Conditional/Weak), 분석 점수(휘도, Laplacian 등), 원본 사진 참조 포함
- **AnalysisSession**: 한 번의 자동 정리 실행 세션. 시작 시간, 스캔된 사진 수, 발견된 후보 수, 종료 사유 포함
- **CleanupMode**: 판별 모드 설정 (Precision/Recall)
- **TrashItem**: 앱 내 휴지통 항목. 삭제 일시, 원본 위치, 만료 일시 포함

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: Precision 모드에서 오탐률(정상 사진을 저품질로 판정) 5% 이하 유지
- **SC-002**: Precision 모드에서 실제 저품질 사진의 60% 이상 탐지
- **SC-003**: Recall 모드에서 실제 저품질 사진의 85% 이상 탐지
- **SC-004**: 1,000장 라이브러리에서 50장 후보 탐색 완료까지 10초 이내
- **SC-005**: 10,000장 라이브러리에서 최대 스캔 한도(60초) 내 탐색 완료 또는 강제 중단
- **SC-006**: 사용자의 "Keep"(선택 해제) 비율 20% 이하 (오탐 지표)
- **SC-007**: Undo 사용 비율 5% 이하 (삭제 후 후회 지표)
- **SC-008**: 탐색 취소 비율 10% 이하 (대기 시간 불만족 지표)

## Assumptions

- iOS 16 이상을 지원 타겟으로 함
