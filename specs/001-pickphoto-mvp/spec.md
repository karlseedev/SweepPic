## Clarifications

- Q: iOS PhotoKit 삭제 시 시스템 확인 팝업이 강제 표시되는데, "확인 없이 즉시 삭제" 정책을 어떻게 구현할 것인가? → A: 앱 내 휴지통 도입 (2단계 삭제)
- Q: 딤드된 사진(삭제 예정)을 탭하면? → A: 뷰어로 진입하되 "복구/완전삭제" 옵션 표시

- Q: 앱 내 휴지통 자동 정리 정책은? → A: MVP는 수동 비우기, 향후 앱 종료 시 확인 옵션 추가 예정

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 사진 브라우징 (Priority: P1)

사용자가 앱을 실행하면 즉시 전체 사진 라이브러리를 그리드 형태로 탐색할 수 있다. 5만 장 이상의 대용량 라이브러리에서도 iOS 기본 사진 앱과 유사한 부드러운 스크롤 경험을 제공한다.

**Why this priority**: 사진 갤러리 앱의 핵심 기능. 브라우징이 없으면 앱의 가치가 없음.

**Independent Test**: 앱 실행 후 사진 그리드가 표시되고, 스크롤이 부드럽게 동작하는지 확인. 5만 장 라이브러리에서 끊김 없이 스크롤 가능.

**Acceptance Scenarios**:

1. **Given** 사용자가 사진 접근 권한을 허용한 상태, **When** 앱을 실행하면, **Then** All Photos 그리드가 최신 사진(맨 아래)부터 표시된다.
2. **Given** 그리드가 표시된 상태, **When** 빠르게 스크롤하면, **Then** 스크롤이 부드럽고 이전 이미지가 잘못 표시되지 않는다(오표시 0).
3. **Given** 그리드가 표시된 상태, **When** 핀치 제스처를 사용하면, **Then** 1열/3열/5열 사이에서 그리드 밀도가 자연스럽게 변경된다.
4. **Given** 핀치 줌 중, **When** 손가락 중심점 아래 사진이 있으면, **Then** 줌 변경 후에도 해당 사진이 화면에 유지된다(앵커 유지).
5. **Given** 앱 내 휴지통에 사진이 있는 상태, **When** 그리드를 보면, **Then** 해당 사진은 딤드(어둡게) 처리되어 표시된다.

---

### User Story 2 - 뷰어에서 사진 탐색 및 삭제 (Priority: P2)

사용자가 그리드에서 사진을 탭하면 전체 화면 뷰어로 진입하여 사진을 자세히 보고, 좌우 스와이프로 이전/다음 사진을 탐색하고, 위 스와이프로 빠르게 삭제(앱 내 휴지통으로 이동)할 수 있다.

**Why this priority**: 사진 정리 특화 기능의 핵심. 위 스와이프 삭제는 SweepPic의 차별점.

**Independent Test**: 그리드에서 사진 탭 → 뷰어 진입 → 좌우 스와이프 탐색 → 위 스와이프 삭제 → 이전 사진으로 자동 이동 확인.

**Acceptance Scenarios**:

1. **Given** 그리드가 표시된 상태, **When** 일반 사진을 탭하면, **Then** 전체 화면 뷰어로 줌 전환 애니메이션과 함께 진입한다.
2. **Given** 뷰어가 표시된 상태, **When** 좌우로 스와이프하면, **Then** 이전/다음 사진으로 부드럽게 전환된다.
3. **Given** 뷰어가 표시된 상태, **When** 위로 화면 높이의 20% 이상 스와이프하면, **Then** 사진이 앱 내 휴지통으로 이동하고 이전 사진으로 자동 이동한다.
4. **Given** 뷰어에서 첫 번째 사진을 보는 상태, **When** 위 스와이프로 삭제하면, **Then** 다음 사진으로 이동한다(이전 사진이 없으므로).
5. **Given** 뷰어에서 마지막 남은 사진을 보는 상태, **When** 삭제하면, **Then** 그리드로 복귀한다.
6. **Given** 뷰어가 표시된 상태, **When** 아래로 스와이프하면, **Then** 뷰어가 닫히고 그리드로 복귀한다.
7. **Given** 그리드에서 딤드된 사진(휴지통 상태)을 탭하면, **When** 뷰어로 진입하면, **Then** "복구" 및 "완전삭제" 옵션이 표시된다.

### User Story 3 - 그리드에서 다중 선택 삭제 (Priority: P3)

사용자가 그리드에서 Select 모드로 진입하여 여러 사진을 선택하고 한 번에 삭제(앱 내 휴지통으로 이동)할 수 있다.

**Why this priority**: 대량 사진 정리 시 필수 기능. 하나씩 삭제하는 것보다 효율적.

**Independent Test**: Select 버튼 탭 → 여러 사진 탭/드래그 선택 → Delete 버튼 탭 → 선택된 사진 앱 내 휴지통으로 이동 확인.

**Acceptance Scenarios**:

1. **Given** 그리드가 표시된 상태, **When** 상단 플로팅 타이틀바의 Select 버튼을 탭하면, **Then** 선택 모드로 진입하고 하단 캡슐 탭바가 Select 툴바(Cancel/Delete)로 대체된다.
2. **Given** 선택 모드인 상태, **When** 일반 사진을 탭하면, **Then** 해당 사진이 선택되고 체크마크가 표시된다.
3. **Given** 선택 모드인 상태, **When** 딤드된 사진(휴지통 상태)을 탭하면, **Then** 선택되지 않는다.
4. **Given** 선택 모드인 상태, **When** 사진 위에서 드래그하면, **Then** 지나가는 일반 사진들이 연속으로 선택된다(딤드 사진 제외).
5. **Given** 사진이 선택된 상태, **When** Delete 버튼을 탭하면, **Then** 선택된 사진이 앱 내 휴지통으로 이동한다(팝업 없음).
6. **Given** 선택 모드인 상태, **When** Cancel 버튼을 탭하면, **Then** 선택이 해제되고 일반 모드로 복귀한다.

---

### User Story 4 - 앨범 브라우징 (Priority: P4)

사용자가 앨범 탭에서 사용자 앨범, 스마트 앨범(Screenshots), 앱 내 휴지통을 탐색하고 앨범 내 사진을 볼 수 있다.

**Why this priority**: 사진 정리 시 특정 앨범(예: 스크린샷)만 정리하고 싶은 경우가 많음.

**Independent Test**: 앨범 탭 진입 → 앨범 목록 확인 → 앨범 탭하여 진입 → 앨범 내 사진 그리드 확인.

**Acceptance Scenarios**:

1. **Given** 앱이 실행된 상태, **When** Albums 탭을 탭하면, **Then** 사용자 앨범 목록, 스마트 앨범(Screenshots), 휴지통이 표시된다.
2. **Given** 앨범 목록이 표시된 상태, **When** 앨범을 탭하면, **Then** 해당 앨범의 사진이 그리드로 표시된다.
3. **Given** 앨범 그리드가 표시된 상태, **When** 사진을 삭제하면, **Then** 앱 내 휴지통으로 이동한다.

---

### User Story 5 - 앱 내 휴지통 관리 (Priority: P5)

사용자가 앱 내 휴지통에서 삭제 예정 사진을 확인하고, 복구하거나 완전 삭제(iOS 휴지통으로 이동)할 수 있다.

**Why this priority**: 실수로 삭제한 사진 복구 및 최종 삭제 확정 기능.

**Independent Test**: 휴지통 진입 → 사진 목록 확인 → 개별 복구/삭제 → 일괄 비우기 시 시스템 팝업 확인.

**Acceptance Scenarios**:

1. **Given** Albums 탭이 표시된 상태, **When** "휴지통" 앨범을 탭하면, **Then** 앱 내 휴지통의 사진 목록이 표시된다.
2. **Given** 휴지통 화면에서, **When** 사진을 탭하면, **Then** 뷰어로 진입하고 "복구" 및 "완전삭제" 옵션이 표시된다.
3. **Given** 휴지통 뷰어에서, **When** "복구" 버튼을 탭하면, **Then** 사진이 일반 상태로 복구되고 그리드에서 딤드 효과가 제거된다.
4. **Given** 휴지통 뷰어에서, **When** "완전삭제" 버튼을 탭하면, **Then** 시스템 확인 팝업이 표시되고 확인 시 iOS "최근 삭제됨"으로 이동한다.
5. **Given** 휴지통 화면에서, **When** "비우기" 버튼을 탭하면, **Then** 시스템 확인 팝업이 표시되고 확인 시 모든 사진이 iOS "최근 삭제됨"으로 이동한다.
6. **Given** 휴지통이 비어있는 상태, **When** 휴지통 화면을 보면, **Then** "휴지통이 비어 있습니다" 메시지가 표시된다.

---

### User Story 6 - 권한 관리 (Priority: P6)

사용자가 앱을 처음 실행할 때 사진 접근 권한을 요청받고, 권한 상태에 따라 적절한 UI를 볼 수 있다.

**Why this priority**: 권한 없이는 앱이 동작하지 않음. 하지만 한 번 설정하면 반복되지 않으므로 상대적으로 낮은 우선순위.

**Independent Test**: 앱 최초 실행 → 권한 요청 화면 → 허용/거부 후 적절한 화면 표시 확인.

**Acceptance Scenarios**:

1. **Given** 권한이 결정되지 않은 상태, **When** 앱을 실행하면, **Then** 권한 요청 화면이 표시된다.
2. **Given** 권한 요청 화면에서, **When** "사진 접근 허용" 버튼을 탭하면, **Then** 시스템 권한 팝업이 표시된다.
3. **Given** 전체 접근 권한이 허용된 상태, **When** 앱을 실행하면, **Then** 전체 라이브러리가 표시된다.
4. **Given** 제한된 접근 권한(Limited)인 상태, **When** 앱을 실행하면, **Then** 권한 거부와 동일하게 설정 앱으로 이동을 안내하는 화면이 표시된다.
5. **Given** 권한이 거부된 상태, **When** 앱을 실행하면, **Then** 설정 앱으로 이동을 안내하는 화면이 표시된다.

---

### Edge Cases

- 사진이 0장일 때: 빈 상태 메시지("사진이 없습니다") 표시
- 앨범에 사진이 0장일 때: 빈 상태 메시지 표시
- 외부에서 사진이 삭제되었을 때: 뷰어에서 "이전 사진 우선" 규칙으로 자동 이동, 앱 내 휴지통 상태도 자동 정리
- 메모리 경고 발생 시: 캐시 즉시 해제
- 빠른 연속 삭제 시: UI가 안정적으로 수렴
- iCloud-only 사진: placeholder로 표시(다운로드 시도 안함)
- 앱 내 휴지통에 있는 사진이 외부에서 삭제됨: 휴지통에서 자동 제거

## Requirements *(mandatory)*

### Functional Requirements

**그리드 브라우징**
- **FR-001**: 시스템은 전체 사진 라이브러리를 3열 기본 그리드로 표시해야 한다. iOS 18 사진 앱과 동일한 레이아웃(셀 간격 2pt, 정사각형 비율)을 사용한다.
  - **Edge-to-Edge**: 콘텐츠는 safe area를 무시하고 화면 전체를 채운다
  - **구현 방식**: 시스템 바(네비/탭) 숨김 + 커스텀 플로팅 오버레이
  - **상단 오버레이**: 플로팅 타이틀바 (타이틀 + Select 버튼), 44pt + safe area, 블러+딤, Select 버튼만 터치 반응
  - **하단 오버레이**: 플로팅 캡슐 탭바 (iOS 18 Photos 스타일), 블러+딤, 버튼만 터치 반응
  - **contentInset**: 플로팅 UI 높이만큼 설정 (첫/마지막 줄 가림 방지)
  - **iOS 버전별 정책**: iOS 16~25는 커스텀 플로팅 오버레이, iOS 26+는 시스템 기본 외관 사용
- **FR-002**: 시스템은 오래된 사진이 위, 최신 사진이 아래인 순서로 정렬해야 한다. 최신 사진(맨 아래) 기준 꽉 차게 정렬하며, 3의 배수가 아닐 경우 빈칸은 맨 위 행(가장 오래된 사진 쪽, 스크롤 끝까지 올려야 보임)에 생성한다.
- **FR-003**: 시스템은 첫 진입 시 맨 아래(최신)로 스크롤해야 한다
- **FR-004**: 시스템은 핀치 제스처로 1열/3열/5열 사이에서 그리드 밀도를 변경할 수 있어야 한다. Select 모드에서도 핀치 줌을 허용하며 선택 상태는 유지된다.
- **FR-005**: 시스템은 핀치 중심점 아래 콘텐츠를 앵커로 유지해야 한다
- **FR-006**: 시스템은 5만 장 라이브러리에서도 hitch < 5 ms/s (Good 등급) 스크롤을 제공해야 한다
- **FR-007**: 시스템은 빠른 스크롤 중에도 잘못된 이미지를 표시하지 않아야 한다(오표시 0)
- **FR-008**: 시스템은 앱 내 휴지통에 있는 사진을 그리드에서 딤드 처리(50% opacity 다크레드 오버레이)하여 표시해야 한다

**뷰어**
- **FR-009**: 시스템은 그리드에서 일반 사진 탭 시 전체 화면 뷰어로 전환해야 한다
- **FR-010**: 시스템은 좌우 스와이프로 이전/다음 사진 탐색을 지원해야 한다
- **FR-011**: 시스템은 위 스와이프(화면 높이 20% 이상, UIScreen.main.bounds.height 기준)로 사진을 앱 내 휴지통으로 이동해야 한다. 뷰어 줌 상태(scale > 1)에서도 위 스와이프 삭제를 허용한다.
- **FR-012**: 시스템은 아래 스와이프로 뷰어를 닫고 그리드로 복귀해야 한다
- **FR-013**: 시스템은 뷰어에서 삭제 후 "이전 사진 우선" 규칙으로 이동해야 한다
- **FR-014**: 시스템은 더블탭/핀치로 이미지 확대/축소를 지원해야 한다
- **FR-015**: 시스템은 일반 사진 뷰어 하단에 원형 플로팅 삭제 버튼을 상시 표시해야 한다. 딤드된 사진(휴지통 상태) 뷰어에서는 "복구/완전삭제" 옵션을 표시한다.

**선택 모드**
- **FR-016**: 시스템은 Select 모드에서 탭으로 단일 선택을 지원해야 한다. iOS 사진 앱과 동일한 선택 표시 스타일을 사용한다.
- **FR-017**: 시스템은 Select 모드에서 드래그로 연속 선택을 지원해야 한다. MVP에서 화면 경계 도달 시 자동 스크롤은 미지원하며, 추후 지원을 검토한다.
- **FR-018**: 시스템은 선택된 사진 수를 상단 플로팅 타이틀바에 표시해야 한다. Select 모드 진입 시 하단 캡슐 탭바는 Select 툴바(Cancel / N개 선택됨 / Delete)로 대체된다. Select 모드 종료(취소/삭제 완료) 시 캡슐 탭바로 원상복귀한다.
- **FR-019**: 시스템은 선택된 사진을 앱 내 휴지통으로 일괄 이동할 수 있어야 한다
- **FR-020**: 시스템은 Select 모드에서 딤드된 사진(휴지통 상태)을 선택 불가로 처리해야 한다

**앱 내 휴지통**
- **FR-021**: 시스템은 삭제된 사진을 앱 내 휴지통으로 이동해야 한다(PhotoKit에서 실제 삭제하지 않음)
- **FR-022**: 시스템은 앱 내 휴지통 상태를 로컬에 저장해야 한다. 삭제/복원 등 상태 변경 행동마다 즉시 저장한다.
- **FR-023**: 시스템은 휴지통에서 개별 사진 복구를 지원해야 한다. MVP에서 휴지통 다중 선택(Select 모드)은 미지원하며 개별 처리만 가능하다.
- **FR-024**: 시스템은 휴지통에서 개별 사진 완전삭제 시 앱 자체 확인 없이 iOS 시스템 팝업만 표시해야 한다
- **FR-025**: 시스템은 휴지통 비우기 시 앱 자체 확인 없이 iOS 시스템 팝업만 표시하고, 확인 시 iOS "최근 삭제됨"으로 이동해야 한다
- **FR-026**: 시스템은 권한이 충분할 때만 삭제 기능을 활성화해야 한다

**앨범**
- **FR-027**: 시스템은 사용자 앨범 목록을 2열 그리드로 표시해야 한다. iOS 사진 앱 앨범 목록과 동일한 레이아웃을 사용한다.
- **FR-028**: 시스템은 스마트 앨범 중 Screenshots를 표시해야 한다
- **FR-029**: 시스템은 앨범 목록에 "휴지통" 앨범을 표시해야 한다
- **FR-030**: 시스템은 앨범 진입 시 해당 앨범의 사진을 그리드로 표시해야 한다
- **FR-031**: 시스템은 앨범 내 삭제 시 앱 내 휴지통으로 이동해야 한다

**권한**
- **FR-032**: 시스템은 권한 미결정 시 권한 요청 화면을 표시해야 한다
- **FR-033**: 시스템은 제한 접근(Limited) 시 권한 거부와 동일하게 설정 앱 이동 안내 화면을 표시해야 한다
- **FR-034**: 시스템은 권한 거부 시 설정 앱 이동 안내("사진 접근 권한이 필요합니다. 설정에서 권한을 허용해주세요." + "설정 열기" 버튼)를 표시해야 한다. 권한 거부 후 앱 재실행 시에도 동일한 안내 화면을 표시한다.

**미디어 타입**
- **FR-035**: 시스템은 비디오를 iOS 사진 앱과 동일한 스타일로 표시해야 한다(썸네일에 재생 아이콘 및 duration 표시). 뷰어에서는 첫 프레임만 표시한다(재생 미지원).
- **FR-036**: 시스템은 Live Photo를 배지 없이 정지 이미지로만 표시해야 한다(애니메이션 미지원, MVP)

### Key Entities

- **Photo Asset**: 사진/비디오/Live Photo. 로컬 식별자, 생성일, 미디어 타입, 썸네일/원본 이미지
- **Trash State**: 앱 내 휴지통 상태. 사진 로컬 식별자 목록, 삭제 시각 (로컬 저장)
- **Album**: 사용자 생성 앨범. 이름, 생성일, 포함된 사진 목록, 대표 썸네일
- **Smart Album**: 시스템 정의 앨범(Screenshots). 동적 필터 조건으로 사진 포함

**기능 기준**:

- **SC-001**: 5만 장 라이브러리에서 30초간 빠르게 스크롤해도 hitch < 5 ms/s 유지
- **SC-002**: 빠른 스크롤 중 잘못된 이미지가 표시되는 경우가 0건 (오표시 0)
- **SC-003**: 핀치 줌 시 앵커 사진이 화면에서 0px drift로 유지 (Gate 3 검증 완료)
- **SC-004**: 뷰어에서 위 스와이프 삭제 후 이전 사진으로 자동 이동이 100% 정확 (테스트 코드로 검증)
- **SC-005**: 다중 선택 삭제 후 선택 상태가 올바르게 해제되고 그리드가 정상 표시
- **SC-006**: 휴지통에서 복구한 사진이 그리드에서 정상(딤드 해제) 표시
- **SC-007**: 휴지통 비우기 시 시스템 팝업이 100% 표시
- **SC-008**: iOS 18에서 그리드가 노치까지 edge-to-edge로 표시되고, 상/하단은 블러+딤의 플로팅 UI로 보임
- **SC-009**: 상단 오버레이에서 스크롤 제스처 충돌 없음 (터치 통과), 하단은 버튼만 터치 반응

## Assumptions

- 사용자는 iOS 16 이상 기기를 사용한다
- 사용자의 사진은 주로 로컬에 저장되어 있다(iCloud 전용 사진은 placeholder 표시)
- 앱 내 휴지통 상태는 앱 로컬 저장소에 저장된다 (파일 기반, 대용량 ID Set 대응)
- 앱 내 휴지통 항목 수 제한 없음 (MVP)
- 앱 내 휴지통은 자동 정리되지 않으며, 사용자가 수동으로 비운다 (MVP)
- 휴지통 비우기 또는 완전삭제 시 iOS 시스템 확인 팝업이 표시된다 (PhotoKit 필수 동작)
- 사진 라이브러리 크기는 최대 5만 장을 기준으로 성능을 보장한다
- 앨범 그리드는 MVP에서 3열 고정 (핀치 줌은 M2 이후)
