## 1. 심사위원 평가 핵심 포인트

### 1-1. 심사위원이 중점적으로 보는 3가지
1. 창업자가 해당 분야에서 **경험과 전문성**을 갖췄는가
2. 사업 아이템의 **혁신성과 차별성 및 경쟁력**이 있는가
3. 실제 **매출과 성장 가능성**이 있는가

> 출처: [2025 예비창업패키지 합격 심사기준 2가지](https://blog.xn--9i1b674cwc38r6pa.com/2025-%EC%98%88%EB%B9%84%EC%B0%BD%EC%97%85%ED%8C%A8%ED%82%A4%EC%A7%80-%ED%95%A9%EA%B2%A9-%EC%8B%AC%EC%82%AC%EA%B8%B0%EC%A4%80-2%EA%B0%80%EC%A7%80--44358)

### 1-2. 실현가능성(Solution) 섹션 핵심
- **"현재 이렇게까지 해놨습니다"** 내용이 가장 설득력 있음
  - 프로토타입, 베타 버전, 3D 이미지 등 완성 결과물
  - 타겟 고객 확보 증거 (커뮤니티, 팔로워 등)
  - 랜딩페이지를 통한 고객 반응 데이터
- 비슷한 아이디어 사이에서 **실행력과 가능성을 보여줄 수 있는 '한 끗'**이 필요

> 출처: [예비창업패키지 합격 전략 - 전문가 사업계획서 작성법](https://imweb.me/blog?idx=215)

### 1-3. 차별성 평가 기준
- **"아이템이 좋다"가 아니라 "비즈니스 모델을 이렇게 세워서 잘 해내겠다"** 가 핵심
- 모든 논리는 **객관적, 정량적, 계획적**이어야 함
- 주관적 표현 금지 ("더 좋다", "더 편하다")
- 근거 제시 필수: "블라인드 테스트 결과 80% 선택" 같은 구체적 데이터

## 2. 차별성 작성 시 주의사항

### 2-1. 절대 하지 말 것
| 실수 | 설명 |
|------|------|
| "국내 최초", "경쟁사 없음" | 시장 조사 부족으로 판단되어 감점 |
| 단순 기능 비교표 (✅/-) | 기능 부재 지적일 뿐, 차별성을 보여주지 못함 |
| 주관적 우월성 표현 | "더 좋다", "더 편하다" → 정량적 근거로 대체 |
| 검증 안 된 주장 | 경쟁사 기술 스택, 시장 점유율 등 추측 금지 |

### 2-2. 해야 할 것
- **"어설픈 최초보다 확실한 최고"** — 구체적이고 검증 가능한 우위 강조
- **기존 솔루션의 한계를 A방식으로 해결한 구체적 접근** 서술
- **기술 기반 진입장벽**: 기술과 혁신이 정책적 중점 주제이므로, 전문적 기술 활용에 의한 진입장벽 제시
- **고객의 지불 가치에 대한 경쟁력의 근거** 제시

> 출처: [PSST 사업계획서 작성법](https://brunch.co.kr/@asdkorea1811/23)

### 2-3. 실현가능성 = "업계 1, 2위와 뭐가 다른데?"
- 전체 시장이 아니라 **업계 1~2위 기업을 면밀히 조사** 후 분석 결과 제시
- 경쟁사의 **불편한 사용 경험, 느린 속도, 고가 비용, 제한적 기능** 등 약점 파악
- 그 약점을 **어떻게 보완하는지** 구체화

## 3. 좋은 작성 vs 나쁜 작성

| 구분 | 나쁜 예 | 좋은 예 |
|------|--------|--------|
| 우월성 주장 | "튼튼합니다" | "내구성 테스트 결과 30% 향상" |
| 마케팅 전략 | "SNS 마케팅" | "OOO 채널, OOO명 규모, OOO 예산" |
| 목표 | "크게 성공하겠습니다" | "1년 내 월 OOO만원 매출" |
| 개발 계획 | "김치 100포기를 만들겠습니다" | 도매 구매처 리스트업 → 공장 숙성 → 레시피 제조 → 패키징 단계별 계획 |
| 차별성 | "국내 최초 AI 정리 앱" | "경쟁사 A의 N점 만족도를 M방식으로 개선" |

## 4. ERRC + 전략캔버스 (경쟁사 분석 시각화 도구)

> ※ 이 도구는 섹션 2(차별성)보다 **섹션 3(성장전략)의 경쟁사 분석**에 더 적합

### 4-1. ERRC 프레임워크
| 항목 | 내용 |
|------|------|
| 제거 (Eliminate) | 고객에게 가치 없는데 비용만 발생시키는 요소 제거 |
| 축소 (Reduce) | 기존 요소의 비용/수준 축소 |
| 증가 (Raise) | 기존 요소에서 더 높은 가치 제공 |
| 창조 (Create) | 기존에 없던 새로운 가치 창출 |

### 4-2. 전략캔버스 작성법
1. **경쟁자 식별** — 같은 카테고리 + 고객의 과업을 수행하는 모든 대안
2. **경쟁 요소 식별** — 고객이 중요하게 여기는 제품 속성 나열
3. **자사와 경쟁자 비교** — 각 요소별로 점수화
4. **가치곡선 도출** — 점수를 연결해 그래프로 시각화

**핵심**: 경쟁사 곡선과 자사 곡선이 **서로 교차하는 모양**이 나와야 제대로 된 차별화 전략. 평가위원은 하루에 수십 개 계획서를 검토하므로 텍스트보다 그래프가 이해가 훨씬 빠름.

> 출처: [블루오션 전략캔버스 활용가이드 - LeanSPRINT](https://leansprint.kr/startup-strategy-canvas-competitive-advantage-guide/)

## 5. 섹션 2 vs 섹션 3 역할 구분

| 구분 | 섹션 2: 실현가능성 (Solution) | 섹션 3: 성장전략 (Scale-up) |
|------|------|------|
| 관점 | 제품·기술 | 시장·사업 |
| 차별성 | 우리 제품이 왜 다른가, 어떻게 유지할 것인가 | 시장에서 어떻게 경쟁할 것인가 |
| 경쟁사 | 기능/기술 수준 비교 | 시장 포지셔닝, ERRC, 전략캔버스 |
| 핵심 질문 | "이걸 정말 만들 수 있어?" | "이걸 팔 수 있어?" |
| 증거 | MVP, 프로토타입, 기술 스택 | 고객 검증, 시장 규모, 수익 모델 |

## 6. 피해야 할 실수 (팀 구성 관련)

- 팀은 **개발(만드는 사람), 판매(파는 사람), 운영(관리하는 사람)** 균형 필요
- "판매는 외부 위탁" → 판매 역량 내재화 계획 필수
- 개발진만 구성된 팀은 감점 요소

## 7. 기타 효과적인 기법

- **간트차트**: 사업추진 일정을 시각적으로 표현, 심사위원이 한눈에 파악
- **PEST 프레임워크**: 문제 인식에서 거시적 근거 제시 (정치/경제/사회/기술)
- **진입 단계 vs 성장 단계 분리**: 성장전략을 두 단계로 나눠 현실성 확보
- **스크린샷/이미지 첨부**: 실제 구동 화면이 가장 강력한 실현가능성 증거
