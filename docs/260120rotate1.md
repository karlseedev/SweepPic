# 회전 시 까만 화면 문제 분석

## 문제 현상

- **증상**: 최초 가로 회전 시 화면이 까맣게 되었다가 회전된 상태가 나타남
- **재현 조건**: 앱 실행 후 첫 번째 세로→가로 회전
- **이후 동작**: 두 번째 회전부터는 정상 동작
- **발생 시점**: 5ba09e6 커밋 이전에도 발생 (회전 처리 코드와 무관한 기존 문제)

## 현재 코드 상태

### BaseGridViewController.swift - viewWillTransition

```swift
override func viewWillTransition(
    to size: CGSize,
    with coordinator: UIViewControllerTransitionCoordinator
) {
    super.viewWillTransition(to: size, with: coordinator)

    // 회전 전: 화면 중앙 셀의 indexPath 저장
    saveScrollAnchorIndexPath()

    // 회전 후 크기로 새 레이아웃 미리 생성 (size 파라미터가 회전 후 크기)
    let newLayout = createLayout(columns: currentGridColumnCount, explicitWidth: size.width)

    coordinator.animate(alongsideTransition: { [weak self] _ in
        guard let self = self else { return }

        // 회전 애니메이션과 동기화하여 레이아웃 변경
        self.collectionView.setCollectionViewLayout(newLayout, animated: false)

        // 셀 크기 캐시 업데이트
        self.updateCellSize()

        // contentInset 재계산 (FloatingUI 높이 반영)
        self.updateContentInset()

        // 저장된 indexPath로 스크롤 복원
        self.restoreScrollAnchorIndexPath()
    }, completion: nil)
}
```

## 원인 분석 (가설)

### 가설 1: prepareForReuse에서 이미지 초기화
- 회전 시 셀 재사용 → `prepareForReuse` 호출 → `imageView.image = nil`
- 새 이미지 로드 완료 전까지 까만 화면
- **검증 결과**: imageView.image = nil 주석처리해도 여전히 까맣게 됨 → 원인 아님

### 가설 2: 메모리 캐시 크기별 키 미스
- 세로 3열: 셀 130x130 → 키: "asset123_130x130"
- 가로 3열: 셀 260x260 → 키: "asset123_260x260"
- 회전 후 다른 크기로 조회 → 캐시 미스 → 비동기 로드 대기
- **검증 결과**: fallback 로직 추가해도 여전히 까맣게 됨 → 원인 아님

### 가설 3: 레이아웃 재계산 지연 (GPT 의견)
- 첫 회전 시 UICollectionViewCompositionalLayout이 새 크기 기준 레이아웃을 처음 계산
- 레이아웃 속성이 아직 준비되지 않아 visible cell 수가 0으로 떨어지는 프레임 발생
- 배경이 black이라 검게 깜빡임
- 이후 회전은 캐시된 레이아웃 경로를 재사용해서 개선
- **검증 결과**: layoutIfNeeded() 추가해도 여전히 까맣게 됨 → 원인 아님

### 시도 3: 스냅샷으로 가리기 (임시방편)

```swift
// BaseGridViewController.swift - viewWillTransition
override func viewWillTransition(
    to size: CGSize,
    with coordinator: UIViewControllerTransitionCoordinator
) {
    super.viewWillTransition(to: size, with: coordinator)

    saveScrollAnchorIndexPath()

    // 회전 전: 스냅샷으로 까만 화면 가리기
    let snapshot = collectionView.snapshotView(afterScreenUpdates: false)
    if let snapshot = snapshot {
        snapshot.frame = collectionView.frame
        view.addSubview(snapshot)
    }

    let newLayout = createLayout(columns: currentGridColumnCount, explicitWidth: size.width)

    coordinator.animate(alongsideTransition: { [weak self] _ in
        guard let self = self else { return }
        self.collectionView.setCollectionViewLayout(newLayout, animated: false)
        self.updateCellSize()
        self.updateContentInset()
        self.restoreScrollAnchorIndexPath()
    }, completion: { _ in
        // 회전 완료 후: 스냅샷 제거
        snapshot?.removeFromSuperview()
    })
}
```

### 시도 5: 메모리 캐시 fallback (다른 크기 이미지 반환)

```swift
// MemoryThumbnailCache.swift

// assetID별 캐시 키 인덱스 추가
private var assetKeyIndex: [String: Set<NSString>] = [:]
private let indexLock = NSLock()

public func get(assetID: String, pixelSize: CGSize) -> UIImage? {
    let key = cacheKey(assetID: assetID, pixelSize: pixelSize)
    var image = cache.object(forKey: key)

    // 정확한 크기 미스 시, 같은 assetID의 다른 크기 이미지로 fallback
    if image == nil {
        indexLock.lock()
        if let keys = assetKeyIndex[assetID] {
            for otherKey in keys {
                if let fallbackImage = cache.object(forKey: otherKey) {
                    image = fallbackImage
                    break
                }
            }
        }
        indexLock.unlock()
    }

    // 통계 업데이트
    // ...
    return image
}

public func set(image: UIImage, assetID: String, pixelSize: CGSize) {
    let key = cacheKey(assetID: assetID, pixelSize: pixelSize)
    let cost = Int(image.size.width * image.size.height * image.scale * image.scale * 4)
    cache.setObject(image, forKey: key, cost: cost)

    // 인덱스에 키 추가
    indexLock.lock()
    if assetKeyIndex[assetID] == nil {
        assetKeyIndex[assetID] = []
    }
    assetKeyIndex[assetID]?.insert(key)
    indexLock.unlock()
}

public func removeAll() {
    cache.removeAllObjects()

    indexLock.lock()
    assetKeyIndex.removeAll()
    indexLock.unlock()
}
```

**결과**: 실패 (여전히 까맣게 됨)

## 추가 조사 필요 사항

1. **디버그 로그로 정확한 상태 확인**
   - 회전 시작/중간/완료 시점에 로그:
     - `collectionView.numberOfItems(inSection: 0)`
     - `collectionView.visibleCells.count`
     - visible cells의 `imageView.image` 상태

2. **UICollectionView 내부 동작 분석**
   - 회전 시 셀이 재생성되는지, 재사용되는지 확인
   - `cellForItemAt`이 호출되는 시점 확인

3. **기본 사진 앱 동작 비교**
   - 기본 사진 앱은 회전 시 까맣게 되지 않음
   - 어떤 방식으로 처리하는지 분석
