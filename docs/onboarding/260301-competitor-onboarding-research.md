### 1-1. Slidebox (스와이프 사진 정리)

**온보딩 요소: 1개**

| # | 요소 | 유형 | 설명 |
|---|------|------|------|
| 1 | ? 버튼 (on-demand 도움말) | 상시 접근 | Unsorted 탭 좌측 상단. 탭하면 제스처 동작 안내 표시 |

**온보딩 플로우:**
- 첫 실행 → 시스템 권한 다이얼로그 → 즉시 Unsorted 탭 진입
- 웰컴 화면, 기능 소개 슬라이드, 코치마크 **없음**
- 디자인 철학: "learn by doing" (해보면서 배우기)

**제스처 체계 (별도 교육 없음):**
- 위로 스와이프 = 삭제(Trash)
- 아래로 스와이프 = 즐겨찾기
- 좌/우 스와이프 = 다음/이전 사진
- 앨범 탭 = 앨범 분류
- 길게 누르기 = 공유

**동기 부여:**
- 총 사진 수 + 앨범 정리 비율(%) 표시
- 별도 게이미피케이션 없음

**안전 장치:**
- 앱 내부 Trash에 보관 → 일괄 삭제 가능
- iOS 최근 삭제 항목에서 30일 복구

**사용자 피드백:**
- "세부 기능을 다루는 빠른 트레이닝 하이라이트가 있었으면 좋겠다"

**출처:** slidebox.co, App Store, TapSmart, TechCrunch, appPicker

| — | 없음 | — | 공식 온보딩/코치마크/튜토리얼 없음 |

**온보딩 플로우:**
- 첫 실행 → 시스템 권한 다이얼로그 → 즉시 카드 스택 표시
- 단 하나의 화면, 단 하나의 동작(스와이프)

**제스처 체계 (별도 교육 없음):**
- 왼쪽 스와이프 = 삭제(Trash)
- 오른쪽 스와이프 = 유지(Keep)
- Keep/Trash 버튼도 병행 제공 (가시적 대안)

**동기 부여: 3개**
- 삭제할 때마다 **실시간 저장 공간 절약량** 표시
- **누적 절약 용량 카운터** 지속 추적
- **월별 정리 알림** (Push Notification)

**안전 장치:**
- 2단계: 스와이프 → 앱 내 Trash → Empty Trash로 최종 삭제
- Undo 즉시 가능

**특이 사항:**
- 유머러스한 맥락 메시지: "Does this really look like a Kodak moment to you??"
- 무료 버전 일일 100장 제한
- 일부 리뷰에서 좌/우 방향 혼란 보고

**출처:** Macworld, AppleInsider, PetaPixel, TechCrunch, TIME

### 1-5. Gemini Photos → CleanMyPhone (MacPaw, AI 정리)

**온보딩 요소: Gemini 0개 / CleanMyPhone 1개**

| # | 요소 | 유형 | 앱 | 설명 |
|---|------|------|----|------|
| 1 | 기능 워크스루 | 캐러셀 | CleanMyPhone | 화려한 애니메이션 배경 + 주요 기능 소개 슬라이드 |

**온보딩 플로우:**
- Gemini: 권한 → 즉시 스캔 → Your Library 화면 (UI 자체가 기능 설명)
- CleanMyPhone: 워크스루 → 권한 → 3일 무료 체험 → 즉시 스캔

**AI 기능 설명 방식 (코치마크 없이 UI로 전달):**
- 카드 UI로 카테고리 표시: Similar, Blurred, Notes, Screenshots
- "Best Result" 배지: 유사 사진 그룹에서 AI가 최적 사진 자동 선정
- 나머지 사진은 자동 사전 선택(preselect)

**스캔 애니메이션:**
- **버블(Bubble) 애니메이션**: 발견 항목이 버블로 실시간 표시 (프로그레스 바 대신)
- 약 65초 스캔 시간이 더 짧게 느껴지는 효과

**프라이버시: 2개**
- "사진을 업로드하거나 저장하지 않습니다 — 모든 정리 마법은 기기에서 로컬로 수행"
- "인터넷 연결 없이도 분석 가능"
- 앱 슬로건: **"Careful iPhone cleaner"** ("조심스러운"이라는 단어로 삭제 불안 해소)

**동기 부여:**
- 메인 화면 하단 "Unneeded Photos" 총 개수 + 카테고리별 내역
- "Clean" 버튼의 만족감 ("the satisfying Clean button")
- 스캔 리마인더 알림

**약점:**
- 총 회수 가능 용량을 한 곳에서 보여주지 않음 → "분절되고 혼란스러움"

**출처:** MacPaw, MacStories, 9to5Mac, TechCrunch, InsanelyMac, UX Design Award 2024

**온보딩 플로우:**
- 첫 실행 → 권한 → **즉시 사진 라이브러리 그리드** (임포트 불필요)
- "명백한 디자인(Obvious Design)" 철학: "가장 명백한 구현이 무엇인가?"

**핵심 디자인 원칙:**
- "액션당 탭 수"를 경쟁 앱과 비교 측정하여 최적화
- "쉽게 만들거나, 숨기거나": 고급 기능은 쉽게 만들고, 불가능하면 명시적 액션 뒤에 숨김
- 디바이스별 적응형 UI (iPad vs iPhone)

**수익 모델 — "Try Before Buy":**
- 프리미엄 기능을 모두 사용해볼 수 있으나 **내보내기(Export) 시에만** 결제 필요
- 구독을 강제하지 않는 점이 긍정 평가

**출처:** Apple Developer ("Behind the Design"), PetaPixel, RevenueCat, MacRumors

## 3. UX 리서치 핵심 인사이트

### 3-1. NNGroup 코치마크 연구 결과

- **튜토리얼은 사용자를 더 빠르게 만들지 않음**
- **오히려 앱을 더 어렵게 인식하게 함**: 튜토리얼 본 그룹 편의성 4.92 vs 건너뛴 그룹 5.49
- **단기 기억 한계**: 힌트를 읽으면서 동시에 사용 불가, 20초 내 기억 소실
- **한 번에 3-4개만 기억 가능**
- **효과적 코치마크**: 단일 상호작용에 집중, 컨텍스트에 맞게 하나씩, 과도한 사용 금지
- **전략적 구현 시 기능 채택률 40-60% 향상** (Netflix, Flipkart 사례)

### 3-2. 컨텍스트 온보딩 vs 프론트로딩 튜토리얼

NNGroup 핵심 결론:
> "꽤 복잡한 모바일 앱에서도, 튜토리얼보다 **컨텍스트에서 팁을 보여주는 것**이 더 효과적이다."

프론트로딩 문제점:
- 사용자를 방해함
- 작업 수행 능력 향상시키지 않음
- 빠르게 잊혀짐
- 인터페이스가 실제보다 더 복잡해 보이게 만듦

### 3-3. 스와이프 제스처 교육 베스트 프랙티스

**Tinder의 핵심 메커니즘:**
- 스와이프 시작 시점부터 방향별 **컬러 오버레이** (녹색/빨간색, opacity 0.5)
- **스탬프 라벨**: "LIKE"/"NOPE"가 스와이프에 따라 투명도 변화로 나타남
- **카드 회전 + 투명도 변화**: 중앙=불투명, 양쪽으로 갈수록 투명

**일반 가이드라인:**
- 제스처가 유일한 방법이 되면 안 됨 → 항상 가시적 대안(버튼) 제공
- 앱 전체에서 동일 제스처 = 동일 동작 일관성
- 스와이프 동작 중 시각적 피드백 필수
- 콘텐츠 엿보기(Content Tease): 옆으로 살짝 보이는 다음 콘텐츠

### 3-4. 프로그레시브 디스클로저

- 인터페이스 탐색하면서 점진적으로 복잡한 기능 노출
- 온보딩 시 고급/자주 사용하지 않는 기능 노출하지 않음
- "앞에서 모든 세부사항을 표시하는 대신, 필요한 것만 제공"

### 3-5. 빈 상태(Empty State) 활용

- 빈 상태는 디자인 관점에서 **절대 비어있으면 안 됨**
- 행동 유도 CTA 포함 필수
- 콘텐츠가 채워졌을 때의 모습을 보여주거나 명확한 지시사항 전달
- 사용자는 **첫 3-7일 내** 앱 사용 여부 결정

### 3-6. 파괴적 동작의 신뢰 구축

핵심 인용:
> "사람들이 실수로부터 복구할 수 있다는 것을 알 때, 더 많이 탐색하고, 더 빠르게 클릭하며, 목표에 집중한다."

단계별 안전 설계:
| 영향 수준 | 패턴 | 예시 |
|----------|------|------|
| 낮음 (되돌림 가능) | 스낵바 + Undo | 이메일 삭제 후 "실행 취소" |
| 중간 (소프트 삭제) | 휴지통/삭제대기함 | Notion: 수일~수주 보관 |
| 높음 (되돌림 불가) | 명시적 확인 단계 | 확인 다이얼로그 |

### 3-7. 게이미피케이션 효과 (Duolingo 사례)

- 가치를 먼저 체험하게 하고, 가입은 나중에 → 리텐션 급증
- 7일 스트릭 유지 사용자는 장기 참여 가능성 **3.6배**
- 게이미피케이션된 온보딩 → **월간 활성 사용자 62% 증가**
- 스트릭+마일스톤 이중 시스템 → **30일 이탈률 35% 감소**

### 3-8. Halide의 Intelligent Activation (주목할 패턴)

Halide만의 독특한 패턴으로 별도 정리:
- 사용자 액션에 반응하여 UI가 나타나고 사라짐
- 모든 토글 액션에 짧은 설명 레이블 → **확인 + 교육 동시 수행**
- "방해하지 않기" 철학
