## 1. 방향

**앱 열면 바로 사진. 쓰다가 필요할 때 알려준다.**

- 첫 실행 Welcome/소개 화면: 없음
- 권한 Priming 화면: 없음 (기존 PermissionVC 그대로)
- 온보딩 = 컨텍스트 코치마크 5개 (4가지 특장점 커버) + 삭제 시스템 안내 3개

### 설계 원칙

- **[확인] 필수**: 모든 온보딩 요소는 [확인] 버튼을 눌러야만 닫힘. 배경 탭이나 스와이프로 dismiss 불가
- **점진적 전달**: 핵심 개념("삭제대기함")을 코치마크 → 행동 피드백 → 현장 확인 순으로 형태를 바꿔가며 반복 전달
- **시각적 강조**: 카피 전체를 안 읽어도 핵심 키워드(볼드/강조 컬러)가 1초 안에 눈에 들어오도록 설계

## 2. 코치마크 A — 그리드 스와이프 삭제

```
트리거: 메인 그리드 첫 진입 후 2초 경과 시
위치: 화면 중앙부 셀 위 오버레이

   ← [손가락 애니메이션: 셀을 옆으로 스와이프]

   "사진을 밀어서 바로 정리하세요
    다시 밀면 복원돼요
    정리한 사진은 삭제대기함으로 이동돼요"

          [확인]
```

- 사용자가 가장 먼저 마주치는 화면 = 그리드
- 실제 스와이프 시도 시 자동 dismiss
- 핵심 전달: 열지 않고 정리 + 복원 가능 + "삭제대기함" 개념 최초 노출

## 3. 코치마크 B — 뷰어 스와이프 삭제

```
트리거: 뷰어에서 사진을 처음 열었을 때
위치: 화면 하단부

   ↑ [손가락 애니메이션: 위로 스와이프]

   "위로 밀면 바로
    삭제대기함으로 이동해요"

          [해볼게요]
```

- 그리드 삭제와 다른 방식(위로 스와이프)임을 명확히
- "삭제대기함" 개념 재확인 (코치마크 A에서 최초 노출된 단어)
- 실제 스와이프 성공 시 자동 dismiss

## 4. 코치마크 C — 유사 사진·얼굴 비교 (2단계)

### C-1: 그리드 뱃지

```
트리거: 유사 사진 뱃지가 1초 이상 안정 표시될 때
위치: 해당 셀 하이라이트

   [뱃지+테두리 하이라이트 + 말풍선]

   "비슷한 사진을 자동으로 찾았어요
    탭해서 비교해보세요"

          [확인]
```

### C-2: 얼굴 + 버튼

```
트리거: 뷰어에서 얼굴 + 버튼이 처음 표시될 때
위치: + 버튼 하이라이트

   [+ 버튼 하이라이트 + 말풍선]

   "얼굴별로 비교할 수 있어요
    인물이 여러 명이면 좌우로 넘겨 전환"

          [확인]
```

- C-1, C-2는 각각 독립 플래그로 관리 (순서 의존성 없음, 어느 쪽이 먼저 트리거되어도 무방)
- C-2 하이라이트 대상: 얼굴 위의 +버튼 (iOS 버전 무관하게 동일 위치)

## 5. 코치마크 D — 저품질 자동 정리

**상세: `docs/onboarding/260211onboarding-D.md`**

### 트리거 1 (자동)

```
조건: A 완료 + E-1 완료 + 그리드 3초 체류 + 사전 스캔 결과 1장 이상
위치: 정리 버튼 하이라이트 (evenOdd 구멍)

   [정리 버튼 하이라이트 + 저품질 썸네일 3장]

   "보관함에서 저품질 사진이 발견됐어요"
   "흔들리거나 초점이 맞지 않은 사진들을
    AI가 자동으로 찾아주는 정리 기능을 사용해보세요"

          [확인] → 탭 모션 → 정리 플로우 자동 진입
```

### 트리거 2 (수동)

```
조건: D 미완료 + 정리 버튼 탭 + 사전 스캔 결과 1장 이상
위치: 전체 딤 (하이라이트 없음)

   [저품질 썸네일 3장]

   "보관함에서 저품질 사진이 발견됐어요"
   "흔들리거나 초점이 맞지 않은 사진들을
    AI가 자동으로 찾아주는 정리 기능입니다"

          [확인] → 가로챈 정리 플로우 이어서 진행
```

- 사전 스캔: 앱 시작 직후 백그라운드에서 사진 분석 (T2 파이프라인)
- 스캔 0건이면 두 트리거 모두 D 표시 안 함
- A/E-1 완료 전이라도 스캔은 독립적으로 진행

## 6. 삭제 시스템 안내 (E-1+E-2 통합 시퀀스, E-3)

코치마크 A~D는 기능 안내이고, E는 **삭제 행동의 결과 피드백**입니다.
사용자가 행동한 직후에 표시되므로 맥락이 명확하고, 코치마크를 안 읽었더라도 이 단계에서 삭제 시스템을 이해하게 됩니다.

### E-1+E-2: 삭제 시스템 안내 (연속 시퀀스)

E-1과 E-2는 **하나의 연속 시퀀스**로 실행된다. 단일 오버레이가 시작부터 끝까지 유지되며, 사용자가 중간에 이탈할 수 없다.

```
트리거: 생애 첫 moveToTrash() 호출 직후 (그리드/뷰어 무관, 1회)

[Step 1] 딤 + 텍스트 + 탭바 손가락 모션 + [확인]
   "방금 삭제한 사진은 삭제대기함으로 이동됐어요.
    아래 탭을 눌러 삭제대기함으로 가볼까요?"
   (탭바 삭제대기함 탭을 가리키는 손가락 모션)
          [확인]

       ── [확인] 탭 ──
   오버레이 유지, 탭 전환 (삭제대기함)

[Step 2] (+0.3s) 텍스트 전환
   "보관함에서 삭제하면 여기에 임시 보관돼요."

[Step 3] (+1.3s) 비우기 버튼 하이라이트 + 텍스트 추가 + [확인] 나타남
   "[비우기]를 누르면 사진이 최종 삭제돼요."
          [확인]

       ── [확인] 탭 ──
   오버레이 dismiss
```

- 오버레이가 처음부터 끝까지 유지 → 입력 차단 갭 없음
- [확인] 2번으로 전체 시퀀스 완료
- Step 1~3 동안 [확인] 버튼 외 모든 입력 차단

### E-3: 첫 비우기 완료 안내

```
트리거: emptyTrash() 또는 permanentlyDelete() 첫 실행 완료 직후 (1회)
형태: 중앙 팝업 (CoachMarkOverlay 공용)

   "✓ 삭제 완료

    애플 사진앱의 '최근 삭제된 항목'에서
    30일 후 완전히 삭제됩니다."

          [확인]
```

- 비우기 완료 직후 표시 → "왜 용량 안 줄어?" 선제 대응
- 이후 비우기에서는 표시하지 않음 (1회)

## 8. 표시 순서와 충돌 방지

동시에 여러 온보딩 요소가 뜨면 안 되므로:

```
첫 세션:
  그리드 진입 → [A: 그리드 스와이프] (2초 후)
  사진 탭 → [B: 뷰어 스와이프] (즉시)
  첫 삭제 실행 → [E-1: 첫 삭제 안내] (즉시)
  [확인] 탭 → 삭제대기함 이동 → [E-2: 삭제대기함 안내] (즉시)
  뱃지 안정 → [C-1: 유사 뱃지] (1초 유지 후)
  + 버튼 표시 → [C-2: 얼굴 비교] (즉시)

첫 비우기:
  비우기 완료 → [E-3: 비우기 완료 안내] (즉시)

A+E-1 완료 후:
  그리드 3초 체류 → [D: 자동 정리] (스캔 결과 있을 때, 1회)
  또는 정리 버튼 탭 → [D: 자동 정리] (D 미완료 + 스캔 결과 있을 때)
```

규칙: 온보딩 요소는 한 번에 하나만 표시. `CoachMarkOverlay`를 싱글톤 + 내부 큐로 설계하여 전역 관리. 현재 것이 닫힌 후 다음 표시.

### 구현 점검 결과

#### ✅ 정상 구현 항목

| 항목 | 구현 방식 | 비고 |
|------|----------|------|
| 싱글톤 구조 | `CoachMarkManager.shared` + `isShowing` (weak 참조) | 오버레이 deinit 시 자동 nil |
| 동시 표시 방지 | 모든 트리거에 `!isShowing` 가드 존재 | A, B, C, D, E 전부 |
| E-1→E-2 시퀀스 보호 | `isDeleteGuideSequenceActive` 플래그 | 중간 dismiss 차단 |
| C-1→C-2 전환 보호 | `isWaitingForC2` 플래그 | B 끼어들기 차단 |
| C-3 Step 전환 보호 | `isC3TransitionActive` 플래그 | 전환 중 dismiss 차단 |
| C-1 선행조건 | A + B + E-1 완료 후에만 표시 | 3가지 `hasBeenShown` 가드 |
| D 선행조건 | A + E-1 완료 후에만 표시 | 2가지 `hasBeenShown` 가드 |
| D 재시도 | `isShowing` 시 0.5초 간격 재시도 (최대 40회) | 다른 코치마크 완료 후 표시 |
| B의 C-2 이중 방어 | `!isWaitingForC2` + `!isShowing` 순차 가드 | C-2 대기 중 B 차단 |

#### 📝 의도된 설계 (미비가 아님)

1. **B는 A 선행조건 없음 (의도됨)**: 사용자가 스크롤 없이 바로 사진을 탭하면 B가 A보다 먼저 표시될 수 있음. 위 순서는 "일반적인 첫 세션 흐름"이며, 엄격한 순서 강제가 아님. 뷰어 진입 맥락에서 즉시 안내하는 것이 UX상 자연스럽기 때문.

2. **E-1은 그리드 스와이프 삭제에서만 트리거 (의도됨)**: 뷰어 삭제(위로 스와이프)에서는 E-1을 트리거하지 않음. 뷰어 삭제 직후에는 뷰어 화면에 머물러 있어 E-1의 삭제대기함 탭 전환 동선이 부자연스러우므로, 그리드 스와이프 삭제 시에만 E-1+E-2 시퀀스를 시작함.

3. **D의 `isDeleteGuideSequenceActive` 별도 체크 없음 (의도됨)**: D 재시도 중 E 시퀀스가 진행 중이면 `isShowing` 가드로 이미 차단됨. E 시퀀스 중에는 `isDeleteGuideSequenceActive=true`로 오버레이 dismiss가 차단되어 `isShowing`이 계속 true를 유지하므로, D가 끼어들 틈이 없음. D 재시도는 최대 40회(약 95초) 후 포기하지만, 다음 viewDidAppear에서 새 사이클이 시작되므로 영구 누락 없음. E-1+E-2 시퀀스 자체가 사용자 [확인] 탭 몇 번이면 끝나므로 95초를 넘기는 것은 현실적으로 불가능.

| D | "AI가 자동으로 찾아주는 정리 기능" + 썸네일 3장 | 실제 썸네일로 시각화 | 완벽 |

| E-2 | "삭제대기함에 임시 보관" + "[비우기]를 누르면 최종 삭제" | — | 완벽 |
| E-3 | "✓ 삭제 완료" | "30일 후 완전히 삭제됩니다" | 완벽 |

#### 📝 의도된 설계

1. **B 안심 요소 미포함**: 카피에 "복구 가능" 문구 없음. B는 뷰어 맥락에서 간결한 행동 안내에 집중. 복구 개념은 A("다시 밀면 복원")와 E-1("삭제대기함으로 이동")에서 전달하므로 B에서 중복하지 않음.
2. **E-1 안심 톤**: "안전"이라는 직접 표현 대신 "삭제대기함으로 이동됐어요"로 즉시 삭제가 아님을 암시. 이어지는 E-2에서 "임시 보관"을 명시하여 단계적으로 안심감 전달.

## 11. 트리거 가드 조건

모든 온보딩 요소(코치마크 + 시스템 피드백)는 아래 조건을 만족해야 표시:

| 가드 | 설명 |
|------|------|
| `!UIAccessibility.isVoiceOverRunning` | VoiceOver 시 제스처/유사사진 기능 비활성화이므로 코치마크도 표시하지 않음 |
| 현재 표시 중인 온보딩 없음 | `CoachMarkOverlay.shared`의 큐로 관리 |
| `targetVC.view.window != nil` | 표시 직전 해당 화면이 여전히 활성 상태인지 재검증 |

화면별 추가 가드:

| 화면 | 가드 | 대상 | 설명 |
|------|------|------|------|
| Grid | `!isScrolling` | A, C-1, D | 스크롤 중에는 표시하지 않음 |
| Viewer | `!isTransitioning && !isDismissing` | B, C-2 | 페이지 전환 중이거나 닫기 제스처 중에는 표시하지 않음 |
| E-1 | 추가 가드 없음 | E-1 | 삭제 직후 즉시 표시 (행동 피드백) |
| E-2 | `trashedAssetIDs.count > 0` | E-2 | 삭제대기함에 아이템 존재 시 |
| E-3 | 추가 가드 없음 | E-3 | 비우기 완료 직후 즉시 표시 (행동 피드백) |

## 12. 재방문 & 스킵 정책

| 항목 | 정책 |
|------|------|
| 코치마크 각각 | 개별 UserDefaults 플래그 — 1회만 |
| 시스템 피드백 각각 | 개별 UserDefaults 플래그 — 1회만 |
| 닫기 방식 | [확인]/[해볼게요] 버튼만 가능. 배경 탭/스와이프 dismiss 불가 |
| 자동 dismiss | 코치마크 A, B는 실제 제스처 수행 시에도 자동 dismiss |
| 앱 업데이트 | 새 기능 추가 시 해당 플래그만 리셋 |

### 구현 점검 결과

| # | 규칙 | 상태 | 비고 |
|---|------|------|------|
| 1 | 코치마크 개별 플래그 | ✅ | CoachMarkType enum 7개 키, `hasBeenShown`/`markAsShown`/`resetShown` |
| 2 | 시스템 피드백 개별 플래그 | ✅ | E-1(`firstDeleteGuide`), E-3(`firstEmpty`) 각각 독립 관리 |
| 3 | [확인] 버튼만 닫힘 | ✅ | hitTest에서 배경 탭 흡수, 스와이프 제스처 없음 |
| 4 | A, B 자동 dismiss | — | 설계상 해당 없음 (아래 참고) |
| 5 | 앱 업데이트 플래그 리셋 | — | 현재 불필요 (아래 참고) |

#### 📝 의도된 설계

1. **4번 "자동 dismiss" 해당 없음**: A, B 코치마크 표시 중에는 오버레이가 전체 화면 터치를 흡수(`hitTest → return self`)하므로, 사용자가 실제 스와이프 제스처를 수행할 수 없음. [확인] 버튼만 누를 수 있는 구조이므로 "실제 제스처 수행 시 자동 dismiss" 시나리오 자체가 발생하지 않음.

2. **5번 "앱 업데이트 리셋" 현재 불필요**: 새 코치마크 추가 시 UserDefaults에 해당 키가 없으므로 `hasBeenShown`이 자동으로 false — 기존 사용자도 새 코치마크를 자동으로 보게 됨. 리셋 메커니즘은 기존 코치마크 내용을 크게 변경하여 재표시할 때 필요하며, 현재 해당 사항 없음.

## 14. 전체 플로우

```
앱 실행 → PermissionVC (기존 그대로) → 메인 그리드
                                         │
                                    [A: 그리드 스와이프] (1화면 스크롤 후, 1회)
                                         │
                                    [A-1: 스와이프 실습] (A 완료 후 3초 내 미실행 시)
                                         │
                                    사진 탭 → 뷰어
                                         │
                                    [B: 뷰어 스와이프] (1회)
                                         │
                                    첫 그리드 스와이프 삭제 실행
                                         │
                                    [E-1: 첫 삭제 안내] (1회)
                                         │
                                    [확인] → 삭제대기함 탭 전환
                                         │
                                    [E-2: 삭제대기함 안내] (1회)
                                         │
                                    ← 그리드 복귀

                                    그리드에서 뱃지 안정 노출
                                         │
                                    [C-1: 유사 뱃지] (A+B+E-1 완료 후, 1회)
                                         │
                                    [확인] → 뷰어 자동 이동
                                         │
                                    [C-2: 얼굴 비교] (+버튼 폴링 후)
                                         │
                                    [확인] → 얼굴 비교 화면 자동 진입
                                         │
                                    [C-3: 셀 선택 + Pic 라벨] (Step 1→2)

첫 비우기:
  비우기 완료 → [E-3: 비우기 완료 안내] (1회)

A+E-1 완료 후:
  그리드 3초 체류 → [D: 자동 정리] (스캔 결과 있을 때, 1회)
  또는 정리 버튼 탭 → [D: 자동 정리] (D 미완료 + 스캔 결과 있을 때)
```
