# 코치마크 D — 사전 스캔 벤치마크 결과

> 작성일: 2026-02-17

## 목적

코치마크 D에서 사용자에게 보여줄 저품질 사진 3장을 사전에 스캔하는 최적 파이프라인을 결정하기 위한 벤치마크.

---

## 테스트 설계

### 3종 파이프라인 비교

| 테스트 | 구성 | 설명 |
|--------|------|------|
| **T1 (Full)** | MetadataFilter → Exposure → SKIP필터 → Blur → SafeGuard | QualityAnalyzer 전체 파이프라인 |
| **T2 (SKIP필터만)** | MetadataFilter → Exposure → SKIP필터 → Blur | SafeGuard 제외 |
| **T3 (최경량)** | MetadataFilter → Exposure → Blur | SKIP필터도 SafeGuard도 없음 |

### 비교 의미

- **T1 - T2** = SafeGuard 비용 (Vision 얼굴 감지)
- **T2 - T3** = SKIP필터 비용 (유틸리티/화이트배경/텍스트 감지)

### SKIP필터 내용 (QualityAnalyzer 내부)

| 필터 | 조건 | 판정 |
|------|------|------|
| `isUtilityImage()` | 극단 휘도 + 낮은 RGB표준편차 | SKIP (단색 배경 메모/문서) |
| `detectTextScreenshot()` | 극단 노출 + 텍스트 블록 다수 | SKIP (블로그 캡처 등) |
| `isWhiteBackgroundImage()` | 극밝음 + 모서리 순백 + 모서리>중앙 | SKIP (일러스트/상품 사진) |

### SafeGuard 내용 (SafeGuardChecker)

| 체크 | 조건 | 효과 |
|------|------|------|
| depthEffect | PHAsset에 포트레이트 모드 표시 | 블러 신호 무효화 |
| clearFace | Vision 얼굴 품질 >= 0.4 | 블러 신호 무효화 |

- 블러 신호가 있을 때만 실행됨
- 메타데이터 체크(depthEffect) → 빠름, 얼굴 품질 체크(Vision) → 느림

### 테스트 조건

- 각 1000장, 이미지만 (비디오 미포함)
- 순차 처리 (병렬 없음)
- Precision 모드 (Strong 신호만 저품질 판정)
- 이미지 로딩: CleanupImageLoader (짧은변 360px)

→ 세 방식 모두 동일한 결과, 시간도 거의 동일

저품질 신호: severeBlur 4장, extremeDark 1장

| | T1 (Full) | T2 (SKIP필터만) | T3 (최경량) |
|---|---|---|---|
| 총 시간 | 21.43초 | 17.80초 | 16.38초 |
| 평균/장 | 23.3ms | 22.0ms | 22.3ms |
| 저품질 | 48장 | 48장 | 55장 |

| SafeGuard 적용 | 0장 | - | - |

```
SafeGuard 비용 (T1-T2): +3.62초, 구제 0장
SKIP필터 비용 (T2-T3): +1.43초, 구제 7장 (모두 extremeBright)
→ SafeGuard는 비용만 크고 효과 없음
→ SKIP필터는 흰 배경 이미지 오판 7장 방지
```

SKIP필터가 구제한 7장: 모두 `extremeBright` → 실제로는 흰 배경/유틸리티 이미지

## 분석

### SafeGuard

| | 앨범 A | 앨범 B |
|---|---|---|
| 시간 비용 | +0.18초 | +3.62초 |
| 구제한 사진 | 0장 | 0장 |

- 두 앨범 모두 SafeGuard가 구제한 사진 **0장**
- 앨범 B에서 3.62초 추가 비용 (블러 판정 48장에 대해 Vision 얼굴 감지 실행)
- **결론: 사전 스캔에서는 SafeGuard 제외**

### SKIP필터

| | 앨범 A | 앨범 B |
|---|---|---|
| 시간 비용 | -0.05초 (오차) | +1.43초 |
| 구제한 사진 | 0장 | 7장 |

- 깨끗한 앨범에서는 영향 없음
- 다양한 앨범에서는 1.43초로 7장 오판 방지
- 코치마크 D에서 흰 배경 이미지를 "저품질"로 보여주면 신뢰도 하락
- **결론: SKIP필터 포함 필수**

### 처리 시간 병목

- 평균 14~23ms/장 중 분석 자체는 ~1ms 수준
- **대부분의 시간은 이미지 로딩** (PHAsset → CGImage 변환)
- 분석 방식 변경보다 이미지 로딩 최적화 또는 병렬 처리가 효과적

## 최적 파이프라인: T2

```
MetadataFilter → Exposure → SKIP필터 → Blur (SafeGuard 제외)
```

- T1 대비 SafeGuard 생략으로 0.18~3.62초 절약
- 판정 결과는 T1과 동일 (SafeGuard 구제 0건이므로)
- SKIP필터로 오판 방지

---

## 미해결 과제

### 1. 속도

- 1000장 순차: 14~18초
- 코치마크 D 트리거까지 약 10~15초 (앱 시작 → 스크롤 → A → A dismiss)
- 깨끗한 앨범(0.5% 비율)에서 3장 확보에 ~600장 = ~8초 필요
- **병렬 처리(4개 동시)로 시간을 1/3~1/4로 줄일 수 있음**

### 2. 수량 부족 가능성

- 0.5% 비율 앨범에서 1000장 스캔해도 5장
- 더 깨끗한 앨범이면 1000장에서 3장 미달 가능
- **대책**: 3장 미달 시 발견된 만큼 표시 / 0건이면 텍스트 폴백

### 3. 비디오 미포함

- 현재 이미지만 스캔
- 1초 미만 비디오는 메타데이터만으로 즉시 lowQuality 판정 (이미지 로드 불필요, ~0ms)
- 비디오 포함 시 빠른 수량 확보에 도움될 수 있음
- **단, 코치마크 D 썸네일에 비디오 프레임을 보여줄지는 UX 판단 필요**

### 4. 속도 개선 후보

| 방안 | 예상 효과 | 비고 |
|------|-----------|------|
| 병렬 처리 (4개 동시) | 시간 1/3~1/4 | `analyzeBatch` 패턴 활용 가능, 3장 조기 종료 필요 |
| 2단계 접근 | Exposure 선행 (64x64, 초경량) → Blur 후행 | 극단 노출을 먼저 빠르게 확보 |
| 이미지 해상도 축소 | 360px → 256px 등 | 로딩 시간 단축, 분석 정확도 영향 검토 필요 |
| 비디오 포함 | 1초 미만 비디오 즉시 판정 | 이미지 로드 불필요 |
