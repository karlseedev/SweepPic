# D-1 코치마크 구현 계획: 자동정리 그리드 3단계 안내

## Context

자동정리 그리드(PreviewGridViewController)에 처음 진입한 사용자에게 3단계 품질 등급 시스템과 화면 사용법을 안내하는 4단계 포커싱 온보딩(D-1)을 추가한다. 기존 코치마크 C/D/E의 멀티스텝 포커싱 패턴을 재사용한다.

**핵심 요구사항**: 화면이 완전히 뜨고 나서, 다른 클릭을 차단한 후 온보딩이 표시되어야 한다.

---

## 4단계 시퀀스 요약

| Step | 포커싱 대상 | Shape | 구멍 margin | 텍스트 위치 |
|------|-----------|-------|-----------|-----------|
| 1 | 헤더 타이틀 | pill | 8pt | 포커싱 아래 |
| 2 | 더보기/제외 버튼 (secondaryStack) | pill | 8pt | 포커싱 위 |
| 3 | 중앙 셀 1개 + 스와이프 모션 | rect | 0pt (C-1/A 패턴) | C-2식 자동 판단 |
| 4 | 삭제대기함 이동 버튼 (primaryButton) | pill | 8pt | 포커싱 위 |

## 포커싱 전환 방식: 확대→축소 패턴 (C-2 검증 패턴)

기존 검증된 패턴만 사용한다. 구멍 직접 이동(미검증)은 사용하지 않는다.

**Step 1 진입 (D의 animateDFocus 패턴):**
```
t=0.00: overlay.alpha = 0.01 (터치 즉시 차단)
t=0.00: dimLayer = 화면 전체 크기 pill (딤 없음 상태)
t=0.00: window.addSubview(overlay)
t=0.00: UIView.animate alpha → 1.0 (0.3s) + animateDFocus pill 축소 (0.9s) 동시
t=0.90: 포커싱 완료 → 0.5s 대기
t=1.40: 텍스트 + 확인 버튼 페이드인 (0.3s)
t=1.70: confirmButton.isEnabled = true
```

**Step N → Step N+1 전환 (확대→축소):**
```
t=0.00: isD1SequenceActive = true
t=0.00: 텍스트 + 확인 버튼 페이드아웃 (0.2s)
t=0.20: animateD1Expand — 현재 구멍 확대 → 화면 전체 (0.3s)
t=0.50: dimLayer path를 다음 위치의 큰 구멍으로 즉시 교체
t=0.50: animateD1Shrink — 큰 구멍 → 타겟 크기로 축소 (0.7s)
t=1.20: 0.5s 대기 (C-2와 동일)
t=1.70: 텍스트 + 확인 버튼 페이드인 (0.3s)
t=2.00: d1CurrentStep 업데이트, isD1SequenceActive=false, confirmButton.isEnabled=true
```

**Step 2→3 추가 처리**: 축소 완료 후 스냅샷+녹색딤드 배치(alpha 0) → 스냅샷 페이드인(0.2s) → 텍스트 페이드인 → 스와이프 루프 시작
**Step 3→4 추가 처리**: 페이드아웃 시 스냅샷+녹색딤드+fingerView도 함께, `d1SwipeLoopActive=false`로 루프 중단 (shouldStopAnimation은 dismiss 전용으로 사용하지 않음 — 전환 체인 중단 방지)

**Reduce Motion 대응:**
- 포커싱 축소 애니메이션 생략 → dimLayer path 즉시 교체
- alpha 즉시 1.0으로 복원 (C-3 패턴)
- Step 3 스와이프 모션: 녹색 딤드 55% + 화살표 아이콘 (A 패턴)

    }
}
