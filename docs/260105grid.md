# 보관함/앨범 뷰어 휴지통 사진 표시 일관성 수정

**상태**: 적용 완료

---

## 문제
- **그리드**: 휴지통 사진이 표시됨 (마룬 딤드 오버레이)
- **뷰어**: 휴지통 사진이 제외됨 (스와이프로 건너뛰어짐)
- **논리적 불일치**: 그리드에서 보이면 뷰어에서도 보여야 함

## 해결 방안
- 보관함/앨범 뷰어에서도 휴지통 사진 표시
- ~~휴지통 사진은 **마룬(#800000) 10pt 두께 테두리**로 표시~~
- **실제 구현**: 휴지통 사진은 **마룬 배경색**으로 표시 (테두리 대신)
- 휴지통 사진에서는 **복구 버튼만** 표시 (삭제 버튼 대신)
- 휴지통 탭 뷰어에서는 마룬 배경 표시 안 함

---

### Phase 5: ViewerViewController 수정
**파일**: `PickPhoto/Features/Viewer/ViewerViewController.swift`

#### 5.1 테두리 표시 조건 (`createPageViewController`)
```swift
// 보관함(.normal)에서만 테두리 표시, 휴지통 탭에서는 표시 안 함
let showBorder = (viewerMode == .normal) && coordinator.isTrashed(at: index)
return PhotoPageViewController(asset: asset, index: index, showTrashedBorder: showBorder)
```

#### 5.2 iOS 16~25: 커스텀 버튼 동적 교체
- `.normal` 모드: 삭제 버튼 + 복구 버튼 둘 다 생성
- **복구 버튼도 중앙 배치** (삭제 버튼과 동일 위치)
- `isHidden`으로 토글
```swift
// 복구 버튼 제약 (중앙 배치 - 삭제 버튼과 동일)
restoreButton.centerXAnchor.constraint(equalTo: view.centerXAnchor),
restoreButton.bottomAnchor.constraint(equalTo: view.safeAreaLayoutGuide.bottomAnchor, constant: -Self.deleteButtonBottomMargin)
```

#### 5.3 iOS 26+: 시스템 툴바 동적 교체
```swift
@available(iOS 26.0, *)
private func updateToolbarItemsForCurrentPhoto() {
    guard viewerMode == .normal else { return }

    // nil guard: setupSystemUIIfNeeded() 이전 호출 방지
    guard toolbarDeleteItem != nil else { return }

    let isTrashed = coordinator.isTrashed(at: currentIndex)
    let flexSpace = UIBarButtonItem(systemItem: .flexibleSpace)

    if isTrashed {
        // 복구 버튼만 (중앙 배치)
        let restoreItem = UIBarButtonItem(title: "복구", primaryAction: UIAction { [weak self] _ in
            self?.restoreButtonTapped()
        })
        restoreItem.tintColor = .systemGreen
        toolbarItems = [flexSpace, restoreItem, flexSpace]
    } else {
        // 삭제 버튼만 (중앙 배치)
        toolbarItems = [flexSpace, toolbarDeleteItem!, flexSpace]
    }
}
```

**초기화 순서 (iOS 26)**
1. `viewWillAppear()` → `setupSystemUIIfNeeded()` 호출
2. `setupSystemUIIfNeeded()` → `setupNormalModeToolbar()` → `toolbarDeleteItem` 초기화
3. 이후 `updateToolbarForCurrentPhoto()` 호출 가능

#### 5.4 updateToolbarForCurrentPhoto 호출 시점
```swift
private func updateToolbarForCurrentPhoto() {
    let isTrashed = coordinator.isTrashed(at: currentIndex)

    // iOS 16~25: 커스텀 버튼 토글
    if viewerMode == .normal {
        deleteButton.isHidden = isTrashed
        restoreButton.isHidden = !isTrashed
    }

    // iOS 26+: 시스템 툴바 교체 (nil guard 필수)
    if #available(iOS 26.0, *) {
        updateToolbarItemsForCurrentPhoto()
    }
}
```

**호출 위치 (중요: iOS 26 초기화 순서 고려)**
- ~~`viewDidLoad()`~~ → **사용 금지** (iOS 26에서 `toolbarDeleteItem`이 아직 nil)
- `viewWillAppear()` → **초기 표시** (iOS 26: `setupSystemUIIfNeeded()` 이후)
- `pageViewController(_:didFinishAnimating:)` → 스와이프 탐색 후
- `moveToNextAfterDelete()` 완료 후 → 버튼/스와이프 삭제 후 이동 시

#### 5.5 삭제 시 동작 (버튼 + 스와이프 공통)

**버튼 삭제:**
```swift
@objc private func deleteButtonTapped() {
    // ... 삭제 요청 ...
    moveToNextAfterDelete()
    updateToolbarForCurrentPhoto()  // 다음 사진 기준으로 버튼 갱신
}
```

**스와이프 삭제 (handleSwipeDelete):**
```swift
private func handleSwipeDelete() {
    // ... 삭제 요청 ...
    moveToNextAfterDelete()
    updateToolbarForCurrentPhoto()  // 스와이프 삭제 후에도 버튼 갱신 필수!
}
```

> **중요**: 스와이프 삭제 경로도 반드시 `updateToolbarForCurrentPhoto()` 호출 필요.
> 다음 사진이 휴지통이면 복구 버튼으로 바뀌어야 함.

#### 5.6 복구 시 모드별 분기 처리
- `.trash` 모드: 다음 사진으로 이동 (목록에서 사라짐)
- `.normal` 모드: 제자리 유지, 테두리 제거 + 버튼 교체
```swift
@objc private func restoreButtonTapped() {
    // ... 복구 요청 ...

    if viewerMode == .trash {
        moveToNextAfterDelete()
    } else {
        // .normal: 제자리에서 UI만 업데이트
        updateCurrentPageTrashedState(isTrashed: false)
        updateToolbarForCurrentPhoto()
    }
}
```

#### 5.7 현재 페이지 테두리 즉시 업데이트
```swift
private func updateCurrentPageTrashedState(isTrashed: Bool) {
    guard let currentVC = pageViewController.viewControllers?.first else { return }
    if let photoVC = currentVC as? PhotoPageViewController {
        photoVC.updateTrashedState(isTrashed: isTrashed)
    } else if let videoVC = currentVC as? VideoPageViewController {
        videoVC.updateTrashedState(isTrashed: isTrashed)
    }
}
```
