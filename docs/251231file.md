## 7. 주의사항

### 7.1 접근자 변경 원칙
- `private` → `internal` (같은 모듈 내 extension에서 접근)
- `fileprivate`는 같은 파일 내에서만 유효하므로 사용 불가
- 외부 노출이 필요 없는 것은 그대로 `internal` (기본값)

### 7.2 Git 커밋 규칙
- CLAUDE.md: "50줄 이상 수정 전 커밋"
- 각 Phase 완료 후 커밋

### 7.3 빌드 테스트
- 각 분리 단계마다 `xcodebuild` 또는 Xcode에서 빌드 확인
- 컴파일 에러 발생 시 즉시 수정

### 7.4 #selector 핸들러 규칙
- `#selector`로 연결되는 메서드는 `@objc func` (internal) 필수
- `@objc private func`는 같은 파일에서만 selector 접근 가능
- 핸들러 이동 시 반드시 private 제거

### 7.5 static 상수 접근
- `private static` → `static` (internal)
- 또는 상수를 사용하는 메서드를 같은 파일에 유지

### 7.6 setupGestures() 처리
`setupGestures()`는 메인 파일에 유지하고, 핸들러만 각 파일로 이동:

```swift
// 메인 파일 (유지)
private func setupGestures() {
    let pinchGesture = UIPinchGestureRecognizer(
        target: self,
        action: #selector(handlePinchGesture(_:))  // GridGestures.swift
    )
    let dragGesture = UIPanGestureRecognizer(
        target: self,
        action: #selector(handleDragSelectGesture(_:))  // GridSelectMode.swift
    )
}

// GridGestures.swift
@objc func handlePinchGesture(_ gesture: ...)  // internal

// GridSelectMode.swift
@objc func handleDragSelectGesture(_ gesture: ...)  // internal
```
