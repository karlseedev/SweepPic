# Auto-Delete Candidate Plan (v1)

## Goal
Identify "obvious delete" photos using on-device signals. Output a ranked list of candidates (Strong/Medium). No auto-delete; user confirms.

## Scope
Single-photo quality classification (not duplicate grouping).

## Categories (with signals and initial thresholds)

1) Near-black / Overexposed (Strong)
- mean luma < 0.05 or > 0.95
- luma std < 0.03
- optional: highlight/shadow clipping ratio > 0.60

2) Lens occlusion / Pocket / Finger (Strong)
- dominant color ratio > 0.85
- edge density < 0.01
- optional: skin-tone ratio > 0.60 and low edge density

3) Severe blur / Focus failure (Medium~Strong)
- Laplacian variance < 40~60 (analysis size ~256px)
- edge density < 0.02
- if face present: VNDetectFaceCaptureQuality < 0.15
- Strong when no faces; Medium when faces exist

4) Low-information background (Medium)
- image entropy < 1.8
- edge density < 0.015
- no faces or text detected

5) Partial frame / Framing collapse (Medium~Strong)
- one side (>= 40% area) is near-uniform in luma
- meaningful content concentrated in <= center 20% area

6) Very low face quality (Medium)
- face detected AND face quality < 0.20
- plus blur signal or low luma std

## Scoring
Compute a weighted score in [0, 1].
- Strong if score >= 0.75
- Medium if 0.55 <= score < 0.75
- Weak: ignore

Suggested weights (sum capped at 1.0):
- Extreme luma: 0.35
- Low luma std: 0.20
- Blur: 0.25
- Low edge density: 0.10
- Dominant color: 0.30
- Face quality low: 0.10

## Guardrails (reduce false positives)
- Exclude favorites, edited photos, screenshots by default.
- If a face is present and face quality is high, never mark Strong.
- If text detection is high (document-like), demote or skip.
- Live Photo / video excluded unless explicitly enabled.

## Pipeline (on-device)
1) Prefilter assets by media type and flags.
2) Load downscaled CGImage (long side 224~320px).
3) Extract features:
   - mean/std luma
   - Laplacian variance
   - edge density
   - dominant color ratio (histogram)
   - entropy
   - face count + face quality
4) Compute score + category label.
5) Output candidate list with Strong/Medium tags.

## Initial Parameters
- Analysis size: 256px long side
- Laplacian var threshold: 50
- Edge density threshold: 0.015
- Dominant color ratio: 0.85
- Entropy threshold: 1.8
- Face quality low threshold: 0.20

## Future Tuning
- Collect user confirmations/rejections to adjust thresholds.
- Optional: train a small CoreML classifier using the above features.
- Add device-specific normalization (different sensor noise).
