# 얼굴 매칭 개선안 v1 (docs/llm/2 기반 수정안)

## 1. 목적
- `docs/llm/2.md`를 기반으로 **최소 변경 + 성능 유지** 전제의 개선안을 확정한다.
- 목표: 오매칭/누락 감소, 위치 변화 대응, 연산량 폭증 방지.

---

## 2. 문서 선정 근거
- `docs/llm/2.md`는 Top-K, FP 1회 생성, 점수 분리 등 **현 구조 유지형 개선**을 담고 있음.
- `docs/llm/3.md`는 전면 재설계(전역 최적화)로 리스크/비용이 큼.
- 따라서 **2를 수정해 적용 → 부족 시 3 검토**가 최적 순서.

---

## 3. 핵심 수정 방향

### 3.1 순서 의존성(선착순 매칭) 완화
- 문제: 현재는 얼굴 루프 순서대로 슬롯이 점유되어 더 좋은 매칭이 배제될 수 있음.
- 해결: 사진 1장 내에서 **후보 (face, slot) 쌍을 fpDistance 기준으로 정렬**한 뒤,
  아직 매칭되지 않은 얼굴/슬롯만 순차 확정하는 방식으로 변경.
- 전역 최적화는 아님(성능 유지), 하지만 순서 편향은 제거.

### 3.2 fallback 규칙 명확화
- `face FP 없음` / `slot FP 없음`을 구분:
  - face FP 없음: 위치 기반 매칭만 허용
  - slot FP 없음: 다른 slot FP와 비교는 가능, 해당 slot은 FP 기반 비교 제외
- FP 기반 매칭이 성립한 경우 **posDistance가 fpDistance 결과를 덮어쓰지 않음**.

### 3.3 파라미터 변경은 보류
- `personMatchThreshold` 변경은 이번 단계에서 **보류**.
- 분포 로그 확보 후 **데이터 기반 튜닝**으로 이동.
- 위치 필터는 `0.15 → 0.3` 같은 하드 컷오프 확장이 아니라,
  **Top-K 후보 방식으로 완화**하고 숫자 변경은 하지 않음.

### 3.4 검증 지표 추가
- 성능/정확도 지표를 반드시 기록:
  - 얼굴당 FP 생성 횟수 (목표: 1회)
  - 후보 슬롯 수(K)
  - FP 매칭 성공률 / fallback 비율
  - 오매칭/누락 케이스 재현 여부

---

## 4. 개선 로직 (요약)

1) 후보 슬롯 선정
- 모든 슬롯과의 posDistance 계산 → 거리순 정렬
- Top-K 슬롯만 FP 비교 대상으로 선정 (기본 K=3~4)
- `posDistance < 0.15`는 **우선순위 힌트**로만 사용:
  0.15 이내 슬롯을 먼저 포함하고, 부족하면 가까운 슬롯으로 채움
- 하드 컷오프는 사용하지 않음

2) 얼굴 FP 1회 생성
- 얼굴마다 FP 생성 1회만 수행, 후보 슬롯 비교에 재사용

3) 매칭 결정
- fpDistance 최소값 기준으로 slot 선택
- 동률/근접 시 posDistance로 tie-breaker
- FP 비교 실패 시에만 위치 기반 fallback

4) 동적 슬롯 생성 (Phase 2)
- 모든 후보 slot의 fpDistance가 임계값 초과인 경우에만 새 슬롯 생성
- 새 슬롯은 `isValidSlot=false`로 시작, 유효 슬롯 조건 충족 시 승격

---

## 5. 적용 단계

### Phase 1 (이번 수정)
- Top-K 후보 선정
- 얼굴 FP 1회 생성
- 매칭 규칙 분리 + 순서 의존성 완화

### Phase 2 (필요 시)
- 동적 슬롯 생성 + 슬롯 상한 적용

### Phase 3 (안정화)
- EMA 기반 슬롯 위치 갱신
- 임계값 튜닝 (로그 기반)

---

## 6. 기대 효과
- 위치 이동에 대한 강건성 개선
- 첫 사진 의존성 완화
- 불필요한 FP 재생성 제거로 성능 유지
- 테두리/뷰어 +버튼 일관성 개선
