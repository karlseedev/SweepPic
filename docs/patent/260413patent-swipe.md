# 발명제안서

## 1. 발명의 명칭

**[국문]**  
그리드형 썸네일 화면에서 선택 모드 전환 없이 스와이프 기반 상태 변경과 상세 열람을 공존시키는 이미지 선택 및 삭제 방법

**[영문]**  
Method for Selecting and Deleting Images in a Grid Thumbnail View Without Selection Mode Switching While Coexisting With Detailed Viewing

## 2. 기술분야

본 발명은 터치스크린 기반 전자 장치에서 사진, 영상 또는 기타 미디어 항목을 관리하는 사용자 인터페이스 기술에 관한 것이다. 보다 구체적으로는, 그리드 형태로 배열된 썸네일 항목에 대해 스와이프 제스처를 이용하여 선택, 선택 해제, 삭제 대상 지정, 복원 등의 상태 변경을 수행하면서도, 별도의 선택 모드 전환 없이 탭 입력에 의한 상세 열람이 유지되도록 하는 인터페이스 방법 및 장치에 관한 것이다.

## 3. 발명의 배경이 되는 기술

모바일 사진 앱이나 파일 관리 앱에서는 다수의 항목을 한 화면에 썸네일 그리드로 표시한 뒤, 사용자가 일부 항목을 선택하거나 삭제하는 방식이 일반적이다. 종래 방식은 대체로 다음과 같은 절차를 따른다.

1. 사용자가 별도의 `선택 모드`에 진입한다.
2. 각 항목을 탭하여 선택한다.
3. 삭제, 이동, 공유 등의 작업 버튼을 누른다.
4. 일반 보기 상태로 돌아오기 위해 선택 모드를 종료한다.

또 다른 방식으로는 리스트에서의 스와이프 삭제, 전체화면 뷰어에서의 단일 항목 삭제, 두 손가락 드래그 기반의 복수 선택 등이 있으나, 사진 정리 작업에서 자주 필요한 `선택`, `삭제`, `상세 확인`, `재선택`의 반복 흐름을 자연스럽게 연결하지 못하는 한계가 있다.

특히 종래의 선택 모드는 화면 전체를 모달 상태로 전환하는 경우가 많기 때문에, 사용자가 여러 항목을 선택하던 도중 특정 사진을 크게 보고 싶어도 즉시 뷰어로 진입하기 어렵다. 많은 경우 썸네일 탭은 뷰어 진입이 아니라 선택 토글로 해석되며, 사용자는 선택을 취소하거나 작업을 먼저 확정한 뒤에야 상세 보기로 이동할 수 있다.

## 4. 종래기술의 문제점

종래기술은 다음과 같은 문제점을 가진다.

### 4.1 별도 선택 모드 진입 필요

복수 항목 처리 기능을 사용하기 위해 일반 브라우징 상태를 벗어나야 하므로, 사용 흐름이 단절된다.

### 4.2 선택과 열람의 상충

선택 중인 상태에서는 썸네일 탭이 선택/해제 기능으로 소비되어, 사용자가 원하는 시점에 특정 사진의 상세 내용을 확인하기 어렵다.

### 4.3 작업 재개의 비연속성

상세 뷰어에 들어갔다가 나온 뒤 다시 선택 모드에 진입하거나 상태를 복구해야 하는 경우가 많아, 실제 정리 작업의 리듬이 깨진다.

### 4.4 정리 작업에 비효율적

사진 정리 사용자는 `대충 훑기`, `몇 장 골라내기`, `애매한 사진 크게 보기`, `다시 정리 계속하기`를 반복하는데, 종래기술은 이 흐름을 하나의 연속 작업으로 지원하지 못한다.

### 4.5 상태 변경 인터랙션의 분산

선택은 탭, 삭제는 버튼, 상세 열람은 별도 탭, 복원은 또 다른 메뉴 등으로 기능이 분산되어 있어 사용자가 학습해야 할 규칙이 많다.

### 5.2 선행기술 대비 시사점

선행기술에는 그리드에서의 다중 선택, 스와이프 기반 삭제, 자동 스크롤을 수반하는 연속 선택 등이 일부 존재할 수 있다. 그러나 본 발명은 아래 조합에서 차별화될 수 있다.

1. 별도의 선택 모드 진입 없이 상태 변경이 시작된다는 점
2. 상태 변경이 일부 이루어진 후에도 탭 기반 상세 열람이 유지된다는 점
3. 상세 뷰어 복귀 후 별도 모드 복구 없이 후속 상태 변경을 즉시 재개할 수 있다는 점
4. 선택 또는 삭제와 열람이 하나의 연속 작업 흐름 안에서 공존한다는 점

즉, 권리화의 핵심은 `스와이프 선택` 또는 `스와이프 삭제` 자체보다, `상태 변경과 상세 열람의 비충돌 구조`, `그리드-뷰어-그리드 왕복 중 작업 연속성`에 있다.

## 6. 해결하고자 하는 과제

본 발명은 상기한 문제를 해결하기 위해 다음 과제를 달성하고자 한다.

1. 별도의 선택 모드 없이 그리드 화면에서 직접 상태 변경을 시작할 수 있도록 할 것
2. 선택 또는 삭제 작업 도중에도 특정 항목을 바로 상세 뷰어로 열람할 수 있도록 할 것
3. 뷰어에서 복귀한 후 별도 모드 재진입 없이 후속 선택 또는 삭제를 즉시 재개할 수 있도록 할 것
4. 단일 항목 상태 변경과 복수 항목 상태 변경을 하나의 제스처 흐름으로 통합할 것
5. 사용자가 현재 상태 변경의 적용 정도와 범위를 직관적으로 인지할 수 있도록 시각적 피드백을 제공할 것

## 7. 과제의 해결 수단

상기 과제를 해결하기 위하여, 본 발명은 다음과 같은 구성을 갖는 방법 및 장치를 제공한다.

### 7.1 비모달(non-modal) 인터페이스 구조

본 발명은 별도의 선택 모드 전환 없이, 상태 변경 이후에도 탭 입력에 의한 상세 열람이 유지되는 비모달(non-modal) 인터페이스 구조에 관한 것이다.

구체적으로, 그리드형 썸네일 화면에서 사용자의 수평 성분을 갖는 스와이프 입력은 특정 항목의 상태 변경 제스처로 해석되고, 탭 입력은 여전히 특정 항목의 상세 뷰어 실행 명령으로 유지된다. 이에 따라 사용자는 일부 항목에 대한 선택 또는 삭제 상태를 부여한 이후에도 다른 항목을 탭하여 즉시 열람할 수 있으며, 상세 뷰어에서 복귀한 후에도 별도의 선택 모드 재진입 없이 후속 상태 변경을 계속할 수 있다.

### 7.2 제스처 역할 고정 구조

본 발명의 그리드 인터페이스에서 각 제스처 유형은 다음과 같이 고정된 역할을 가진다.

- 탭 제스처: 해당 항목의 상세 뷰어를 실행
- 수평 스와이프 제스처: 해당 항목에 대한 선택 또는 소정의 작업 수행
- 수직 스와이프 제스처: 그리드 스크롤 수행

상기 제스처 역할은 항목 상태 변경 전후를 포함한 어떠한 시점에서도 바뀌지 않는다.

### 7.3 상태 변경의 종류

상기 상태 변경은 선택, 선택 해제, 삭제 대상 지정, 삭제 복원, 앱 내부 삭제대기함 이동, 이동 후보 지정, 제외 처리 등으로 구현될 수 있다. 동일한 제스처 엔진이 복수의 상태 변경 유형에 공통 적용될 수 있다.

### 7.4 진행률 기반 피드백

스와이프 이동량 또는 방향에 따라 해당 썸네일 위에 오버레이 또는 커튼형 시각 효과를 표시하여, 사용자가 제스처의 진행도와 확정 가능성을 실시간으로 인식하도록 한다.

### 7.5 단일 항목에서 복수 항목으로의 확장

사용자의 손가락이 최초 대상 셀을 벗어나 인접 셀 영역으로 이동하면, 시스템은 자동으로 복수 항목 처리 모드로 전환할 수 있다. 이 경우 시작 셀과 현재 셀의 상대 위치를 기준으로 연속 범위 또는 사각형 범위를 결정하여 동일 상태 변경을 여러 항목에 적용한다.

### 7.6 확정 및 취소 판단

스와이프 종료 시 이동 거리, 이동 속도, 진행률 또는 이들의 조합에 기초하여 상태 변경의 확정 여부를 결정할 수 있다. 확정 요건을 충족하지 않으면 대상 항목은 원래 상태로 복귀한다.

### 7.7 실행 취소 및 중간 저장 구조

삭제 또는 제외와 같은 상태 변경은 즉시 영구 반영되는 대신 앱 내부 삭제대기함, 중간 저장소, 또는 되돌리기 가능한 임시 상태로 우선 기록될 수 있다. 이에 따라 사용자는 오조작에 대한 부담을 줄인 상태에서 연속 작업을 수행할 수 있다.

## 8. 발명의 핵심 구성

본 발명의 일 실시예에 따른 시스템은 다음 요소를 포함할 수 있다.

- 복수의 미디어 항목을 그리드 썸네일로 표시하는 표시부
- 터치 및 스와이프 입력을 감지하는 입력 감지부
- 스크롤 입력과 상태 변경 입력을 구분하는 제스처 판정부
- 선택, 삭제, 복원 등 항목 상태를 관리하는 상태 관리부
- 진행률 기반 시각 효과를 출력하는 피드백 처리부
- 탭 입력에 따라 특정 항목의 상세 뷰어를 여는 열람 처리부
- 상세 뷰어 복귀 시 그리드 상태를 유지하거나 재구성하는 복귀 처리부

## 9. 발명의 작동 방식

### 9.1 기본 브라우징 상태

사용자는 그리드 화면에서 임의의 썸네일을 탭하여 언제든지 상세 뷰어로 들어갈 수 있다.

### 9.2 스와이프에 의한 상태 변경 시작

사용자가 특정 썸네일 위에서 수평 방향 또는 수평 우세 방향의 스와이프를 수행하면, 시스템은 이를 해당 항목의 상태 변경 제스처로 인식한다.

### 9.3 진행도 반영

스와이프가 진행되는 동안 대상 썸네일에는 커튼형 오버레이, 색상 변화, 채움 효과 또는 기타 시각 피드백이 적용된다.

### 9.4 복수 항목 처리

스와이프가 인접 셀로 확장되면, 시스템은 복수 항목 처리 모드로 전환되어 둘 이상의 썸네일에 대해 선택 또는 삭제 예정 상태를 연속적으로 부여한다.

### 9.5 상세 뷰어 즉시 진입

사용자는 위와 같은 상태가 일부 부여된 이후에도 다른 썸네일을 탭하여 즉시 상세 뷰어로 들어갈 수 있다. 이때 탭은 선택 모드용 탭이 아니라 일반 열람 명령으로 동작한다.

### 9.6 상세 뷰어에서 복귀 후 즉시 재개

사용자가 상세 뷰어에서 돌아오면, 별도 모드 전환 없이 그리드 상에서 다시 스와이프를 수행하여 후속 선택, 삭제, 복원 또는 제외를 계속할 수 있다.

## 10. 본 발명의 핵심 차별점

본 발명의 핵심은 `그리드에서 스와이프로 상태 변경이 가능하다`는 점만이 아니다. 진정한 차별점은 `상태 변경 작업과 일반 열람 흐름을 분리하지 않고 공존시킨다`는 데 있다.

### 10.1 별도의 선택 모드 전환 없는 구조

종래의 선택 모드는 전체 화면 의미를 바꾸는 모달 상태이지만, 본 발명은 개별 항목의 상태만 변경하고 화면 전체는 일반 탐색 상태로 유지한다.

### 10.2 선택과 상세 확인의 양립

사용자는 일부 항목을 스와이프로 선택 또는 삭제 지정한 상태에서도 다른 항목을 탭하여 즉시 상세 뷰어로 진입할 수 있다.

### 10.3 왕복 가능한 작업 흐름

상세 뷰어 진입이 작업 중단이 아니라 작업 중간의 확인 단계로 기능하고, 복귀 후에도 사용자는 다시 곧바로 후속 선택 및 삭제를 진행할 수 있다.

### 10.4 실제 사진 정리 문맥에 부합

사용자가 `선별 -> 확인 -> 재선별`을 반복하는 실사용 패턴에 가장 잘 맞는 구조를 제공한다.

## 11. 실시예

### 실시예 1: 사진 관리 앱에서의 스와이프 삭제

사용자의 모바일 장치에 저장된 사진들이 그리드 형태의 썸네일로 표시된다. 사용자가 삭제하고자 하는 사진의 썸네일 위에서 한 손가락으로 수평 방향으로 스와이프한다.

1. 터치 입력이 감지되고 팬 제스처가 시작된다.
2. 이동 방향과 이동량이 분석되어 상태 변경 제스처로 판별된다.
3. 대상 셀에 진행률 기반 오버레이가 적용된다.
4. 종료 시 확정 조건이 충족되면 삭제가 성립되고, 해당 사진은 삭제대기함으로 이동된다.
5. 필요시 되돌리기 인터페이스가 함께 제공될 수 있다.

### 실시예 2: 다중 사진 연속 삭제

사용자가 첫 번째 썸네일에서 스와이프를 시작한 후 손가락을 인접 셀로 이동시키면, 시스템은 자동으로 복수 항목 처리 모드로 전환한다. 동일 행에서는 연속 범위가 선택되고, 다른 행으로 이동하면 사각형 범위가 선택될 수 있다. 종료 시 선택된 여러 항목에 대해 일괄 삭제가 수행된다.

### 실시예 3: 삭제된 항목의 복구

삭제대기함 화면 또는 삭제 상태가 시각적으로 표시된 그리드에서, 사용자가 특정 항목을 수평 스와이프하면 시스템은 이를 복구 작업으로 해석할 수 있다. 이 경우 오버레이는 반전된 방향으로 해제되거나 다른 색상으로 변경될 수 있으며, 확정 시 해당 항목은 복구된다.

### 실시예 4: 선택 기능으로의 확장

파일 관리 앱 또는 사진 선별 앱에서, 사용자가 그리드 형태로 표시된 항목들 중 일부를 선택하여 공유, 이동, 분류, 즐겨찾기 등의 후속 작업을 수행하고자 할 때, 동일한 스와이프 제스처 엔진을 적용할 수 있다.

### 실시예 5: 스와이프 작업과 탭 열람의 교차 수행

사용자가 사진 그리드에서 불필요한 사진을 정리하면서, 중간중간 특정 사진의 상세 내용을 확인하고자 한다.

1. 사용자가 첫 번째 사진을 스와이프하여 삭제 또는 삭제 예정 상태로 만든다.
2. 사용자가 다음 사진을 탭하면 별도의 모드 해제 없이 즉시 상세 뷰어가 실행된다.
3. 사용자가 상세 뷰어에서 그리드 화면으로 복귀한다.
4. 사용자는 별도의 선택 모드 재진입 없이 바로 다른 사진을 다시 스와이프하여 후속 선택 또는 삭제를 계속한다.

본 실시예는 선택 또는 삭제 작업과 상세 열람이 하나의 흐름 안에서 교차 수행되는 본 발명의 핵심 특징을 나타낸다.

### 실시예 6: 미리보기 화면에서의 제외 처리

자동 정리 기능의 미리보기 화면에서, 시스템이 삭제를 권장하는 사진들이 그리드로 표시되고 사용자가 일부 사진을 제외 대상으로 바꾸고자 할 때, 동일한 스와이프 제스처를 이용하여 제외 상태를 부여하거나 해제할 수 있다.

## 12. 발명의 효과

본 발명에 의하면 다음과 같은 효과를 얻을 수 있다.

1. 별도의 선택 모드 진입 및 종료 과정이 감소하여 조작 단계가 줄어든다.
2. 선택과 열람이 서로 충돌하지 않아 작업 흐름이 부드럽다.
3. 사용자가 애매한 항목을 즉시 확대 확인하고 바로 정리 흐름으로 복귀할 수 있다.
4. 단일 항목 처리와 복수 항목 처리를 같은 제스처 문법으로 통합할 수 있다.
5. 진행률 기반 시각 피드백에 의해 사용자가 현재 작업 상태를 직관적으로 인지할 수 있다.
6. 사진 정리, 선별, 검토 같은 반복 작업에 특히 적합한 사용자 경험을 제공할 수 있다.
7. 삭제대기함 또는 되돌리기 구조와 결합할 경우 안전성까지 확보할 수 있다.

**[도 3]** 단일 항목 모드에서 복수 항목 모드로 자동 전환되는 과정을 나타내는 화면 구성도

**[도 4]** 시작 셀과 현재 셀의 위치 관계에 기초한 복수 항목 범위 산출 예시도

## 14. 권리화 포인트 제안

### 14.1 독립항 포인트

그리드형 썸네일 화면에서 스와이프 입력에 의해 하나 이상의 항목 상태를 변경하면서도, 별도의 선택 모드 전환 없이 탭 입력에 따른 상세 뷰어 진입 기능을 유지하고, 상세 뷰어 복귀 후 후속 상태 변경을 재개하는 방법

### 14.2 종속항 포인트

- 상태 변경이 선택, 선택 해제, 삭제 대상 지정, 복원을 포함하는 점
- 단일 항목 상태 변경에서 복수 항목 상태 변경으로 자동 전환되는 점
- 복수 항목 범위를 시작 셀과 현재 셀의 위치 관계로 결정하는 점
- 진행률 기반 오버레이 또는 커튼형 피드백을 제공하는 점
- 삭제 처리 결과를 앱 내부 삭제대기함에 반영하는 점
- 상세 뷰어 진입 전후에도 탭 입력의 의미가 변경되지 않는 점

## 15. 청구항 방향 초안

### 독립항 초안

터치스크린을 구비한 전자 장치에서 그리드 형태로 배열된 복수의 이미지 항목을 관리하는 방법에 있어서,

- 특정 썸네일에 대한 스와이프 입력을 감지하는 단계,
- 상기 스와이프 입력에 응답하여 하나 이상의 썸네일 상태를 변경하는 단계,
- 상기 상태 변경 이후에도 썸네일 탭 입력에 응답하여 상세 뷰어를 실행하는 단계,
- 상기 상세 뷰어로부터 복귀한 후 추가 스와이프 입력에 응답하여 후속 상태 변경을 수행하는 단계를 포함하는 이미지 관리 방법.

### 종속항 초안

- 상기 상태 변경이 선택 상태 부여, 삭제 대상 지정, 복원 상태 전환, 또는 제외 상태 전환을 포함하는 방법
- 상기 스와이프 입력이 인접 셀로 확장되면 복수 항목 처리 모드로 전환되는 방법
- 상기 복수 항목 처리 모드에서 복수 항목 범위를 사각형 영역으로 산출하는 방법
- 상기 상태 변경 진행도에 비례하는 시각적 오버레이를 표시하는 방법
- 상기 삭제 대상 지정이 앱 내부 삭제대기함 이동을 포함하는 방법
- 상기 탭 입력이 상태 변경 전후에 동일하게 상세 열람 명령으로 유지되는 방법

## 16. 요약

본 발명은 그리드 썸네일 화면에서 스와이프를 통해 항목의 선택 또는 삭제 상태를 부여하면서도, 별도의 선택 모드 전환 없이 상태 변경 이후에도 탭 입력에 의한 상세 열람이 유지되도록 하여, 특정 항목을 즉시 상세 뷰어로 열람하고 다시 돌아와 후속 선택 및 삭제를 계속할 수 있게 하는 이미지 관리 인터페이스에 관한 것이다.

즉, 본 발명의 본질은 `선택 또는 삭제가 가능하다`는 점보다, `선택 또는 삭제와 보기 작업이 서로 충돌하지 않고 연속적으로 공존한다`는 데 있다.
