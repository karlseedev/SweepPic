# 260113 뷰어 전환 배경 페이드인 작업 기록

## 문제 상황
- 기본 사진 앱에서는 썸네일 클릭 시 확대 모션에서 상하단 색상이 자연스럽게 페이드인됨
- 현재 앱에서는 바로 딱 나타남

## 분석

### 현재 구현 상태
- **iOS 18+**: `preferredTransition = .zoom` 사용 (GridViewController.swift:956)
- **iOS 16~17**: 표준 push 애니메이션만 사용

### 관련 코드 구조
```
ViewerViewController.view
├─ backgroundView (.black 배경)
├─ UIPageViewController.view
│  └─ _UIQueuingScrollView (내부 자동 생성)
│     └─ PhotoPageViewController.view
│        └─ scrollView
│           └─ imageView
└─ 버튼들 (삭제, 뒤로가기 등)
```

## 수정 시도

### 1차 수정: transitionCoordinator 활용
ViewerViewController에서:
- `backgroundView` 초기 alpha를 0으로 설정
- `viewWillAppear`에서 `transitionCoordinator.animate(alongsideTransition:)` 사용

```swift
private func animateBackgroundFadeIn() {
    if let coordinator = transitionCoordinator {
        coordinator.animate(alongsideTransition: { _ in
            self.backgroundView.alpha = 1
        }, completion: nil)
    } else {
        backgroundView.alpha = 1
    }
}
```

### 2차 수정: 배경 투명화
- `ViewerViewController.view.backgroundColor = .clear`
- `pageViewController.view.backgroundColor = .clear`
- `PhotoPageViewController.view.backgroundColor = .clear`
- `VideoPageViewController.view.backgroundColor = .clear`

### 3차 수정: UIPageViewController 내부 뷰 투명화
```swift
for subview in pageViewController.view.subviews {
    subview.backgroundColor = .clear
}
```

### 4차 수정: 재귀적 투명화
```swift
private func makeAllSubviewsTransparent(_ view: UIView) {
    for subview in view.subviews {
        subview.backgroundColor = .clear
        makeAllSubviewsTransparent(subview)
    }
}
```

## 현재 문제
- 코드상으로 alongside animation은 정상 동작
- PhotoPageViewController 배경은 투명 (isClear: true)
- UIPageViewController 내부 뷰들도 backgroundColor: nil
- **하지만 빨간색 테스트 배경이 보이지 않음**

## 추가 시도 (GPT 솔루션)

### 5차 수정: containerView에 dimming 뷰 추가
GPT 분석 결과, iOS 18 zoom transition은 **스냅샷 기반**으로 동작하여 실제 뷰 계층의 alpha 변경이 보이지 않음.
해결책으로 `transitionCoordinator.containerView`에 직접 dimming 뷰를 추가.

```swift
private func installTransitionFadeIfNeeded() {
    guard disableCustomFadeAnimation,
          let coordinator = transitionCoordinator else { return }

    let container = coordinator.containerView
    let dimming = UIView(frame: container.bounds)
    dimming.backgroundColor = .black
    dimming.alpha = 0
    dimming.autoresizingMask = [.flexibleWidth, .flexibleHeight]

    container.insertSubview(dimming, belowSubview: view)
    transitionDimmingView = dimming

    coordinator.animate(alongsideTransition: { _ in
        dimming.alpha = 1
    }, completion: { [weak self] context in
        dimming.removeFromSuperview()
        self?.transitionDimmingView = nil
        if !context.isCancelled {
            self?.backgroundView.alpha = 1
        }
    })
}
```

### 타이밍 불일치 문제 발견
디버그 로그 결과:
```
transitionDuration: 0.34초 (시스템이 보고하는 전환 시간)
alongsideTransition 실행: 0.01초 (즉시)
completion 호출: 0.91초 (실제 완료 시점)
```

**문제**: alongside animation은 0.34초 기준으로 끝나지만, 실제 zoom transition은 ~0.9초까지 진행됨.
결과적으로 배경 페이드인이 이미지 확대보다 먼저 완료되어 타이밍이 맞지 않음.

### 웹 검색 결과: iOS 18 zoom transition API 한계

> "The transition might look even better if the black background didn't zoom at all and instead cross-faded in place while only the image was zoomed... **Unfortunately this isn't what Apple implemented.**"

> "This is different from the effect in **Apple's Photos app** where only the image gets shrunk, whereas the background and toolbars fade out during the transition. **The Photos app looks a bit nicer. But anyway, for now, this is not what Apple has built into the SDK.**"

**결론**:
- 기본 사진 앱은 내부 API 또는 커스텀 transition 사용
- SDK의 `preferredTransition = .zoom`은 배경 분리 불가
- alongside animation으로는 완전 동기화 불가능

### 6차 수정: UIView.animate로 더 긴 duration
alongside 대신 별도의 UIView.animate(duration: 0.7)로 시도.
→ **실패**: 페이드인 자체가 동작하지 않음 (transition 컨텍스트 문제 추정)

### 7차 수정: alongside 내부에서 delay 적용
alongside 블록 안에서 UIView.animate(delay: 0.2)로 시도.
→ **실패**: 마찬가지로 페이드인 동작 안 함

### 8차 수정: 뷰 투명화 + 중복 방지 로직
- `view.backgroundColor = .clear`, `view.isOpaque = false`
- `pageViewController.view.backgroundColor = .clear`
- `removeTransitionDimmingView()` 메서드 추가
→ **실패**: 여전히 기본 사진앱 품질에 미치지 못함

## 최종 결론

**원복 결정** (커밋 4306c34로 복원)

### 원복 사유
1. iOS 18 `preferredTransition = .zoom` API 자체의 한계
2. alongside animation과 실제 zoom transition의 타이밍 불일치 (0.34초 vs ~0.9초)
3. 기본 사진 앱은 SDK에 없는 내부 API 사용 추정
4. 여러 시도에도 "자연스러운 페이드인" 품질 달성 실패

### 향후 대안
1. iOS 18 zoom transition 포기 → 커스텀 `UIViewControllerAnimatedTransitioning` 구현
2. 현재 상태 유지 (기본 사진 앱과 동일하지 않음을 수용)
