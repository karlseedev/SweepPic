# PhotoPageViewController "반쯤 잘림" 현상 해결 기획

## 1. 문제 정의

### 증상
스와이프 시 다음 이미지 대신 **좌측 1/3 정도가 검게** 따라오다가 이미지로 바뀌는 현상

### 근본 원인 (로그로 확정)
```
[Photo] 📸 pending@will tid=6 idx=3352 hasImage=false frame=(0.0, 0.0, 0.0, 0.0)
[Photo] 📸 pending@nextRunLoop tid=6 idx=3352 hasImage=false frame=(0.0, 0.0, 0.0, 0.0)
...27ms 후...
[Photo] ✅ 이미지 완료 - index: 3352, 24ms, since init: 37ms
[Photo] 📦 frame 변경 - old: (0.0, 0.0, 0.0, 0.0), new: (0.0, 0.0, 390.0, 520.0)
```

| 시점 | 상태 | 문제 |
|------|------|------|
| willTransition | hasImage=false, frame=(0,0,0,0) | 빈 페이지로 전환 시작 |
| 이미지 로드 | 25-40ms 소요 | 그 사이 검은 영역 노출 |
| 이미지 완료 | frame 변경 발생 | 레이아웃 점프 |

### 왜 배경색 변경은 효과 없었나
- 배경색 .black으로 변경해도 `frame=(0,0,0,0)`인 imageView가 여전히 문제
- 배경색은 증상을 숨길 뿐, 근본 해결 아님

---

## 2. 현재 코드 문제점

### 타이밍 흐름
```
t=0.0ms   init()
t=0.2ms   viewDidLoad() → setupUI()만 호출
t=10ms    willTransition (스와이프 시작) ← 이미지 없음!
t=12ms    viewDidLayoutSubviews() → 이미지 요청 시작
t=37ms    이미지 완료 → frame 변경 발생
t=250ms   didFinishAnimating
```

### 핵심 문제 2가지
1. **이미지 요청 시점이 늦음**: viewDidLayoutSubviews에서 시작 (willTransition 후)
2. **레이아웃이 이미지 도착마다 변경됨**: LOD0→LOD1→LOD2 각 단계에서 frame 재계산

---

## 3. 해결 전략

### 원칙
> "전환 시점에 pending 페이지가 절대 빈 상태가 아니게"

### 핵심 변경 2가지

| 변경 | 현재 | 개선 |
|------|------|------|
| 이미지 요청 시점 | viewDidLayoutSubviews | **viewDidLoad** |
| 레이아웃 계산 | 이미지 도착마다 | **1회만 (PHAsset 비율 기반)** |

---

## 4. 구현 상세

### Phase 1: Early LOD0 (viewDidLoad에서 즉시 요청)

```swift
override func viewDidLoad() {
    super.viewDidLoad()
    setupUI()

    // 1. 레이아웃 1회 고정 (PHAsset 비율 기반)
    applyInitialLayout()

    // 2. 즉시 LOD0 요청 (bounds가 0이어도 가능)
    requestLOD0Image()
}

/// PHAsset 비율 기반 레이아웃 (1회만 실행)
private func applyInitialLayout() {
    guard !hasAppliedInitialLayout else { return }

    let assetWidth = CGFloat(asset.pixelWidth)
    let assetHeight = CGFloat(asset.pixelHeight)
    guard assetWidth > 0, assetHeight > 0 else { return }

    // bounds가 0일 수 있으므로 UIScreen.main.bounds fallback
    let containerSize = view.bounds.size.width > 0
        ? view.bounds.size
        : UIScreen.main.bounds.size

    // aspect fit 계산
    let assetRatio = assetWidth / assetHeight
    let containerRatio = containerSize.width / containerSize.height

    let fitSize: CGSize
    if assetRatio > containerRatio {
        fitSize = CGSize(width: containerSize.width,
                        height: containerSize.width / assetRatio)
    } else {
        fitSize = CGSize(width: containerSize.height * assetRatio,
                        height: containerSize.height)
    }

    // frame 확정
    imageView.frame = CGRect(origin: .zero, size: fitSize)
    scrollView.contentSize = fitSize

    // 중앙 정렬
    let hInset = max(0, (containerSize.width - fitSize.width) / 2)
    let vInset = max(0, (containerSize.height - fitSize.height) / 2)
    scrollView.contentInset = UIEdgeInsets(top: vInset, left: hInset,
                                           bottom: vInset, right: hInset)
    scrollView.contentOffset = CGPoint(x: -hInset, y: -vInset)

    hasAppliedInitialLayout = true
}

/// LOD0 즉시 요청 (.fast, opportunistic)
private func requestLOD0Image() {
    let screenSize = UIScreen.main.bounds.size
    let scale = UIScreen.main.scale
    let targetSize = CGSize(width: screenSize.width * scale,
                           height: screenSize.height * scale)

    requestCancellable = ImagePipeline.shared.requestImage(
        for: asset,
        targetSize: targetSize,
        contentMode: .aspectFit,
        quality: .fast  // opportunistic → degraded 먼저 표시
    ) { [weak self] image, isDegraded in
        guard let self = self, let image = image else { return }

        // 이미지만 교체 (레이아웃 변경 없음)
        self.imageView.image = image
        self.imageSize = image.size  // maxZoomScale 계산용
        self.updateMaxZoomScale()
    }
}
```

### Phase 2: 레이아웃 변경 금지 정책

```swift
override func viewDidLayoutSubviews() {
    super.viewDidLayoutSubviews()
    scrollView.frame = view.bounds

    // 초기 레이아웃이 아직 안 됐으면 적용
    if !hasAppliedInitialLayout {
        applyInitialLayout()
        requestLOD0Image()
    }

    // 줌 중에는 레이아웃 변경 보류
    if isZoomInteractionActive {
        needsLayoutUpdateAfterZoom = true
        return
    }

    // 이미 레이아웃 적용됨 → 이미지 요청만 (레이아웃 변경 없음)
    // requestImageForCurrentBoundsIfNeeded() 호출하지 않음
}
```

### Phase 3: LOD1/LOD2 정책

| LOD | 시점 | 동작 |
|-----|------|------|
| LOD0 | viewDidLoad | 즉시 요청, .fast |
| LOD1 | didFinishAnimating + 150ms idle | 고품질 요청, **이미지만 교체** |
| LOD2 | 줌 시작 시 | 원본 요청, **이미지만 교체** |

```swift
// LOD1: 이미지만 교체 (레이아웃 변경 없음)
func requestHighQualityImage() {
    // ... 요청 ...

    completion: { [weak self] image, isDegraded in
        guard let self = self, let image = image else { return }

        // 레이아웃 변경 없이 이미지만 교체
        self.imageView.image = image
        self.imageSize = image.size
        self.updateMaxZoomScale()

        // updateImageLayout() 호출하지 않음!
    }
}
```

## 6. 2차 최적화 (1차 성공 후)

### Preheat (선택)
```swift
// ViewerViewController.willTransitionTo
func pageViewController(_ pvc, willTransitionTo pending) {
    // pending ±1~2 페이지 preheat
    let indices = [pendingIndex - 1, pendingIndex, pendingIndex + 1]
    let assets = indices.compactMap { coordinator.asset(at: $0) }
    ImagePipeline.shared.preheatAssets(assets, targetSize: viewerTargetSize)
}
```

### LOD1 디바운스 (선택)
```swift
// didFinishAnimating 후 150ms idle 확인
func pageViewController(_ pvc, didFinishAnimating...) {
    lod1DebounceTimer?.invalidate()
    lod1DebounceTimer = Timer.scheduledTimer(withTimeInterval: 0.15, repeats: false) {
        // 150ms 동안 새 전환 없으면 LOD1 요청
        currentPhotoVC.requestHighQualityImage()
    }
}
```
