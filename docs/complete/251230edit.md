# BarsVisibilityPolicy 중앙 집중식 가시성 제어 계획

## 문제
- `hidesBottomBarWhenPushed = true` 사용 시 pop할 때 iOS가 TabBar를 자동 복원
- 각 VC가 `tabBar.isHidden`을 직접 건드리면 iOS 자동 복원과 충돌
- 코드 중복 및 재발 가능성 높음

## 해결 원칙
**"VC는 정책만 선언, TabBarController가 실행"** - 단일 책임 원칙
**"채택하지 않으면 아무것도 안 함"** - 안전한 기본값 원칙

---

## 구현 계획

### 1. BarsVisibilityControlling 프로토콜 정의
**파일**: `Protocols/BarsVisibilityControlling.swift` (신규)

```swift
/// Bar 가시성 정책을 선언하는 프로토콜
/// - 채택하지 않은 VC: TabBarController 기본 정책 적용
/// - 채택한 VC: 명시적으로 선언한 값 적용, nil이면 기본 정책
protocol BarsVisibilityControlling {
    /// nil: 기본 정책, true: 숨김, false: 표시
    var prefersFloatingOverlayHidden: Bool? { get }
    var prefersSystemTabBarHidden: Bool? { get }
    var prefersToolbarHidden: Bool? { get }
}

// 기본값: 모두 nil (TabBarController 기본 정책 따름)
extension BarsVisibilityControlling {
    var prefersFloatingOverlayHidden: Bool? { nil }
    var prefersSystemTabBarHidden: Bool? { nil }
    var prefersToolbarHidden: Bool? { nil }
}
```

**프로토콜 채택 정책:**
- **일반 VC**: 채택 불필요 (TabBarController 기본 정책 적용)
- **특별한 요구사항 있는 VC만 채택**: ViewerVC처럼 floatingOverlay를 숨겨야 하는 경우

**TabBarController 기본 정책:**
- iOS 16~25: 시스템 탭바 숨김, floatingOverlay 표시, 툴바 숨김
- iOS 26+: 시스템 탭바 표시, 툴바 숨김

### 2. ViewerViewController에서 프로토콜 채택
**파일**: `ViewerViewController.swift`

```swift
extension ViewerViewController: BarsVisibilityControlling {
    var prefersFloatingOverlayHidden: Bool? { true }  // Viewer에서만 floatingOverlay 숨김
    // prefersSystemTabBarHidden, prefersToolbarHidden은 nil (TabBarController 기본 동작)
}
```

### 3. hidesBottomBarWhenPushed iOS 버전별 분기
**파일**: `ViewerViewController.swift` - init()

```swift
init(...) {
    // ...

    // iOS 16~25: hidesBottomBarWhenPushed 사용 안 함 (자동 복원 차단)
    // iOS 26+: 시스템 UX 유지
    if #available(iOS 26.0, *) {
        hidesBottomBarWhenPushed = true
    }
    // else: 기본값 false 유지
}
```

### 4. TabBarController에 UINavigationControllerDelegate 구현
**파일**: `TabBarController.swift`

```swift
// setupTabs()에서 delegate 설정
photosNavController.delegate = self
albumsNavController.delegate = self
trashNavController.delegate = self

// UINavigationControllerDelegate 확장
extension TabBarController: UINavigationControllerDelegate {

    /// willShow에서 전달되는 viewController는 "곧 보여질 VC"로, 정확한 대상임
    /// (topViewController는 전환 중에는 아직 이전 VC일 수 있음)
    func navigationController(_ navigationController: UINavigationController,
                              willShow viewController: UIViewController,
                              animated: Bool) {
        applyBarsVisibilityPolicy(for: viewController, in: navigationController)
    }

    private func applyBarsVisibilityPolicy(for viewController: UIViewController,
                                            in navigationController: UINavigationController) {
        let policy = viewController as? BarsVisibilityControlling

        if useFloatingUI {
            // ===== iOS 16~25 =====
            // 시스템 탭바: 항상 숨김 (floatingTabBar 사용)
            tabBar.isHidden = true

            // floatingOverlay: 정책이 있으면 적용, 없으면 표시
            if let floatingOverlay = floatingOverlay {
                floatingOverlay.isHidden = policy?.prefersFloatingOverlayHidden ?? false
            }
        } else {
            // ===== iOS 26+ =====
            // 시스템 탭바: 항상 표시 강제 (자동 복원 문제 방어)
            tabBar.isHidden = false
            // floatingOverlay는 iOS 26에서 nil이므로 처리 불필요
        }

        // 툴바: 정책이 있으면 적용, nil이면 기본 정책(숨김)
        let hideToolbar = policy?.prefersToolbarHidden ?? true
        navigationController.setToolbarHidden(hideToolbar, animated: false)
    }
}
```

### 5. 개별 VC에서 가시성 코드 제거
- `ViewerViewController.swift`: viewWillAppear/viewWillDisappear의 floatingOverlay.isHidden 제거
- `GridViewController.swift`, `TrashAlbumViewController.swift`, `AlbumGridViewController.swift`:
  개별 tabBar.isHidden 코드 제거 (있다면)

## iOS 버전별 동작 요약

| 항목 | iOS 16~25 | iOS 26+ |
|------|-----------|---------|
| 시스템 TabBar | 항상 숨김 | 항상 표시 (강제) |
| floatingOverlay | 정책 따름 (기본 표시) | 사용 안 함 (nil) |
| hidesBottomBarWhenPushed | false (자동 복원 차단) | true (시스템 UX) |

## 장점
- **안전한 기본값**: 프로토콜 미채택 시 아무것도 안 건드림 → 예기치 않은 동작 방지
- **iOS 26 방어**: `tabBar.isHidden = false` 강제로 자동 복원 문제 완전 차단
- **예방적 해결**: iOS 16~25에서 hidesBottomBarWhenPushed 미사용 → 자동 복원 자체 차단
- **단일 책임**: VC는 선언만, TabBarController가 실행
- **확장성**: 특별한 요구사항 있는 화면만 프로토콜 채택
- **재발 방지**: push/pop/탭 전환/백그라운드 복귀 모든 경로에서 일관된 동작
