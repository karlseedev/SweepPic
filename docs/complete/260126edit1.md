**날짜**: 2026-01-26
**브랜치**: 001-auto-cleanup
**상태**: 분석 완료, 수정 대기
**최종 검증**: 2026-01-26 완료

---

## 배경

사용자가 "연도별 정리"를 수행한 후 "이어서 정리" 버튼이 활성화되는 문제 발견.
스펙 문서 분석 결과, 이어서 정리의 동작 방식을 명확히 정의하고 구현 수정이 필요함.

---

## 스펙 정의 (확정)

### 메인 화면 "이어서 정리" 버튼
- **적용 대상**: `fromLatest` 또는 `continueFromLast` 정리만
- **활성화 조건**:
  - 이전 세션이 존재
  - `endReason`이 `maxFound`(50장) 또는 `maxScanned`(1000장)
  - `endReason`이 `endOfRange`(범위 끝)이면 비활성화

### 연도 선택 화면 "이어서" 버튼
- **적용 대상**: `byYear` 정리
- **표시 조건**:
  - 이전 세션이 `byYear(year)` 방식
  - `endReason`이 `maxFound` 또는 `maxScanned`
- **동작**: 해당 연도 내에서 `lastAssetDate`부터 이어서 탐색

// 변경 전 (475-487줄)
case .byYear(let year):
    let startOfYear = Calendar.current.date(from: DateComponents(year: year))!
    let endOfYear = Calendar.current.date(from: DateComponents(year: year + 1))!

    options.predicate = NSPredicate(

case .byYear(let year, let continueFrom):
    let startOfYear = Calendar.current.date(from: DateComponents(year: year))!
    let endOfYear = Calendar.current.date(from: DateComponents(year: year + 1))!

    if let fromDate = continueFrom {
        // 이어서: 해당 연도 + fromDate 이전
        options.predicate = NSPredicate(
            format: "(mediaType == %d OR mediaType == %d) AND creationDate >= %@ AND creationDate < %@ AND creationDate < %@",

            PHAssetMediaType.video.rawValue,
            startOfYear as NSDate,
            endOfYear as NSDate,
            fromDate as NSDate
        )
    } else {
        // 새로 시작: 해당 연도 전체

            PHAssetMediaType.video.rawValue,
