# PickPhoto 동영상 재생 기능 구현 계획

## 개요

뷰어에서 동영상 재생 기능 추가 (기본 사진 앱과 유사한 UX)

### 사용자 선택
- 자동 재생: ✅ (기본 음소거로 시작)
- 핀치 줌: ✅ (Aspect Fit ↔ Aspect Fill 토글)
- 컨트롤 UI: **AVPlayerViewController** (시스템 기본 컨트롤)

### 구현 범위
- ✅ 일반 동영상 재생
- ❌ 슬로모션 (1차 제외)
- ❌ 라이브포토 (1차 제외)

### 기본 사진 앱 동작 기준
- **무음 모드**: 자동 재생 시 기본 음소거, 사용자가 해제하면 세션 내 유지
- **페이지 스와이프**: 전환 완료 시(`didFinishAnimating`) 이전 비디오 정지
- **iCloud 다운로드**: 스피닝 인디케이터 표시, 진행률/재시도 UI 없음

```swift
public final class VideoPipeline {
    public static let shared = VideoPipeline()

    /// 기본 비디오 요청 옵션
    private lazy var defaultOptions: PHVideoRequestOptions = {
        let options = PHVideoRequestOptions()
        options.deliveryMode = .highQualityFormat
        options.isNetworkAccessAllowed = true  // iCloud 다운로드 허용
        options.version = .current
        return options
    }()

    /// AVPlayerItem 요청
    /// - Parameters:
    ///   - asset: PHAsset
    ///   - options: 요청 옵션 (nil이면 기본값 사용)
    ///   - completion: 결과 콜백
    ///     - playerItem: 성공 시 AVPlayerItem
    ///     - info: PHImageManager info 딕셔너리 (에러/취소/다운로드 상태 확인용)
    func requestPlayerItem(
        for asset: PHAsset,
        options: PHVideoRequestOptions? = nil,
        completion: @escaping (AVPlayerItem?, [AnyHashable: Any]?) -> Void
    ) -> Cancellable
}
```
- PHImageManager.requestPlayerItem 래핑
- ImagePipeline의 Cancellable 패턴 재사용
- **info 딕셔너리 노출**: 취소/다운로드 진행/에러 구분 가능
  - `PHImageCancelledKey`: 요청 취소 여부
  - `PHImageErrorKey`: 에러 객체
  - `PHImageResultIsInCloudKey`: iCloud 다운로드 필요 여부

## Phase 2: VideoPageViewController

### 2-1. 기본 구조
- **파일**: `/Users/karl/Project/Photos/iOS/PickPhoto/PickPhoto/Features/Viewer/VideoPageViewController.swift`

```swift
final class VideoPageViewController: UIViewController {
    let asset: PHAsset
    let index: Int

    private var playerViewController: AVPlayerViewController?
    private var player: AVPlayer?
    private var requestCancellable: Cancellable?

    // Lifecycle
    viewDidLoad()       → AVPlayerViewController embed, 오토레이아웃 설정
```

### 2-2. AVPlayerViewController embed
- AVPlayerViewController를 child VC로 추가
- **오토레이아웃으로 배치** (회전/세이프에어리어 대응):
```swift
playerVC.view.translatesAutoresizingMaskIntoConstraints = false
NSLayoutConstraint.activate([
    playerVC.view.topAnchor.constraint(equalTo: view.topAnchor),
    playerVC.view.leadingAnchor.constraint(equalTo: view.leadingAnchor),
    playerVC.view.trailingAnchor.constraint(equalTo: view.trailingAnchor),
    playerVC.view.bottomAnchor.constraint(equalTo: view.bottomAnchor)
])
```
- `entersFullScreenWhenPlaybackBegins = false` (인라인 재생)
- `exitsFullScreenWhenPlaybackEnds = false`

### 2-3. 줌 지원
- **AVPlayerViewController 기본 제공 (iOS 16+)**:
  - 핀치 아웃 → Aspect Fill (화면 채우기)
  - 핀치 인 → Aspect Fit (세이프 에어리어 내)
  - 더블탭 → Aspect 토글
- **제한사항**: 자유로운 확대/축소(1x~4x)는 미지원
  - 기본 사진 앱 수준의 자유 줌 필요 시 커스텀 구현 필요 (2차 구현 검토)
- **1차 구현**: AVPlayerViewController 기본 동작 사용

### 2-4. 오디오 처리

## Phase 3: ViewerViewController 통합

### 3-1. 미디어 타입 분기
- **파일**: `ViewerViewController.swift`
- **수정 위치**: `createPhotoViewController(at:)` → `createPageViewController(at:)`로 **메서드명 변경**

```swift
private func createPageViewController(at index: Int) -> UIViewController? {
    guard let asset = coordinator.asset(at: index) else { return nil }

    switch asset.mediaType {
    case .video:
        return VideoPageViewController(asset: asset, index: index)
    default:
        return PhotoPageViewController(asset: asset, index: index)
    }
}
```

### 3-2. 페이지 전환 시 처리
- `pageViewController(_:didFinishAnimating:)` 델리게이트에서:
  - 이전 페이지가 VideoPageViewController면 → `pause()`
  - 새 페이지가 VideoPageViewController면 → 자동 재생은 viewDidAppear에서 처리
- **willTransitionTo에서 선제 정지 불필요**: 스와이프 취소 시 계속 재생해야 함

### 3-3. 삭제 동작
- 기존 SwipeDeleteHandler 그대로 사용
- 동영상 삭제 시 재생 중지 후 삭제
