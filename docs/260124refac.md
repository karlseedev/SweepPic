# 네비게이션 바 버튼: 초기 설정 함수 재사용

**작성일**: 2026-01-24
**상태**: 적용 예정

---

## 1. 문제점

### 1.1 버튼 하나 수정에 4곳 변경 필요

```
버튼 구성 정의 위치:
├── 초기 설정
│   ├── iOS 26+: setupSystem...()
│   └── iOS 16~25: configureFloating...()
│
└── Select 복원
    ├── iOS 26+: restoreNavigationBarAfterSelectMode()
    └── iOS 16~25: exitSelectModeFloatingUI()
```

### 1.2 원인

"복원 시에는 버튼만 바꾸면 된다"는 최적화 의도로 분리했으나:
- 버튼 구성 변경 시 4곳 동기화 필요
- 실수하면 초기/복원 불일치 발생

---

## 2. 해결책: 초기 설정 함수 재사용

### 2.1 핵심 아이디어

복원에서 초기 설정 함수를 호출하여 동일한 버튼 구성 보장.

```swift
// Before: 복원에서 별도 코드
override func restoreNavigationBarAfterSelectMode() {
    navigationItem.rightBarButtonItem = selectButton  // 별도 구현
}

// After: 초기 설정 함수 재사용
override func restoreNavigationBarAfterSelectMode() {
    if #available(iOS 26.0, *) {
        setupCleanupButton()  // 초기 설정과 동일
    }
}
```

### 2.2 변경 효과

| 항목 | Before | After |
|------|--------|-------|
| 버튼 변경 시 수정 포인트 | 4곳 | **2곳** (iOS 버전별 초기 설정) |
| 신규 코드 | - | 없음 (기존 함수 재사용) |
| 초기/복원 동기화 | 수동 | **자동 보장** |

## 3. 적용 방법

### 3.1 Grid (GridSelectMode.swift)

```swift
// iOS 26+
override func restoreNavigationBarAfterSelectMode() {
    if #available(iOS 26.0, *) {
        setupCleanupButton()  // GridViewController+Cleanup.swift
    }
}

// iOS 16~25
override func exitSelectModeFloatingUI() {
    guard let tabBarController = tabBarController as? TabBarController,
          let overlay = tabBarController.floatingOverlay else { return }

    overlay.titleBar.exitSelectMode()
    overlay.tabBar.exitSelectMode(animated: true)

    // 초기 설정 함수 재사용
    configureFloatingOverlay()
}
```

### 3.2 Album (GridSelectMode.swift 내 AlbumGridViewController extension)

**주의**: 초기 설정 함수가 `private`이므로 접근 범위를 `fileprivate` 또는 `internal`로 변경 필요.

```swift
// iOS 26+
override func restoreNavigationBarAfterSelectMode() {
    if #available(iOS 26.0, *) {
        setupSystemNavigationBarForAlbum()  // 접근 범위 변경 필요
    }
}

// iOS 16~25
override func exitSelectModeFloatingUI() {
    guard let tabBarController = tabBarController as? TabBarController,
          let overlay = tabBarController.floatingOverlay else { return }

    overlay.titleBar.exitSelectMode()
    overlay.tabBar.exitSelectMode(animated: true)

    configureFloatingOverlayForAlbum()  // 접근 범위 변경 필요
}
```

### 3.3 Trash (TrashSelectMode.swift)

**주의**: 초기 설정 함수가 `private`이므로 접근 범위 변경 필요.

```swift
// iOS 26+
override func restoreNavigationBarAfterSelectMode() {
    if #available(iOS 26.0, *) {
        setupSystemNavigationBar()  // 접근 범위 변경 필요
    }
}

// iOS 16~25
override func exitSelectModeFloatingUI() {
    guard let tabBarController = tabBarController as? TabBarController,
          let overlay = tabBarController.floatingOverlay else { return }

    overlay.titleBar.exitSelectMode()
    overlay.tabBar.exitTrashSelectMode(animated: true)

    // delegate 복원 (필수)
    overlay.tabBar.delegate = overlay

    configureFloatingOverlayForTrash()  // 접근 범위 변경 필요
}
```

## 4. 리퀴드 글래스 UI 대응

### 4.1 왜 이 방식이 적합한가

리퀴드 글래스는 **컴포넌트 레벨 변경**:
```swift
// 현재
let button = UIBarButtonItem(title: "Select", ...)

// 리퀴드 글래스 적용 후
let button = LiquidGlassButton(...)  // 새 컴포넌트
```

### 4.2 변경 시 영향

| 방식 | 리퀴드 글래스 적용 시 |
|------|---------------------|
| 헬퍼 추상화 | 헬퍼 API 변경 + 호출부 확인 |
| **초기 설정 재사용** | **초기 설정 함수 내부만 수정** |

→ 컴포넌트 교체는 각 VC의 초기 설정 함수 안에서 처리, 복원은 자동 적용

## 7. 함께 수정해야 할 버그

### Grid: Select 종료 후 정리 버튼 누락

**현상**: Select 모드 종료 시 정리 버튼이 사라지고 Select 버튼만 남음

**원인**: GridViewController가 `restoreNavigationBarAfterSelectMode()`, `exitSelectModeFloatingUI()`를 오버라이드하지 않아 Base 기본 구현 사용

| 시점 | iOS 26+ | iOS 16~25 |
|------|---------|-----------|
| 초기 | `[Select, 정리]` | `setTwoRightButtons()` |
| 복원 (현재) | `[Select]` 만 | 버튼 복원 없음 |

**해결**: 이 리팩토링으로 복원에서 초기 설정 함수를 재사용하면 자동 수정됨
