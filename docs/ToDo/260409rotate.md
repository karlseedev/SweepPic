# 260409 Rotate Plan

## 목표

- 앱의 기본 정책은 `세로 고정`
- `ViewerViewController`에서만 `가로/세로` 지원
- iOS 16~25, iOS 26 경로를 모두 일관되게 처리

## 현재 상태

- orientation 설정은 현재 `project.pbxproj` build settings에서 관리 중
- `Info.plist`의 중복 orientation 키는 제거 완료
- 현재 정책은 `iPhone/iPad 모두 portrait only`
- 런타임에서 orientation을 제어하는 코드(`supportedInterfaceOrientations`, `shouldAutorotate`)는 없음

## 문제 구조

- 뷰어는 iOS 버전에 따라 진입 방식이 다름
  - iOS 26: `UINavigationController.pushViewController`
  - iOS 16~25: `present(viewerVC, animated: true)`
- 따라서 `ViewerViewController`만 수정해서는 모든 경로를 안정적으로 커버하기 어려움
- Apple rotation 정책상 앱 마스크와 top/root 컨테이너가 함께 orientation을 결정함

## 최적 방안

### 1. 앱 전체 지원 마스크는 다시 넓힌다

- build settings에서 orientation을 다시 확장
- 이유:
  - 앱 전체 마스크가 `portrait only`이면 viewer도 landscape 진입 불가

예상 방향:

- `iPhone`: `Portrait + LandscapeLeft + LandscapeRight`
- `iPad`: `Portrait + LandscapeLeft + LandscapeRight (+ 필요 시 UpsideDown)`

## 2. orientation 정책은 런타임 중앙 관리로 분리

- `AppDelegate.application(_:supportedInterfaceOrientationsFor:)` 추가
- 현재 표시 중인 최상단 화면 기준으로 orientation mask 반환

권장 정책:

- 기본: `.portrait`
- viewer: `.allButUpsideDown`

## 3. 컨테이너가 leaf VC 정책을 포워딩

- `TabBarController`에서 현재 활성 VC 기준으로 orientation 정책 전달
- 필요 메서드:
  - `supportedInterfaceOrientations`
  - `shouldAutorotate`
  - `preferredInterfaceOrientationForPresentation`

필요 시 `selectedViewController -> UINavigationController -> visible/topViewController` 순으로 추적

## 4. ViewerViewController만 예외 허용

- `ViewerViewController`에 orientation override 추가

예상:

- `supportedInterfaceOrientations = .allButUpsideDown`
- `shouldAutorotate = true`
- 필요 시 `preferredInterfaceOrientationForPresentation = .portrait`

## 5. 진입/복귀 시 orientation 재평가

- viewer 표시 직전/직후
- viewer 종료 직전/직후

필요 시 검토:

- `setNeedsUpdateOfSupportedInterfaceOrientations()`
- `UIViewController.attemptRotationToDeviceOrientation()`
- iOS 16+에서 더 강한 제어가 필요하면 `UIWindowScene.requestGeometryUpdate`

## 구현 우선순위

1. build settings를 viewer 가능 상태로 확장
2. `AppDelegate`에 중앙 orientation 정책 추가
3. `TabBarController`에 현재 화면 orientation 포워딩 추가
4. `ViewerViewController`에 예외 허용 추가
5. viewer 진입/복귀 시 회전 갱신 안정화

## 비추천 방안

- `ViewerViewController`에만 `supportedInterfaceOrientations` 추가
  - push 경로에서 컨테이너 정책과 충돌할 수 있음
- 모든 일반 화면에 portrait override를 개별 추가
  - 빠뜨릴 가능성이 높고 유지보수 나쁨
- `requestGeometryUpdate`만으로 해결
  - 회전 강제는 가능하지만 정책 위치로는 부적절

## 검증 포인트

- 보관함 -> viewer 진입 후 가로 회전 가능
- 앨범 -> viewer 진입 후 가로 회전 가능
- 삭제대기함 -> viewer 진입 후 가로 회전 가능
- viewer 종료 후 다시 일반 화면은 세로 고정 유지
- iOS 16~25 modal 경로에서 정상 동작
- iOS 26 push 경로에서 정상 동작
- face comparison, paywall, preview 등 다른 full-screen 화면은 여전히 세로 유지

## 결론

최적안은 `viewer만 특별취급`하되, 그 예외를 `개별 화면 땜질`이 아니라 `앱/컨테이너/뷰어` 3단 구조에서 중앙 관리하는 것이다.
