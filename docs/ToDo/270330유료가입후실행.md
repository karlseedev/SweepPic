## 1. Entitlements 복원

무료 개발자 계정에서 빌드 에러가 발생하여 주석 처리한 항목을 복원해야 합니다.

**파일**: `SweepPic/SweepPic/SweepPic.entitlements`

주석 해제할 내용:
```xml
<!-- Associated Domains -->
<key>com.apple.developer.associated-domains</key>
<array>
    <string>applinks:sweeppic.link</string>
</array>

<!-- Push Notifications -->
<key>aps-environment</key>
<string>development</string>
```

---

## 2. Apple Developer 설정

| 항목 | 설정 위치 | 용도 | 필요 시점 |
|------|----------|------|----------|
| Associated Domains | Apple Developer > Identifiers > App ID | `sweeppic.link` Universal Link 작동 | Phase 7~8 |
| APNs Key (.p8) | Apple Developer > Keys | 초대자 보상 도착 푸시 알림 | Phase 8 |
| ASC API Key | App Store Connect > Users and Access > Keys | Promotional Offer 서명 + Offer Code 관리 | Phase 3~ |

## 3. 미완료 테스트 — Phase 3 (초대 링크 생성 및 공유)

Phase 3 구현은 코드 작성 + 빌드 성공까지 완료했으나, 아래 테스트는 미실행 상태입니다.

### 3-1. Edge Function 배포 및 테스트

```bash
# Supabase CLI로 함수 배포
supabase functions deploy referral-api
```

배포 후 확인:
- [ ] POST `/functions/v1/referral-api/create-link` — 새 user_id로 요청 → 코드 생성 확인 (x0{6}9j 형식)
- [ ] 같은 user_id로 재요청 → 기존 코드 반환 확인 (중복 생성 안 됨)
- [ ] Rate limit 테스트 — 분당 5회 초과 요청 → HTTP 429 응답 확인
- [ ] 잘못된 요청 (user_id 누락) → 에러 응답 확인

### 3-2. UI 시각 확인 (시뮬레이터)

- [ ] `ReferralExplainViewController`를 모달로 띄워서:
  - 제목 "친구에게 초대하고 함께 프리미엄 받기!" 표시
  - 나/친구 보상 설명 행 배치
  - [초대하기] 버튼 + "이미 구독 중이어도 14일 무료 연장" 부가 문구
- [ ] 로딩 상태: 버튼 비활성화 + 스피너 표시
- [ ] 에러 상태: 빨간 에러 메시지 + "다시 시도" 버튼
- [ ] 닫기 버튼(X) 및 배경 탭으로 dismiss

### 3-3. 공유 시트 테스트

- [ ] [초대하기] 탭 → API 호출 → 공유 시트 표시
- [ ] 공유 메시지 5개 요소 확인:
  1. "SweepPic으로 사진 정리하고 14일 프리미엄 무료 받기!"
  2. "초대 코드: x0______9j"
  3. "아래 링크로 앱 설치" + "링크를 한 번 더 누르면 무료혜택 자동 적용"
  4. "앱 > 설정 > 초대 코드 입력에 붙여넣기"
  5. 공유 URL
- [ ] 공유 완료 → Push 프리프롬프트 진행
- [ ] 공유 취소 → 아무 동작 없음

### 3-4. Push 프리프롬프트 테스트

- [ ] `.notDetermined` 상태 → "친구가 가입하면 알려드릴까요?" 프리프롬프트 표시
  - [알림 받기] → 시스템 Push 권한 팝업
  - [닫기] → 닫힘
- [ ] `.denied` 상태 → 프리프롬프트 표시
  - [알림 받기] → "알림이 꺼져 있어요" + [설정으로 이동] / [나중에]
- [ ] `.authorized` 상태 → 프리프롬프트 미표시
- [ ] 2회째부터 프리프롬프트 미표시 (`hasAskedPushPermission` 동작)

## 4. 이후 Phase에서 유료 계정이 필요한 항목

| Phase | 필요 항목 | 이유 |
|-------|----------|------|
| Phase 3~ | ASC API Key | Promotional Offer 서명 생성 |
| Phase 4 | Offer Code | App Store Connect에서 Offer 생성 + 코드 발급 |
| Phase 7 | Associated Domains | Universal Link (`sweeppic.link/r/{code}`) 작동 |
| Phase 8 | APNs Key (.p8) | Push 알림 발송 |
| Phase 9 | ASC API | Offer Code 자동 보충 (offer-code-replenish) |
