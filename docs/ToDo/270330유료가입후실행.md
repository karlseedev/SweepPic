## 1. Entitlements 복원

무료 개발자 계정에서 빌드 에러가 발생하여 주석 처리한 항목을 복원해야 합니다.

**파일**: `SweepPic/SweepPic/SweepPic.entitlements`

주석 해제할 내용:
```xml
<!-- Associated Domains -->
<key>com.apple.developer.associated-domains</key>
<array>
    <string>applinks:sweeppic.link</string>
</array>

<!-- Push Notifications -->
<key>aps-environment</key>
<string>development</string>
```

| 항목 | 설정 위치 | 용도 | 필요 시점 |

### 3-1. Edge Function 배포 및 테스트


배포 후 확인:
- [ ] POST `/functions/v1/referral-api/create-link` — 새 user_id로 요청 → 코드 생성 확인 (x0{6}9j 형식)
- [ ] 같은 user_id로 재요청 → 기존 코드 반환 확인 (중복 생성 안 됨)

- [ ] 잘못된 요청 (user_id 누락) → 에러 응답 확인

### 3-2. UI 시각 확인 (시뮬레이터)

  - [초대하기] 버튼 + "이미 구독 중이어도 14일 무료 연장" 부가 문구

### 3-3. 공유 시트 테스트

- [ ] [초대하기] 탭 → API 호출 → 공유 시트 표시
- [ ] 공유 메시지 5개 요소 확인:

  5. 공유 URL
- [ ] 공유 완료 → Push 프리프롬프트 진행
- [ ] 공유 취소 → 아무 동작 없음

### 3-4. Push 프리프롬프트 테스트

## 5. 카카오톡 SDK 연동 (공유 시트 개선)

현재 iOS 기본 UIActivityViewController로 카카오톡에 공유하면, 카카오톡 공유 화면에서 메시지 미리보기가 표시되지 않는 문제가 있음 (카카오톡 share extension의 텍스트 처리 방식 한계).

### 해결: Kakao SDK 연동으로 카드형 공유

카카오 링크(Kakao Talk Sharing)를 사용하면 카드형 미리보기가 표시됨:
```
┌─────────────────────┐
│ [앱 아이콘/이미지]     │
│ SweepPic 초대         │
│ Pro멤버십 14일 무료... │
│ [초대 받기] 버튼       │
└─────────────────────┘
```

### 필요한 작업

1. **Kakao Developers 앱 등록** — [developers.kakao.com](https://developers.kakao.com)
2. **네이티브 앱 키 발급** — Kakao Developer Console에서 iOS 앱 등록
3. **Kakao SDK 설치** — SPM으로 `KakaoSDK` 추가
4. **메시지 템플릿 생성** — Kakao Developer Console에서 카드형 공유 템플릿 디자인
5. **ReferralShareManager 확장** — 카카오톡 설치 여부 감지 → 설치 시 Kakao Link, 미설치 시 기본 공유 시트

### 우선순위

- 릴리즈 필수는 아님 (기본 공유 시트로도 메시지 전달은 정상 동작)
- UX 개선 항목으로 출시 후 진행 가능

## 6. 미완료 테스트 — Phase 5 (초대자 보상 수령)

Phase 5 구현은 코드 작성 + 빌드 성공 + API 테스트 + 실기기 UI 테스트까지 완료했으나, 아래 항목은 유료 계정이 필요합니다.

### 6-1. Promotional Offer 실제 동작

- [ ] Supabase Vault에 `ASC_P8_KEY` 환경변수 설정 (P8 키 내용)
- [ ] `ASC_KEY_ID`, `BUNDLE_ID` 환경변수 확인
- [ ] 구독 중인 사용자로 claim-reward API 호출 → `reward_type: "promotional"` + 서명 반환 확인
- [ ] 클라이언트에서 서명으로 Product.purchase(options: [promotionalOffer]) 실행 → 14일 연장 적용 확인

### 6-2. Offer Code 실제 리딤

- [ ] App Store Connect에서 `referral_reward_01` Offer 생성 (14일 무료, pro_monthly 기반)

- [ ] 비구독자로 claim-reward → 리딤 URL → App Store 시트 → 구독 시작 확인

### 6-3. StoreKit Sandbox 테스트

- [ ] Sandbox 계정으로 Promotional Offer 구매 플로우 전체 확인
- [ ] 구매 성공 후 SubscriptionStore.refreshSubscriptionStatus() → isProUser 갱신 확인
- [ ] 구매 실패(취소) → 에러 메시지 + pending 상태 유지 확인
