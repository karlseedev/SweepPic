## 1. 솔루션 선정

### 1.1 최종 조합

| 역할 | 도구 | 비용 | 상태 |
|------|------|------|------|
| **사용 패턴 분석** | TelemetryDeck | 무료~99유로/월 | **확정** |
| **크래시 리포팅** | Firebase Crashlytics | 무료 무제한 | 기획서 유지 (별도 확정 필요) |
| **유입 분석** | Apple App Analytics | 무료 | 기획서 유지 (SDK 불필요) |
| **원시 데이터 저장** | Supabase | 무료~$25/월 | **미확정 (추후 검토)** |

### 1.2 TelemetryDeck 선정 근거

**프라이버시 우선 분석 도구 4종 비교 (iOS SDK + 퍼널/리텐션 지원 기준):**

| | TelemetryDeck | PostHog | Aptabase | Countly |
|--|:---:|:---:|:---:|:---:|
| **프라이버시 라벨** | Not Linked | 구성 복잡 | **가장 깔끔** | 구성에 따라 다름 |
| **퍼널 분석** | O | O | **X** | O |
| **리텐션 분석** | O | O | **X** | O |
| **무료 한도** | 100K/월 | **1M/월** | 20K/월 | 무제한(셀프호스팅) |
| **DAU 10K 비용** | ~42,000원 | ~0~200,000원 | ~14,000~55,000원 | ~260,000~480,000원 |
| **설정 난이도** | 매우 쉬움 | 보통 | 쉬움 | 어려움 |
| **서버 위치** | EU 전용 | US/EU 선택 | EU/US 선택 | 셀프호스팅/클라우드 |

> Plausible, Umami는 iOS SDK 미제공으로 탈락

**탈락 사유:**

| 도구 | 탈락 사유 |
|------|----------|
| **Aptabase** | 퍼널/리텐션 분석 불가 (사용자 식별자 없음) — 사진 정리 앱의 사용 패턴 이해에 필수 |
| **Countly** | 클라우드 비용 과다, 셀프호스팅은 서버 관리 부담 — 1인 개발에 부적합 |
| **PostHog** | 기능 최강이나 프라이버시 라벨 구성 복잡, SDK 무거움 |
| **Firebase Analytics** | IDFA 제거 가능하나(`FirebaseAnalyticsWithoutAdIdSupport`) Device ID·Usage Data·Coarse Location 여전히 수집 → "Data Not Collected" 불가, "Not Linked" 구성 복잡 |

**TelemetryDeck 선정 이유:**
- 프라이버시 라벨 "Not Linked to Identity" 선언 가능 → App Store 경쟁 우위
- ATT 팝업 불필요
- 퍼널/리텐션/코호트 분석 내장
- EU 서버 전용 (GDPR 네이티브 준수)
- Swift-first SDK, 설정 간편
- 1인 개발자에게 합리적인 가격

### 1.3 Supabase — 추후 검토 사항

현재 결정 사항 없음. 추후 필요 시 검토할 용도:

| 용도 | 설명 |
|------|------|
| 원시 데이터 저장 | TelemetryDeck에서 제공하지 않는 세부 쿼리가 필요할 때 |
| 커스텀 분석 | SQL로 자유롭게 분석하고 싶을 때 |
| 데이터 소유권 | 분석 데이터를 내 DB에도 보관하고 싶을 때 |

결정 시점: 출시 후 TelemetryDeck 대시보드만으로 부족하다고 느낄 때

## 2. 비용 시뮬레이션

### 2.1 시그널 수 재계산

세션 요약 방식 도입으로 기획서 원래 추정보다 시그널이 줄었다.

**세션당 시그널 추정:**

| # | 이벤트 | 세션당 시그널 |
|---|--------|------------|
| 1 | 앱 실행 | 1건 |
| 2 | 권한 | ~0건 (초기 1~2회) |
| 3 | 사진 열람 | 0~1건 |
| 4-1 | 보관함/앨범 삭제·복구 | 0~1건 |
| 4-2 | 휴지통 뷰어 | 0~1건 |
| 5-1 | 유사 분석 | 0~1건 |
| 5-2 | 유사 그룹 행동 | 0~3건 |
| 6 | 앱 오류 | 0~1건 |
| 7-1 | 기존 정리 | 0~1건 |
| 7-2 | 미리보기 정리 | 0~1건 |
| **합계** | | **약 5~8건/세션** |

**사용자당 월간 시그널:**

| 사용자 유형 | 일 세션 | 세션당 시그널 | 월간 시그널 |
|-----------|--------|------------|-----------|
| 활성 (DAU) | 2회 | ~6건 | ~360건 |
| 비활성 (월 2~3회) | 0.1회 | ~6건 | ~18건 |
| **MAU 가중 평균** | | | **~80건** |

> 산정 기준: MAU 평균 ~80 시그널/월, DAU:MAU = 1:3

### 2.3 기획서 대비 비용 변화

| | 기획서 추정 (매번 기록) | 확정 (세션 요약) | 차이 |
|--|---------------------|---------------|------|
| MAU당 시그널 | ~100건 | ~80건 | **20% 절약** |
| 무료 한도 DAU | ~330명 | ~400명 | **여유 증가** |

세션 요약 방식 도입으로 동일 사용자 수 대비 시그널이 줄어, 무료 한도를 더 오래 사용할 수 있다.

## 5. 사진 규모 구간 정의

사진 라이브러리 규모는 보관함(카메라롤)만 기준이며, 10단계 구간으로 전송한다.

| 구간 | 범위 |
|------|------|
| 1 | 0 ~ 500장 |
| 2 | 500 ~ 2,000장 |
| 3 | 2,000 ~ 10,000장 |
| 4 | 10,000 ~ 19,999장 |
| 5 | 20,000 ~ 29,999장 |
| 6 | 30,000 ~ 39,999장 |
| 7 | 40,000 ~ 59,999장 |
| 8 | 50,000 ~ 79,999장 |
| 9 | 80,000 ~ 109,999장 |
| 10 | 110,000장 이상 |

**설계 의도:**
- 정확한 수치 대신 구간으로 전송하여 개인 식별 방지
- 1만장 이상부터 촘촘하게 분류 — 사진 많은 사용자가 앱의 핵심 타겟이므로 세밀한 행동 분석 필요

## 6. 이벤트 상세

### 6.1 앱 실행

**수집 방식:** 매번 기록
**수집 시점:** 앱이 포그라운드로 활성화되는 순간

| 데이터 항목 | 설명 | 카운트 시점 |
|-----------|------|-----------|
| 실행 횟수 | 이번이 몇 번째 실행인지 | 앱 활성화 순간 |
| 사진 규모 구간 | 보관함 사진 수 기준 10단계 (섹션 3 참조) | 앱 활성화 순간 |

**분석 목적:**
- DAU/WAU/MAU 산출
- 리텐션 계산 (D1/D7/D30)
- 사진 규모별 사용자 분포 파악

---

### 6.2 사진 접근 권한

**수집 방식:** 매번 기록
**수집 시점:** 시스템 권한 팝업 응답 순간 / 설정에서 돌아온 순간

| 데이터 항목 | 설명 | 카운트 시점 |
|-----------|------|-----------|
| 결과 | 전체 허용 / 일부 허용 / 거부 | 권한 응답 순간 |
| 구분 | 최초 요청 / 설정 변경 | 권한 응답 순간 |

**분석 목적:**
- 권한 거부율 → 온보딩 화면 설득력 진단
- 설정 변경으로 돌아온 비율 → 재유도 UX 효과

---

### 6.3 사진 열람

**수집 방식:** 세션 요약 1건
**수집 시점:** 세션 종료 시 (백그라운드 진입)

| 데이터 항목 | 설명 | 카운트 시점 |
|-----------|------|-----------|
| 총 열람 수 | 세션 중 사진을 크게 본 총 횟수 | 뷰어가 열리는 순간 |
| 보관함에서 열람 | 메인 보관함 그리드에서 탭하여 연 횟수 | 뷰어가 열리는 순간 |
| 앨범에서 열람 | 앨범 상세 그리드에서 탭하여 연 횟수 | 뷰어가 열리는 순간 |
| 휴지통에서 열람 | 휴지통 그리드에서 탭하여 연 횟수 | 뷰어가 열리는 순간 |

**분석 목적:**
- 사용자가 주로 어디서 사진을 보는지 경로 비율 파악
- 안 쓰는 경로 개선 또는 제거 판단

### 6.4 삭제·복구

#### 4-1 보관함/앨범에서의 삭제·복구

**수집 방식:** 세션 요약 1건
**수집 시점:** 세션 종료 시 (백그라운드 진입)

| 데이터 항목 | 설명 | 카운트 시점 |
|-----------|------|-----------|
| 그리드 스와이프 삭제 | 그리드에서 스와이프하여 휴지통 이동한 횟수 | 스와이프 확정 순간 |
| 그리드 스와이프 복구 | 그리드에서 스와이프하여 복구한 횟수 | 스와이프 확정 순간 |
| 뷰어 스와이프 삭제 | 뷰어에서 스와이프하여 휴지통 이동한 횟수 | 스와이프 확정 순간 |
| 뷰어 휴지통 버튼 | 뷰어에서 휴지통 버튼으로 이동한 횟수 | 버튼 탭 순간 |
| 뷰어 복구 버튼 | 뷰어에서 복구 버튼으로 복구한 횟수 | 버튼 탭 순간 |
| 진입 경로 — 보관함 | 보관함에서 진입하여 삭제/복구한 총 횟수 | 각 행동 발생 순간 |
| 진입 경로 — 앨범 | 앨범에서 진입하여 삭제/복구한 총 횟수 | 각 행동 발생 순간 |

**분석 목적:**
- 핵심 기능(스와이프 삭제) 사용률
- 스와이프 vs 버튼 비율 → 스와이프 사용률 낮으면 제스처 안내 부족
- 복구 비율 → 높으면 실수 삭제가 많다는 신호

#### 4-2 휴지통 뷰어에서의 행동

**수집 방식:** 세션 요약 1건
**수집 시점:** 세션 종료 시 (백그라운드 진입)

| 데이터 항목 | 설명 | 카운트 시점 |
|-----------|------|-----------|
| 완전삭제 | 휴지통 뷰어에서 완전삭제한 횟수 | 버튼 탭 순간 |
| 복귀 | 휴지통 뷰어에서 보관함으로 복귀한 횟수 | 버튼 탭 순간 |

**분석 목적:**
- 휴지통에서 실제로 완전삭제하는 비율
- 복귀 비율 → 높으면 자동 정리 정확도 개선 필요

### 6.5 유사 사진

#### 5-1 분석

**수집 방식:** 세션 요약 1건
**수집 시점:** 세션 종료 시 (백그라운드 진입)

| 데이터 항목 | 설명 | 카운트 시점 |
|-----------|------|-----------|
| 분석 완료 횟수 | 유사 사진 분석이 완료된 횟수 | 분석 완료 순간 (취소된 건 제외) |
| 분석 취소 횟수 | 유사 사진 분석이 중간에 취소된 횟수 | 분석 취소 순간 |
| 발견된 총 그룹 수 | 완료된 분석에서 발견된 유사 그룹 총 수 | 분석 완료 순간 |
| 평균 소요시간 | 완료된 분석의 평균 소요 시간 (초) | 분석 완료 순간 |

**분석 목적:**
- 유사 사진 기능 사용 빈도
- 분석 취소율 → 높으면 분석 시간이 너무 길다는 신호
- 평균 소요시간 → 성능 개선 기준

#### 5-2 그룹 행동

**수집 방식:** 그룹별 1건
**수집 시점:** 얼굴 비교 화면이 닫히는 순간

| 데이터 항목 | 설명 | 카운트 시점 |
|-----------|------|-----------|
| 그룹 전체 장수 | 해당 유사 그룹의 전체 사진 수 | 화면 닫히는 순간의 최종 상태 |
| 삭제 장수 | 해당 그룹에서 삭제한 사진 수 (0이면 안 지움) | 화면 닫히는 순간의 최종 상태 |

**분석 목적:**
- 그룹별 삭제 비율 → 낮으면 UI 개선 또는 추천 정확도 문제
- "보기만 하고 안 지운" 그룹의 특성 파악 (그룹 크기와 삭제율 상관관계)

### 6.7 정리 기능

#### 7-1 기존 정리

**수집 방식:** 정리 종료 시 1건
**수집 시점:** 정리 흐름에서 빠져나오는 순간 (완료/취소/중간 이탈 모두 포함)

| 데이터 항목 | 설명 | 카운트 시점 |
|-----------|------|-----------|
| 도달 단계 | 버튼만 탭 / 휴지통 경고에서 이탈 / 방식 선택까지 / 정리 완료 / 결과 행동까지 | 흐름 이탈 순간 |
| 휴지통 경고 표시 여부 | 휴지통이 비어있지 않아 경고가 표시되었는지 | 경고 표시 순간 |
| 선택 방식 | 최신 / 이어서 / 연도별 (방식 선택 전 이탈이면 없음) | 방식 선택 순간 |
| 결과 | 완료 / 0장 발견 / 취소 (정리 미진행이면 없음) | 정리 종료 순간 |
| 발견 수 | 저품질로 판별되어 휴지통으로 이동한 사진 수 | 정리 종료 순간의 최종 값 |
| 소요시간 | 정리 분석 소요 시간 (초) | 정리 종료 순간 |
| 취소 시 진행률 | 취소한 경우 몇 % 진행 시점이었는지 | 취소 순간 |
| 결과 행동 | 확인 / 휴지통 보기 (결과 화면 미도달이면 없음) | 결과 화면 버튼 탭 순간 |

**분석 목적:**
- 퍼널 분석: 각 단계별 이탈률
- 인기 정리 방식 파악
- 취소 시점 → 분석이 너무 오래 걸리는지 진단
- 결과 행동 → 휴지통 확인 비율

#### 7-2 미리보기 정리

**수집 방식:** 미리보기 종료 시 1건
**수집 시점:** 미리보기 화면에서 빠져나오는 순간

| 데이터 항목 | 설명 | 카운트 시점 |
|-----------|------|-----------|
| 도달 단계 | 분석까지 / 그리드 표시 / 최종 행동까지 | 화면 이탈 순간 |
| 발견 수 | 분석에서 찾은 저품질 사진 수 | 분석 완료 순간 |
| 소요시간 | 분석 소요 시간 (초) | 분석 완료 순간 |
| 최종 도달 단계 | 완화 / 기본 / 강화 (어디까지 확장했는지) | 화면 닫히는 순간의 최종 상태 |
| 더보기 횟수 | "더 보기" 버튼을 누른 횟수 | 버튼 탭 순간 |
| 제외 횟수 | "제외하기" 버튼을 누른 횟수 | 버튼 탭 순간 |
| 뷰어 열람 횟수 | 미리보기 그리드에서 사진을 탭해 크게 본 횟수 | 뷰어 열리는 순간 |
| 최종 행동 | 휴지통 이동 / 닫기 | 화면 닫히는 순간 |
| 이동한 사진 수 | 실제 휴지통으로 이동한 사진 수 (닫기면 0) | 이동 완료 순간 |

**분석 목적:**
- 미리보기 기능 효과 측정 (기존 정리 대비)
- 단계 확장 패턴 → 기본 임계값이 적절한지 판단
- 뷰어 열람 → 사진을 직접 확인하는 비율 (신뢰도 지표)
- 더보기/제외 횟수 → 단계 시스템 UX 적절성
