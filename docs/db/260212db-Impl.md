### 2.6 권한 처리 위치 (이벤트 2)

**호출 흐름:**
```
PermissionViewController.requestPhotoAccess() (라인 285)
  → PermissionStore.requestAuthorization() (라인 90)
    → PhotoLibraryService.requestAuthorization() (라인 81)
      → PHPhotoLibrary.requestAuthorization(for: .readWrite)
```

**firstRequest vs settingsChange 구분:**

| timing | 판별 방법 | 코드 위치 |
|--------|----------|----------|
| `firstRequest` | `PermissionViewController.requestPhotoAccess()` 결과 수신 후 직접 호출 | PermissionViewController.swift:293 |
| `settingsChange` | `sceneWillEnterForeground`에서 `checkAndNotifyIfChanged()` 전후 상태 비교 | SceneDelegate.swift:230 |

> **⚠️ 중복 집계 주의:** `requestAuthorization()`도 내부적으로 `notifyChange`를 호출하므로, `handlePermissionChange`에 analytics를 넣으면 firstRequest에서도 settingsChange가 중복 발생한다.
>
> **해결책:** settingsChange 추적은 `handlePermissionChange`가 아닌, `sceneWillEnterForeground`에서 `checkAndNotifyIfChanged()` 호출 전후로 상태를 비교하여 변경이 있을 때만 직접 호출한다.
>
> ```swift
> // sceneWillEnterForeground에서:
> let before = PermissionStore.shared.currentStatus
> PermissionStore.shared.checkAndNotifyIfChanged()
> let after = PermissionStore.shared.currentStatus
> if before != after {
>     AnalyticsService.shared.trackPermissionResult(result: mapped(after), timing: .settingsChange)
> }
> ```
