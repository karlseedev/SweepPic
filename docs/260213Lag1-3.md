# 정리버튼 첫 클릭 랙 — v4 원인 추적

**날짜**: 2026-02-13
**이전 문서**: `docs/260212Lag1-2.md` (v1~v3 실패)
**증상**: 정리버튼 첫 클릭 시 ~2.4초 UI 프리즈 (iOS 18 / iOS 26 공통)

---

## 완료 기준

1. 최초 클릭 후 **0.5초 내** alert/시트가 표시된다
2. iOS hang detector에 hang이 보고되지 않는다
3. 2회차 이후와 체감 차이가 없다

---

## v1~v3 실패 요약

| 버전 | 접근 | 결과 |
|------|------|------|
| v1 | VisionExecutor로 Vision 호출 백그라운드화 | 효과 없음 |
| v2 | nonisolated + VisionExecutor + yield | 효과 없음 |
| v3 | CleanupService.shared 우회 + SessionStore 동기화 | 효과 없음 |

**공통 실패 원인**: 모두 `cleanupButtonTapped()` 내부 코드가 느리다고 가정했으나, 실제 코드 실행은 ~40ms에 불과. 랙은 우리 코드 밖에서 발생.

---

## v4 진단 과정

### 1단계: 코드 타이밍 측정

`cleanupButtonTapped()` 내부에 `CACurrentMediaTime` 타이밍 삽입.

**결과**:
```
[CleanupLag] isTrashEmpty: 24.3ms, methodSheet: 19.0ms, total: 43.3ms
Hang detected: 2.60s (debugger attached, not reporting)
```

우리 코드 **43.3ms** vs 시스템 행 **2.60초** → 2.56초는 우리 코드 밖에서 발생.

### 2단계: 첫 번째 present() 가설 검증

"선택" 버튼에 더미 UIAlertController를 추가하여 "정리" 보다 먼저 present() 호출.

**결과**:
- "선택" (첫 번째 present): **2.42초 행 발생**
- "정리" (두 번째 present): **38.3ms, 랙 없음**

→ **어떤 버튼이든 첫 번째 `present(alert)` 에서 랙 발생** 확인.

### 3단계: 원인 분리 테스트

| 테스트 | 내용 | Hang |
|--------|------|------|
| 원래 | UIAlertController, `animated: true` | **2.42s** |
| A | UIAlertController, `animated: false` | **0.87s** |
| B | 빈 UIViewController, `animated: true` | **없음** |

**결론**:
- `present()` 자체는 문제 없음 (B가 증명)
- 애니메이션 자체도 문제 없음 (B가 증명)
- **UIAlertController 고유의 문제** — `animated: false`에서도 0.87초

### 비용 분해

| 구성요소 | 시간 | 비율 |
|---------|------|------|
| UIAlertController 내부 초기화 (블러 등) | 0.87s | 36% |
| 블러 애니메이션 렌더링 (매 프레임) | 1.55s | 64% |
| **합계** | **2.42s** | 100% |

## 현재까지 확인된 사실

| 항목 | 확인 |
|------|------|
| iOS 18 / iOS 26 공통 | O |
| 첫 번째 present()에서만 발생 | O |
| 2회차부터 정상 | O |
| 우리 코드 실행 시간 | ~40ms |
| 시스템 행 시간 | ~2.4s |
| `present()` 자체가 원인 | X |
| `animated: true` 가 원인 | X (빈 VC는 빠름) |
| UIAlertController가 원인 | **O** |
| UIAlertController와 빈 VC의 차이 | **UIVisualEffectView (블러)** |

## 남은 의문

1. **왜 우리 앱에서만 UIAlertController 첫 사용이 느린가?**
   - 다른 앱에서 2.4초 걸리면 전 세계 앱이 느려야 함
   - 우리 앱 특유의 환경(뷰 계층, MTKView 등)이 블러를 느리게 만들 가능성

2. **UIVisualEffectView 블러의 어떤 단계가 느린가?**

   - 배경 캡처(스냅샷)?
   - GPU 렌더링?
