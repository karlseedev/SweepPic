# Feature Specification: 유사 사진 정리 기능

**Feature Branch**: `001-similar-photo`
**Created**: 2025-12-31
**Status**: Draft
**Input**: docs/prd8.md 기반 - 연속 촬영된 유사 사진 중 얼굴 표정을 비교하여 불필요한 사진을 쉽게 삭제할 수 있는 기능

---

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 그리드에서 유사 사진 발견 (Priority: P1)

사용자가 사진보관함 또는 앨범을 스크롤하다가 멈추면, 화면에 보이는 사진 중 유사사진썸네일그룹에 속하는 셀에 빛이 회전하는 테두리 애니메이션이 표시된다. 이를 통해 정리가 필요한 사진을 시각적으로 쉽게 인지할 수 있다.

**Why this priority**: 유사 사진을 발견하는 것이 기능의 시작점이며, 이 기능 없이는 다른 모든 기능이 무의미함

**Independent Test**: 연속 촬영된 얼굴 사진 3장 이상이 포함된 앨범에서 스크롤 후 멈추면 테두리 애니메이션이 표시되는지 확인

**Acceptance Scenarios**:

1. **Given** 사진보관함에 연속 촬영된 얼굴 사진 5장이 있음, **When** 스크롤하다가 0.3초 이상 멈춤, **Then** 해당 사진들에 빛이 회전하는 테두리 애니메이션이 표시됨
2. **Given** 그리드에 유사 사진 테두리가 표시됨, **When** 다시 스크롤을 시작함, **Then** 테두리 애니메이션이 즉시 해제됨
3. **Given** 화면에 풍경 사진만 보임 (얼굴 없음), **When** 스크롤 멈춤, **Then** 테두리가 표시되지 않음
4. **Given** 유사 사진이 2장만 있음, **When** 스크롤 멈춤, **Then** 테두리가 표시되지 않음 (3장 이상 필요)

---

### User Story 2 - 뷰어에서 유사사진정리버튼 표시 (Priority: P1)

사용자가 사진을 전체 화면으로 볼 때, 현재 사진 앞뒤 7장 범위 내에 3장 이상의 유사한 얼굴 사진이 있으면 우측 상단에 유사사진정리버튼이 표시된다.

**Why this priority**: 사진 정리의 핵심 진입점으로, 그리드 트리거와 함께 기능의 핵심 시작점

**Independent Test**: 유사 사진 3장 중 하나를 뷰어에서 열면 정리버튼이 표시되는지 확인

**Acceptance Scenarios**:

1. **Given** 뷰어에서 유사사진썸네일그룹(3장 이상)의 하나를 보고 있음, **When** 화면이 표시됨, **Then** 우측 상단에 유사사진정리버튼이 표시됨
2. **Given** 현재 사진에 얼굴이 없음, **When** 앞뒤 7장에 유사 사진이 있음, **Then** 버튼이 표시되지 않음
3. **Given** 유사사진정리버튼이 표시됨, **When** 다른 사진으로 스와이프, **Then** 새 사진 기준으로 조건 재평가하여 버튼 표시/숨김

---

### User Story 3 - 얼굴 위 + 버튼 표시 (Priority: P2)

사용자가 유사사진정리버튼을 탭하면, 현재 사진에서 감지된 얼굴 위에 + 버튼이 표시된다. 이를 통해 비교하고 싶은 인물을 선택할 수 있다.

**Why this priority**: 얼굴 비교 기능의 진입점이지만, 단순히 전체 사진을 비교하는 것보다 상세한 UX 제공

**Independent Test**: 유사사진정리버튼 탭 시 감지된 얼굴마다 + 버튼이 표시되는지 확인

**Acceptance Scenarios**:

1. **Given** 3명의 얼굴이 있는 사진에서 유사사진정리버튼 탭, **When** 얼굴 분석 완료, **Then** 3개의 + 버튼이 각 얼굴 위에 표시됨
2. **Given** 10명의 얼굴이 있는 사진, **When** 유사사진정리버튼 탭, **Then** 크기가 큰 상위 5개 얼굴에만 + 버튼 표시
3. **Given** 화면 너비의 3% 크기인 작은 얼굴이 있음, **When** 분석, **Then** 해당 얼굴에는 + 버튼 미표시 (5% 미만)
4. **Given** + 버튼들이 표시됨, **When** 유사사진정리버튼 다시 탭, **Then** + 버튼 오버레이 닫힘 (토글)
5. **Given** + 버튼 표시 중, **When** 다른 사진으로 스와이프, **Then** + 버튼 자동 해제

---

### User Story 4 - 얼굴 비교 화면에서 정리 (Priority: P2)

사용자가 특정 인물의 + 버튼을 탭하면 해당 인물의 얼굴만 크롭한 2열 그리드 화면이 표시된다. 여기서 삭제할 사진을 선택하고 Delete를 눌러 정리할 수 있다.

**Why this priority**: 핵심 가치 실현 - 얼굴 표정을 상세히 비교하여 불필요한 사진을 삭제

**Independent Test**: 인물 + 버튼 탭 → 얼굴 비교 화면 진입 → 사진 선택 → Delete로 삭제 완료

**Acceptance Scenarios**:

1. **Given** 인물 1의 + 버튼 탭, **When** 화면 전환, **Then** "인물 1 (N장)" 제목의 2열 그리드에 해당 인물 얼굴 크롭 이미지 표시
2. **Given** 얼굴 비교 화면에서 2장 선택, **When** Delete 탭, **Then** 2장이 앱 내 휴지통으로 이동, 뷰어로 복귀
3. **Given** 3장 선택 후, **When** Cancel 탭, **Then** 선택만 해제, 화면 유지
4. **Given** 순환 버튼(↻) 탭, **When** 인물 1 화면 표시 중, **Then** 인물 2 화면으로 크로스페이드 전환, 선택 상태 유지

---

### User Story 5 - 현재 사진 삭제 후 자동 이동 (Priority: P3)

얼굴 비교 화면에서 현재 보고 있던 사진을 삭제하면, 뷰어로 복귀 시 자동으로 이전/다음 사진으로 이동한다.

**Why this priority**: 삭제 후 빈 화면을 방지하는 UX 처리

**Independent Test**: 현재 사진 삭제 후 뷰어 복귀 시 적절한 사진이 표시되는지 확인

**Acceptance Scenarios**:

1. **Given** 현재 사진을 삭제함, **When** 뷰어로 복귀, **Then** 이전 사진으로 자동 이동
2. **Given** 첫 번째 사진을 삭제함, **When** 뷰어로 복귀, **Then** 다음 사진으로 자동 이동
3. **Given** 모든 사진을 삭제함, **When** 삭제 완료, **Then** 뷰어 닫힘, 그리드로 복귀

---

### User Story 6 - 접근성 지원 (Priority: P3)

VoiceOver 사용자 및 모션 감소 설정 사용자도 유사 사진 정리 기능을 동일하게 사용할 수 있다.

**Why this priority**: 접근성은 중요하지만 핵심 기능 완료 후 개선 가능

**Independent Test**: VoiceOver로 모든 기능에 접근 가능하고 적절한 라벨이 읽히는지 확인

**Acceptance Scenarios**:

1. **Given** VoiceOver 활성화, **When** 유사사진썸네일그룹 셀에 포커스, **Then** "유사 사진 있음" 음성 안내
2. **Given** 모션 감소 설정 활성화, **When** 유사 사진 발견, **Then** 정적 테두리로 대체 (애니메이션 없음)
3. **Given** VoiceOver 활성화, **When** 얼굴 비교 화면 진입, **Then** 모든 버튼/셀에 적절한 라벨 읽힘

---

### Edge Cases

- 유사 사진 분석 중 네이티브 사진 앱에서 사진이 삭제되면 분석 취소 및 유사사진썸네일그룹 무효화
- 빠른 스크롤 후 멈춤 반복 시 메모리 누수 없이 정상 동작
- Delete 진행 중 앱이 백그라운드로 전환되면 삭제 완료 후 정상 처리
- 유사 사진 중 일부에서만 특정 인물 얼굴이 감지되면 해당 사진만 비교에서 제외
- 뷰어와 그리드에서 분석 결과 불일치 시 (해상도 차이로 인해) 뷰어 기준 최종 판정

---

## Requirements *(mandatory)*

### Functional Requirements

**유사 사진 감지**
- **FR-001**: 시스템은 현재 사진 기준 앞 7장 + 뒤 7장 범위(총 14장) 내에서 유사 사진을 감지해야 함
- **FR-002**: 유사도 판정은 이미지 피처 벡터 기반 비교를 사용하며, 거리 10.0 이하를 유사로 판정 (0=동일, 0~5=거의 같음, 5~10=유사)
- **FR-002a**: Feature Print 생성 시 이미지 해상도 - **480x480** (성능과 정확도 균형)
- **FR-003**: 현재 사진 포함 3장 이상 유사해야 기능 활성화
- **FR-004**: 얼굴이 있는 사진만 유사사진정리 대상이 됨 (풍경/음식 등 제외)
- **FR-005**: 앱 내 휴지통의 사진은 분석 대상에서 제외

**그리드 트리거**
- **FR-006**: 스크롤 멈춤 후 0.3초 디바운싱 후 분석 시작
- **FR-007**: **유사사진썸네일그룹에 속하는 셀**에만 테두리 애니메이션 표시 (흰색 그라데이션이 사각형 테두리를 시계방향으로 회전)
- **FR-008**: 스크롤 재시작 시 분석 취소 및 테두리 해제
- **FR-009**: 테두리가 있든 없든 셀 탭 시 해당 사진의 뷰어로 이동

**뷰어 트리거**
- **FR-010**: **현재 사진이 유사사진썸네일그룹에 속하면** 우측 상단에 유사사진정리버튼 표시 (SF Symbols `square.stack.3d.up`, 반투명 원형 배경, 배지 없음)
- **FR-011**: 사진 스와이프 시 유사사진썸네일그룹 소속 여부 재평가하여 버튼 표시/숨김
- **FR-012**: 버튼 탭 시 0.5초 이내 얼굴 위 + 버튼 표시

**얼굴 + 버튼**
- **FR-013**: 뷰어에 표시되는 이미지 기준 화면 너비 5% 이상 크기의 얼굴만 감지 (5% 미만은 미검출 처리)
- **FR-014**: 최대 5개 얼굴까지 + 버튼 표시 (크기 큰 순 선택 후 위치순 재정렬)
- **FR-015**: + 버튼은 좌→우, 위→아래 순서로 인물 번호 부여 (x좌표 우선 정렬)
- **FR-016**: 버튼 겹침 시 자동 위치 조정 - 이동 순서: 좌→우→아래, 버튼 지름 1.2배씩 이동, 최대 4회 시도 후 현재 위치 유지
- **FR-017**: 유사사진정리버튼 재탭 또는 스와이프로 + 버튼 오버레이 닫힘
- **FR-017a**: + 버튼은 반투명 원형 배경에 + 아이콘, 기본 위치는 얼굴 위 중앙
- **FR-017b**: 동일 인물이 여러 얼굴로 감지되는 경우(정면+옆모습) 감지된 얼굴 그대로 처리 (별도 클러스터링 없음)

**얼굴 비교 화면**
- **FR-018**: 2열 정사각형 그리드로 얼굴 크롭 이미지 표시
- **FR-019**: 얼굴 크롭 시 bounding box 기준 30% 여백 추가
- **FR-020**: 헤더에 뒤로/인물명(N장)/순환 버튼 표시
- **FR-021**: 하단바에 Cancel/선택개수/Delete 표시
- **FR-022**: 인물 순환 시 선택한 사진 상태 유지
- **FR-023**: Delete 시 선택된 사진들 앱 내 휴지통으로 이동 후 뷰어 복귀
- **FR-024**: Cancel 시 선택만 해제, 화면 유지

**성능**
- **FR-025**: 그리드 스크롤 시 기기 네이티브 주사율 유지 (일반 60Hz, ProMotion 120Hz)
- **FR-026**: 테두리 10개 이상 동시 표시해도 성능 저하 없음
- **FR-027**: Vision 동시 분석 제한 - 그리드: 5개, 뷰어: 3개 (메모리 관리)
- **FR-027a**: 처리 시간 목표 (상세: [prd8algorithm.md](../../docs/prd8algorithm.md) §7.1)
  - Feature Print 생성: 50ms 이하 (480x480 해상도)
  - 얼굴 감지: 30ms 이하
  - 얼굴 크롭 Feature Print: 30ms 이하
  - 거리 계산: 1ms 이하

**그룹 유형**
- **FR-028**: 유사사진썸네일그룹 - 그리드 테두리/뷰어 버튼 표시용, 분석 범위 내 전체 (그리드: 화면+앞뒤7장, 뷰어: 앞뒤7장)
- **FR-028a**: 유사사진썸네일그룹 판정 조건 (모든 조건 충족 필요):
  1. 현재 사진에 얼굴 1개 이상 감지
  2. 앞뒤 7장 범위 내 Feature Print 거리 10.0 이하 유사 사진 존재
  3. 현재 사진 포함 유사 사진 3장 이상
  4. 유사사진썸네일그룹 내 얼굴 있는 사진 3장 이상 (얼굴 필터 적용 후)
- **FR-029**: 유사사진정리그룹 - 얼굴 비교 화면용, 최대 8장 제한
- **FR-030**: 유사사진정리그룹 선택 알고리즘 - 현재 사진 기준 인덱스 거리순, 동일 거리면 앞쪽 우선
- **FR-031**: 유사사진정리그룹 결정 시점 - 유사사진정리버튼 탭 시

**얼굴 비교 화면 상세**
- **FR-032**: 헤더 레이아웃 - 좌측: ← 뒤로 (뷰어 복귀), 중앙: "인물 N (M장)", 우측: ↻ 순환 버튼
- **FR-033**: 하단바 레이아웃 - 좌측: Cancel (흰색 텍스트), 중앙: "N개 선택됨" (전체 선택 사진 수), 우측: Delete (빨간색, 0개면 비활성화)
- **FR-034**: 인물 순환 시 크로스페이드 애니메이션 지속 시간: 0.3초
- **FR-035**: 얼굴 크롭 시 이미지 경계에 걸치면 클램핑 처리 (이미지 범위 내로 제한)
- **FR-035a**: Vision 좌표계 변환 - Vision boundingBox는 좌하단 원점, Y축 반전 필요 (`y = 1 - boundingBox.maxY`)

**iOS 버전별 UI**
- **FR-036**: iOS 16~25 - 커스텀 FloatingTitleBar/FloatingTabBar 사용
- **FR-037**: iOS 26+ - 시스템 네비바/툴바 사용 (Liquid Glass 자동 적용)
- **FR-038**: 기능적으로 동등하게 동작 (버튼 위치/동작 동일)

**예외 처리**
- **FR-039**: 화면 이동 시 (스와이프, 뒤로가기) 진행 중인 분석 취소
- **FR-040**: 사진 라이브러리가 3장 미만이거나 유사 사진이 3장 미만이면 기능 미표시
- **FR-041**: PhotoKit 권한 취소 시 기존 권한 없음 화면 표시 (분석 중단, 에러 안내)

**접근성**
- **FR-042**: VoiceOver 라벨 - 테두리: "유사 사진 있음", 유사사진정리버튼: "유사 사진 N장, 정리하기, 버튼", + 버튼: "인물 N 얼굴 비교, 버튼", 얼굴 크롭 셀: "사진 N, 선택됨/선택 안됨", 순환 버튼: "다음 인물, 버튼"
- **FR-043**: 모션 감소 설정 시 정적 테두리로 대체 (흰색, 애니메이션 테두리와 동일 스펙)
- **FR-044**: Dynamic Type - MVP 미지원 (추후 개선)

**인물 매칭**
- **FR-045**: 위치 기반 인물 매칭 사용 (x좌표 우선, 좌→우, 위→아래)
- **FR-046**: 인물 미검출 시 해당 사진만 비교에서 제외 (나머지 사진은 정상 표시)
- **FR-047**: 자동 매칭 검증 - 얼굴 크롭 Feature Print 거리 비교 (상세: [prd8algorithm.md](../../docs/prd8algorithm.md))
- **FR-048**: 검증 거리 임계값 - 0.6 이하(높음), 0.6~1.0(중간), 1.0 초과(낮음)
- **FR-049**: 신뢰도 낮음(거리 > 1.0) 발견 시 해당 사진에 경고 배지 표시 (⚠️ 노란색, 셀 우측 하단)
- **FR-049a**: 1개 이상의 사진에서 신뢰도 낮음 발견 시 경고 헤더 표시
  - 위치: 얼굴 비교 화면 상단
  - 내용: "⚠️ 인물 매칭 확인 필요 - 일부 사진에서 다른 사람이 포함되었을 수 있습니다"
  - [확인] 버튼 탭 시 닫힘, 개별 배지는 유지
- **FR-050**: 검증은 얼굴 비교 화면 진입 후 백그라운드에서 수행

### Key Entities

- **SimilarThumbnailGroup (유사사진썸네일그룹)**: 그리드 테두리 표시용, 최대 크기 제한 없음
- **SimilarCleanupGroup (유사사진정리그룹)**: 얼굴 비교 화면용, 최대 8장, 인덱스 거리순 선택
- **FaceRegion**: 사진 내 감지된 얼굴 영역 (bounding box, 크기 비율)
- **PersonComparison**: 특정 인물의 얼굴 크롭 이미지 목록 및 선택 상태
- **CroppedFace**: 얼굴 크롭 이미지 + Feature Print + 인물 번호 (상세: [prd8algorithm.md](../../docs/prd8algorithm.md) §4)
- **FaceMatch**: 인물 매칭 결과 (인물 번호, 얼굴 observation, Feature Print, 거리, 신뢰도)
- **ValidationResult**: 검증 결과 (isValid, 전체 신뢰도, 불일치 사진 목록, 상세 결과)

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 사용자가 연속 촬영 사진 중 원하지 않는 사진을 4탭 이내로 삭제할 수 있음 (유사사진정리버튼→+버튼→사진선택→Delete)
- **SC-002**: 스크롤 멈춤 시점부터 1초 이내 테두리 애니메이션 표시 (0.3초 디바운싱 포함)
- **SC-003**: 유사사진정리버튼 탭 후 0.5초 이내 + 버튼 표시
- **SC-004**: 5만 장 라이브러리에서도 그리드 스크롤 네이티브 주사율 유지 (테스트 환경: iPhone SE 2세대 이상, 인물 사진 60% / 풍경·음식 30% / 스크린샷 10%)
- **SC-005**: VoiceOver 사용자도 모든 기능에 접근 가능

## Assumptions

- 사용자의 사진 라이브러리에 접근 권한이 부여되어 있음
- 연속 촬영된 사진은 촬영 시간 순으로 정렬되어 있어 앞뒤 7장 범위 분석이 효과적임
- 이미지 피처 벡터 거리 10.0이 "유사 사진"의 적절한 임계값임 (추후 조정 가능)
- 위치 기반 인물 매칭이 연속 촬영 사진에서 충분히 정확함 (구도가 거의 동일)
- MVP에서는 분석 결과 캐싱 없이 실시간 분석 수행 (성능 이슈 발생 시 Phase 7에서 캐싱 추가)

## Scope Boundaries

### In Scope
- 사진보관함 및 앨범 상세 화면에서의 유사 사진 정리
- 얼굴이 있는 사진에 대한 유사 사진 감지 및 비교
- 앱 내 휴지통으로의 삭제 (휴지통 복구 지원)
- iOS 16~26+ 버전 지원 (버전별 UI 분기)

### Out of Scope
- 휴지통 화면에서의 유사 사진 정리 기능
- 풍경/음식 등 얼굴 없는 사진의 유사 사진 정리
- 선택 모드에서의 유사 사진 기능
- 실제 인물 인식/클러스터링 (MVP는 위치 기반 매칭)
- 유사 사진 자동 삭제 추천
