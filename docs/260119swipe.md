# FaceComparisonViewController 스와이프 인물 전환 구현 계획

## 문제 분석

### 이전 스냅샷 방식 실패 원인
- `snapshotView(afterScreenUpdates: true)` 호출 시점에 이미지 로딩 미완료
- `reloadData()` 후 `loadCroppedFaceImage()`가 비동기로 실행되어 타이밍 불일치
- 결과: 빈 셀(회색 배경)이 스냅샷에 캡처됨

### GPT 리뷰 피드백 반영
1. **High**: UIPageViewController로 바꿔도 빈 셀 문제 가능 → 플레이스홀더 방식으로 해결 (현재 회색 배경 + 스피너 추가 고려)
2. **Medium**: photoFaces/validPersonIndices 비동기 로딩 → 부모가 단일 소스로 관리, 페이지는 참조만
3. **Medium**: 선택 상태 불일치 → 페이지 전환 시 reloadData() 호출
4. **Low**: 수평/수직 제스처 충돌 → UIPageViewController 기본 처리로 해결

### 해결 방향
스냅샷 방식 대신 **UIPageViewController** 사용
- 스냅샷과 달리 실제 셀이므로 이미지 로드 완료 시 자동 업데이트
- **이전/다음 페이지 프리페치**로 스와이프 시 빈 화면 최소화
- 앱 내 ViewerViewController에서 이미 사용 중인 검증된 패턴

## 구현 방식

### 구조 변경

```
FaceComparisonViewController (컨테이너, 데이터 단일 소스)
├─ photoFaces: [String: [CachedFace]]     ← 부모가 관리
├─ selectedAssetIDs: Set<String>          ← 부모가 관리
├─ validPersonIndices: [Int]              ← 부모가 관리
│
├─ UIPageViewController (페이지 전환 담당)
│  └─ PersonPageViewController[] (각 인물별 페이지)
│     └─ UICollectionView (부모 데이터 참조)
│
└─ 하단바 (Cancel/Delete) - 모든 페이지 공유
```

### 수정 파일

1. **FaceComparisonViewController.swift** (리팩토링)
   - CollectionView 관련 코드를 PersonPageViewController로 이동
   - UIPageViewController 컨테이너로 변경
   - UIPageViewControllerDataSource/Delegate 구현
   - **데이터 단일 소스 유지** (photoFaces, selectedAssetIDs, validPersonIndices)

2. **PersonPageViewController.swift** (새로 생성)
   - 단일 인물(personIndex)의 그리드 표시
   - 기존 CollectionView 로직 이동
   - **부모의 데이터를 참조** (중복 로드 방지)
   - 셀 선택 시 부모에게 알림

### 주요 구현 포인트

1. **데이터 단일 소스 (부모 관리)**
   ```swift
   // FaceComparisonViewController
   private var photoFaces: [String: [CachedFace]] = [:]
   private var selectedAssetIDs: Set<String> = []
   private var validPersonIndices: [Int] = []
   ```

2. **PersonPageViewController가 부모 참조 (읽기 전용 + 콜백)**
   ```swift
   // PersonPageViewController
   weak var dataSource: FaceComparisonDataSource?

   protocol FaceComparisonDataSource: AnyObject {
       var photoFaces: [String: [CachedFace]] { get }
       var selectedAssetIDs: Set<String> { get }  // 읽기 전용
       func photosForPerson(_ personIndex: Int) -> [String]
       func asset(for assetID: String) -> PHAsset?
       func toggleSelection(for assetID: String)  // 선택 변경 API
   }
   ```

3. **선택 상태 변경 흐름 (콜백 기반)**
   ```
   셀 탭 → dataSource.toggleSelection(assetID)
        → 부모가 selectedAssetIDs 업데이트
        → 부모가 하단바 라벨 갱신
        → 부모가 현재 페이지에 reloadData() 호출
   ```

4. **데이터 로딩 완료 시 리로드**
   - `loadPhotoFaces()` 완료 → 현재 페이지 reloadData()
   - `loadValidPersonIndices()` 완료 → 페이지 재구성 + 타이틀 갱신

5. **validPersonIndices 비었을 때 처리**
   - 비어있으면 빈 화면 + "얼굴 없음" 메시지
   - 현재 personIndex가 없으면 첫 번째 유효 인물로 이동

6. **명시적 프리페치 (크롭 결과 캐시)**
   - UIPageViewController 기본 동작에 의존하지 않음
   - **loadPhotoFaces() 완료 후** 현재 + 인접 인물의 크롭 이미지 프리페치
   - **willTransitionTo에서** 전환 대상 인물의 크롭 이미지 사전 로드
   - **크롭 결과 캐시**: `[String: UIImage]` (assetID → 크롭된 이미지)
   - PHCachingImageManager + FaceCropper 결과까지 캐싱

7. **데이터 로딩 게이트**
   - photoFaces와 validPersonIndices 둘 다 완료 후 1회 페이지 구성
   - 완료 전에는 로딩 상태 표시
   ```swift
   var isDataReady: Bool {
       return !photoFaces.isEmpty && !validPersonIndices.isEmpty
   }
   ```

8. **"얼굴 없음" 표시**
   - validPersonIndices 비었을 때 placeholder VC 표시
   - 별도 EmptyStateViewController 또는 오버레이 레이블

## 구현 단계

### Step 1: PersonPageViewController 생성 (~400줄)
- FaceComparisonDataSource 프로토콜 정의
- CollectionView 관련 코드 이동
- 부모 데이터 참조 방식으로 구현
- 셀 선택 시 `dataSource.toggleSelection()` 호출

### Step 2: FaceComparisonViewController 리팩토링 (~200줄 변경)
- UIPageViewController 컨테이너로 변경
- DataSource/Delegate 구현
- FaceComparisonDataSource 프로토콜 구현
- 하단바는 기존 위치 유지
- `toggleSelection()` 구현 (selectedAssetIDs 업데이트 + UI 갱신)

### Step 3: 데이터 로딩 및 리로드 연결
- `loadPhotoFaces()` 완료 시 현재 페이지 reloadData()
- `loadValidPersonIndices()` 완료 시 페이지 재구성
- validPersonIndices 비었을 때 빈 화면 처리

### Step 4: 프리페치 및 크롭 캐시 구현
- 크롭 결과 캐시 `[String: UIImage]` 추가
- `loadPhotoFaces()` 완료 후 현재 + 인접 인물 프리페치 시작
- `willTransitionTo`에서 전환 대상 인물 프리페치
- 캐시 히트 시 즉시 셀 표시, 미스 시 비동기 로드

### Step 5: 제스처 우선순위 확인
- UIPageViewController 내부 스크롤과 CollectionView 수직 스크롤 충돌 확인
- 필요시 gestureRecognizer(_:shouldRecognizeSimultaneouslyWith:) 구현

### Step 6: 기존 기능 연결
- cycleButton 탭 → pageViewController.setViewControllers() 호출
- 디버그 버튼 등 기존 기능 유지
- iOS 26 시스템 네비게이션바 호환

## 검증 방법

1. 좌우 스와이프로 인물 전환 확인
2. 드래그 중 다음 인물 화면이 옆에서 보이는지 확인 (프리페치로 이미지 로드됨)
3. 프리페치 캐시 히트 시 즉시 표시 확인
4. 선택 상태가 페이지 전환 후에도 유지되는지 확인
5. 셀 선택 시 하단바 라벨 즉시 갱신되는지 확인
6. 하단바 Delete 버튼이 전체 선택에 대해 동작하는지 확인
7. iOS 26과 iOS 18에서 모두 테스트
8. 수직 스크롤과 수평 스와이프 제스처 충돌 없는지 확인
9. validPersonIndices 비었을 때 빈 화면 + 메시지 표시 확인
10. 데이터 로딩 완료 전 로딩 상태 표시 확인
