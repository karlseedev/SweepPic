# Visible Cells Transform 핀치줌 구현 계획 (통합본)

**작성일**: 2026-01-01
**목표**: 핀치 제스처 중 visible cells에 transform을 적용하여 Apple 사진앱에 근접한 UX 제공
**상태**: 설계 완료 ✅

---

## 설계 결정사항 (확정)

8가지 핵심 설계 항목에 대한 결정 (zoom1 + zoom2 비교 분석 결과 확정)

| # | 항목 | 결정 | 상세 |
|---|------|------|------|
| 1 | **스케일 매핑** | 이중 처리 | 임계값 판단=raw scale, 시각적 transform=클램프(0.85~1.15) |
| 2 | **점프 처리** | 동시 처리 | transform+layout을 UIView.animate 안에서 함께 처리 |
| 3 | **셀 겹침** | 무시 | scale>1.0 시 겹침 허용. 잠깐이라 문제 없음 |
| 4 | **앵커 추적** | 고정 | 핀치 시작 위치로 고정. 이동하지 않음 |
| 5 | **제스처 충돌** | 핀치 우선 | 핀치 중 드래그선택/스와이프삭제 무시 |
| 6 | **스크롤 방지** | isScrollEnabled | 핀치 시작 시 false, 종료 시 true |
| 7 | **패딩 핀치** | 허용 | 빈 영역에서도 핀치 가능. 화면 중앙을 앵커로 사용 |
| 8 | **Transform 분리** | 필수 | 핀치=cell.transform, 선택=contentView.transform |

### 스케일 이중 처리 (결정 #1 상세)

임계값 판단과 시각적 transform을 분리하여 처리:

- **임계값 판단**: `gesture.scale` (raw) 사용 → 정확한 전환 시점 결정
- **시각적 transform**: 클램프된 값(0.85~1.15) 사용 → 과도한 변형 방지

```swift
// 임계값 판단: raw scale 사용
if gesture.scale < 0.85 { performZoom(to: .zoomIn) }
else if gesture.scale > 1.15 { performZoom(to: .zoomOut) }

// 시각적 transform: 클램프된 scale 적용
let visualScale = min(max(gesture.scale, 0.85), 1.15)
applyTransformToVisibleCells(scale: visualScale)
```

**이유**: 0.7~1.3 범위는 임계값(0.85/1.15)과 괴리가 커서 전환 전 과도한 변형 발생. 임계값 근접 범위가 사진앱 느낌에 더 부합.

### Transform 분리 (결정 #8 상세)

**문제**: PhotoCell.updateSelectionUI()가 `self.transform` 사용 중 → 핀치 transform과 충돌

**해결**: 핀치와 선택 transform을 분리 적용
- 핀치: `cell.transform`
- 선택: `cell.contentView.transform`

```swift
// GridGestures.swift - 핀치 transform
cell.transform = CGAffineTransform(scaleX: scale, y: scale)...

// PhotoCell.swift - 선택 transform (수정 필요)
private func updateSelectionUI() {
    UIView.animate(withDuration: 0.15) {
        self.contentView.transform = self.isSelectedForDeletion
            ? CGAffineTransform(scaleX: 0.95, y: 0.95)
            : .identity
    }
}
```

## 1. 현재 구현 분석

### 1.1 현재 동작 방식

**파일**: `SweepPic/SweepPic/Features/Grid/GridGestures.swift`

```swift
// 현재 핀치줌 핸들러 (85-126줄)
@objc func handlePinchGesture(_ gesture: UIPinchGestureRecognizer) {
    switch gesture.state {
    case .began:
        // 앵커 assetID 저장
        pinchAnchorAssetID = ...

    case .changed:
        // 임계값 체크 (1.15/0.85)
        if scale > 1.15 { performZoom(to: zoomIn) }
        else if scale < 0.85 { performZoom(to: zoomOut) }

    case .ended:
        pinchAnchorAssetID = nil
    }
}
```

**문제점**:
- 임계값 도달 전까지 시각적 피드백 없음
- 핀치 동작이 "반응 없음 → 갑자기 전환"으로 느껴짐

### 1.2 현재 파라미터

| 파라미터 | 값 | 설명 |
|----------|-----|------|
| zoomInThreshold | 0.85 | scale < 0.85 → 확대 (열 수 감소) |
| zoomOutThreshold | 1.15 | scale > 1.15 → 축소 (열 수 증가) |
| cooldown | 200ms | 전환 간 최소 간격 |

### 2.2 Visible Cells Transform 방식

```
핀치 시작 (scale=1.0)
    ↓
핀치 진행 (scale 변화)
    ↓ visible cells에 transform 적용
    ↓ 앵커 중심으로 셀들이 확대/축소되는 효과
    ↓
임계값 도달 (scale < 0.85 또는 > 1.15)
    ↓
레이아웃 전환 + transform 초기화
```

## 3. 구현 설계

### 3.1 핵심 로직

```swift
// MARK: - Pinch Zoom with Visible Cells Transform

/// 핀치 줌 상태
struct PinchZoomState {
    var isActive: Bool = false
    var anchorPoint: CGPoint = .zero      // 핀치 중심점
    var anchorAssetID: String?            // 앵커 에셋 ID
    var initialScale: CGFloat = 1.0       // 시작 스케일
    var currentScale: CGFloat = 1.0       // 현재 스케일
    var lastAppliedScale: CGFloat = 1.0   // 마지막 적용 스케일 (스로틀링용)
}

@objc func handlePinchGesture(_ gesture: UIPinchGestureRecognizer) {
    switch gesture.state {
    case .began:
        beginPinchZoom(gesture)

    case .changed:
        updatePinchZoom(gesture)

    case .ended, .cancelled:
        endPinchZoom(gesture)

    default:
        break
    }
}
```

### 3.2 Phase별 구현

#### Phase 1: 핀치 시작 (began)

```swift
private func beginPinchZoom(_ gesture: UIPinchGestureRecognizer) {
    let location = gesture.location(in: collectionView)

    // 1. 앵커 저장 (결정 #4: 고정 - 이후 업데이트하지 않음)
    pinchState.anchorPoint = location

    // 2. 앵커 assetID 결정 (결정 #7: 패딩 영역이면 화면 중앙 사용)
    if let indexPath = collectionView.indexPathForItem(at: location),
       indexPath.item >= paddingCellCount {
        pinchState.anchorAssetID = assetIDForCollectionIndexPath(indexPath)
    } else {
        // 패딩 영역 또는 빈 공간 → 화면 중앙 셀을 앵커로
        let centerPoint = CGPoint(
            x: collectionView.bounds.midX,
            y: collectionView.bounds.midY + collectionView.contentOffset.y
        )
        pinchState.anchorPoint = centerPoint  // 앵커 포인트도 중앙으로 조정
        if let centerIndexPath = collectionView.indexPathForItem(at: centerPoint) {
            pinchState.anchorAssetID = assetIDForCollectionIndexPath(centerIndexPath)
        }
    }

    pinchState.initialScale = 1.0
    pinchState.lastAppliedScale = 1.0
    pinchState.isActive = true

    // 3. 스크롤 비활성화 (결정 #6: isScrollEnabled 사용)
    collectionView.isScrollEnabled = false

    // 4. 다른 제스처 비활성화 (결정 #5: 핀치 우선)
    swipeDeleteState.swipeGesture?.isEnabled = false
    dragSelectGesture?.isEnabled = false
}
```

#### Phase 2: 핀치 진행 (changed) - 핵심

```swift
private func updatePinchZoom(_ gesture: UIPinchGestureRecognizer) {
    guard pinchState.isActive else { return }

    // 결정 #1: 이중 처리 - raw scale과 visual scale 분리
    let rawScale = gesture.scale
    pinchState.currentScale = rawScale

    // 1. 임계값 체크 - raw scale 사용 (정확한 전환 시점)
    if shouldTransitionLayout(scale: rawScale) {
        transitionToNewLayout(scale: rawScale, gesture: gesture)
        return
    }

    // 2. 임계값 미도달 - 클램프된 visual scale로 transform 적용
    let visualScale = min(max(rawScale, 0.85), 1.15)
    applyTransformToVisibleCells(scale: visualScale)
}

private func applyTransformToVisibleCells(scale: CGFloat) {
    // 스로틀링: 스케일 변화가 미미하면 스킵 (0.01 미만)
    let scaleDelta = abs(scale - pinchState.lastAppliedScale)
    guard scaleDelta > 0.01 else { return }
    pinchState.lastAppliedScale = scale

    // 결정 #4: 앵커 고정 - pinchState.anchorPoint는 변경하지 않음
    let anchorPoint = pinchState.anchorPoint

    // CATransaction으로 암시적 애니메이션 비활성화
    CATransaction.begin()
    CATransaction.setDisableActions(true)

    // 결정 #3: 셀 겹침 무시 - zIndex 처리 없이 그대로 둠
    for cell in collectionView.visibleCells {
        guard let photoCell = cell as? PhotoCell else { continue }

        // 앵커 중심 기준 상대 위치 계산
        let cellCenter = cell.center
        let dx = cellCenter.x - anchorPoint.x
        let dy = cellCenter.y - anchorPoint.y

        // 결정 #1: 클램프된 scale 값을 transform에 적용
        let scaledDx = dx * (scale - 1)
        let scaledDy = dy * (scale - 1)

        // 결정 #8: 핀치 transform은 cell.transform에 적용
        // (선택 transform은 contentView.transform에 별도 적용됨)
        let transform = CGAffineTransform(scaleX: scale, y: scale)
            .translatedBy(x: scaledDx / scale, y: scaledDy / scale)
        photoCell.setPinchTransform(transform)
    }

    CATransaction.commit()
}
```

#### Phase 3: 핀치 종료 (ended)

```swift
private func endPinchZoom(_ gesture: UIPinchGestureRecognizer) {
    guard pinchState.isActive else { return }

    let scale = gesture.scale

    // 1. 임계값 도달 여부에 따라 분기
    if shouldTransitionLayout(scale: scale) {
        // 레이아웃 전환 (이미 처리됨)
    } else {
        // 원래 상태로 스냅백
        snapBackToOriginal()
    }

    // 2. 상태 초기화
    pinchState.isActive = false

    // 3. 스크롤 재활성화 (결정 #6)
    collectionView.isScrollEnabled = true

    // 4. 다른 제스처 재활성화 (결정 #5)
    updateSwipeDeleteGestureEnabled()  // 기존 메서드 활용
    dragSelectGesture?.isEnabled = isSelectMode
}

private func snapBackToOriginal() {
    // UIViewPropertyAnimator로 부드러운 스냅백 (스프링 효과)
    let animator = UIViewPropertyAnimator(
        duration: 0.3,
        dampingRatio: 0.8
    ) {
        for cell in self.collectionView.visibleCells {
            cell.transform = .identity
        }
    }
    animator.startAnimation()
}
```

### 3.3 레이아웃 전환 시 처리

```swift
private func transitionToNewLayout(scale: CGFloat, gesture: UIPinchGestureRecognizer) {
    // 1. 쿨다운 체크
    guard canPerformZoom() else { return }

    // 2. 새 열 수 결정
    let newColumns = scale < 0.85 ? currentColumnCount.zoomIn : currentColumnCount.zoomOut
    guard let columns = newColumns, columns != currentColumnCount else { return }

    // 3. 결정 #2: 동시 처리 - transform 초기화 + 레이아웃 전환을 함께 애니메이션
    UIView.animate(withDuration: 0.25) {
        // transform 초기화와 레이아웃 전환을 동시에 처리
        for cell in self.collectionView.visibleCells {
            cell.transform = .identity
        }

        // 레이아웃 전환
        self.currentColumnCount = columns
        self.updateCellSize()
        self.collectionView.setCollectionViewLayout(
            self.createLayout(columns: columns),
            animated: false
        )

        // 앵커 스크롤
        if let anchorIndexPath = self.anchorIndexPath() {
            self.collectionView.scrollToItem(
                at: anchorIndexPath,
                at: .centeredVertically,
                animated: false
            )
        }
    } completion: { [weak self] _ in
        guard let self = self else { return }

        // 상태 복원 (결정 #5, #6)
        self.collectionView.isScrollEnabled = true
        self.updateSwipeDeleteGestureEnabled()
        self.dragSelectGesture?.isEnabled = self.isSelectMode
        self.pinchState.isActive = false

        self.refreshVisibleCellsAfterZoom()
    }

    // 4. 스케일/상태 리셋
    pinchState.initialScale = 1.0
    gesture.scale = 1.0
    lastPinchZoomTime = Date()
}
```

## 4. 경계 케이스 처리

### 4.1 새로 나타나는 셀 처리 (정책 변경 대비)

> **현재 정책**: 핀치 중 스크롤 비활성화 (결정 #6)
> 따라서 신규 셀 처리는 기본적으로 불필요하나, 정책 변경 시 아래 코드 활용

**문제**: 스크롤 허용 정책으로 변경 시 새 셀이 transform 없이 나타남

**해결**: `willDisplayCell`에서 현재 핀치 상태 적용

```swift
// GridViewController+DataSource.swift
func collectionView(_ collectionView: UICollectionView,
                    willDisplay cell: UICollectionViewCell,
                    forItemAt indexPath: IndexPath) {
    // 기존 로직...

    // 핀치 중이면 transform 적용 (스크롤 허용 정책 시 활성화)
    if pinchState.isActive {
        let visualScale = min(max(pinchState.currentScale, 0.85), 1.15)
        applySingleCellTransform(cell, scale: visualScale)
    }
}

private func applySingleCellTransform(_ cell: UICollectionViewCell, scale: CGFloat) {
    let anchorPoint = pinchState.anchorPoint
    let cellCenter = cell.center
    let dx = cellCenter.x - anchorPoint.x
    let dy = cellCenter.y - anchorPoint.y
    let scaledDx = dx * (scale - 1)
    let scaledDy = dy * (scale - 1)

    cell.transform = CGAffineTransform(scaleX: scale, y: scale)
        .translatedBy(x: scaledDx / scale, y: scaledDy / scale)
}
```

### 4.2 스크롤 + 핀치 동시 발생 방지

**결정 #6 확정**: `isScrollEnabled` 토글 방식 사용

```swift
// 핀치 시작 시

// 핀치 종료 시
```

- Apple 사진앱에 근접한 동작
- 스크롤 중 핀치 시작하면 자동으로 스크롤 중단

## 6. UIViewPropertyAnimator 활용

### 6.1 스냅백 애니메이션 품질 개선

```swift
private var snapBackAnimator: UIViewPropertyAnimator?

private func snapBackToOriginal() {
    // 기존 애니메이터 취소
    snapBackAnimator?.stopAnimation(true)

    // 새 애니메이터 생성 (스프링 효과)
    snapBackAnimator = UIViewPropertyAnimator(
        duration: 0.3,
        dampingRatio: 0.8
    ) {
        for cell in self.collectionView.visibleCells {
            cell.transform = .identity
        }
    }

    snapBackAnimator?.addCompletion { _ in
        self.snapBackAnimator = nil
    }

    snapBackAnimator?.startAnimation()
}
```

## 12. 롤백 계획

### Feature Flag 기반 롤백

운영 리스크 감소를 위해 Feature Flag 방식 사용:

```swift
// FeatureFlags.swift
enum FeatureFlags {
    static var enableVisibleCellsTransform: Bool {
        // UserDefaults 또는 Remote Config 기반
        return UserDefaults.standard.bool(forKey: "pinch_transform_enabled")
    }
}

// GridGestures.swift
case .changed:
    // Feature flag 체크
    if FeatureFlags.enableVisibleCellsTransform {
        let visualScale = min(max(rawScale, 0.85), 1.15)
        applyTransformToVisibleCells(scale: visualScale)
    }
    // 임계값 체크는 항상 실행 (기존 동작)
    if shouldTransitionLayout(scale: rawScale) { ... }
```

### 롤백 기준

| 기준 | 임계값 | 조치 |
|------|--------|------|
| 프레임 드랍 | hitch 2f 이상 반복 | Flag off |
| 앵커 드리프트 | 20px 초과 | Flag off |
| 크래시 | 1회 발생 | Flag off + 조사 |

### 롤백 장점

- 앱 재배포 없이 서버에서 즉시 비활성화 가능
- A/B 테스트로 점진적 롤아웃 가능
- 문제 발생 시 즉각 대응 가능
