# iOS 26 Zoom Transition 계단현상 조사 및 시도 결과

## 문제 정의

- **증상**: iOS 26에서 그리드 → 뷰어 줌 트랜지션 시 이미지가 부드럽게 확대되지 않고 계단현상(stepping artifact) 발생
- **iOS 18**: 심하게 느껴지지 않음
- **iOS 26**: 심하게 느껴짐

---

## 원인 가설

| 가설 | 설명 | 확인 방법 |
|------|------|----------|
| **A. 프레임 드롭** | 애니메이션이 60/120fps로 재생 안 됨 | CADisplayLink, Instruments |
| **B. 저해상도 썸네일 확대** | 작은 이미지가 풀화면으로 확대되며 픽셀화 | 로그로 이미지 크기 비교 |

---

## 로그 확인 결과

```
[ZoomTransition] imageSize: 396x528px, screenPixelSize: 1206x2622px, scale: 3.0x
```

- 썸네일: 396x528px
- 화면: 1206x2622px
- 확대 비율: 약 3~5배

**초기 결론**: 저해상도 썸네일 확대가 원인으로 추정

---

## Apple Photos 앱 동작 조사

### 조사 방법
- WebSearch, Perplexity 리서치
- WWDC24 세션, Douglas Hill 블로그, Apple Developer 문서

### 조사 결과

1. **PHCachingImageManager 프리히트 사용**
   - 화면에 보이는 셀 + 주변 셀을 display-size로 미리 캐시
   - 스크롤하면서 캐시를 갱신

2. **전환 시작 시점에 고해상도 이미지 준비**
   - 썸네일을 그대로 확대하지 않음
   - 전환 직전에 display-size 이미지를 확보해둠

3. **opportunistic → highQuality 단계 교체**
   - 빠른 저해상도로 먼저 표시
   - 전환 시작 전/중에 고해상도로 교체

4. **전환 중 레이아웃/리로드 억제**
   - zoom transition 도중 셀 재구성 방지
   - sourceView 변경으로 인한 모션 끊김 방지

### 핵심 포인트
> **"전환 시작 시점에 sourceView의 이미지가 충분히 큰가"**

## 시도한 해결책

### 1차: 터치다운 시 display-size 프리페치

**구현 방식:**
```swift
func collectionView(_ collectionView: UICollectionView,
                    didHighlightItemAt indexPath: IndexPath) {
    prefetchZoomTransitionImageIfNeeded(at: indexPath)
}

private func prefetchZoomTransitionImageIfNeeded(at indexPath: IndexPath) {
    // display-size (화면 픽셀 크기) 요청
    let targetSize = CGSize(
        width: view.bounds.width * scale,
        height: view.bounds.height * scale
    )

    // 완료 시 셀 이미지 교체
    ImagePipeline.shared.requestImage(for: asset, targetSize: targetSize, ...) { image, _ in
        cell.thumbnailImageView.image = image
    }
}
```

**결과:**
- 섬네일이 더 큰 이미지(display-size)로 로드되는 것 확인
- **하지만 계단현상 여전히 발생**

## 현재 결론

**썸네일 크기 문제가 아님**

섬네일을 display-size로 교체해도 계단현상이 발생하므로, 다른 원인을 찾아야 함.

### 남은 가능성

| 가능성 | 설명 | 확인 방법 |
|--------|------|----------|
| **프레임 드롭** | 메인 스레드 히치로 애니메이션 끊김 | Instruments - Animation Hitches |
| **iOS 26 특유 문제** | Liquid Glass 등 새 렌더링과 충돌 | iOS 26 베타 릴리즈 노트 확인 |
| **Core Animation 보간** | 스케일 애니메이션 보간 방식 문제 | 커스텀 트랜지션으로 우회 |
| **sourceView 문제** | sourceViewProvider 호출 타이밍/빈도 | 로그로 호출 횟수/타이밍 확인 |

   - Core Animation 프레임 타이밍

   - 더 세밀한 제어 가능
