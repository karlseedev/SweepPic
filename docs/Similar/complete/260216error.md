# 유사사진 얼굴 그리드 — 선택 상태 동기화 버그 분석

## 현상
- 얼굴 비교 그리드에서 사진을 선택한 후, 스와이프로 다른 인물 페이지로 넘어가면 선택 상태가 바로 반영되지 않음
- 한두 번 왔다갔다 해야 반영되는 경우가 있음
- 순환 버튼으로 전환 시에는 항상 정상 (새 VC 생성)

## 근본 원인 (2가지)

### 원인 A: UIPageViewController prefetch
- UIPageViewController(scroll style)가 인접 VC를 미리 생성
- 인접 VC의 `cellForItemAt`이 **미리 생성 시점**의 `selectedAssetIDs`를 읽어 셀 구성
- 이후 현재 페이지에서 선택 변경해도, 이미 구성된 인접 VC의 셀에는 반영 안 됨
- 순환 버튼은 `navigateToPerson`에서 매번 새 VC 생성 → 항상 최신 데이터

### 원인 B: 비동기 이미지 로드 클로저의 캡처된 isSelected (핵심)

`PersonPageViewController.swift` `cellForItemAt` (213~227줄):

```swift
let isSelected = dataSource.isSelected(assetID)  // ← cellForItemAt 시점에 캡처

dataSource.loadFaceImage(assetID: assetID, personIndex: personIndex) { [weak cell] image in
    guard cell?.currentAssetID == assetID else { return }
    DispatchQueue.main.async {
        cell?.configure(with: image, isSelected: isSelected, ...)  // ← 캡처된 과거 값!
    }
}

cell.configure(with: nil, isSelected: isSelected, ...)  // ← 동기, 같은 캡처 값
```

- `isSelected`가 `cellForItemAt` 호출 시점에 로컬 변수로 고정됨
- 비동기 이미지 로드 완료 시에도 **캡처된 과거 값**으로 `configure` 재호출
- 이로 인해:
  - 인접 VC: 미리 생성 시 캡처된 false → 이후 선택해도 이미지 완료 시 false로 덮어씀
  - 현재 VC: 탭으로 setSelected(true) 한 후, 이미지 완료 시 캡처된 false로 덮어쓸 수 있음
- 이미지 로드 타이밍에 따라 발생 여부 달라짐 → **"경우가 있어"의 원인**

## 검토한 5가지 방안

### 방안 1: `didFinishAnimating`에서 `reloadData()`
- **동작**: 전환 완료 후 모든 셀 재생성
- **원인 A**: 전환 후 갱신 (애니메이션 중 stale)
- **원인 B**: 재생성하므로 그 시점은 OK, 하지만 새 클로저도 또 캡처
- **부작용**: 이미지 전체 재로드 → 깜빡임
- **확실성**: 확실히 동작

### 방안 2: `viewWillAppear`에서 `visibleCells` 갱신
- **동작**: VC가 나타나기 직전에 visible cells 순회
- **원인 A**: 불확실
- **원인 B**: 안 다룸
- **부작용**: 없음
- **확실성**: **불확실** — viewWillAppear 시점에 visibleCells 유효성 미보장

### 방안 3: `willDisplay cell`에서 선택 상태 갱신
- **동작**: 셀이 화면에 표시되기 직전마다 최신 상태 적용
- **원인 A**: 불확실
- **원인 B**: 안 다룸
- **부작용**: 없음
- **확실성**: **불확실** — UIPageViewController 스와이프는 collectionView 내부 스크롤이 아니므로 willDisplay 재호출 미보장

### 방안 4: `toggleSelection`에서 `pageViewController.children` 순회
- **동작**: 선택 변경 시 모든 자식 VC의 셀 직접 갱신
- **원인 A**: 불확실
- **원인 B**: 안 다룸
- **부작용**: 없음
- **확실성**: **불확실** — children이 인접 VC를 포함하는지 미보장 (문서화 안 됨)

### 방안 5: 이미지/선택 상태 분리 + didFinishAnimating 경량 갱신 (채택)
- **동작**: 비동기 클로저는 이미지만 설정 (선택 상태 안 건드림) + 전환 완료 시 visible cells 선택 갱신
- **원인 A**: didFinishAnimating에서 선택 상태 갱신
- **원인 B**: **해결** (비동기 클로저가 선택 상태를 아예 건드리지 않으므로 덮어쓸 수 없음)
- **부작용**: 없음 (이미지 로딩과 선택 상태가 분리되어 깜빡임 없음)
- **확실성**: 확실 (공식 API만 사용)

## 비교표

| | 원인 A (prefetch) | 원인 B (캡처 isSelected) | 부작용 | 동작 확실성 |
|---|---|---|---|---|
| 방안 1 | 전환 후 갱신 | 재생성이라 일시 OK | 이미지 깜빡임 | 확실 |
| 방안 2 | ? | X | 없음 | 불확실 |
| 방안 3 | ? | X | 없음 | 불확실 |
| 방안 4 | ? | X | 없음 | 불확실 |
| **방안 5** | **전환 후 갱신** | **해결 (클로저가 선택 상태 미접촉)** | **없음** | **확실** |

## 방안 5 커버리지

| 케이스 | 해결 메커니즘 |
|--------|------------|
| 이미지 로드가 선택 변경 **후** 완료 | 수정 1: 클로저가 `setImage()`만 호출 → 선택 상태 미접촉 → 덮어쓰기 불가 |
| 이미지 로드가 선택 변경 **전** 이미 완료 | 수정 3: didFinishAnimating에서 visible cells 선택 갱신 |
| 현재 페이지에서 탭 후 이미지 완료 시 덮어쓰기 | 수정 1: 클로저가 선택 상태를 건드리지 않으므로 덮어쓰기 불가 |
| 스와이프 취소 후 재시도 | 수정 1 + 수정 3 조합: 이미지 완료 시 선택 미접촉 + 다음 전환 완료 시 갱신 |
| 순환 버튼 전환 | 새 VC 생성이므로 보험 불필요 (cellForItemAt이 최신 데이터) |

한계: 스와이프 **애니메이션 중** ~0.3초 stale 가능 → 전환 완료 즉시 갱신 (스와이프 경로 기준)

## 구현 가이드

### 핵심 원칙: 비동기 클로저에서 선택 상태를 건드리지 않는다

이미지 로딩과 선택 상태 관리를 **완전히 분리**한다.
- 비동기 콜백(이미지 로드 완료)에서는 `setImage()`만 호출 — 선택 상태 미접촉
- 선택 상태는 동기 시점(cellForItemAt, didSelectItemAt)과 `refreshSelectionStates()`에서만 관리

이전 방식(콜백에서 `configure(image, isSelected)` 전체 호출)의 문제:
- 이미지 교체 + 선택 오버레이 변경이 동시에 일어나 **깜빡임** 발생
- 캡처된 `isSelected`든 최신 조회든, `configure()` 호출 자체가 시각적 부하

### 수정 0: FaceComparisonViews.swift — setImage() 메서드 추가

FaceComparisonCell에 이미지만 설정하는 경량 메서드를 추가한다.

```swift
/// 이미지만 설정 (선택 상태, debugText 등 기존 상태 유지)
/// 비동기 이미지 로드 완료 시 사용 — 선택 상태를 건드리지 않아 깜빡임 방지
func setImage(_ image: UIImage?) {
    imageView.image = image
}
```

위치: `configure()` 바로 아래, `setSelected()` 바로 위

### 수정 1: PersonPageViewController.swift — cellForItemAt 클로저 수정

**before** (버그):
```swift
let isSelected = dataSource.isSelected(assetID)  // 로컬 변수

dataSource.loadFaceImage(assetID: assetID, personIndex: personIndex) { [weak cell] image in
    guard cell?.currentAssetID == assetID else { return }
    DispatchQueue.main.async {
        cell?.configure(with: image, isSelected: isSelected, ...)
        //                        ^^^^^^^^^^^ 이미지 + 선택 상태 동시 변경 → 깜빡임
    }
}

cell.configure(with: nil, isSelected: isSelected, ...)  // 동기 placeholder
```

**after** (수정):
```swift
let isSelected = dataSource.isSelected(assetID)  // 로컬 변수 유지 (placeholder용)

dataSource.loadFaceImage(assetID: assetID, personIndex: personIndex) { [weak cell] image in
    DispatchQueue.main.async {
        guard let cell = cell,
              cell.currentAssetID == assetID else { return }
        cell.setImage(image)  // 이미지만 설정, 선택 상태 미접촉
    }
}

cell.configure(with: nil, isSelected: isSelected, ...)  // 동기 placeholder — 변경 없음
```

변경 포인트:
- `configure()` 대신 `setImage()` 호출 → 선택 상태를 건드리지 않음
- `[weak self]` 불필요 (dataSource 접근 안 함)
- 셀 재사용 가드를 `DispatchQueue.main.async` **안**으로 이동

### 수정 2: PersonPageViewController.swift — refreshSelectionStates() 추가

```swift
/// 화면에 보이는 셀의 선택 상태를 최신 값으로 갱신합니다.
/// 이미지 재로드 없이 체크마크만 갱신하므로 깜빡임이 없습니다.
func refreshSelectionStates() {
    guard let dataSource = dataSource else { return }
    for cell in collectionView.visibleCells {
        if let faceCell = cell as? FaceComparisonCell,
           let assetID = faceCell.currentAssetID {
            faceCell.setSelected(dataSource.isSelected(assetID))
        }
    }
}
```

### 수정 3: FaceComparisonViewController.swift — didFinishAnimating에서 호출

```swift
func pageViewController(
    _ pageViewController: UIPageViewController,
    didFinishAnimating finished: Bool,
    previousViewControllers: [UIViewController],
    transitionCompleted completed: Bool
) {
    guard completed,
          let currentPage = pageViewController.viewControllers?.first as? PersonPageViewController else {
        return
    }

    if let index = validPersonIndices.firstIndex(of: currentPage.personIndex) {
        currentPersonArrayIndex = index
    }

    updateTitleBar()
    currentPage.refreshSelectionStates()  // ← 추가: 선택 상태 경량 갱신
}
```

참고: `didFinishAnimating`은 gesture 전환에만 호출됨 (Apple SDK 헤더 명시).
순환 버튼(`setViewControllers`)은 매번 새 VC를 생성하므로 이 보험이 불필요.
