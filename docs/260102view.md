# PhotoPageViewController 성능 개선 계획

## 문제 현상

좌우 스와이프 시 사진이 부드럽지 않고 세로로 반쯤 끊겨서 보임. 비디오는 정상 동작.

---

## 근본 원인 분석

### 로그 기반 타임라인

```
t=0ms     willTransition          ← 전환 시작
t=20ms    1차 이미지 로드 완료      ← frame (0,0,0,0) → (390,520) 변경
t=55ms    2차 원본 이미지 로드      ← updateImageLayoutPreservingZoom 호출
t=277ms   didFinishAnimating      ← 전환 완료
```

**문제**: 전환 애니메이션 중(0~277ms)에 레이아웃이 2번 변경됨

### PhotoPageViewController vs VideoPageViewController 비교

| 항목 | PhotoPageViewController (문제) | VideoPageViewController (정상) |
|------|-------------------------------|-------------------------------|
| **배경색** | `.clear` (투명) | `.black` (검정) |
| **1차 품질** | `.high` (고품질만) | `.fast` (opportunistic) |
| **이미지 로드 시점** | `viewDidLayoutSubviews` (전환 중) | `viewDidLoad` (전환 시작) |
| **초기 frame** | `(0, 0, 0, 0)` → 이미지 로드 후 변경 | 포스터가 이미 설정됨 |
| **고해상도 요청** | 즉시 시작 (1차 이미지 로드 후) | `didFinishAnimating` 후 |

### 핵심 원인 3가지

1. **투명 배경** (`.clear`): 인접 페이지의 레이아웃 변경이 비쳐 보임
2. **고품질 우선** (`.high`): degraded 없이 20~30ms 대기, 즉시 표시 불가
3. **전환 중 풀사이즈 요청**: 55ms에 원본 로드 → 레이아웃 갱신 발생

---

## 해결 방향: LOD(Level of Detail) 파이프라인

### Apple Photos 앱 동작 방식

1. PHCachingImageManager로 인접 페이지 이미지를 **미리 캐싱**
2. 페이지 전환 **시작 전**에 이미 이미지가 메모리에 있음
3. **저해상도 → 고해상도** 다단계 로딩 (즉시 표시 후 개선)
4. 풀사이즈는 **줌 시에만** 요청

### 현재 문제점

```swift
// PhotoPageViewController.swift - 현재 구조
override func viewDidLayoutSubviews() {
    requestImageForCurrentBoundsIfNeeded()  // ← 전환 중에 호출됨
}

// 1차도 .high → degraded 없이 20~30ms 기다림
quality: .high

// 2차는 항상 풀사이즈 → 전환 중에도 요청
private func requestFullSizeImage() {
    targetSize: PHImageManagerMaximumSize  // ← 항상 최대 크기
}
```

## 최종 통합안

### LOD 파이프라인 구조

```
┌─────────────────────────────────────────────────────────┐
│                    LOD 파이프라인                        │
├─────────────────────────────────────────────────────────┤
│ LOD0: viewDidLoad                                       │
│       - quality: .fast (opportunistic)                  │
│       - targetSize: 화면 크기                            │
│       - 즉시 표시 (degraded 허용)                        │
├─────────────────────────────────────────────────────────┤
│ LOD1: didFinishAnimating (현재 페이지만)                 │
│       - quality: .high                                  │
│       - targetSize: 화면 크기                            │
├─────────────────────────────────────────────────────────┤
│ LOD2: scrollViewWillBeginZooming (줌 시작 시)            │
│       - quality: .high                                  │
│       - targetSize: PHImageManagerMaximumSize           │
└─────────────────────────────────────────────────────────┘
```

### 전환 관리

```
┌─────────────────────────────────────────────────────────┐
│                    전환 관리                             │
├─────────────────────────────────────────────────────────┤
│ willTransition:                                         │
│   1. 인접 페이지 LOD0 프리페치                           │
│   2. in-flight LOD1/LOD2 요청 취소/보류                  │
├─────────────────────────────────────────────────────────┤
│ didFinishAnimating:                                     │
│   1. 현재 페이지 LOD1 요청                               │
│   2. 이전 페이지 요청 취소                               │
└─────────────────────────────────────────────────────────┘
```

### 기본 설정

```
┌─────────────────────────────────────────────────────────┐
│                    기본 설정                             │
├─────────────────────────────────────────────────────────┤
│ view.backgroundColor = .black                           │
│ imageView 초기 이미지 = nil (검정 배경 위)              │
└─────────────────────────────────────────────────────────┘
```

## 구현 계획

### Phase 1: 즉각적 개선

1. **배경색 변경**: `.clear` → `.black`
2. **1차 품질 변경**: `.high` → `.fast`
3. **포스터 로드 위치 변경**: `viewDidLayoutSubviews` → `viewDidLoad`

### Phase 2: LOD1 분리

4. **LOD1 요청 타이밍**: `didFinishAnimating` 후 현재 페이지만
5. **ViewerViewController에서 트리거**: `requestHighQualityIfNeeded()` 추가

### Phase 3: LOD2 최적화

6. **풀사이즈 조건 변경**: 항상 → `zoomScale > 1.0` 시작 시점에만
7. **scrollViewWillBeginZooming에서 트리거**

### Phase 4: 전환 관리

8. **willTransition에서 프리페치**: 인접 페이지 LOD0
9. **in-flight 요청 취소**: 전환 시작 시 LOD1/LOD2 취소

## 예상 효과

| 개선 항목 | Before | After |
|----------|--------|-------|
| 전환 중 프레임 변경 | 2회 (20ms, 55ms) | 0회에 가깝게 |
| 빈 프레임 노출 | 20ms (0,0,0,0) | 최소화 |
| 인접 페이지 비침 | Yes (.clear) | No (.black) |
| 불필요한 풀사이즈 요청 | 모든 페이지 | 줌한 페이지만 |
| 메모리/배터리 효율 | 낮음 | 높음 |

## 주의사항

1. **viewDidLoad 타이밍**: UIPageViewController가 인접 페이지를 전환 시작 시점에 생성하므로, viewDidLoad도 전환 중에 호출될 수 있음. 다만 `.fast` + 검정 배경이라 비용이 훨씬 작음.

2. **프리페치 완료 보장**: 포스터가 전환 전에 완전히 로드되어야 프레임 변경 0회 달성. willTransition에서 프리페치해도 전환 중 도착할 수 있음.

3. **줌 시 풀사이즈 지연**: LOD2를 줌 시작 시점으로 미루면 줌 시작 시 약간의 지연 발생 가능. 허용 가능한 수준인지 테스트 필요.
