# PRD: 자동 정리 기능 - 저품질 사진 정의 (Low Quality Definition)

**목적**: 사용자가 '정리' 버튼을 눌렀을 때 선별될 **'저품질 사진'의 구체적이고 기술적인 기준**을 정의한다. 오탐지(False Positive)를 최소화하여 사용자의 소중한 추억을 보호하는 것을 최우선으로 한다.

---

## 1. 저품질 사진 판별 기준 (Criteria)

다음 3가지 카테고리를 순차적으로 검사하여 '저품질'로 판정한다.

### 1.1 스크린샷 (Screenshots)
가장 명확한 정리 대상. 사용자가 임시 정보 저장을 위해 캡처한 이미지.

- **판별 방법**: `PHAsset` 메타데이터 활용
- **조건**: `asset.mediaSubtypes.contains(.photoScreenshot)`
- **신뢰도**: **매우 높음 (100%)**

### 1.2 어두운 사진 (Underexposed / Dark)
식별이 불가능할 정도로 광량이 부족한 사진. (단, 의도적인 로우키 촬영은 제외하기 위해 아주 낮은 임계값 적용)

- **판별 방법**: 썸네일 이미지의 픽셀 휘도(Luminance) 분석
- **알고리즘**:
  1. 이미지를 작은 크기(예: 64x64)로 다운샘플링.
  2. Grayscale로 변환.
  3. 전체 픽셀의 **평균 휘도(Average Luminance)** 계산 (0~255 범위).
- **임계값 (Threshold)**: **평균 휘도 < 30** (약 12% 미만 밝기)
- **신뢰도**: 높음

### 1.3 흔들린 사진 (Blurry)
초점이 맞지 않거나 모션 블러로 인해 피사체의 윤곽이 흐릿한 사진.

- **판별 방법**: Laplacian 연산의 분산 (Variance of Laplacian)
- **알고리즘**:
  1. 이미지를 Grayscale로 변환.
  2. OpenCV 또는 Accelerate 프레임워크를 사용하여 Laplacian 필터 적용 (Edge 검출).
  3. 결과 이미지 픽셀값의 **분산(Variance)** 계산.
  4. 분산값이 클수록 Edge가 선명함(Sharp). 작을수록 흐릿함(Blur).
- **임계값 (Threshold)**: **Variance < 100.0**
- **신뢰도**: 중간 (심도 효과 사진 등 예외 처리 필수)

---

## 2. 예외 처리 (Exclusion Rules) - '삭제 불가' 조건

위 기준에 해당하더라도, 다음 조건 중 하나라도 만족하면 **절대로 '저품질'로 분류하지 않는다.**

1.  **즐겨찾기 (Favorite)**: `asset.isFavorite == true`
2.  **편집됨 (Edited)**: 사용자가 후보정을 거친 사진 (`asset.hasAdjustments` 등 확인 필요)
3.  **심도 효과 (Depth Effect)**: 인물 모드 등으로 촬영된 사진은 배경이 흐릿하므로 Blur 오탐지 가능성 높음.
    - 조건: `asset.mediaSubtypes.contains(.depthEffect)`
4.  **얼굴이 선명한 경우 (Face Saved)**: 배경은 흐려도(Blurry) 인물은 선명할 수 있음.
    - 조건: Vision 프레임워크 `VNDetectFaceCaptureQualityRequest` 점수가 **0.4 이상**인 얼굴이 하나라도 감지된 경우.

---

## 3. 탐색 및 판별 알고리즘 (Algorithm)

'정리' 버튼 탭 시 수행되는 탐색 로직.

**입력**: 전체 사진 라이브러리 (오래된 순 정렬)
**출력**: 저품질 사진 후보 50장

1.  **Batch Fetch**: 가장 오래된 사진부터 100장씩 Fetch (메모리 효율 고려).
2.  **1차 필터 (Metadata)**:
    - `isFavorite` 체크 → True면 Pass.
    - `isScreenshot` 체크 → True면 **[저품질 후보 추가]**.
    - `mediaSubtypes`에 `.depthEffect` 포함 → Pass (Blur 검사 생략).
3.  **2차 필터 (Pixel Analysis)** - *스크린샷이 아니고, 심도 효과가 없는 사진 대상*:
    - 썸네일 로드 및 다운샘플링.
    - **Darkness Check**: 평균 휘도 < 30 → **[저품질 후보 추가]**.
    - **Blur Check** (Dark가 아닌 경우만 수행):
        - Variance of Laplacian 계산.
        - Variance < 100.0 이면:
            - 얼굴 감지 수행.
            - 선명한 얼굴(Quality > 0.4)이 없으면 → **[저품질 후보 추가]**.
4.  **종료 조건**:
    - 후보 리스트 크기 == 50 도달.
    - 또는, 분석 대상 범위(예: 최근 1년치) 초과 시.

---

## 4. 파라미터 튜닝 계획

실제 기기 테스트를 통해 다음 임계값을 조정한다.

| 파라미터 | 초기값 | 튜닝 목표 |
|---|---|---|
| Dark Threshold | 30 / 255 | 야간 촬영 사진이 포함되지 않는 최댓값 찾기 |
| Blur Threshold | 100.0 | 텍스트 문서나 약간 흔들린 사진을 구분하는 최적값 찾기 |
| Face Quality | 0.4 / 1.0 | 블러된 사진 중 '인물은 건질만한' 사진을 살리는 기준 |
