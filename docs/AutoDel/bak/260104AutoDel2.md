# 저품질 사진 자동 정리 (Plan v2)

## 범위
- 단일 사진 품질 분류(중복 그룹핑 제외)
- 사진, 라이브포토, 비디오 포함(대표 프레임 기준 분석)

## 사용자 흐름
1) 사용자가 상단 "정리" 버튼을 탭한다.
2) 현재 인덱스에서 과거 방향으로 스캔한다.
3) 저품질 50장을 찾아 앱 내 휴지통으로 이동한다.

## 외부 근거 자료
- Google Files: 저품질 사진은 **흐림**, **저해상도**, **너무 어둡거나 밝음**.
  - https://support.google.com/files/answer/11544776
- Wikipedia(Exposure/Underexposure):
  - 과다 노출 = 하이라이트 디테일 손실("blown-out highlights" / "clipped whites")
  - 노출 부족 = 섀도우 디테일 손실("blocked-up shadows" / "clipped blacks")
  - https://en.wikipedia.org/wiki/Underexposure
- Wikipedia(Image quality): 이미지 품질 핵심 속성은 sharpness, noise, contrast, exposure accuracy.
  - https://en.wikipedia.org/wiki/Image_quality
- Wikipedia(Motion blur): 노출 중 움직임으로 인한 스트리킹.
  - https://en.wikipedia.org/wiki/Motion_blur
- PyImageSearch: blur 판별은 **Laplacian variance**가 임계값보다 낮으면 흐림.
  - https://pyimagesearch.com/2015/09/07/blur-detection-with-opencv/

## 정량 정의(운영 기준)
저품질 = 아래 조건 중 하나 이상을 만족하는 경우.

1) 흐림/흔들림(선명도 저하)
- 지표: Laplacian variance(그레이스케일)
- 판정: blur_score < T_blur
- 근거: Laplacian variance는 blur 판별 기준. 낮을수록 엣지/디테일이 적음.
- 비고: PyImageSearch 예시 threshold=100은 데이터셋 의존. 최종 T_blur는 캘리브레이션 필요.

2) 과다 노출 / 노출 부족(클리핑)
- 지표: luma 히스토그램의 클리핑 비율
- 과다 노출: clip_white_ratio > T_white
- 노출 부족: clip_black_ratio > T_black
- 근거: 하이라이트/섀도우 디테일 손실이 과다/부족 노출의 정의

3) 저해상도
- 지표: min(pixel_width, pixel_height) 또는 메가픽셀
- 판정: min_dim < R_min 또는 MP < MP_min
- 근거: Google Files의 저품질 분류에 "low resolution" 포함
- 비고: R_min/MP_min은 제품 UX 기준으로 결정

4) 과다 노이즈 / 저대비(디테일 손실)
- 지표:
  - 노이즈: 평탄 영역 분산 또는 고주파 잔차
  - 대비: percentile(luma, 95) - percentile(luma, 5)
- 판정: noise_sigma > T_noise AND contrast < T_contrast
- 근거: Image quality 속성에 noise/contrast 포함

## 결정 규칙(랭킹)
- 하드 조건(blur 또는 clipping) 만족 시 즉시 저품질 후보
- 그 외는 종합 점수로 판단
- 심각도 순으로 정렬 후 50장 선택

## 임계값 캘리브레이션(필수)
- 샘플 라벨링(삭제/유지) 데이터 수집
- 지표 분포 분석 후 precision/recall 최적화로 T_blur/T_white/T_black 등 결정
- 오탐 최소화를 위해 보수적 기준 적용

## 미디어 타입 처리
- 사진: 다운스케일(긴 변 256~320px)
- 라이브포토: 키 프레임 분석
- 비디오: 대표 키프레임(또는 일정 간격 프레임) 분석

## 비범위
- 중복/유사 사진 정리는 제외
- 사용자 확인 없는 자동 삭제는 제외
