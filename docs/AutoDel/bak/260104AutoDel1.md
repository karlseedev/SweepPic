# 저품질 사진 자동 정리 기능

## 1. 개요

### 목표
그리드에서 "정리" 버튼을 누르면 저품질 사진을 자동 분석하여 휴지통으로 이동

### 동작 흐름
```
1. 사용자가 화면 상단의 "정리" 버튼 탭
2. 현재 화면 기준 오래된 사진부터 저품질 사진 탐색
3. 50장을 찾아서 휴지통에 넣음 → 끝
```

### 탐색 범위
- **시간 범위**: 현재 스크롤 위치 이후(과거) 모든 사진
- **공간 범위**: 현재 앨범/폴더 내에서만
- **미디어 유형**: 사진, 라이브포토, 비디오 모두 포함

### 핵심 과제
1. **'저품질 사진'의 구체적인 정의**
2. **정의된 저품질 사진을 판별하는 로직 및 기능**

## 2. 저품질 사진 정의

### 2.1 상용 앱 기준 비교

| 앱 | 저품질 유형 |
|----|------------|
| **Google Files** | Blurry, Low resolution, Too dark, Too bright |
| **Gemini Photos** | Blurry, Closed eyes, Similar photos, Screenshots, Photos of text |
| **Apple Photos** | 중복 (동일 픽셀, 메타데이터, 해시) |

**출처:**
- [Google Files Help](https://support.google.com/files/answer/11544776)
- [App Store - CleanMyPhone](https://apps.apple.com/us/app/gemini-photos/id1277110040)
- [Macworld - Apple Photos Duplicate](https://www.macworld.com/article/1366211/how-to-use-duplicate-detection-apple-photos.html)

### 2.2 삭제 대상 유형 (5가지)

| 유형 | 설명 | 발생 원인 |
|------|------|----------|
| **흐릿한 사진** | 초점이 안 맞거나 흔들린 사진 | 카메라 흔들림, 피사체 움직임, 초점 오류 |
| **과다 노출** | 너무 밝아서 디테일이 날아간 사진 | 밝은 환경에서 노출 보정 실패 |
| **노출 부족** | 너무 어두워서 내용 파악 불가 | 어두운 환경, 주머니 속 촬영 |
| **렌즈 가림** | 손가락이나 물체가 렌즈를 가린 사진 | 잘못된 그립, 케이스 간섭 |
| **단색/무의미** | 검은 화면, 천장, 바닥 등 | 실수로 버튼 누름, 아기가 만짐 |

### 2.3 삭제 대상 제외 (보수적 판정)

- **스크린샷**: 메타데이터로 필터링 (의도적 저장)
- **의도적 야경/실루엣**: 어둡지만 예술적 의도
- **문서 촬영**: 단색 배경이지만 유용
- **얼굴 포함 사진**: 더 보수적으로 판정

## 3. 감지 알고리즘 및 임계값

### 3.1 흐릿함 감지 (Laplacian Variance)

**원리**
```
Laplacian 연산자로 이미지의 2차 미분(엣지) 계산
- 고분산 = 선명한 엣지 = 선명한 이미지
- 저분산 = 적은 엣지 = 흐릿한 이미지
```

**임계값 (학술 자료 기반)**

| Laplacian 분산 | 판정 | 출처 |
|----------------|------|------|
| **< 100** | 흐릿함 | PyImageSearch 권장 |
| **< 150** | 흐릿함 | GeeksforGeeks 권장 |
| **64 ~ 83** | 흐릿한 이미지 예시 | 실험 결과 |
| **> 1000** | 선명한 이미지 예시 | 실험 결과 |

⚠️ **주의**: 이미지 480px 리사이즈 후 계산, 데이터셋마다 튜닝 필요

**출처:**
- [PyImageSearch - Blur Detection](https://pyimagesearch.com/2015/09/07/blur-detection-with-opencv/)
- [GeeksforGeeks - Laplacian Method](https://www.geeksforgeeks.org/computer-vision/how-to-check-for-blurry-images-in-your-dataset-using-the-laplacian-method/)
- [Medium - Laplacian Blur Detection](https://medium.com/@sagardhungel/laplacian-and-its-use-in-blur-detection-fbac689f0f88)

### 3.2 노출 문제 감지 (히스토그램 분석)

**방법 1: 평균 밝기 기반**

| 평균 밝기 (0~1) | 판정 |
|-----------------|------|
| **< 0.10** | 매우 어두움 (주머니 속) |
| **< 0.15** | 노출 부족 |
| **0.15 ~ 0.85** | 정상 |
| **> 0.85** | 과다 노출 |
| **> 0.90** | 매우 밝음 (하얀 화면) |

**방법 2: 픽셀 분포 비율 기반**

| 상태 | 어두운 픽셀 비율 (0-50) | 밝은 픽셀 비율 (200-255) |
|------|------------------------|-------------------------|
| **좋은 이미지** | 6.7% | 20% |
| **노출 부족** | **40%+** | 0.9% |
| **과다 노출** | 2.5% | **65%+** |

**출처:**
- [Medium - Bad Lighting Detection](https://medium.com/@fanzongshaoxing/detect-face-in-bad-lighting-condition-using-color-histograms-2571df5fc53b)
- [OpenCV Q&A - Exposure Detection](https://answers.opencv.org/question/62479/determining-if-a-color-image-is-overexposed-or-underexposed/)

### 3.3 렌즈 가림 감지 (영역 밝기 비교)

**알고리즘**
```
1. 이미지를 9개 영역 분할 (3x3 그리드)
2. 각 영역 평균 밝기 계산
3. 모서리 밝기 < 중앙 × 0.4 → 가림 판정
4. 한 모서리만 < 0.15 → 손가락 가림
```

| 패턴 | 특징 | 원인 |
|------|------|------|
| 한쪽 모서리 어두움 | 한 방향만 급격히 어두움 | 손가락 가림 |
| 전체 가장자리 어두움 | 비네팅 패턴 | 케이스 간섭 |

**출처:**
- [Explore Landscape Photography - Vignetting](https://explorelandscapephotography.com/what-causes-dark-corners-in-photos-how-to-fix-vignetting/)

### 3.4 단색/무의미 감지 (RGB 분산)

| RGB 표준편차 | 판정 |
|--------------|------|
| **< 10** | 거의 단색 (삭제 후보) |
| **10 ~ 20** | 저대비 |
| **> 20** | 정상 |

**출처:**
- [PyImageSearch - Image Colorfulness](https://pyimagesearch.com/2017/06/05/computing-image-colorfulness-with-opencv-and-python/)

### 3.5 주머니 속 사진 (복합 조건)

```
AND 조건 모두 충족 시 주머니 속 판정:
- 평균 밝기 < 0.10
- RGB 표준편차 < 15
- Laplacian 분산 < 50
- 모서리/중앙 밝기 차이 < 0.05
```

### 3.6 임계값 요약표

| 지표 | 임계값 | 조건 |
|------|--------|------|
| Laplacian 분산 | < 100 | 흐릿함 |
| 평균 밝기 | < 0.15 | 노출 부족 |
| 평균 밝기 | > 0.85 | 과다 노출 |
| RGB 표준편차 | < 10 | 단색 |
| 모서리/중앙 비율 | < 0.4 | 렌즈 가림 |

## 4. iOS Vision Framework (iOS 18+)

### 4.1 CalculateImageAestheticsScoresRequest

Apple이 iOS 18에서 제공하는 **공식 이미지 품질 평가 API**

**반환값: ImageAestheticsScoresObservation**
| 속성 | 타입 | 설명 |
|------|------|------|
| `overallScore` | Float | -1 ~ 1 (높을수록 좋은 이미지) |
| `isUtility` | Bool | 유틸리티 이미지 여부 (스크린샷, 영수증 등) |

**분석 요소**
- 블러 (Blur)
- 노출 (Exposure)
- 기타 미적 요소

**점수 해석**
- **낮은 점수 (< 0)**: 초점 안 맞음, 실수로 촬영된 것 같은 이미지
- **isUtility = true**: 스크린샷, 영수증, 문서 (공유/기억 가치 낮음)

**출처:**
- [Apple Developer Documentation](https://developer.apple.com/documentation/vision/calculateimageaestheticsscoresrequest)
- [WWDC24 - Vision Framework](https://developer.apple.com/videos/play/wwdc2024/10163/)
- [Create with Swift - Image Aesthetics](https://www.createwithswift.com/scoring-the-aesthetics-of-an-image-with-the-vision-framework/)

### 4.2 VNFaceCaptureQuality (얼굴 품질)

**faceCaptureQuality 속성**
- 범위: 0 ~ 1
- 높을수록: 조명 좋음, 선명함, 중앙 배치

**출처:**
- [Medium - Face Capture Quality](https://medium.com/@kamil.tustanowski/detecting-face-capture-quality-using-the-vision-framework-e94af66f31e3)

## 5. 비디오/라이브포토 품질 평가

### 5.1 비디오 품질 지표 (TLVQM 기준)

| 지표 | 설명 |
|------|------|
| 센서 노이즈 | 저조도 환경에서 발생 |
| 모션 블러 | 빠른 움직임으로 인한 흐림 |
| 카메라 흔들림 | 손떨림으로 인한 불안정 |

### 5.2 프레임 단위 분석

```
1. 대표 프레임 추출 (시작, 중간, 끝)
2. 각 프레임에 Laplacian 분석 적용
3. 최솟값 또는 평균값으로 품질 판정
```

**출처:**
- [Frontiers - Video Quality Assessment](https://www.frontiersin.org/journals/signal-processing/articles/10.3389/frsip.2023.1193523/full)

## 6. 구현 전략 옵션

### Option A: iOS 18+ 전용 (단순)

```swift
// CalculateImageAestheticsScoresRequest 사용
if overallScore < 0 {
    // 저품질
}
if isUtility {
    // 유틸리티 이미지
}
```

**장점**: 간단, Apple 최적화
**단점**: iOS 18+ 필수

### Option B: iOS 16+ 호환 (커스텀)

| 지표 | 알고리즘 | 임계값 |
|------|----------|--------|
| 흐릿함 | Laplacian Variance | < 100 |
| 저노출 | 평균 밝기 | < 0.15 |
| 과노출 | 평균 밝기 | > 0.85 |
| 단색 | RGB 표준편차 | < 10 |
| 렌즈 가림 | 9영역 밝기 비교 | 모서리/중앙 < 0.4 |

**장점**: iOS 16+ 지원
**단점**: 직접 구현 필요, 튜닝 필요
