# 계획: ViewerMode.cleanup + 제외 버튼 + 탭바/툴바 숨김

## Context

미리보기 그리드(PreviewGridViewController)에서 썸네일을 탭하면 ViewerViewController가 push되는데:
1. **탭바가 보임** — iOS 26에서 `prefersSystemTabBarHidden` 미선언
2. **삭제 버튼이 보임** — `.normal` 모드 기본값으로 열려서 삭제 버튼/스와이프삭제 활성화 (미리보기는 읽기 전용)

**해결:** `ViewerMode.cleanup` 추가 + "제외" 버튼으로 개별 사진을 후보에서 제거하는 기능 구현

## 수정 파일 4개

### 1. `ViewerViewController.swift` (Features/Viewer/)

**ViewerMode enum (줄 26):**
```swift
enum ViewerMode {
    case normal    // 삭제 버튼
    case trash     // 복구/완전삭제 버튼
    case cleanup   // 제외 버튼 (정리 미리보기용)
}
```

**ViewerViewControllerDelegate (줄 36) — 기본 구현 추가:**

```swift
// extension에 기본 빈 구현 → 기존 Grid/Album/Trash 3곳 수정 불필요
extension ViewerViewControllerDelegate {
    func viewerDidRequestExclude(assetID: String) {}
}
```

**제외 버튼 프로퍼티 추가 (줄 ~155):**
```swift
private lazy var excludeButton: GlassTextButton = {
    let button = GlassTextButton(title: "제외", style: .plain, tintColor: .white)
    button.translatesAutoresizingMaskIntoConstraints = false
    button.addTarget(self, action: #selector(excludeButtonTapped), for: .touchUpInside)
    return button

```

**setupActionButtons() (줄 420) — `.cleanup` case 추가:**
- 하단 중앙에 excludeButton 배치 (deleteButton과 동일 위치)

**setupSwipeDeleteHandler() (줄 464):**
- `.cleanup`은 스와이프 제외 안 함 → 기존 `guard viewerMode == .normal` 유지 (변경 없음)

**excludeButtonTapped 핸들러 추가:**
```swift
@objc private func excludeButtonTapped() {
    guard let assetID = coordinator.assetID(at: currentIndex) else { return }
    let generator = UINotificationFeedbackGenerator()
    generator.notificationOccurred(.success)

    // 1. delegate에 제외 알림
    delegate?.viewerDidRequestExclude(assetID: assetID)

    // 2. 코디네이터에서 에셋 제거 (moveToNextAfterDelete보다 먼저 실행해야 함)
    //    — nextIndexAfterDelete()가 제거 후 count 기준으로 계산하므로 순서 중요
    (coordinator as? PreviewViewerCoordinator)?.removeAsset(id: assetID)

    // 3. 다음 사진으로 이동 (기존 메서드 재사용)
    moveToNextAfterDelete()
    updateToolbarForCurrentPhoto()
}
```
**주의: 실행 순서** — `removeAsset` → `moveToNextAfterDelete` 순서 필수.
`moveToNextAfterDelete()`는 내부에서 `nextIndexAfterDelete(currentIndex:)`로 다음 인덱스를 계산한 뒤
`refreshFilteredIndices()`를 호출하므로, 제거가 먼저 완료되어야 인덱스가 정확함.

**setupSystemToolbar() (줄 815) — `.cleanup` case 추가:**
- iOS 26 시스템 툴바에 "제외" 버튼 표시

**BarsVisibilityControlling (줄 1200):**
```swift
var prefersSystemTabBarHidden: Bool? { true }  // 추가 — 모든 뷰어 모드에서 탭바 숨김
```

### 2. `PreviewViewerCoordinator.swift` (Features/AutoCleanup/Preview/)

assets를 `var`로 변경 + 제외 지원:
```swift
private var assets: [PHAsset]  // let → var

/// 에셋 제거 (제외 기능용)
func removeAsset(id: String) {
    assets.removeAll { $0.localIdentifier == id }
}
```

`nextIndexAfterDelete` — `removeAsset` 호출 후 assets.count가 줄어든 상태에서 동작.
기존 구현이 count 기반이라 순서만 맞으면 정상 동작.

**ViewerViewControllerDelegate 채택:**

```swift
    func viewerDidRequestExclude(assetID: String) {

    }
    func viewerDidRequestDelete(assetID: String) {}
```

**viewWillAppear에서 그리드 갱신:**

```swift
    super.viewWillAppear(animated)
    if !excludedAssetIDs.isEmpty {
        applyExclusions()
    }
}
```

**applyExclusions():**
- `previewResult = previewResult.excluding(excludedAssetIDs)` — 새 인스턴스 재할당
- collectionView.reloadData()
- updateBottomView() + updateHeader()
- excludedAssetIDs.removeAll() (적용 완료)

### 4. `PreviewResult.swift` (Features/AutoCleanup/Preview/)

**제외 필터링 메서드 추가:**
```swift
/// 특정 assetID들을 제외한 새 결과 반환
func excluding(_ assetIDs: Set<String>) -> PreviewResult {
    return PreviewResult(
        lightCandidates: lightCandidates.filter { !assetIDs.contains($0.assetID) },
        standardCandidates: standardCandidates.filter { !assetIDs.contains($0.assetID) },
        deepCandidates: deepCandidates.filter { !assetIDs.contains($0.assetID) },
        scannedCount: scannedCount,
        totalTimeSeconds: totalTimeSeconds
    )
}
```

## 동작 흐름

1. 그리드에서 썸네일 탭 → `ViewerViewController(mode: .cleanup)` push
2. 뷰어 하단에 "제외" 버튼만 표시 (삭제/스와이프 없음, 탭바 없음)
3. "제외" 탭 → ① delegate로 assetID 전달 → ② coordinator.removeAsset → ③ moveToNextAfterDelete
4. 뷰어 pop → viewWillAppear에서 previewResult 재할당 + reloadData
5. 모든 사진 제외 시 → 뷰어 자동 닫힘 (기존 moveToNextAfterDelete 로직)

  2. 제외 탭 → 다음 사진 이동, 햅틱 피드백

  6. 기존 그리드/앨범/휴지통 뷰어 동작 영향 없음 (delegate 기본 구현으로 컴파일 안전)
