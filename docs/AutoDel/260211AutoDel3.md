# 자동 정리 미리보기 구현 계획 (방향 3: 인라인 배너 + 자동 스크롤)

## 설계 문서
`docs/autodel/260211AutoDel2.md`

## 흐름 변경
```
현재:  정리 시작 → 분석 + 즉시 휴지통 이동 → 완료 Alert
변경:  정리 시작 → 분석 (이동 없이) → 미리보기 그리드 → 사용자 확인 → 휴지통 이동
```

## 3모드 임계값
- 1단계 (완화): path1 OR path2(< -0.3) — 확실한 저품질
- 2단계 (기본): path1 OR path2(< 0.0) — 1단계 + 추가분
- 3단계 (강화): path1 OR path2(< 0.2) — 1+2단계 + 추가분

---

## 핵심 설계 결정

### 1. BaseGridViewController 상속 안 함 (별도 VC)
- BaseGrid는 모든 셀이 PhotoCell 전제 + GridDataSource(PHFetchResult 기반)
- 미리보기는 PhotoCell + BannerCell 혼합 + 배열 기반 + 하단 고정 버튼 영역
- 차이가 크므로 별도 UIViewController로 구현
- PhotoCell은 재사용 (register + configure 호출) — 내부에서 ImagePipeline.shared 사용하므로 외부 주입 불필요

### 2. 분석 서비스: 새 서비스 생성 (CleanupService 수정 안 함)
- CleanupService는 기존 즉시삭제 흐름 그대로 유지 (안정성)
- `CleanupPreviewService` 신규 생성: 분석만 수행, 3모드 분류, 이동 없음
- ModeComparisonTester의 evaluatePath1/evaluatePath2/detectTextScreenshot 로직을 **독립적으로 복사** (ModeComparisonTester는 `#if DEBUG` 안에 있으므로 참조 불가)
- 의존성은 모두 싱글톤: `QualityAnalyzer.shared`, `AestheticsAnalyzer.shared` (iOS 18+), `CleanupImageLoader.shared`

### 3. CollectionView 구조: 멀티 섹션
```
Section 0: 1단계 사진 (3열 그리드)         — 항상 표시
Section 1: 배너 "↓ 기준을 낮춰 N장 추가"   — stage >= 2일 때
Section 2: 2단계 추가분 (3열 그리드)        — stage >= 2일 때
Section 3: 배너 2                          — stage >= 3일 때
Section 4: 3단계 추가분 (3열 그리드)        — stage >= 3일 때
```
- CompositionalLayout: 사진 섹션은 3열 정사각형, 배너 섹션은 전체 너비 1행

### 4. 개발 전략: 서비스는 production, UI 진입만 DEBUG
- CleanupPreviewService, 모델, PreviewGridVC 등은 **`#if DEBUG` 밖** (추후 정식 교체 대비)
- CleanupMethodSheet의 "미리보기 정리" 메뉴 버튼만 `#if DEBUG`
- 테스트 완료 후 기존 정리 흐름 교체 (별도 작업)

### 5. 이미지만 분석 (비디오 제외)
- ModeComparisonTester 패턴 그대로: `mediaType == .image`
- AestheticsScore는 비디오에 적용 불가, 비디오 품질 판정은 기존 CleanupService에서만

    }

## Phase 2: 분석 서비스

### 새 파일

**`Features/AutoCleanup/Preview/CleanupPreviewService.swift`**

```swift
final class CleanupPreviewService {
    // 싱글톤 의존성 직접 사용 (생성자 주입 불필요)
    // - QualityAnalyzer.shared
    // - AestheticsAnalyzer.shared (iOS 18+, #available 가드 필요)
    // - CleanupImageLoader.shared

    // 취소 지원
    private var isCancelled = false
    private let lock = NSLock()

    /// 메인 API — 분석만 수행, 이동 없음
    func analyze(
        method: CleanupMethod,
        progressHandler: @escaping (CleanupProgress) -> Void
    ) async throws -> PreviewResult

    func cancel()
}
```

**초기화:**
- `QualityAnalyzer.shared.setMode(.precision)` 호출 (싱글톤 상태 고정, 다른 곳에서 변경 가능하므로)

**분석 루프 (analyze 내부):**
1. `createFetchResult(for:)` — **자체 구현** (CleanupService의 동일 메서드는 private)
   - `mediaType == .image` (비디오 제외 — ModeComparisonTester 패턴)
   - `creationDate` 내림차순 정렬
   - .fromLatest / .continueFromLast / .byYear 모두 처리 (byYear는 컴파일 위해 case 처리, UI 메뉴에서는 미노출)
2. 배치 순회 (batchSize = 100, CleanupConstants.batchSize 재사용)
3. 각 사진:
   - 극단적 비율 체크 (> 5.0 or < 0.2) → skip
   - `QualityAnalyzer.shared.analyze(asset)` → path1 판정
   - **iOS 18+ 에서만**: 이미지 로드 + AestheticsScore + 텍스트 감지 수행
     - `CleanupImageLoader.shared.loadImage(for: asset)` → CGImage 로드
     - `AestheticsAnalyzer.shared.analyze(image)` → AestheticsMetrics
     - `detectTextScreenshot(image)` → VNRecognizeTextRequest (1회, 결과 재사용)
     - path2 판정 3회 (threshold만 다름: -0.3, 0.0, 0.2)
     - 3모드 OR 계산 → 카테고리 분류 → PreviewCandidate 생성
   - **iOS 16~17**: path1 결과만으로 light 판정 (이미지 로드/텍스트 감지 스킵 → 성능 최적화)
4. 종료 조건: `maxScanCount(2000)` 도달 또는 범위 끝 (maxFoundCount 제한 없음)
5. 결과 분리: lightCandidates / standardCandidates(추가분만) / deepCandidates(추가분만)

**내부 메서드 (ModeComparisonTester에서 독립 복사):**

`AestheticsMetrics`가 `@available(iOS 18.0, *)`이므로 이를 참조하는 메서드도 `@available` 처리 필요.

```swift
// 경로1: Strong 신호 → 확정, Weak/Conditional → AestheticsScore < 0.2 동의 필요
// aestheticsScore를 Float?로 받아 iOS availability 문제 회피
private func evaluatePath1(oldResult: QualityResult, aestheticsScore: Float?) -> Bool

// 경로2: score + isUtility + 텍스트 스크린샷 필터
// AestheticsMetrics 대신 개별 값으로 받아 iOS availability 문제 회피
private func evaluatePath2(score: Float?, isUtility: Bool, isTextScreenshot: Bool, threshold: Float) -> Bool

// VNRecognizeTextRequest로 텍스트 블록 >= 5 감지
private func detectTextScreenshot(_ image: CGImage) async -> Bool
```

> **iOS availability 해결 전략**: `AestheticsMetrics` 타입을 메서드 시그니처에 노출하지 않고, `analyze()` 루프 내부의 `if #available(iOS 18.0, *)` 블록에서 score/isUtility를 `Float?`/`Bool`로 추출한 뒤 전달.

**iOS 분기:**
- iOS 18+: path1 + path2 → 3단계 결과 (이미지 로드 + AestheticsScore + 텍스트 감지 수행)
- iOS 16~17: path1만 → lightCandidates만, standard/deep은 빈 배열 (이미지 로드/텍스트 감지 스킵)

**프로그레스:**
- 기존 `CleanupProgress` 구조체 재사용 (호환 확인 완료: progressBar에 scan 기반 progress 사용)
- foundCount = deep 기준 합산 (UI에서 "N장 발견" 표시)

**"이어서" 세션:**
- 미리보기 전용 세션 저장 필요 (마지막 스캔 날짜)
- `UserDefaults`에 별도 키로 저장 (`PreviewSession.lastScanDate`)
- 기존 `CleanupSessionStore`와 독립 (기존 "이어서 정리" 흐름 영향 없음)

**thumbnailSize 계산:**
```swift
private func thumbnailSize() -> CGSize {
    let scale = UIScreen.main.scale
    let spacing: CGFloat = 2.0  // BaseGridViewController.cellSpacing
    let columns: CGFloat = 3
    let totalSpacing = spacing * (columns - 1)
    let cellWidth = floor((collectionView.bounds.width - totalSpacing) / columns)
    return CGSize(width: cellWidth * scale, height: cellWidth * scale)
}
```

func previewBottomViewDidTapExpand(_ view: PreviewBottomView) {
    // 1. currentStage 변경 (numberOfSections 먼저 업데이트되도록)
    currentStage = PreviewStage(rawValue: currentStage.rawValue + 1)!

    // 2. 새 섹션 삽입
    collectionView.performBatchUpdates {
        let newSections = (currentStage == .standard)
            ? IndexSet([1, 2]) : IndexSet([3, 4])
        collectionView.insertSections(newSections)

        // 3. 배너 위치로 자동 스크롤

            at: IndexPath(item: 0, section: bannerSection),
            at: .top, animated: true
        )
    }

    // 4. 하단 버튼 + 헤더 업데이트
    updateBottomView()
    updateHeader()
}

```swift
// delegate 구현 추가
func cleanupMethodSheetDidSelectPreview(continueFromLast: Bool) {
    startPreviewCleanup(continueFromLast: continueFromLast)
}

func startPreviewCleanup(continueFromLast: Bool) {
    // 1. CleanupProgressView 표시 (기존 UI 재사용)
    let progressView = CleanupProgressView()
    progressView.delegate = self  // 취소 delegate 연결
    progressView.configure(method: continueFromLast ? .continueFromLast : .fromLatest)
    progressView.show(in: view)

    // 2. CleanupPreviewService 실행 (프로퍼티로 보관 — 취소 접근용)
    let service = CleanupPreviewService()
    self.previewService = service

    Task {
        do {
            let result = try await service.analyze(
                method: continueFromLast ? .continueFromLast : .fromLatest,
                progressHandler: { progress in
                    progressView.update(with: progress)
                }
            )
            await MainActor.run { [weak self] in
                self?.previewService = nil
                progressView.hide {
                    if result.totalCount > 0 {
                        let previewVC = PreviewGridViewController(previewResult: result)
                        previewVC.delegate = self
                        self?.navigationController?.pushViewController(previewVC, animated: true)
                    } else {
                        self?.showNoPreviewResultAlert()
                    }
                }
            }
        } catch {
            await MainActor.run { [weak self] in
                self?.previewService = nil
                progressView.hide {
                    self?.showCleanupError(.analysisFailed(error.localizedDescription))
                }
            }
        }
    }
}
```

## 구현 후 수정 내역 (2026-02-12)

구현 완료 후 테스트 과정에서 아래 수정사항이 반영됨.

### 1. PreviewBottomView 버튼 리디자인
- UIButton.Configuration 기반 3개 버튼 → **GlassTextButton** 3개로 교체
- 세로 3줄 레이아웃(140pt) → **primaryButton + 가로 스택(collapse, expand) 2줄**(110pt)
- 배경색: `UIColor(white: 0.09)` (상단 썸네일 영역과 구분)

### 2. GlassTextButton 확장
- `multiline` 파라미터 추가 (2줄 텍스트 지원, `numberOfLines = 0`)
- `fontSize` 파라미터 추가 (보조 버튼 16pt)
- `glassTintColor` 파라미터 추가 (화면별 커스텀 Glass tintColor)
- `setGlassBackground()` 메서드 추가 (iOS 18 전용 배경색)

### 3. iOS 버전 분기 처리
- **iOS 26**: 네이티브 `UIGlassEffect`가 tintColor를 독립 처리 → `glassTintColor: (white: 0.5, alpha: 0.15)`
- **iOS 18**: Metal 셰이더가 어두운 배경에서 투명 → `setGlassBackground(white: 0.25, alpha: 0.3)`로 대체 배경 제공
- `contentInsetAdjustmentBehavior = .never` (PreviewGridVC 전용, 이중 inset 방지)

### 4. 버튼 텍스트 변경
- primaryButton: `"탐색된 N장 휴지통으로 이동"` → `"N점 이하 사진 N장 휴지통 이동"` (상단 타이틀 점수와 동일)
- 보조 버튼: 1개만 보일 때 1줄, 둘 다 보일 때 2줄 (동적 separator)
- 화살표(→) 제거
- 보조 스택 높이 48pt 고정

### 5. 팝업 문구 수정
- 제목: `"N장을 정리할까요?"` → `"N장을 휴지통으로 이동할까요?"`
- 메시지: 삭제 (nil)
- 버튼: `"정리하기"` → `"이동하기"`

### 6. 스크롤/레이아웃 수정
- expand 시 `scrollToItem`을 `performBatchUpdates` **completion** 안으로 이동 (애니메이션 충돌 방지)

- `configure()` 끝에 `layoutIfNeeded()` + `LiquidGlassOptimizer.preload(in:)` 호출 (버튼 크기 변경 시 블러 오버레이 frame 동기화)
