# Interactive Layout Transition 핀치줌/회전 구현 계획

**작성일**: 2026-01-21
**목표**: iOS Photos 앱과 동일한 점진적 레이아웃 전환 구현
**상태**: 설계 중

---

## 1. 목표 동작

### iOS Photos 앱 핀치줌/회전 동작 분석

```
핀치 시작
    ↓
셀들이 점진적으로 크기/위치 변화 (보간)
    ↓ transitionProgress: 0.0 → 1.0
    ↓ 3열 레이아웃 → 5열 레이아웃 사이를 부드럽게 전환
    ↓
핀치 종료
    ↓
진행도에 따라 완료(finish) 또는 취소(cancel)
```

### 핵심 특징

| 항목 | 현재 구현 | 목표 구현 |
|-----|----------|----------|
| 전환 방식 | 임계값에서 즉시 전환 | 점진적 전환 (0~1 보간) |
| 셀 크기 변화 | 없음 → 갑자기 변경 | 점진적으로 변화 |
| 셀 위치 변화 | 이동 애니메이션 | 점진적으로 새 위치로 이동 |
| 사용자 피드백 | 임계값 전까지 없음 | 핀치에 따라 실시간 피드백 |

## 3. 구현 설계

### 3.1 핀치 제스처 → transitionProgress 매핑

```swift
/// 핀치 스케일을 transitionProgress로 변환
/// - Parameters:
///   - initialScale: 핀치 시작 시 스케일 (보통 1.0)
///   - currentScale: 현재 핀치 스케일
///   - direction: 줌 방향 (.zoomIn 또는 .zoomOut)
/// - Returns: 0.0 ~ 1.0 사이의 진행도
func transitionProgress(
    initialScale: CGFloat,
    currentScale: CGFloat,
    direction: ZoomDirection
) -> CGFloat {
    switch direction {
    case .zoomIn:  // 3열 → 1열 (확대)
        // scale > 1.0: 손가락 벌림
        // 1.0 → 1.5 를 0.0 → 1.0으로 매핑
        let progress = (currentScale - initialScale) / 0.5
        return min(max(progress, 0), 1)

    case .zoomOut:  // 3열 → 5열 (축소)
        // scale < 1.0: 손가락 모음
        // 1.0 → 0.5 를 0.0 → 1.0으로 매핑
        let progress = (initialScale - currentScale) / 0.5
        return min(max(progress, 0), 1)
    }
}
```

### 3.2 줌 방향 결정

```swift
enum ZoomDirection {
    case zoomIn   // 열 수 감소 (확대)
    case zoomOut  // 열 수 증가 (축소)
}

/// 핀치 스케일 변화에 따라 줌 방향 결정
func determineZoomDirection(scale: CGFloat) -> ZoomDirection? {
    if scale > 1.05 {
        return .zoomIn   // 확대 (열 수 감소)
    } else if scale < 0.95 {
        return .zoomOut  // 축소 (열 수 증가)
    }
    return nil  // 아직 방향 미결정
}
```

### 3.3 Interactive Transition 상태 관리

```swift
/// Interactive Transition 상태
struct InteractiveTransitionState {
    var isActive: Bool = false
    var transitionLayout: UICollectionViewTransitionLayout?
    var initialScale: CGFloat = 1.0
    var direction: ZoomDirection?
    var targetColumns: GridColumnCount?
    var anchorPoint: CGPoint = .zero      // 핀치 중심점 - **화면 좌표** (contentOffset 무관)
    var anchorAssetID: String?            // 앵커 에셋 ID
}

private var interactiveState = InteractiveTransitionState()
```

## 4. 핀치 제스처 핸들러 재설계

### 4.1 Phase: began

**좌표계 규칙:**
- `anchorPoint`: **화면 좌표** (0,0 = collectionView 좌측 상단, contentOffset 무관)
- `contentLocation`: **content 좌표** (indexPathForItem 등에 사용)

```swift
case .began:
    // content 좌표 (indexPathForItem용)
    let contentLocation = gesture.location(in: collectionView)

    // 화면 좌표 (앵커 고정용) - contentOffset을 빼서 화면 기준으로 변환
    let screenLocation = CGPoint(
        x: contentLocation.x - collectionView.contentOffset.x,
        y: contentLocation.y - collectionView.contentOffset.y
    )

    // 1. 앵커 저장 (화면 좌표로 고정, 이후 업데이트하지 않음)
    interactiveState.anchorPoint = screenLocation

    // 2. 앵커 assetID 결정
    //    - 셀 위에서 핀치: 해당 셀을 앵커로
    //    - 패딩 영역 또는 빈 공간: 화면 중앙 셀을 앵커로
    if let indexPath = collectionView.indexPathForItem(at: contentLocation),
       indexPath.item >= paddingCellCount {
        interactiveState.anchorAssetID = assetIDForCollectionIndexPath(indexPath)
    } else {
        // 패딩 영역 또는 빈 공간 → 화면 중앙 셀을 앵커로
        let screenCenter = CGPoint(
            x: collectionView.bounds.width / 2,
            y: collectionView.bounds.height / 2
        )
        interactiveState.anchorPoint = screenCenter  // 화면 좌표

        // content 좌표로 변환하여 셀 찾기
        let contentCenter = CGPoint(
            x: collectionView.contentOffset.x + screenCenter.x,
            y: collectionView.contentOffset.y + screenCenter.y
        )
        if let centerIndexPath = collectionView.indexPathForItem(at: contentCenter) {
            interactiveState.anchorAssetID = assetIDForCollectionIndexPath(centerIndexPath)
        }
    }

    // 3. 초기 스케일 저장
    interactiveState.initialScale = gesture.scale  // 보통 1.0

    // 4. 스크롤 비활성화 (핀치 중 스크롤 방지)
    collectionView.isScrollEnabled = false

    // 5. 다른 제스처 비활성화 (핀치 우선)
    swipeDeleteState.swipeGesture?.isEnabled = false
    dragSelectGesture?.isEnabled = false
```

### 4.2 Phase: changed

```swift
case .changed:
    let scale = gesture.scale

    // 1. 방향 결정 (아직 미결정이면)
    if interactiveState.direction == nil {
        if let direction = determineZoomDirection(scale: scale) {
            // ⚠️ 중요: 목표 열 수를 먼저 확인 (direction 설정 전에!)
            // 최소/최대 열에서 해당 방향 전환 불가능하면 direction 설정 안함
            let targetColumns = direction == .zoomIn
                ? currentGridColumnCount.zoomIn
                : currentGridColumnCount.zoomOut

            guard let target = targetColumns else {
                // 전환 불가 (최소/최대 열) → direction 설정하지 않음
                // 이렇게 하면 반대 방향 핀치 시 다시 시도 가능
                return
            }

            // 목표 열 수 확인 후 direction 설정
            interactiveState.direction = direction
            interactiveState.targetColumns = target

            // 2. Interactive Transition 시작
            let targetLayout = createLayout(columns: target)
            interactiveState.transitionLayout = collectionView.startInteractiveTransition(
                to: targetLayout,
                completion: { [weak self] completed, finished in
                    self?.handleTransitionCompletion(completed: completed, finished: finished)
                }
            )
            interactiveState.isActive = true
        }
    }

    // 3. 진행도 업데이트 + 앵커 위치 유지
    if interactiveState.isActive,
       let transitionLayout = interactiveState.transitionLayout,
       let direction = interactiveState.direction {

        let progress = transitionProgress(
            initialScale: interactiveState.initialScale,
            currentScale: scale,
            direction: direction
        )

        transitionLayout.transitionProgress = progress

        // ⚠️ 앵커 드리프트 방지: 전환 중 contentOffset 조정
        // 앵커 셀이 화면에서 고정되도록 offset 계산
        if let anchorAssetID = interactiveState.anchorAssetID,
           let anchorIndexPath = collectionIndexPath(for: anchorAssetID) {
            adjustContentOffsetForAnchor(
                anchorIndexPath: anchorIndexPath,
                anchorPoint: interactiveState.anchorPoint,
                transitionLayout: transitionLayout
            )
        }

        transitionLayout.invalidateLayout()
    }
```

### 4.3 Phase: ended / cancelled

```swift
case .ended, .cancelled:
    guard interactiveState.isActive,
          let transitionLayout = interactiveState.transitionLayout else {
        resetInteractiveState()
        return
    }

    // 진행도에 따라 완료/취소 결정
    let progress = transitionLayout.transitionProgress

    if progress > 0.5 {
        // 50% 이상 진행 → 목표 레이아웃으로 완료
        if let target = interactiveState.targetColumns {
            currentGridColumnCount = target
        }
        collectionView.finishInteractiveTransition()
    } else {
        // 50% 미만 → 원래 레이아웃으로 취소
        collectionView.cancelInteractiveTransition()
    }

    // ⚠️ 중요: resetInteractiveState()를 여기서 호출하지 않음!
    // completion에서 호출하므로 interactiveState는 아직 유효함

default:
    break
```

### 4.4 상태 초기화 및 Completion 처리

```swift
private func resetInteractiveState() {
    interactiveState = InteractiveTransitionState()

    // 스크롤 재활성화
    collectionView.isScrollEnabled = true

    // 다른 제스처 재활성화
    updateSwipeDeleteGestureEnabled()  // 기존 메서드 활용
    dragSelectGesture?.isEnabled = isSelectMode
}

/// ⚠️ 중요: 이 메서드는 startInteractiveTransition의 completion에서 호출됨
/// - completed: 전환이 완료되었는지 (중단 없이)
/// - finished: 목표 레이아웃으로 완료(true) vs 취소로 원래 레이아웃 복귀(false)
private func handleTransitionCompletion(completed: Bool, finished: Bool) {
    // ⚠️ 중요: 이 시점에서 interactiveState는 아직 유효함
    // (resetInteractiveState는 .ended에서 호출하지 않고 여기서 호출)

    if finished {
        // 목표 레이아웃으로 완료된 경우만 처리

        // 1. 앵커 위치로 스크롤 조정
        if let assetID = interactiveState.anchorAssetID,
           let indexPath = collectionIndexPath(for: assetID) {
            collectionView.scrollToItem(at: indexPath, at: .centeredVertically, animated: false)
        }

        // 2. 셀 크기 업데이트 (전환 완료 후)
        updateCellSize()

        // 3. 고해상도 썸네일 재요청
        didPerformZoom(to: currentGridColumnCount)
    }
    // else: 취소된 경우 - 원래 레이아웃으로 복귀, 추가 작업 없음

    // 4. 상태 초기화 (completion에서 호출해야 anchorAssetID 사용 가능)
    resetInteractiveState()
}
```

## 5. UICollectionViewCompositionalLayout 호환성

### 5.1 잠재적 이슈

`startInteractiveTransition`은 주로 `UICollectionViewFlowLayout`과 함께 사용됨.
`UICollectionViewCompositionalLayout`과의 호환성은 검증 필요.

### 5.2 테스트 계획

1. **Phase 1**: 현재 CompositionalLayout으로 시도
2. **Phase 2**: 문제 발생 시 FlowLayout 기반 대안 검토

### 5.3 FlowLayout 대안 (필요시)

```swift
/// 핀치줌 전용 FlowLayout 생성
func createFlowLayout(columns: GridColumnCount) -> UICollectionViewFlowLayout {
    let layout = UICollectionViewFlowLayout()

    let spacing = Self.cellSpacing
    let columnCount = CGFloat(columns.rawValue)
    let totalSpacing = spacing * (columnCount - 1)
    let availableWidth = view.bounds.width - totalSpacing
    let cellWidth = floor(availableWidth / columnCount)

    layout.itemSize = CGSize(width: cellWidth, height: cellWidth)
    layout.minimumInteritemSpacing = spacing
    layout.minimumLineSpacing = spacing

    return layout
}
```

## 6. 회전 적용

### 6.1 회전도 동일한 원리

회전 시에도 열 수가 변경되면 Interactive Transition 적용 가능:

```swift
override func viewWillTransition(
    to size: CGSize,
    with coordinator: UIViewControllerTransitionCoordinator
) {
    super.viewWillTransition(to: size, with: coordinator)

    // 회전 전 앵커 저장
    saveScrollAnchorAssetID()

    let isLandscape = size.width > size.height
    let newColumnCount = isLandscape
        ? currentGridColumnCount.landscapeColumnCount
        : currentGridColumnCount.portraitColumnCount

    // 새 레이아웃 준비
    let targetLayout = createLayout(columns: newColumnCount, explicitWidth: size.width)

    coordinator.animate(alongsideTransition: { [weak self] context in
        guard let self = self else { return }

        // 회전 애니메이션과 동기화하여 레이아웃 전환
        // (Interactive가 아닌 animated 전환 사용)
        self.currentGridColumnCount = newColumnCount
        self.collectionView.setCollectionViewLayout(targetLayout, animated: true)
        self.updateCellSize()
        self.updateContentInset()

    }, completion: { [weak self] _ in
        self?.restoreScrollAnchorAssetID()
        self?.refreshVisibleCellsAfterRotation()
    })
}
```

### 6.2 회전 vs 핀치줌 차이

| 항목 | 핀치줌 | 회전 |
|-----|--------|------|
| 트리거 | 사용자 제스처 | 시스템 이벤트 |
| 전환 방식 | Interactive (사용자 제어) | Animated (시스템 동기화) |
| API | `startInteractiveTransition` | `setCollectionViewLayout(animated: true)` |

## 7. TLLayoutTransitioning 참고 구현

### 7.1 transitionProgress 계산 함수 (TLLayoutTransitioning 방식)

```swift
/// TLLayoutTransitioning 스타일의 진행도 계산
/// - Parameters:
///   - initialScale: 시작 스케일
///   - currentScale: 현재 스케일
///   - finalScale: 목표 스케일 (zoomIn: 2.0, zoomOut: 0.5 등)
func transitionProgress(
    _ initialScale: CGFloat,
    _ currentScale: CGFloat,
    _ finalScale: CGFloat
) -> CGFloat {
    let totalRange = finalScale - initialScale
    guard totalRange != 0 else { return 0 }

    let currentRange = currentScale - initialScale
    let progress = currentRange / totalRange

    return min(max(progress, 0), 1)
}
```

### 7.2 앵커 드리프트 방지 (핵심!)

전환 중 앵커 셀이 화면에서 고정되도록 contentOffset을 조정:

**좌표계:** `anchorPoint`는 **화면 좌표** (0,0 = collectionView 좌측 상단)

```swift
/// 전환 중 앵커 셀이 화면의 원래 위치에 고정되도록 contentOffset 조정
/// - Parameters:
///   - anchorIndexPath: 앵커 셀의 indexPath
///   - anchorPoint: 핀치 시작 시 앵커의 **화면 좌표** (contentOffset 무관)
///   - transitionLayout: 현재 전환 레이아웃
private func adjustContentOffsetForAnchor(
    anchorIndexPath: IndexPath,
    anchorPoint: CGPoint,
    transitionLayout: UICollectionViewTransitionLayout
) {
    // 전환 중인 레이아웃에서 앵커 셀의 현재 위치 계산 (content 좌표)
    guard let attributes = transitionLayout.layoutAttributesForItem(at: anchorIndexPath) else {
        return
    }

    // 앵커 셀이 원래 화면 위치(anchorPoint)에 오도록 offset 계산
    // cellCenter(content 좌표) - anchorPoint(화면 좌표) = 필요한 contentOffset
    let cellCenter = attributes.center
    let newOffset = CGPoint(
        x: cellCenter.x - anchorPoint.x,
        y: cellCenter.y - anchorPoint.y
    )

    // offset 범위 제한 (bounce 방지)
    // ⚠️ transition 중에는 transitionLayout의 contentSize 사용
    let contentSize = transitionLayout.collectionViewContentSize
    let maxOffsetX = max(0, contentSize.width - collectionView.bounds.width)
    let maxOffsetY = max(0, contentSize.height - collectionView.bounds.height)

    collectionView.contentOffset = CGPoint(
        x: min(max(0, newOffset.x), maxOffsetX),
        y: min(max(0, newOffset.y), maxOffsetY)
    )
}
```

### 7.3 Content Offset 계산 (completion용)

```swift
/// 앵커 셀이 화면 중앙에 유지되도록 contentOffset 계산 (완료 후 사용)
func calculateToContentOffset(
    for anchorIndexPath: IndexPath,
    in targetLayout: UICollectionViewLayout
) -> CGPoint {
    // 목표 레이아웃에서 앵커 셀의 위치 계산
    guard let attributes = targetLayout.layoutAttributesForItem(at: anchorIndexPath) else {
        return collectionView.contentOffset
    }

    // 화면 중앙에 오도록 offset 계산
    let cellCenter = attributes.center
    let viewCenter = CGPoint(
        x: collectionView.bounds.width / 2,
        y: collectionView.bounds.height / 2
    )

    return CGPoint(
        x: cellCenter.x - viewCenter.x,
        y: cellCenter.y - viewCenter.y
    )
}
```

## 12. 롤백 계획

### Feature Flag 기반 롤백

```swift
// FeatureFlags.swift
enum FeatureFlags {
    static var enableInteractiveZoom: Bool {
        // UserDefaults 또는 Remote Config 기반
        return UserDefaults.standard.bool(forKey: "interactive_zoom_enabled")
    }
}

// BaseGridViewController.swift - handlePinchGesture
// ⚠️ 중요: 전체 핸들러를 분기해야 일관성 유지
// (.began에서 스크롤/제스처 비활성화 로직이 레거시 방식과 다를 수 있음)
@objc func handlePinchGesture(_ gesture: UIPinchGestureRecognizer) {
    if FeatureFlags.enableInteractiveZoom {
        handleInteractivePinchZoom(gesture)
    } else {
        handleLegacyPinchZoom(gesture)
    }
}

// 새로운 Interactive Transition 방식
private func handleInteractivePinchZoom(_ gesture: UIPinchGestureRecognizer) {
    switch gesture.state {
    case .began:
        // Interactive 전용 초기화
        setupInteractiveTransition(gesture)
    case .changed:
        updateInteractiveTransition(gesture)
    case .ended, .cancelled:
        finalizeInteractiveTransition(gesture)
    default:
        break
    }
}

// 기존 임계값 방식 (fallback)
private func handleLegacyPinchZoom(_ gesture: UIPinchGestureRecognizer) {
    // 기존 코드 유지
}
```

### 롤백 기준

| 기준 | 임계값 | 조치 |
|------|--------|------|
| 프레임 드랍 | hitch 2f 이상 반복 | Flag off |
| 앵커 드리프트 | 20px 초과 | Flag off |
| 크래시 | 1회 발생 | Flag off + 조사 |
