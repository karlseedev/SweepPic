# PRD 7: 그리드 즉시 삭제/복원

## 개요

그리드 화면에서 선택 모드 진입 없이 섬네일을 직접 조작하여 사진을 휴지통으로 이동하거나 복원하는 기능.

### 배경

현재 그리드에서 사진을 삭제하려면:
1. 선택 모드 진입 → 사진 선택 → 삭제 버튼
2. 또는 뷰어 진입 → 스와이프/버튼으로 삭제

이 과정에서 "선택하다가 확대해서 보고 싶을 때" 선택을 취소해야 하는 불편함이 있음.
2차에서는 모드 전환 없이 그리드에서 바로 삭제/복원할 수 있는 제스처를 추가.

### 목표

- 선택 모드 없이 그리드에서 즉시 삭제/복원
- 직관적인 제스처와 시각적 피드백
- 기존 UX(탭→뷰어, 스크롤, 핀치줌)와 충돌 없음

---

## 기능 요구사항

### FR-101: 섬네일 스와이프 삭제/복원

#### 기본 동작
| 사진 상태 | 스와이프 결과 |
|----------|-------------|
| 일반 사진 | → 빨간 딤드 (휴지통으로 이동) |
| 빨간 딤드 사진 | → 딤드 해제 (복원) |

#### 제스처 사양
- **방향**: 양방향 (←/→) 모두 동작
- **시작 조건**: 수평 각도 15° 이상일 때 스와이프로 인식 (그 미만은 스크롤)
- **확정 조건** (OR):
  - 스와이프 거리 ≥ 섬네일 너비의 50%
  - 스와이프 속도 ≥ 800pt/s
- **취소 조건**: 50% 미만에서 손 떼고, 속도도 800pt/s 미만

#### 시각 효과
- **삭제 시**: 스와이프 방향과 같은 방향에서 빨간 딤드가 커튼처럼 채워짐
  - 예: 오른쪽 스와이프 → 왼쪽에서 빨간색 밀려옴
- **복원 시**: 스와이프 방향과 같은 방향으로 빨간 딤드가 걷힘
  - 예: 오른쪽 스와이프 → 빨간색이 오른쪽으로 사라짐
- **진행 표시**: 손가락 이동에 따라 실시간으로 딤드 영역 변화
- **취소 시**: 딤드가 원래 상태로 슬라이드되어 돌아감
- **확정 시**: 나머지 영역이 빠르게 채워지거나 걷힘

#### 햅틱 피드백
- 확정 시 약한 진동 (UIImpactFeedbackGenerator, style: .light)

---

### FR-102: 투 핑거 탭 삭제/복원

#### 기본 동작
| 사진 상태 | 투 핑거 탭 결과 |
|----------|---------------|
| 일반 사진 | → 빨간 딤드 (휴지통으로 이동) |
| 빨간 딤드 사진 | → 딤드 해제 (복원) |

#### 제스처 사양
- **인식 조건**:
  - 두 손가락 동시 터치
  - 터치 후 0.2초 이내
  - 이동 거리 10pt 미만
- **핀치 줌과 구분**: 0.2초 초과 또는 이동 10pt 이상이면 핀치 줌으로 전환

#### 시각 효과
- **삭제 시**: 빨간 딤드 페이드 인
- **복원 시**: 빨간 딤드 페이드 아웃
- **duration**: 0.15초 (기본값)
  - 대안: 0.2초 (iOS 기본), 0.3초 (변화 강조)
  - ⚠️ 테스트 후 최종 확정 필요

#### 햅틱 피드백
- 동작 시 약한 진동 (UIImpactFeedbackGenerator, style: .light)

---

### FR-103: 적용 범위

| 화면 | 스와이프 | 투 핑거 탭 |
|------|---------|-----------|
| 사진보관함 (GridViewController) | ✅ | ✅ |
| 앨범 (AlbumGridViewController) | ✅ | ✅ |
| 휴지통 탭 (TrashAlbumViewController) | ❌ | ❌ |

- 휴지통 탭에서는 기존 방식 유지 (탭 → 뷰어 → 복구/완전삭제 버튼)

---

### FR-104: 선택 모드와의 관계

- **선택 모드 진입 시**: 스와이프/투 핑거 탭 제스처 비활성화
- **선택 모드에서는 기존 동작 유지**:
  - 탭: 선택/해제 토글
  - 드래그: 연속 선택
- **충돌 방지**: 선택 모드 여부(isSelectMode)로 제스처 활성화 제어

---

### FR-105: 기존 제스처와의 공존

| 제스처 | 용도 | 우선순위 |
|--------|------|---------|
| 한 손가락 탭 | 뷰어 진입 | 기존 유지 |
| 한 손가락 수직 드래그 | 스크롤 | 기존 유지 |
| 한 손가락 수평 드래그 (15°+) | **삭제/복원** | 신규 |
| 두 손가락 탭 (0.2초 이내) | **삭제/복원** | 신규 |
| 두 손가락 핀치 | 줌 (1/3/5열) | 기존 유지 |
| 롱프레스 | (미사용) | - |

## 비기능 요구사항

### NFR-101: 성능
- 스와이프 애니메이션 60fps 유지
- 제스처 인식 지연 < 16ms

### NFR-102: 일관성
- 빨간 딤드 색상/투명도는 기존과 동일 (다크레드 #8B0000, 50% opacity)
- TrashStore 연동 방식 기존과 동일

## 기술 설계 참고

```swift

// .changed에서 translation으로 딤드 영역 업데이트
// .ended에서 거리/속도 체크 후 확정/취소
```

### 투 핑거 탭 제스처 구현
```swift
// UITapGestureRecognizer (numberOfTouchesRequired = 2) 사용
// 또는 커스텀 제스처로 시간/거리 조건 체크
```

## 테스트 시나리오

### TC-101: 스와이프 삭제
1. 일반 사진을 오른쪽으로 50% 이상 스와이프
2. 빨간 딤드가 왼쪽에서 채워짐
3. 손 떼면 딤드 완성 + 햅틱
4. TrashStore에 추가됨

### TC-102: 스와이프 복원
1. 빨간 딤드 사진을 왼쪽으로 50% 이상 스와이프
2. 딤드가 왼쪽으로 걷힘
3. 손 떼면 딤드 완전 제거 + 햅틱
4. TrashStore에서 제거됨

### TC-103: 스와이프 취소
1. 사진을 30%만 스와이프 후 손 뗌 (속도 느리게)
2. 딤드가 원래 상태로 돌아감
3. TrashStore 변경 없음

### TC-104: 빠른 스와이프
1. 사진을 빠르게 (800pt/s 이상) 짧게 스와이프
2. 거리 50% 미만이어도 삭제/복원 확정

### TC-105: 투 핑거 탭 삭제
1. 일반 사진을 두 손가락으로 탭
2. 빨간 딤드 페이드 인 (0.15초) + 햅틱
3. TrashStore에 추가됨

### TC-106: 투 핑거 탭 복원
1. 빨간 딤드 사진을 두 손가락으로 탭
2. 딤드 페이드 아웃 (0.15초) + 햅틱
3. TrashStore에서 제거됨

### TC-107: 선택 모드에서 비활성화
1. 선택 모드 진입
2. 스와이프/투 핑거 탭 시도
3. 제스처 무시됨 (기존 선택 동작만 동작)

### TC-108: 핀치 줌과 구분
1. 두 손가락 터치 후 벌리기 시작
2. 투 핑거 탭이 아닌 핀치 줌으로 인식
3. 열 수 변경 (1/3/5)
