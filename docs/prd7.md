# PRD 7: 그리드 즉시 삭제/복원

## 개요

그리드 화면에서 선택 모드 진입 없이 섬네일을 직접 조작하여 사진을 휴지통으로 이동하거나 복원하는 기능.

### 배경

현재 그리드에서 사진을 삭제하려면:
1. 선택 모드 진입 → 사진 선택 → 삭제 버튼
2. 또는 뷰어 진입 → 스와이프/버튼으로 삭제

이 과정에서 "선택하다가 확대해서 보고 싶을 때" 선택을 취소해야 하는 불편함이 있음.
2차에서는 모드 전환 없이 그리드에서 바로 삭제/복원할 수 있는 제스처를 추가.

### 목표

- 선택 모드 없이 그리드에서 즉시 삭제/복원
- 직관적인 제스처와 시각적 피드백
- 기존 UX(탭→뷰어, 스크롤, 핀치줌)와 충돌 없음

## 기능 요구사항

### FR-101: 섬네일 스와이프 삭제/복원

#### 기본 동작
| 사진 상태 | 스와이프 결과 |
|----------|-------------|
| 일반 사진 | → 빨간 딤드 (휴지통으로 이동) |
| 빨간 딤드 사진 | → 딤드 해제 (복원) |

#### 삭제 후 그리드 상태
- **빨간 딤드 상태로 그리드에 유지** (제거되지 않음)
- 정렬/재배치 없음 (원래 위치 그대로)
- 사용자가 스와이프로 다시 복원 가능

#### 빨간 딤드 상태 지속성
- TrashStore가 파일로 저장됨 (`TrashState.json`)
- **앱 재실행 후에도 휴지통 상태 유지**
- 이전 세션에서 삭제한 항목도 그리드에 빨간 딤드로 표시
- 복원하거나 완전 삭제할 때까지 유지

#### 빨간 딤드 사진 탭 → 뷰어 동작
- 빨간 딤드 사진 탭 시 **ViewerMode.trash**로 뷰어 진입
- 삭제 버튼 대신 **복구/완전삭제 버튼** 표시
- 복구: TrashStore에서 제거 → 딤드 해제
- 완전삭제: iOS 시스템 휴지통으로 이동 (시스템 팝업)

#### 제스처 사양
- **방향**: 양방향 (←/→) 모두 동작
- **시작 위치**: 섬네일 위에서 시작해야 인식 (빈 영역에서 시작한 스와이프는 무시)
- **시작 조건**: 수평선(0°) 기준 ±15° 이내인 경우 스와이프로 인식, 그 외는 스크롤
  - **최소 이동 거리**: 10pt 이상 이동 후 각도 판정 (초기 미세 움직임 오판 방지)
  - 계산: `translation >= 10pt` 후 `angle = atan2(abs(velocity.y), abs(velocity.x))`, angle < 15°이면 스와이프
- **확정 조건** (OR):
  - 스와이프 거리 ≥ 섬네일 너비의 50%
  - 스와이프 속도 ≥ 800pt/s
  - ⚠️ 열 수(1/3/5열)에 따라 셀 크기가 달라 체감이 다를 수 있음, 테스트 후 조정 필요
- **취소 조건**: 스와이프 거리 50% 미만에서 손 떼고, 속도도 800pt/s 미만
- **대상 셀 고정**: 스와이프 중 손가락이 다른 셀로 이동해도 시작한 셀에 대해서만 동작
- **스크롤 중 처리**: 스크롤 momentum 중에는 스와이프 제스처 무시 (momentum 종료 후 인식)

#### 시각 효과
- **삭제 시**: 손가락 뒤에서 빨간 딤드가 따라옴 (손가락이 지나간 자리가 빨갛게 채워짐)
  - 예: 오른쪽 스와이프 → 손가락 뒤(왼쪽)에서 빨간색이 따라옴
- **복원 시**: 손가락이 빨간 딤드를 밀어냄 (손가락 방향으로 빨간색이 걷힘)
  - 예: 오른쪽 스와이프 → 빨간색이 오른쪽으로 밀려나며 사라짐
- **진행 표시**: 손가락 이동에 따라 실시간으로 딤드 영역 변화
- **취소 시**: 딤드가 원래 상태로 슬라이드 (0.2초, spring animation)
- **확정 시**: 나머지 영역이 빠르게 채워지거나 걷힘 (0.15초, ease-out)

#### 햅틱 피드백
- 확정 시 약한 진동 (UIImpactFeedbackGenerator, style: .light)

### FR-102: 투 핑거 탭 삭제/복원

#### 기본 동작
| 사진 상태 | 투 핑거 탭 결과 |
|----------|---------------|
| 일반 사진 | → 빨간 딤드 (휴지통으로 이동) |
| 빨간 딤드 사진 | → 딤드 해제 (복원) |

#### 제스처 사양
- **구현 방식**: iOS 기본 `UITapGestureRecognizer(numberOfTouchesRequired: 2)` 사용
  - iOS가 탭/핀치 구분을 자동 처리
  - 별도 시간/거리 조건 불필요
- **인식 조건**:
  - 두 손가락 동시 터치
  - **두 손가락 모두 같은 섬네일 위에 있어야 함**
- **핀치 줌과 구분**: iOS `UIPinchGestureRecognizer`가 자동 구분
- **다른 섬네일에 걸친 경우**: 두 손가락이 서로 다른 섬네일에 있으면 무시 (동작 안 함)

#### 시각 효과
- **삭제 시**: 빨간 딤드 페이드 인
- **복원 시**: 빨간 딤드 페이드 아웃
- **duration**: 0.15초 (기본값)
  - 대안: 0.2초 (iOS 기본), 0.3초 (변화 강조)
  - ⚠️ 테스트 후 최종 확정 필요

#### 햅틱 피드백
- 동작 시 약한 진동 (UIImpactFeedbackGenerator, style: .light)

### FR-103: 적용 범위

| 화면 | 스와이프 | 투 핑거 탭 | 비고 |
|------|---------|-----------|------|
| 사진보관함 (GridViewController) | ✅ | ✅ | 선택 모드 시 비활성화 |
| 앨범 (AlbumGridViewController) | ✅ | ✅ | 선택 모드 없음, 항상 활성화 |
| 휴지통 탭 (TrashAlbumViewController) | ❌ | ❌ | 기존 방식 유지 |

- 휴지통 탭에서는 기존 방식 유지 (탭 → 뷰어 → 복구/완전삭제 버튼)

#### 휴지통 탭 시각 표현
- **딤드 없이 정상 표시**
- 이유: 휴지통 안에서는 모두 삭제 대상이므로 구분 불필요
- 뷰어 진입 후 복구/완전삭제 버튼 사용

#### 앨범에서의 삭제 동작
- **앨범에서 삭제 = 전체 라이브러리에서 휴지통으로 이동**
- 앨범에서만 제거되는 것이 아님 (iOS 사진앱과 다름)
- TrashStore.moveToTrash() 호출

#### 열 수별 동작
- **1열/3열/5열 모두 동작** (별도 제한 없음)
- 5열에서 투 핑거 탭: 셀 크기(약 77pt)가 작아 두 손가락이 같은 셀에 들어가기 어려움
  - 자연스럽게 인식 실패 → 아무 동작 안 함 (의도된 동작)
  - 스와이프는 정상 동작

---

### FR-104: 선택 모드와의 관계

- **선택 모드 진입 시**: 스와이프/투 핑거 탭 제스처 비활성화
- **선택 모드에서는 기존 동작 유지**:
  - 탭: 선택/해제 토글
  - 드래그: 연속 선택
- **충돌 방지**: 선택 모드 여부(isSelectMode)로 제스처 활성화 제어
- **참고**: AlbumGridViewController는 선택 모드가 없으므로 제스처가 항상 활성화됨

---

### FR-105: 기존 제스처와의 공존

| 제스처 | 용도 | 우선순위 |
|--------|------|---------|
| 한 손가락 탭 | 뷰어 진입 | 기존 유지 |
| 한 손가락 수직 드래그 | 스크롤 | 기존 유지 |
| 한 손가락 수평 드래그 (±15° 이내) | **삭제/복원** | 신규 |
| 두 손가락 탭 | **삭제/복원** | 신규 |
| 두 손가락 핀치 | 줌 (1/3/5열) | 기존 유지 |
| 롱프레스 | (미사용) | - |

#### 충돌 방지 메커니즘
- **스와이프 vs 스크롤**:
  - `gestureRecognizerShouldBegin`에서 각도 판정 (±15° 이내만 스와이프)
  - 대각선 드래그(15° 초과)는 스크롤로 처리
  - `shouldRecognizeSimultaneouslyWith`: false (동시 인식 방지)
- **탭 vs 스와이프**:
  - `UITapGestureRecognizer` 기본값: 이동 10pt 미만이면 탭
  - 스와이프는 의도적 수평 이동이므로 자연스럽게 구분됨
- **투 핑거 탭 vs 핀치 줌**:
  - iOS 기본 제스처 인식기가 자동 구분

### FR-106: TrashStore 연동 및 실패 처리

#### 낙관적 업데이트 + 롤백 패턴
```
1. 스와이프/탭 확정 → 즉시 UI 변경 (빨간 딤드 애니메이션)
2. TrashStore.moveToTrash() 또는 restore() 호출
3-A. 성공 → 완료
3-B. 실패 → 롤백 처리
```

#### 실패 시 롤백 처리
- **롤백 애니메이션**: 빨간 딤드가 원래 상태로 슬라이드/페이드
- **햅틱 피드백**: UINotificationFeedbackGenerator(.error)
- **토스트 메시지**: "저장 실패. 다시 시도해주세요" (하단, 2초 표시 후 자동 사라짐)

#### 실패 가능 상황
- 디스크 공간 부족
- 파일 시스템 오류
- 앱 강제 종료 타이밍

#### 동시/연속 토글 처리 (셀별 잠금)
- **애니메이션 중인 셀에만 추가 제스처 무시**
- 다른 셀은 자유롭게 조작 가능 (여러 사진 연속 삭제 허용)
- 같은 셀에 대한 연속 토글(삭제→즉시복원)만 방지
- 애니메이션 완료 후 해당 셀 잠금 해제

## 비기능 요구사항

### NFR-101: 성능
- 스와이프 애니메이션 60fps 유지
- 제스처 인식 지연 < 16ms
- **측정 조건**:
  - 사진 수: 5만 장 라이브러리
  - 스크롤 상태: idle (momentum 아님)
  - 측정 구간: 터치 시작 → 딤드 애니메이션 첫 프레임

### NFR-102: 일관성
- 빨간 딤드 색상/투명도는 기존과 동일 (다크레드 #8B0000, 50% opacity)
- TrashStore 연동 방식 기존과 동일

### NFR-103: 접근성
- **VoiceOver 활성화 감지 시 스와이프/투 핑거 탭 제스처 비활성화**
  - `UIAccessibility.isVoiceOverRunning` 체크
  - VoiceOver 시스템 제스처와 충돌 방지
  - **전역 override**: 모든 화면에서 적용 (AlbumGridViewController 포함)
- 대체 방법: 선택 모드를 통한 삭제 (GridViewController)
- AlbumGridViewController는 선택 모드가 없으므로 뷰어 진입 후 삭제 버튼 사용
- **향후 고려** (2차 범위 외):
  - 햅틱 피드백 비활성화 옵션
  - 제스처 감도 조정 옵션

## 기술 설계 참고

### 스와이프 제스처 구현
```swift
// UIPanGestureRecognizer 사용
// gestureRecognizerShouldBegin에서:
//   1. 최소 이동 거리 체크 (10pt 이상)
//   2. 각도 판정: angle = atan2(abs(velocity.y), abs(velocity.x))
//   3. return angle < (15.0 * .pi / 180.0) // 수평 ±15° 이내만 인식
// .changed에서 translation으로 딤드 영역 업데이트
// .ended에서 거리/속도 체크 후 확정/취소
```

### 투 핑거 탭 제스처 구현
```swift
// iOS 기본 UITapGestureRecognizer 사용
let twoFingerTap = UITapGestureRecognizer()
twoFingerTap.numberOfTouchesRequired = 2
// iOS가 핀치 줌과 자동 구분
// 두 손가락 위치가 같은 섬네일인지만 확인
```

### 딤드 애니메이션
- 스와이프: CALayer mask 또는 clipsToBounds + frame 조작
- 페이드: UIView.animate with alpha

## 테스트 시나리오

### TC-101: 스와이프 삭제
1. 일반 사진을 오른쪽으로 스와이프 거리 50% 이상 스와이프
2. 손가락 뒤(왼쪽)에서 빨간 딤드가 따라옴
3. 손 떼면 딤드 완성 + 햅틱
4. TrashStore에 추가됨

### TC-102: 스와이프 복원
1. 빨간 딤드 사진을 왼쪽으로 스와이프 거리 50% 이상 스와이프
2. 딤드가 왼쪽으로 밀려나며 걷힘
3. 손 떼면 딤드 완전 제거 + 햅틱
4. TrashStore에서 제거됨

### TC-103: 스와이프 취소
1. 사진을 스와이프 거리 30%만 스와이프 후 손 뗌 (속도 느리게)
2. 딤드가 원래 상태로 돌아감
3. TrashStore 변경 없음

### TC-104: 빠른 스와이프
1. 사진을 빠르게 (800pt/s 이상) 짧게 스와이프
2. 스와이프 거리 50% 미만이어도 삭제/복원 확정

### TC-105: 투 핑거 탭 삭제
1. 일반 사진을 두 손가락으로 탭 (같은 섬네일 위)
2. 빨간 딤드 페이드 인 (0.15초) + 햅틱
3. TrashStore에 추가됨

### TC-106: 투 핑거 탭 복원
1. 빨간 딤드 사진을 두 손가락으로 탭 (같은 섬네일 위)
2. 딤드 페이드 아웃 (0.15초) + 햅틱
3. TrashStore에서 제거됨

### TC-107: 선택 모드에서 비활성화
1. 선택 모드 진입
2. 스와이프/투 핑거 탭 시도
3. 제스처 무시됨 (기존 선택 동작만 동작)

### TC-108: 핀치 줌과 구분
1. 두 손가락 터치 후 벌리기 시작
2. 투 핑거 탭이 아닌 핀치 줌으로 인식
3. 열 수 변경 (1/3/5)

### TC-109: 연속 스와이프 삭제
1. 여러 사진을 연속으로 빠르게 스와이프 삭제
2. 각 사진에 대해 정상적으로 딤드 애니메이션 및 햅틱 동작
3. 모든 사진이 TrashStore에 추가됨

### TC-110: 스와이프 중 다른 셀로 이동
1. 사진 A에서 스와이프 시작
2. 손가락을 사진 B 영역으로 이동
3. 사진 A에 대해서만 딤드 애니메이션 동작 (B는 영향 없음)

### TC-111: 스크롤 momentum 중 스와이프
1. 그리드를 빠르게 스크롤하여 momentum 발생
2. momentum 중에 섬네일 스와이프 시도
3. 스와이프 무시됨 (momentum 유지)
4. momentum 종료 후 스와이프하면 정상 동작

### TC-112: 두 손가락이 다른 섬네일에서 탭
1. 한 손가락은 사진 A, 다른 손가락은 사진 B에서 동시 탭
2. 아무 동작 없음 (두 손가락이 같은 섬네일에 있어야 함)

### TC-113: TrashStore 저장 실패 시 롤백
1. (테스트용) TrashStore 저장 실패 시뮬레이션
2. 스와이프 삭제 시도 → 빨간 딤드 애니메이션 진행
3. 저장 실패 감지
4. 롤백 애니메이션 (딤드가 원래 상태로 돌아감)
5. 에러 햅틱 (진동)
6. 토스트 메시지 표시 ("저장 실패. 다시 시도해주세요")
7. TrashStore 상태 변경 없음

### TC-114: VoiceOver 활성화 시 제스처 비활성화
1. VoiceOver 활성화
2. 스와이프/투 핑거 탭 시도
3. 제스처 무시됨 (VoiceOver 제스처만 동작)

### TC-115: 스와이프 중 앱 백그라운드 진입
1. 스와이프 40% 진행 중
2. 홈 버튼으로 앱 백그라운드 전환
3. 다시 포그라운드로 복귀
4. 딤드가 원래 상태로 리셋됨 (스와이프 취소)
5. TrashStore 변경 없음

### TC-116: 대각선 드래그 시 스크롤 처리
1. 섬네일 위에서 대각선(30°)으로 드래그 시작
