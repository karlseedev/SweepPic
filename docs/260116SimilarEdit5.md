| # | 증상 | 설명 | 심각도 |

## 2. 현행 코드 구조 요약 (2026-01-16 기준)

**파이프라인:**
```
사진 → YuNet 감지 → 랜드마크 추출 → FaceAligner 정렬 → SFace 임베딩 → 슬롯 매칭 → CachedFace 저장
```

**핵심 파일:**
| 파일 | 역할 |
|------|------|
| `SimilarityAnalysisQueue.swift` | 전체 파이프라인 조율, 슬롯 매칭 |
| `YuNetFaceDetector.swift` | 얼굴 감지 + 5-point 랜드마크 |
| `FaceAligner.swift` | 랜드마크 기반 112×112 정렬 |
| `SFaceRecognizer.swift` | 128차원 임베딩 추출 |
| `SimilarityConstants.swift` | 임계값 상수 |

**매칭 전략:**
- 전역 후보 정렬 (Cost 오름차순) + Grey Zone
- Keep Best (고품질 임베딩으로 슬롯 갱신)
- 슬롯 위치 갱신 (매칭마다 center/boundingBox 갱신)
- 저품질 경로 (norm < 7: 위치 우선 매칭)

## 3. 개선해야 할 문제 (Problems to Solve)

### 3.1 얼굴 감지 정확도
- **False Positive**: 얼굴이 아닌 것을 얼굴로 감지
- **False Negative**: 실제 얼굴을 감지하지 못함
- **위치 오차**: 감지된 바운딩 박스가 실제 얼굴과 다름

### 3.2 얼굴 임베딩 품질
- **저품질 얼굴**: 흐릿함, 측면, 가림 등으로 임베딩 품질 저하
- **동일인 거리**: 같은 사람의 임베딩 거리가 threshold보다 멀어지는 경우
- **타인 거리**: 다른 사람의 임베딩 거리가 threshold보다 가까워지는 경우

### 3.3 슬롯 할당 로직
- **초기 슬롯 결정**: 첫 번째 사진에서 잘못된 슬롯 생성
- **점진적 오염**: 잘못된 매칭이 누적되어 슬롯 품질 저하

S3: 얼굴 미검출 ←─ 감지 실패                ←── 1. 얼굴 감지 모델 한계

### 4.5 슬롯 매칭 알고리즘 (Slot Matching)

**현행 로직 (2026-01-16 기준):**
- 전역 후보 정렬 (Cost 오름차순) + Grey Zone 전략 적용됨
- 동적 인물 풀 + Keep Best + 슬롯 위치 갱신 적용됨

| # | 원인 | 설명 | 영향 | 테스트 상태 |
|---|------|------|------|------------|
| 1 | **cost Top-K 후보 편향** | 슬롯 > 5개 시 cost 상위 3개만 후보 | 위치상 가까운 슬롯이 후보 제외 | 테스트 필요 |
| 2 | **저품질 경로 과완화** | posNorm ≤ 0.25 + cost < 0.787이면 매칭 | 다른 사람도 합쳐질 위험 | 테스트 필요 |
| 3 | **저품질 신규 슬롯 금지** | norm < 7이면 신규 슬롯 생성 불가 | 잘못된 슬롯 강제 매칭 또는 누락 | 테스트 필요 |

**과거 3대 결함 (개선됨):**
| 결함 | 현재 상태 |
|------|----------|
| 위치 선행 필터링 | ✅ 해결 - 전역 정렬 후 Grey Zone |
| 정적 기준 슬롯 | ✅ 해결 - 동적 인물 풀 |
| 선착순 탐욕 매칭 | ✅ 해결 - Cost 정렬 후 순차 확정 |
| 슬롯 위치 고정 | ✅ 해결 - updateSlotIfBetter에서 위치 갱신 |

### 4.6 임계값 설정 (현행 값)

| 상수명 | 현재값 | 용도 |
|--------|--------|------|
| `greyZoneThreshold` | **0.45** | 확신/모호 구간 경계 |
| `personMatchThreshold` | **0.637** | 거절 임계값 (LFW 기준) |
| `greyZonePositionLimit` | **0.20** | Grey Zone 위치 조건 |
| `lowQualityPosLimit` | **0.25** | 저품질 위치 조건 (완화) |
| `lowQualityCostLimit` | **0.787** | 저품질 cost 상한 (0.637+0.15) |

**매칭 구간:**
| 구간 | Cost 범위 | 조건 |
|------|-----------|------|
| 확신 | < 0.45 | 즉시 매칭 |
| 모호 (Grey) | 0.45 ~ 0.637 | posNorm < 0.20 시 매칭 |
| 거절 | ≥ 0.637 | 매칭 실패 |
| 저품질 | cost < 0.787 | posNorm ≤ 0.25 시 매칭 |

## 6. 대응 방안 검토

### 6.1 감지 단계 개선안

| 방안 | 장점 | 단점 | 우선순위 |
|------|------|------|----------|
| Score threshold 상향 (0.6→0.7) | FP 감소 | 작은 얼굴 FN 증가 | 검토 중 |
| 가장자리 필터링 (x<0.05 제외) | 경계 FP 방지 | 실제 가장자리 얼굴 놓침 | 검토 중 |
| Vision+YuNet 교집합 사용 | 가장 안전 | 검출률 저하 | 검토 중 |
| Vision+YuNet 합집합 사용 | 검출률 최대 | FP 증가 | 검토 중 |

### 6.2 임베딩 단계 개선안

| 방안 | 장점 | 단점 | 우선순위 |
|------|------|------|----------|
| 저품질 필터 강화 | 오염 방지 | 정보 손실 | 구현됨 |
| 다중 임베딩 비교 | 안정성 증가 | 연산량 증가 | 검토 중 |
| Threshold 동적 조정 | 유연성 | 복잡도 증가 | 검토 중 |

### 6.3 슬롯 매칭 개선안 (GPT 제안)

| 방안 | 설명 | 영향 |
|------|------|------|
| **위치 후보 강제 포함** | Top-K에 pos 기준 가장 가까운 슬롯 1개 추가 | 후보 편향 완화 |
| **저품질 규칙 강화** | posNorm 0.25 → 0.15, cost 상한 낮춤 | over-merge 감소 |
| **YuNet 실패 시 Vision fallback** | YuNet=0이면 rawFacesMap 사용 | 미검출 감소 |
