# 자동 정리 UX 재설계 — 단계적 확인 방식

## 배경

`260130AestheticsScore5.md`에서 3모드 체계(완화/기본/강화)를 설계하고 ModeComparisonTester로 테스트를 진행했습니다.
테스트 과정에서 **"사용자에게 완화/기본/강화 옵션을 그냥 주면 어느 수준인지 모른다"** 는 UX 문제가 발견되었습니다.

### 핵심 문제

- 완화/기본/강화라는 텍스트만으로는 각 모드가 어떤 사진을 잡는지 판단 불가
- 사용자가 적절한 옵션을 선택하려면 **결과를 눈으로 봐야** 함
- 현재 시스템은 분석하면서 즉시 휴지통으로 이동하므로 미리보기 자체가 불가능

---

## 1. 조사 결과 요약 (사진 정리 앱 UX 패턴)

| 앱 | 방식 |
|-----|------|
| Gemini Photos | 도넛 차트 + Smart Cleanup / 수동 리뷰 이중 경로 |
| CleanMyPhone | AI 카테고리 분류 (스크린샷, 흐린 사진, 유사 등) + 그룹별 정리 |
| Slidebox / Flic | 틴더 스타일 스와이프 (한 장씩 판단) |
| Apple Photos (iOS 18.1) | AI 제안 + 민감도 조절 |
| Google Photos | 카테고리별 정리 제안 + 스와이프 리뷰 |

### 주요 인사이트

1. **"정리 강도 슬라이더/모드 선택"은 주류가 아님** — 대부분 카테고리 그룹핑 사용
2. **이중 경로가 표준** — 원탭 자동 + 수동 리뷰 동시 제공
3. **미리보기가 신뢰의 핵심** — 삭제 전 반드시 눈으로 확인 가능해야 함
4. **카테고리 기반 그룹핑이 업계 표준** — 흐린/스크린샷/유사 등으로 분류

## 2. 검토한 대안

| 안 | 방식 | 설명 |
|----|------|------|
| A | 모드별 탭 미리보기 | 3개 탭(신중/일반/적극) 전환하며 그리드 확인 |
| **B** | **기본 + 사후 조절** | **기본 모드로 보여주고 "더 찾기/줄이기" 버튼** |
| C | 카테고리 그룹 | 모드 제거, 잡힌 이유별(흐림/노출/미적) 그룹화 |
| **D** | **단계적 확인** | **확실한 것부터 단계적으로 확장** |
| E | 요약 카드 비교 | 3개 카드(장수 + 대표 썸네일)로 비교 후 선택 |

### 결정: D안 (단계적 확인) 채택

**선택 이유:**

1. **신뢰 순서** — 확실한 저품질부터 보여줘서 "이 앱 판단이 맞네"라는 신뢰가 먼저 생김
2. **판단 방향이 단순** — "이거 괜찮으면 정리, 아니면 더 보기" 한 방향만 고민
3. **자연스러운 탈출** — 보수적 사용자는 1단계에서 바로 끝낼 수 있음
4. **추가분의 맥락이 명확** — 이전 단계와 비교해서 추가분의 수준을 체감 가능

## 3. 흐름 변경

### 현재 흐름
정리 시작 → 분석 + 즉시 휴지통 이동 → 완료 Alert

### 변경 후 흐름
정리 시작 → 분석 (이동 없이 결과 수집) → 미리보기 그리드 → 사용자 확인 → 휴지통 이동

### 3모드 임계값 (기존 유지)

| 단계 | 모드 | path2 임계값 | 로직 |
|------|------|-------------|------|
| 1단계 | 완화 | -0.3 | path1 OR path2(< -0.3) |
| 2단계 | 기본 | 0.0 | path1 OR path2(< 0.0) |
| 3단계 | 강화 | 0.2 | path1 OR path2(< 0.2) |

## 4. UI 설계 — 단계적 확인

### 핵심 원칙

- **미리보기 화면 = 잡힌 사진만 모아놓은 별도 그리드 VC** (메인 그리드에서 push)
- 사진 탭 → 전체화면 보기 가능
- "기준 낮춰서 더 보기" 시 추가분을 사용자가 명확하게 인지할 수 있어야 함

### 추가분 인지 문제

단계를 올릴 때 추가된 사진을 사용자가 얼마나 명확히 인지할 수 있느냐가 핵심 UX 과제.
테두리 색, 배지만으로는 인지가 떨어질 수 있어 3가지 방향을 검토함.

### 방향 1: 추가분만 별도 화면 (push)

추가분만 새 화면으로 push. 기존 사진과 섞이지 않아 인지 문제 없음.

- 1단계: 47장 그리드 → "기준 낮춰서 더 보기" → 2단계 push
- 2단계: 추가 36장만 표시 → "83장 정리하기" 또는 "빼고 47장만" 또는 "더 보기" → 3단계 push
- 3단계: 추가 41장만 표시 → "124장 정리하기" 또는 "빼고 83장만"

장점: 추가분 인지 100% 명확, 화면이 단순
단점: 화면 전환이 많음, 전체 그림을 한눈에 못 봄

### 방향 2: 기존 사진 접고, 추가분에 포커스

한 화면 안에서 기존 사진이 축소/접히고 추가분이 메인이 됨.
"이전 단계 47장 ▶" 접힌 섹션 탭하면 펼쳐짐.

장점: 한 화면에서 맥락 유지, 추가분에 집중
단점: 접기/펼치기 UX가 사진 앱에서 낯설 수 있음

### 방향 3: 그리드 유지 + 인라인 배너 + 자동 스크롤 ← 우선 구현

한 그리드에 합치되, 추가분 시작점에 배너를 삽입하고 자동 스크롤로 추가분 시작점을 보여줌.

- "기준 낮춰서 더 보기" 탭 → 그리드에 추가분 삽입 + 인라인 배너("↓ 기준을 낮춰 36장 추가") + 자동 스크롤
- 3단계까지 가면 배너 2개 + 3구간으로 나뉜 그리드

장점: 화면 전환 없음, 전체를 한 그리드에서 확인, 배너+자동스크롤로 추가분 인지
단점: 추가분 인지가 방향 1보다 약할 수 있음 (실제 테스트 필요)

### 방향 비교

| | 방향 1 (별도 화면) | 방향 2 (접기) | 방향 3 (인라인 배너) |
|---|---|---|---|
| 추가분 인지 | 매우 명확 | 명확 | 보통 (테스트 필요) |
| 전체 맥락 | 못 봄 | 부분적 | 전체 볼 수 있음 |
| 화면 전환 | 많음 (push/pop) | 없음 | 없음 |
| 구현 복잡도 | 높음 (화면 3개) | 중간 | 낮음 (1화면) |
| UX 자연스러움 | 보통 | 낯설 수 있음 | 자연스러움 |

### 진행 계획: 방향 3 우선 구현 후 테스트

방향 3이 가장 구현이 단순하고 자연스러움. 실제 테스트 후 추가분 인지가 부족하면 방향 1로 전환.
**방향 3 → 테스트 → 판단 → (필요 시) 방향 1 전환**

## 5. 사용자 스토리

### 보수적인 사용자 (민수)

1. 메인 그리드에서 정리 버튼 탭
2. 분석 진행 (하단 토스트: "분석 중... 1,847장")
3. 분석 완료 → 미리보기 화면 push ("저품질 사진 47장")
4. 그리드를 스크롤하며 훑어봄 — 전부 확실한 저품질
5. "47장 정리하기" 탭 → 확인 → 휴지통 이동 → 완료
6. 10초 만에 끝

### 적극적인 사용자 (영희)

1. 같은 흐름으로 47장 미리보기 도착
2. "기준 낮춰서 더 보기" 탭 → 배너 삽입 + 자동 스크롤
3. 추가 36장 확인 — 괜찮음
4. "기준 낮춰서 더 보기" 한번 더 → 배너 2 삽입 + 자동 스크롤
5. 추가 41장 확인 — 좀 아까운 게 있음
6. "빼고 83장만 정리" 선택 → 3단계 제외, 1+2단계만 정리

### 바쁜 사용자 (지훈)

1. 47장 미리보기 도착
2. 대충 훑어보고 "47장 정리하기" 바로 탭
3. 끝

| 항목 | 설명 |
|------|------|
