**핵심 피드백 (심각도 순)**
1. `[높음]` C-2 실패 시에도 `markAsShown()`이 먼저 찍히는 설계는 목표 달성 실패를 숨길 수 있습니다.  
`docs/onboarding/260211onboarding-C.md:207`, `docs/onboarding/260211onboarding-C.md:214`  
현재 계획대로면 얼굴 비교 진입 실패(`firstFace` 없음, `present` 충돌 등)에도 재노출이 막힐 수 있습니다. `markAsShown()`은 C-2 네비게이션 성공 이후로 미루는 게 안전합니다.

2. `[높음]` `confirmTapped()` C 분기에 재진입 방지(중복 탭 방지) 설계가 빠져 있습니다.  
`docs/onboarding/260211onboarding-C.md:131`  
C는 0.8초 이상 비동기 시퀀스라 중복 호출 시 이중 push/present 위험이 있습니다. `isConfirmInProgress` + `confirmButton.isEnabled = false`를 명시하세요.

3. `[높음]` C-2 대기 2초 타임아웃 + 실패 시 `markAsShown()`은 실제 환경에서 과도하게 스킵될 가능성이 큽니다.  
`docs/onboarding/260211onboarding-C.md:187`, `docs/onboarding/260211onboarding-C.md:418`, `docs/onboarding/260211onboarding-C.md:421`  
문서도 cache miss 수초 가능성을 인정하고 있어 충돌합니다. 최소한 타임아웃 연장(예: 5~8초) 또는 타임아웃 시 `markAsShown()` 미적용이 필요합니다.

4. `[중간]` `dismissCurrent()` 가드 방향은 맞지만, 상태 수명 관리가 취약합니다.  
`docs/onboarding/260211onboarding-C.md:353`  
`isWaitingForC2`가 true로 고착되면 모든 dismiss 경로가 막힙니다. “모든 실패/중단 경로에서 flag 해제”를 체크리스트로 명시해야 합니다.

5. `[중간]` `dismissCurrent()` 호출 지점 분석이 4곳으로 되어 있으나 실제 코드는 5곳입니다.  
문서: `docs/onboarding/260211onboarding-C.md:327`  
코드 추가 지점: `SweepPic/SweepPic/Features/Viewer/SwipeDeleteHandler.swift:80`  
위협도는 낮아도(오버레이 hitTest 차단) 문서 정확성 관점에서 보완 필요합니다.

6. `[중간]` C-1 네비게이션 타깃을 `indexPath`만 보존하면 데이터 변경 시 오탐 가능성이 있습니다.  
`docs/onboarding/260211onboarding-C.md:97`, `docs/onboarding/260211onboarding-C.md:109`  
`assetID`를 함께 캡처하고 confirm 시점에 재해석하는 방식이 더 안전합니다.

**요청하신 포인트 집중 검토**
- `dismissCurrent` 가드: UIKit 동작 기준으로 **필수에 가깝고 타당**합니다. `viewWillDisappear`는 사용자 터치 없이도(화면 전환만으로) 호출됩니다. 현재 코드에서도 `GridViewController.viewWillDisappear`와 `ViewerViewController.viewWillDisappear`에서 dismiss가 호출됩니다. (`SweepPic/SweepPic/Features/Grid/GridViewController.swift:361`, `SweepPic/SweepPic/Features/Viewer/ViewerViewController.swift:399`)
- `confirmTapped` 분기: **필수 설계이고 방향은 맞습니다.** 현재 구현은 즉시 `dismiss()`라 C 플로우를 구현할 수 없습니다. (`SweepPic/SweepPic/Shared/Components/CoachMarkOverlayView.swift:465`) 다만 위 1~3번(성공 판정, 재진입 방지, 타임아웃 정책) 보완이 필요합니다.
