주요 판단부터 말하면, 계획의 핵심 가정은 맞습니다. 특히 `UIPageViewController`에서 스와이프만 `didFinishAnimating`로 잡고, 순환 버튼의 `setViewControllers`와 구분하려는 방향은 타당합니다. 다만 아래 3가지는 보완이 필요합니다.

- 높음: 힌트/상태가 `FaceScan` 진입 경로까지 새어 나갑니다. 계획은 C-3 완료 여부를 전역 `UserDefaults`로 보고 힌트를 제어하지만 [curious-coalescing-whistle.md](/Users/karl/.claude/plans/curious-coalescing-whistle.md#L12) [curious-coalescing-whistle.md](/Users/karl/.claude/plans/curious-coalescing-whistle.md#L61), 실제 `FaceComparisonViewController`는 `.viewer`뿐 아니라 `.faceScan`에서도 재사용됩니다 [FaceComparisonViewController.swift](/Users/karl/Project/Photos/iOS/SweepPic/SweepPic/Features/SimilarPhoto/UI/FaceComparisonViewController.swift#L35) [FaceScanListViewController.swift](/Users/karl/Project/Photos/iOS/SweepPic/SweepPic/Features/FaceScan/UI/FaceScanListViewController.swift#L636). 지금 설계대로면 `viewer`에서 C-3을 본 적 없는 사용자는 `FaceScan`에서 힌트가 영구 미노출되고, 반대로 한 번 본 뒤에는 `FaceScan`에서도 같은 힌트가 뜹니다. 의도가 `viewer` 전용이면 `showSwipeHintIfNeeded()` 초반에 `guard !mode.isFaceScan`를 명시해야 합니다. 전 경로 공통 기능이면 C-3 게이트를 빼거나 경로별 상태를 분리해야 합니다.

- 중간: `hasEverSwiped` + `postSwipeHintCount` 2키 상태머신의 불변식이 문서에 없습니다. 계획은 `hasEverSwiped == true && postSwipeHintCount < 2`에서 노출과 동시에 `count++` 하도록 되어 있는데 [curious-coalescing-whistle.md](/Users/karl/.claude/plans/curious-coalescing-whistle.md#L13) [curious-coalescing-whistle.md](/Users/karl/.claude/plans/curious-coalescing-whistle.md#L41), 이러면 한 키만 리셋되거나 `showSwipeHintIfNeeded()`가 재호출될 때 카운트가 쉽게 어긋납니다. 최소한 문서에 다음을 박아 두는 게 안전합니다: `hasEverSwiped == false`면 `postSwipeHintCount`는 무시하거나 0으로 정규화, 첫 성공 스와이프에서 `hasEverSwiped = true`만 세팅하고 카운트는 0 유지, `count++`는 “실제 힌트를 붙인 뒤 1회만” 증가. 특히 호출 지점이 [setupInitialPageIfReady()](/Users/karl/Project/Photos/iOS/SweepPic/SweepPic/Features/SimilarPhoto/UI/FaceComparisonViewController.swift#L516)라서, 구현은 `showSwipeHintIfNeeded()`를 사실상 idempotent하게 만들어야 합니다.

- 중간: 검증 항목에 `didFinishAnimating`의 가장 중요한 엣지케이스가 빠졌습니다. 현재 계획은 `completed == true`를 써서 방향은 맞지만 [curious-coalescing-whistle.md](/Users/karl/.claude/plans/curious-coalescing-whistle.md#L23), 테스트 목록에는 “살짝 스와이프 후 취소”가 없습니다 [curious-coalescing-whistle.md](/Users/karl/.claude/plans/curious-coalescing-whistle.md#L83). Apple 기준으로 이 delegate는 gesture-initiated transition 종료 시 오고, `completed`가 실제 완료/취소를 구분합니다. 따라서 반드시 추가해야 할 시나리오는 2개입니다: 부분 스와이프 후 손 떼서 `completed == false`면 `hasEverSwiped`가 바뀌지 않고 힌트가 유지되는지, 순환 버튼 경로는 여전히 delegate를 타지 않아 상태가 안 바뀌는지. 후자는 현재 코드도 같은 전제를 씁니다 [FaceComparisonViewController.swift](/Users/karl/Project/Photos/iOS/SweepPic/SweepPic/Features/SimilarPhoto/UI/FaceComparisonViewController.swift#L692) [ViewerViewController+Setup.swift](/Users/karl/Project/Photos/iOS/SweepPic/SweepPic/Features/Viewer/ViewerViewController+Setup.swift#L268).

추가로 확인한 점 하나는, C-3 완료 판정을 `CoachMarkType.faceComparisonGuide.hasBeenShown`로 보는 자체는 괜찮습니다. 이 코치마크는 dismiss 시 자동으로 `markAsShown()` 됩니다 [CoachMarkOverlayView.swift](/Users/karl/Project/Photos/iOS/SweepPic/SweepPic/Shared/Components/CoachMarkOverlayView.swift#L932). 그래서 “C-3 아직 안 끝났으면 힌트 숨김”이라는 게이트 자체는 맞습니다.

권장 수정은 간단합니다. 계획 문서에 `mode` 범위를 먼저 명시하고, `UserDefaults` 불변식 3개를 적고, 검증 항목에 `completed == false` 취소 케이스를 추가하면 됩니다.
