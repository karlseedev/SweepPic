**Open Questions / Gaps**

- `FaceComparisonViewController`를 정말 재사용하려면, `SimilarityCache`를 공유할지 버릴지 먼저 결정해야 합니다. 공유하면 기존 유사사진 기능과 충돌하고, 버리면 비교 화면에 필요한 얼굴 데이터 주입 경로를 새로 설계해야 합니다.
- 자동 테스트 계획이 없습니다. 최소한 `continueFrom` 동률 날짜 케이스, FaceScan diff 복원, 캐시 격리, iOS 26 present 경로는 테스트 항목으로 추가하는 편이 안전합니다.

총평하면, 이 계획은 방향은 맞지만 그대로 구현에 들어가기에는 핵심 정합성 이슈가 남아 있습니다. 특히 `SimilarityCache` 격리 전략과 `FaceComparisonVC` 데이터 주입 방식이 먼저 보강되어야 설계 문서 목표를 안정적으로 달성할 수 있습니다. 웹검색은 하지 않았고, 로컬 설계 문서와 현재 코드만으로 판단했습니다.
