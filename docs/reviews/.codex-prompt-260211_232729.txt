# SweepPic 온보딩 기획안

## 원칙

**앱 열면 바로 사진. 쓰다가 필요할 때 알려준다.**

- 첫 실행 Welcome/소개 화면: 없음
- 권한 Priming 화면: 없음 (기존 PermissionVC 그대로)
- 온보딩 = 컨텍스트 코치마크 5개 (4가지 특장점 커버)
- 다크모드 전용 (앱 강제 .dark)

---

## 코치마크 A — 그리드 스와이프 삭제

```
트리거: 메인 그리드 첫 진입 후 2초 경과 시
위치: 화면 중앙부 셀 위 오버레이

   ← [손가락 애니메이션: 셀을 옆으로 스와이프]

   "사진을 밀어서 바로 정리하세요
    다시 밀면 복원돼요"

          [해볼게요]
```

- 사용자가 가장 먼저 마주치는 화면 = 그리드
- 실제 스와이프 시도 시 자동 dismiss
- 핵심 전달: 열지 않고 정리 + 복원 가능

---

## 코치마크 B — 뷰어 스와이프 삭제

```
트리거: 뷰어에서 사진을 처음 열었을 때
위치: 화면 하단부

   ↑ [손가락 애니메이션: 위로 스와이프]

   ↑ [손가락 애니메이션: 위로 스와이프]

   "위로 밀면 삭제, 연속으로 밀면 연속 삭제
    삭제된 사진은 휴지통에서 복구할 수 있어요"

          [해볼게요]
```

- 그리드 삭제와 다른 방식(위로 스와이프)임을 명확히
- 복구 안내도 함께 전달 (별도 코치마크 불필요)
- 실제 스와이프 성공 시 자동 dismiss

---

## 코치마크 C — 유사 사진·얼굴 비교 (2단계)

### C-1: 그리드 뱃지

```
트리거: 유사 사진 뱃지가 1초 이상 안정 표시될 때
위치: 해당 셀 하이라이트

   [뱃지+테두리 하이라이트 + 말풍선]

   "비슷한 사진을 자동으로 찾았어요
    탭해서 비교해보세요"

          [확인]
```

### C-2: 얼굴 + 버튼

```
트리거: 뷰어에서 얼굴 + 버튼이 처음 표시될 때
위치: + 버튼 하이라이트

   [+ 버튼 하이라이트 + 말풍선]

   "얼굴별로 비교할 수 있어요
    인물이 여러 명이면 스와이프로 전환"

          [확인]
```

- C-1, C-2는 각각 독립 플래그로 관리 (순서 의존성 없음, 어느 쪽이 먼저 트리거되어도 무방)
- C-2 하이라이트 대상: 얼굴 위의 +버튼 (iOS 버전 무관하게 동일 위치)

---

## 코치마크 D — 저품질 자동 정리

```
트리거: 그리드 2~3회차 세션 시 (콜드런치 기준, scene(willConnectTo:)에서 카운트)
위치: 정리 버튼 하이라이트

   [정리 버튼 하이라이트 + 말풍선]

   "흐리고 어두운 사진을 자동으로 찾아줘요
    소중한 사진은 자동으로 걸러주니 안심하세요"

          [확인]
```

- 첫 진입이 아닌 2~3회차: 첫 방문은 사진 탐색에 집중, 정리는 익숙해진 후
- "자동 보호" 언급으로 사용 허들 낮춤

---

## 표시 순서와 충돌 방지

동시에 여러 코치마크가 뜨면 안 되므로:

```
첫 세션:
  그리드 진입 → [A: 그리드 스와이프] (2초 후)
  사진 탭 → [B: 뷰어 스와이프] (즉시)
  뱃지 안정 → [C-1: 유사 뱃지] (1초 유지 후)
  + 버튼 표시 → [C-2: 얼굴 비교] (즉시)

2~3회차 세션:
  그리드 진입 → [D: 자동 정리] (2초 후)
```

규칙: 코치마크는 한 번에 하나만 표시. `CoachMarkOverlay`를 싱글톤 + 내부 큐로 설계하여 전역 관리. 현재 것이 닫힌 후 다음 표시.

## 코치마크별 카피 정리

| ID | 한 줄 요약 | 핵심 전달 | 안심 요소 |
|----|-----------|----------|----------|
| A | 밀어서 바로 정리 | 열지 않고 삭제 | 다시 밀면 복원 |
| B | 위로 밀면 연속 삭제 | 팝업 없이 빠른 정리 | 휴지통에서 복구 |
| C-1 | 비슷한 사진 자동 발견 | 직접 찾을 필요 없음 | — |
| C-2 | 얼굴별 비교 | 탭 한 번으로 비교 | 여러 인물 스와이프 |
| D | 저품질 자동 탐색 | AI가 찾아줌 | 소중한 사진 자동 필터링 |

## iOS 버전별 고려사항

코치마크 하이라이트 대상이 iOS 버전에 따라 다름:

| 항목 | iOS 16~25 (useFloatingUI) | iOS 26+ |
|------|--------------------------|---------|
| 정리 버튼 (D) | FloatingTitleBar 우측 | NavigationBar 우측 |
| 뱃지 (C-1) | PhotoCell 우상단 (동일) | PhotoCell 우상단 (동일) |
| 얼굴 +버튼 (C-2) | 얼굴 위 FaceButton (동일) | 얼굴 위 FaceButton (동일) |

- 구현 시 정리 버튼(D)만 `useFloatingUI` 분기로 동적 참조 필요
- C-1, C-2는 iOS 버전 무관하게 동일 위치

## 트리거 가드 조건

모든 코치마크는 아래 조건을 만족해야 표시:

| 가드 | 설명 |
|------|------|
| `!UIAccessibility.isVoiceOverRunning` | VoiceOver 시 제스처/유사사진 기능 비활성화이므로 코치마크도 표시하지 않음 |
| `!isScrolling` | 스크롤 중에는 표시하지 않음 |
| 현재 표시 중인 코치마크 없음 | `CoachMarkOverlay.shared`의 큐로 관리 |
