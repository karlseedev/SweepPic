**리뷰 결과**
현재 문서 계획만으로는 목표 달성에 **불충분**합니다. 특히 번호 일관성/데이터 흐름에서 중요한 누락이 있습니다.

1. `심각` 번호 기준(원본 순서)이 현재 캐시 구현에서 깨질 수 있습니다.  
`memberAssetIDs`를 기준으로 번호를 고정한다고 했지만, 그룹 병합 시 `Set -> Array` 변환으로 순서가 비결정적으로 바뀝니다. 그러면 같은 사진 번호가 재분석/병합 후 달라질 수 있습니다.  
참조: `docs/similar/260214numbering.md:13`, `docs/similar/260214numbering.md:16`, `PickPhoto/PickPhoto/Features/SimilarPhoto/Analysis/SimilarityCache.swift:440`, `PickPhoto/PickPhoto/Features/SimilarPhoto/Analysis/SimilarityCache.swift:476`

2. `높음` 문서 데이터 흐름대로면 `FaceComparisonDataSource` 프로토콜 확장이 빠져 컴파일/연결이 끊깁니다.  
문서엔 `dataSource.photoNumber(for:)` 호출이 있는데, 실제 프로토콜에는 해당 메서드가 없습니다.  
참조: `docs/similar/260214numbering.md:89`, `PickPhoto/PickPhoto/Features/SimilarPhoto/UI/PersonPageViewController.swift:28`

3. `높음` 뷰어 비동기 흐름에서 stale 결과로 잘못된 번호가 표시될 수 있습니다.  
`checkAndShowFaceButtons()`에서 비동기 `Task` 후 `showFaceButtons(for: assetID)`를 호출하는데, 그 사이 현재 인덱스가 바뀌어도 검증이 없습니다. 번호 라벨 추가 시 오표시 위험이 커집니다.  
참조: `PickPhoto/PickPhoto/Features/Viewer/ViewerViewController+SimilarPhoto.swift:456`, `PickPhoto/PickPhoto/Features/Viewer/ViewerViewController+SimilarPhoto.swift:466`, `PickPhoto/PickPhoto/Features/Viewer/ViewerViewController+SimilarPhoto.swift:495`, `PickPhoto/PickPhoto/Features/Viewer/ViewerViewController+SimilarPhoto.swift:506`

4. `중간` 뷰어 번호와 그리드 번호의 “동일 기준” 보장이 약합니다.  
그리드가 `viewDidLoad` 시점에 캐시에서 멤버를 다시 읽는 구조라, 전환 사이 캐시/그룹이 바뀌면 뷰어에서 본 번호와 불일치할 수 있습니다. 스냅샷 전달(예: `ComparisonGroup`에 번호 맵 포함)이 더 안전합니다.  
참조: `docs/similar/260214numbering.md:75`, `docs/similar/260214numbering.md:85`, `PickPhoto/PickPhoto/Features/Viewer/ViewerViewController+SimilarPhoto.swift:671`, `PickPhoto/PickPhoto/Features/SimilarPhoto/UI/FaceComparisonViewController.swift:225`

5. `중간` iOS 26 레이아웃 케이스가 빠졌습니다.  
문서는 `safeArea + 68` 고정 오프셋을 제시하지만, iOS 26은 시스템 네비게이션 바/백 버튼 구조가 달라 위치 기준을 분기하거나 앵커 기준(백버튼 하단)으로 잡아야 충돌/과도한 하강을 피할 수 있습니다.  
참조: `docs/similar/260214numbering.md:35`, `docs/similar/260214numbering.md:101`, `PickPhoto/PickPhoto/Features/Viewer/ViewerViewController.swift:901`

6. `중간` 번호 맵 로딩 완료와 초기 페이지 렌더 타이밍 정의가 없습니다.  
현재 화면은 2개 데이터 로드 완료 시 바로 페이지를 띄우는데, 번호 맵을 비동기로 추가하면 초기 셀이 빈/잘못된 번호로 뜰 수 있습니다. 준비 플래그에 번호 맵도 포함하거나 로딩 후 `reloadData`가 필요합니다.  
참조: `docs/similar/260214numbering.md:84`, `PickPhoto/PickPhoto/Features/SimilarPhoto/UI/FaceComparisonViewController.swift:114`, `PickPhoto/PickPhoto/Features/SimilarPhoto/UI/FaceComparisonViewController.swift:428`

7. `낮음` 스펙 문구 간 충돌이 있습니다.  
“+버튼이 보일 때만 표시”와 `clearButtonsOnly()`에서 라벨 “유지”는 엄밀히 충돌합니다(전환 중 예외인지 명시 필요).  
참조: `docs/similar/260214numbering.md:39`, `docs/similar/260214numbering.md:51`

**보완하면 좋은 점**
1. 번호의 기준 순서를 “라이브러리 원본 인덱스” 또는 “초기 그룹 생성 순서”로 명시하고, 병합 로직에서 순서 보존 규칙을 추가.  
2. `FaceComparisonDataSource`에 `photoNumber(for:) -> Int?` 추가 및 `nil` 처리 규칙 명시.  
3. 뷰어 비동기 완료 시 `assetID == coordinator.assetID(at: currentIndex)` 가드 추가.  
4. 뷰어에서 계산한 번호 맵 스냅샷을 FaceComparison으로 전달해 일관성 고정.  
5. iOS 16~25 / iOS 26 라벨 배치 규칙 분리.  
6. 검증 항목에 “그룹 병합 후 번호 안정성”, “빠른 스와이프/취소 시 번호 정확성”, “번호 맵 조회 실패 시 fallback” 추가.
