**Findings**

1. `reloadItems(at:)`를 `isTrashed` 리셋의 유일한 수단으로 두면, 목표한 사이클 안정성이 아직 보장되지 않습니다. 계획은 단일/다중 해제 확정 후 `reloadItems(at:)`로 `isTrashed=true` 잔류를 없애겠다고 적었지만, [zippy-doodling-otter.md](/Users/karl/.claude/plans/zippy-doodling-otter.md#L59) [zippy-doodling-otter.md](/Users/karl/.claude/plans/zippy-doodling-otter.md#L110), 실제 셀 상태는 `setRestoredPreview()`가 바꾸지 않습니다. [PhotoCell.swift](/Users/karl/Project/Photos/iOS/SweepPic/SweepPic/Features/Grid/PhotoCell.swift#L1052) `cancelDimmedAnimation()`은 여전히 `self.isTrashed`를 기준으로 마룬/아이콘 복원을 결정합니다. [PhotoCell.swift](/Users/karl/Project/Photos/iOS/SweepPic/SweepPic/Features/Grid/PhotoCell.swift#L965) 따라서 해제 확정 직후 reload가 반영되기 전에 같은 셀을 다시 제외 스와이프했다가 취소하면, 계획이 피하려는 빨간 딤드 회귀가 다시 가능합니다. 이 경로를 끊으려면 해제 확정 시점에 `reloadItems`와 별개로 `cell.updateTrashState(false)` 같은 즉시 리셋이 필요합니다. [PhotoCell.swift](/Users/karl/Project/Photos/iOS/SweepPic/SweepPic/Features/Grid/PhotoCell.swift#L780)

2. “해제에서 `cancelDimmedAnimation`을 완전히 배제한다”는 전략이 모든 경로를 덮지 못합니다. 계획 본문은 단일 취소, 시스템 cancel, 멀티 취소, 커튼 셀 취소는 다루지만 [zippy-doodling-otter.md](/Users/karl/.claude/plans/zippy-doodling-otter.md#L76) [zippy-doodling-otter.md](/Users/karl/.claude/plans/zippy-doodling-otter.md#L127), 강제 취소 경로인 `cancelActiveSwipeIfNeeded()`는 그대로 `cancelDimmedAnimation()`을 호출합니다. [PreviewGridViewController+SwipeDelete.swift](/Users/karl/Project/Photos/iOS/SweepPic/SweepPic/Features/AutoCleanup/Preview/PreviewGridViewController+SwipeDelete.swift#L246) stage 전환이나 `viewWillAppear`에서 이 함수가 해제 중 셀에 들어오면 전략이 깨집니다. 이 함수도 `targetIsTrashed`/`deleteAction` 기준으로 같은 분기를 가져가야 합니다.

3. `handleMultiSwipeChanged`의 해제 모드 분기는 계획상 아직 불충분합니다. 현재 구현에서 순수 상하 이동 블록은 무조건 `setFullDimmed(false)`를 쓰고, reconciliation도 삭제 모드만 보정합니다. [PreviewGridViewController+MultiSwipe.swift](/Users/karl/Project/Photos/iOS/SweepPic/SweepPic/Features/AutoCleanup/Preview/PreviewGridViewController+MultiSwipe.swift#L213) [PreviewGridViewController+MultiSwipe.swift](/Users/karl/Project/Photos/iOS/SweepPic/SweepPic/Features/AutoCleanup/Preview/PreviewGridViewController+MultiSwipe.swift#L230) 계획은 “block 3 동일”, “reconciliation 동일”이라고만 적어서 [zippy-doodling-otter.md](/Users/karl/.claude/plans/zippy-doodling-otter.md#L98) 실제 구현 시 누락될 여지가 큽니다. 해제 모드에서는 최소한 다음이 명시돼야 합니다: selection filter 반전, removed 셀 green 복구, prev/current/vertical 셀은 `setRestoredPreview()`, reconciliation은 `isDimmedActive == true`일 때 오버레이 제거.

4. `enterMultiSwipeMode` 해제 분기에서 `swipeDirection` 저장을 유지해야 합니다. 이후 앵커 셀 커튼 progress 계산과 ended 판정은 저장된 `swipeDirection`을 사용합니다. [PreviewGridViewController+MultiSwipe.swift](/Users/karl/Project/Photos/iOS/SweepPic/SweepPic/Features/AutoCleanup/Preview/PreviewGridViewController+MultiSwipe.swift#L179) [PreviewGridViewController+MultiSwipe.swift](/Users/karl/Project/Photos/iOS/SweepPic/SweepPic/Features/AutoCleanup/Preview/PreviewGridViewController+MultiSwipe.swift#L280) 계획 스니펫은 애니메이션만 분기하고 있어 [zippy-doodling-otter.md](/Users/karl/.claude/plans/zippy-doodling-otter.md#L160) 그대로 구현하면 좌스와이프 해제에서 방향 판정이 기본값 `.right`로 남을 수 있습니다.

**검증 답변**

1. `confirmDimmedAnimation(toTrashed:false)`는 현재 preview 코드에 아예 호출이 없습니다. [PreviewGridViewController+SwipeDelete.swift](/Users/karl/Project/Photos/iOS/SweepPic/SweepPic/Features/AutoCleanup/Preview/PreviewGridViewController+SwipeDelete.swift#L206) [PreviewGridViewController+MultiSwipe.swift](/Users/karl/Project/Photos/iOS/SweepPic/SweepPic/Features/AutoCleanup/Preview/PreviewGridViewController+MultiSwipe.swift#L357) 그래서 그 부분은 새로 넣지만 않으면 됩니다. 다만 `cancelDimmedAnimation` 배제 전략은 `cancelActiveSwipeIfNeeded()` 때문에 아직 “모든 경로”가 아닙니다.

2. `reloadItems(at:)`가 reload를 수행하면, 이 코드베이스 쪽 준비는 맞습니다. `cellForItemAt`가 항상 `configure(..., isTrashed:false)`를 호출하고 [PreviewGridViewController.swift](/Users/karl/Project/Photos/iOS/SweepPic/SweepPic/Features/AutoCleanup/Preview/PreviewGridViewController.swift#L702), `configure`는 `self.isTrashed = isTrashed`를 실행합니다. [PhotoCell.swift](/Users/karl/Project/Photos/iOS/SweepPic/SweepPic/Features/Grid/PhotoCell.swift#L539) `prepareForReuse`도 `isTrashed=false`로 초기화합니다. [PhotoCell.swift](/Users/karl/Project/Photos/iOS/SweepPic/SweepPic/Features/Grid/PhotoCell.swift#L373) 다만 이건 UIKit reload 타이밍에 의존하므로, 즉시성 보장은 앱 코드만으로는 없습니다.

3. 다중 스와이프 해제 분기는 현재 계획만으로는 “모든 블록이 안전하다”고 보기 어렵습니다. 특히 pure-vertical 블록과 reconciliation 조건을 해제 모드용으로 명시하지 않으면 누락될 수 있습니다.

4. 제외→해제→재제외 사이클은 지금 계획 그대로면 아직 꼬일 수 있습니다. 핵심 원인은 해제 확정 직후에도 셀 객체 내부 `isTrashed`가 잠시 남을 수 있다는 점이고, 그 상태에서 다음 제외 스와이프 취소가 들어오면 `cancelDimmedAnimation()`이 다시 마룬/아이콘 복원을 할 수 있습니다.

앱 실행 없이 코드 정독 기준으로 보면, 이 계획은 방향은 맞지만 아직 충분하지 않습니다. 최소 수정 사항은 해제 확정 시 `isTrashed`를 즉시 false로 내리는 것, `cancelActiveSwipeIfNeeded()`까지 같은 분기를 적용하는 것, `handleMultiSwipeChanged`의 vertical/reconciliation 분기를 해제 모드로 명시하는 것입니다.
