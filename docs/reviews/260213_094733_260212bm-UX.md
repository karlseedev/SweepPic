**주요 발견 (심각도 순)**  
- `Critical` 정책이 문서 간 충돌합니다. `docs/bm/260212bm-UX.md:78`은 Grace Period에 “게이트 있음”인데, 같은 문서에서 `docs/bm/260212bm-UX.md:190`은 방안 A(Grace Period 중 게이트 미적용)를 권장하고 `docs/bm/260212bm-UX.md:446`은 무제한 비우기로 전제합니다. 구현 계획도 아직 `docs/bm/260212bm-plan.md:276`에서 “광고 미표시, 게이트 작동”으로 남아 있어 실행 시 혼선이 큽니다.  
- `Critical` 퍼널/로드맵 시간축이 안 맞습니다. UX 퍼널은 Day 8~30에 트라이얼/유료 전환을 가정(`docs/bm/260212bm-UX.md:415`, `docs/bm/260212bm-UX.md:417`)하지만, Pro는 +1개월 Phase 2 도입(`docs/bm/260211bm1.md:564`), Phase 1에서는 Pro 버튼 비활성(`docs/bm/260212bm-plan.md:248`)입니다. 즉 현재 시뮬레이션 수치는 “출시 초기 실제 운영” 기준으로는 재현이 어렵습니다.  
- `High` 전환 가정 일부가 서로 충돌합니다. UX는 트라이얼→유료 50%(`docs/bm/260212bm-UX.md:417`)인데 BM KPI는 30%(`docs/bm/260211bm1.md:615`)입니다. 활성 구독자 3~5%(`docs/bm/260211bm1.md:616`) 목표와 설치 대비 0.3%(`docs/bm/260212bm-UX.md:419`)도 같은 기간/코호트 기준인지 불명확합니다.  
- `High` 벤치마크 인용 중 과도 낙관/정합성 이슈가 있습니다. ATT opt-in 50~70%(`docs/bm/260211bm1.md:96`)는 최신 공개치 대비 높습니다. Adjust(2025-07-15)는 “프롬프트 노출 기준 평균 35%”, Singular(2025 Q4)는 “첫 오픈 즉시 요청 기준 9.1%”를 제시합니다.  
- `High` Grace Period 방안 A는 “Dead Button 해결”에는 맞지만 단독 최적해는 아닙니다. Day0 마찰 제거 장점은 크지만 Day4 cliff(배신감) 리스크를 키웁니다. 이미 문서도 Day4 급락 위험을 인정(`docs/bm/260212bm-UX.md:539`).  
- `Medium` 이탈 시나리오가 제품 운영 리스크를 충분히 덮지 못합니다. E1~E7는 좋지만 `광고 no-fill/로딩 실패`, `오프라인`, `ATT/알림 거부`, `소량 사용자(항상 10장 이하)`의 저수익 고착, `Phase1 Pro 비활성으로 고의향 사용자 이탈` 시나리오가 빠져 있습니다.  
- `Medium` 개선 우선순위에서 P2(게이미피케이션)보다 선행해야 할 항목이 있습니다. 계측/실험 설계(이벤트 정의), no-fill fallback, Phase1~2 전환기 정책 정합화가 더 상위여야 합니다.

**질문별 결론**  
1. 사용자 여정 시뮬레이션 현실성: 방향은 좋지만 “Phase 분리(출시/한달후)”가 섞여 있어 현재 상태 그대로는 비현실 구간이 있습니다.  
2. 전환 퍼널 수치 타당성: `0.3%` 설치→유료는 freemium 중위 대비 낮은 편이며(RevenueCat 2025 preview의 freemium D35 median 2.18% 대비), 현재 게이트 구조에서는 오히려 보수적으로 타당합니다. 다만 트라이얼→유료 50%는 내부 KPI 30%와 충돌합니다.  
3. Grace Period 방안 A 최선 여부: “Phase 1(Pro 비활성)”에서는 A가 실무적으로 최선에 가깝지만, “장기 운영” 최선은 아님. A + Day3 예고 + Day4 1회 패스 + 소프트 페이월 노출을 묶은 하이브리드가 더 안전합니다.  
4. 이탈 시나리오 충분성: 핵심은 잘 잡았지만 기술/광고 운영 이탈(no-fill, 실패, 권한 거부)과 저사용량 세그먼트 시나리오가 보강 필요합니다.  
5. 개선 우선순위 적절성: P0/P1 재정렬 필요. 특히 “문서 정책 통일 + 계측 + no-fill 대응 + Pro 진입경로”를 P0로 올리는 게 맞습니다.  
6. 놓친 중요 시나리오: `광고 실패`, `권한 거부`, `저사용량 사용자의 영구 무료 고착`, `Phase1에서 결제 의향 사용자 손실`, `Day4 cliff 후 1점 리뷰 확산`이 핵심 누락입니다.

**우선순위 재정렬 제안**  
1. `P0` Grace Period 정책 단일화(UX/BM/Plan 모두 동일 규칙으로 수정).  
2. `P0` 퍼널 기준 통일(D35 install cohort vs active cohort 분리) + KPI 수치 정합화.  
3. `P0` no-fill/오프라인/광고실패 fallback UX 정의.  
4. `P1` Phase1에서도 결제 의향 사용자 수용 경로 마련(대기리스트/알림/사전등록 등).  
5. `P1` Day4 소프트 랜딩(예고 + 1회 패스) 실험.  
6. `P2` 게이미피케이션, 리포트 강화.
