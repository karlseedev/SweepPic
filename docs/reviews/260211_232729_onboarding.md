현재 기획안은 방향은 좋지만, **지금 상태로는 목표(“자연스럽게 배우고 실제 사용으로 이어짐”) 달성에 충분하지 않습니다.**

**핵심 피드백 (심각도 순)**

**사용자 관점 UX에서 특히 위험한 지점**
1. 첫 삭제 경험 전에 “연속 삭제”를 강조하면 실수/불안이 커집니다.
2. “스와이프”가 화면마다 다른 의미(좌우/위/비교화면 전환)를 가져 초반 인지부하가 큽니다.
3. 코치마크가 뜨는 타이밍(2초 고정)이 사용자의 탐색 흐름을 자를 수 있습니다.
4. “자동 보호/AI” 톤은 기대를 과도하게 올려, 실제 결과가 기대보다 낮으면 신뢰 하락으로 이어집니다.

**바로 반영 권장안**
1. `Viewed`와 `Completed`를 분리하세요. 실제 행동 성공 시 `Completed` 처리, dismiss는 재노출 후보 유지.
2. C-2 카피를 “비교 화면에서 좌우 스와이프”처럼 맥락 포함으로 수정하세요.

5. C 계열 미발생 사용자용 백업 진입점(설정 > 사용법, 또는 정리 버튼 롱프레스 힌트)을 추가하세요.
