# 얼굴 매칭 로직 분석 및 문제점

## 1. 현재 로직 요약

### 1.1 전체 흐름
```
그룹 형성 → 얼굴 감지 → 인물 번호 할당 → 유효 슬롯 계산 → 캐시 저장
```

### 1.2 인물 번호 할당 로직 (`assignPersonIndicesForGroup`)

```
1. 기준 슬롯 생성
   - 첫 번째 사진(얼굴이 있는)의 얼굴들을 기준으로 사용
   - 각 얼굴의 위치(center) + Feature Print 저장
   - personIndex는 위치 순서로 부여 (X 오름차순, Y 내림차순)

2. 각 사진의 얼굴 매칭
   a) 위치 기반 후보 필터링
      - posDistance < 0.15 인 슬롯만 후보로 선택
      - 위치가 먼 슬롯은 비교 대상에서 제외됨

   b) Feature Print 비교 (후보 슬롯만)
      - fpDistance < 0.8 이면 매칭
      - 가장 작은 fpDistance를 가진 슬롯 선택

   c) Fallback
      - Feature Print 생성 실패 시 위치 거리만으로 판단
```

---

## 2. 문제점 분석

### 2.1 위치 기반 필터링이 먼저 적용됨 (치명적)

**현상:**
- `posDistance < 0.15` 조건으로 후보 슬롯을 먼저 필터링
- 위치가 먼 슬롯은 Feature Print 비교 대상에서 **완전히 제외**

**문제:**
- 같은 인물이 다른 위치에 있으면 매칭 실패
- 다른 인물이 비슷한 위치에 있으면 잘못 매칭

**예시 (로그 2):**
```
사진 3의 얼굴:
  - 슬롯 1과 위치 거리: > 0.15 → 비교 안 함
  - 슬롯 2와 위치 거리: 0.11 → 비교함 (fpDistance = 0.79)

실제로는 인물 1인데, 위치 때문에 슬롯 2와만 비교되어 인물 2로 할당됨
```

### 2.2 기준 슬롯이 첫 번째 사진에서만 생성됨

**현상:**
- 오직 첫 번째 사진의 얼굴들만 기준 슬롯으로 사용

**문제:**
- 첫 번째 사진에 없는 인물은 기준 슬롯 자체가 없음
- 새로운 인물이 나타나면 어느 슬롯에도 매칭 안 되거나, 잘못된 슬롯에 매칭

**예시 (로그 1):**
```
사진 1 (기준): 인물 A만 있음
사진 5, 6: 인물 B만 있음 (첫 번째 사진에 없음)

→ 인물 B를 위한 기준 슬롯이 없음
→ 위치가 비슷한 인물 A 슬롯과 비교됨
→ fpDistance 0.41, 0.65 < 0.8 이라 인물 A로 잘못 할당
```

### 2.3 Feature Print 임계값 문제

**현상:**
- 현재 임계값: 0.8 (iOS 17+ 기준)
- 같은 인물: fpDistance 0.28 ~ 0.35
- 다른 인물: fpDistance 0.41 ~ 0.79

**문제:**
- 임계값이 너무 높아서 다른 인물도 매칭됨
- 임계값만 낮추면 정상 케이스도 실패할 수 있음

### 2.4 Fallback 로직이 위치 기반

**현상:**
```swift
guard let refFP = candidate.slot.featurePrint else {
    // Feature Print가 없으면 위치만으로 판단
    if Float(candidate.posDistance) < bestScore {
        bestScore = Float(candidate.posDistance)
        bestSlot = candidate.slot.index
    }
    continue
}
```

**문제:**
- Feature Print 생성 실패 시 위치만으로 판단
- 다른 인물이 비슷한 위치에 있으면 잘못 매칭

## 3. 테스트 케이스

### 3.1 케이스 1: 기준 사진에 없는 인물

| 사진 | 인물 | fpDistance | 결과 |
|------|------|------------|------|
| 1 (기준) | A | 0 | 정상 |
| 2 | A | 0.28 | 정상 |
| 3 | A | 0.29 | 정상 |
| 4 | A | 0.32 | 정상 |
| 5 | **B** | 0.65 | **오류** (A로 매칭) |
| 6 | **B** | 0.41 | **오류** (A로 매칭) |

**원인:** 인물 B의 기준 슬롯이 없어서, 위치가 비슷한 인물 A 슬롯과 비교됨

### 3.2 케이스 2: 위치 변화로 인한 잘못된 매칭

| 사진 | personIndex | fpDistance | 실제 인물 |
|------|-------------|------------|----------|
| 1 (기준) | 1 | 0 | A |
| 1 (기준) | 2 | 0 | B |
| 2 | 1 | 0.36 | A |
| 2 | 2 | 0.43 | B |
| 3 | 2 | 0.79 | **A? B?** |
| 4 | 1 | 0.51 | A |

**문제:** 사진 3의 얼굴이 슬롯 1과 비교되지 않아서, 실제 인물 파악 불가

## 4. 근본적 해결 방안

### 4.1 방안 1: Feature Print 우선 매칭 (권장)

```
현재: 위치 필터 → Feature Print 비교 (후보만)
변경: 모든 슬롯 Feature Print 비교 → 가장 가까운 슬롯 선택
```

**로직:**
1. 각 얼굴에 대해 **모든 기준 슬롯**과 Feature Print 거리 계산
2. 가장 작은 fpDistance를 가진 슬롯 선택
3. fpDistance < 임계값 이면 해당 슬롯에 할당
4. 임계값 초과 시 **새로운 슬롯 생성** (동적 슬롯)

**장점:**
- 위치 변화에 강건함
- 더 정확한 인물 매칭

**단점:**
- 계산량 증가 (N개 얼굴 × M개 슬롯)

### 4.2 방안 2: 동적 슬롯 생성

**현재:** 첫 번째 사진에서만 슬롯 생성
**변경:** 매칭 실패 시 새로운 슬롯 동적 생성

**로직:**
1. 기존 슬롯에 매칭 시도
2. 모든 슬롯에서 fpDistance > 임계값 이면
3. 해당 얼굴로 새로운 슬롯 생성 (새 personIndex)

### 4.3 방안 3: 클러스터링 기반 매칭

**현재:** 기준 슬롯 → 매칭
**변경:** 모든 얼굴 간 거리 계산 → 클러스터링 → personIndex 할당

**로직:**
1. 그룹 내 모든 얼굴의 Feature Print 생성
2. N×N 거리 행렬 계산
3. 클러스터링 알고리즘 (DBSCAN, Hierarchical 등)
4. 각 클러스터에 personIndex 할당

**장점:**
- 가장 정확한 인물 분리
- 기준 사진 의존성 제거

**단점:**
- 구현 복잡도 높음
- 계산량 O(N²)

---

## 5. 권장 구현 순서

### Phase 1: Feature Print 우선 매칭 (단기)
1. 위치 필터링 제거
2. 모든 슬롯과 Feature Print 비교
3. 가장 가까운 슬롯에 할당

### Phase 2: 동적 슬롯 생성 (중기)
1. 매칭 실패 시 새 슬롯 생성
2. 첫 번째 사진 의존성 제거

### Phase 3: 임계값 튜닝 (지속)
1. 실제 데이터 기반 최적 임계값 탐색
2. 같은 인물 fpDistance 분포 분석
3. 다른 인물 fpDistance 분포 분석

---

## 6. 디버그 정보 개선 제안

현재 디버그 정보는 **할당된 슬롯과의 거리만** 표시합니다.
더 정확한 분석을 위해 **모든 슬롯과의 거리**를 표시해야 합니다.

```json
{
  "faces": [
    {
      "personIndex": 2,
      "allSlotDistances": [
        { "slotIndex": 1, "fpDistance": 0.35 },
        { "slotIndex": 2, "fpDistance": 0.79 }
      ]
    }
  ]
}
```

이렇게 하면 "왜 이 슬롯에 할당되었는지" 명확하게 파악할 수 있습니다.
