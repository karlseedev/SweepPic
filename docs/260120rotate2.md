# 회전 시 셀 개별 이동 애니메이션 제거 계획 (v2)

## 문제
- **현재 동작:** 회전 시 각 셀이 개별적으로 새 위치로 이동하는 애니메이션 발생
- **목표 동작:** 기본 사진앱처럼 전체 화면이 통째로 회전하는 느낌

## 근본 원인

`setCollectionViewLayout`은 새 레이아웃 객체로 **교체**하면서 셀 이동 애니메이션 발생.
`invalidateLayout`은 기존 레이아웃을 **유지**하면서 재계산만 수행.

```
| 메서드                  | 동작                    | 결과                     |
|------------------------|------------------------|-------------------------|
| setCollectionViewLayout | 새 레이아웃 객체로 교체   | 셀 개별 이동 애니메이션    |
| invalidateLayout        | 기존 레이아웃 유지, 재계산 | 셀 이동 없음, 자연스러운 전환 |
```

## 주의: paddingCellCount 변경 문제

**Critical Issue:** 열 수가 변경되면 `paddingCellCount`도 변경됨 (computed property)

```swift
var paddingCellCount: Int {
    let columns = currentGridColumnCount.rawValue
    // 열 수에 따라 다른 값 반환
}
```

예시:
- 3열: paddingCellCount=2 → indexPath 5 = asset[3]
- 5열: paddingCellCount=4 → indexPath 5 = asset[1]

**`invalidateLayout()`만으로는 `cellForItemAt`이 다시 호출되지 않아 기존 셀 이미지가 그대로 남음 → 중복 문제 발생**

**해결:** completion에서 visible cells 재구성 필수

```swift
// 변경 후
func createLayout() -> UICollectionViewLayout {
    let layout = UICollectionViewCompositionalLayout { [weak self] _, environment in
        // self가 nil이면 기본 3열 사용
        let columns = self?.currentGridColumnCount.rawValue ?? 3
        let columnCount = CGFloat(columns)
        let spacing: CGFloat = 2  // Self.cellSpacing 대신 상수 사용 (클로저 내)
        let totalSpacing = spacing * (columnCount - 1)
        let containerWidth = environment.container.effectiveContentSize.width
        let availableWidth = containerWidth - totalSpacing
        let cellWidth = floor(availableWidth / columnCount)

        // 아이템 크기
        let itemSize = NSCollectionLayoutSize(
            widthDimension: .absolute(cellWidth),
            heightDimension: .absolute(cellWidth)
        )
        let item = NSCollectionLayoutItem(layoutSize: itemSize)

        // 그룹
        let groupSize = NSCollectionLayoutSize(
            widthDimension: .fractionalWidth(1.0),
            heightDimension: .absolute(cellWidth)
        )
        let group = NSCollectionLayoutGroup.horizontal(
            layoutSize: groupSize,
            repeatingSubitem: item,
            count: columns
        )
        group.interItemSpacing = .fixed(spacing)

        // 섹션
        let section = NSCollectionLayoutSection(group: group)
        section.interGroupSpacing = spacing
        section.contentInsetsReference = .none

        return section
    }
    return layout
}
```

### 3단계: 스크롤 앵커를 assetID 기반으로 변경

**파일:** `BaseGridViewController.swift`

현재 indexPath 기반 → assetID 기반으로 변경 (padding 변경에 안전)

```swift
// 변경 전
private var scrollAnchorIndexPath: IndexPath?

// 변경 후
private var scrollAnchorAssetID: String?

/// 화면 중앙 셀의 assetID 저장
private func saveScrollAnchorAssetID() {
    let visibleRect = CGRect(
        origin: collectionView.contentOffset,
        size: collectionView.bounds.size
    )
    let centerPoint = CGPoint(x: visibleRect.midX, y: visibleRect.midY)

    // 1차: 정확히 centerPoint에 있는 셀
    if let indexPath = collectionView.indexPathForItem(at: centerPoint) {
        scrollAnchorAssetID = assetIDForCollectionIndexPath(indexPath)
        return
    }

    // 2차: visible cells 중 중앙에 가장 가까운 셀
    let visibleIndexPaths = collectionView.indexPathsForVisibleItems
    var closestIndexPath: IndexPath?
    var closestDistance: CGFloat = .greatestFiniteMagnitude

    for indexPath in visibleIndexPaths {
        guard let cell = collectionView.cellForItem(at: indexPath) else { continue }
        let distance = hypot(cell.center.x - centerPoint.x, cell.center.y - centerPoint.y)
        if distance < closestDistance {
            closestDistance = distance
            closestIndexPath = indexPath
        }
    }

    scrollAnchorAssetID = closestIndexPath.flatMap { assetIDForCollectionIndexPath($0) }
}

/// assetID 기준으로 스크롤 복원
private func restoreScrollAnchorAssetID() {
    guard let assetID = scrollAnchorAssetID else { return }
    scrollAnchorAssetID = nil

    // 새 padding 기준으로 indexPath 계산
    guard let indexPath = collectionIndexPath(for: assetID) else { return }
    guard indexPath.item < collectionView.numberOfItems(inSection: 0) else { return }

    collectionView.scrollToItem(at: indexPath, at: .centeredVertically, animated: false)
}
```

### 4단계: viewWillTransition 수정

**파일:** `BaseGridViewController.swift` (line 250-291)

```swift
override func viewWillTransition(to size: CGSize, with coordinator: UIViewControllerTransitionCoordinator) {
    super.viewWillTransition(to: size, with: coordinator)

    // 회전 전: assetID 저장 (indexPath 대신)
    saveScrollAnchorAssetID()

    let isLandscape = size.width > size.height
    let newColumnCount = isLandscape
        ? currentGridColumnCount.landscapeColumnCount
        : currentGridColumnCount.portraitColumnCount

    coordinator.animate(alongsideTransition: { [weak self] _ in
        guard let self = self else { return }

        // 열 수 업데이트
        self.currentGridColumnCount = newColumnCount

        // invalidateLayout으로 레이아웃 재계산 (셀 이동 애니메이션 없음)
        self.collectionView.collectionViewLayout.invalidateLayout()

        self.updateCellSize()
        self.updateContentInset()

        // assetID 기준으로 스크롤 복원
        self.restoreScrollAnchorAssetID()
    }, completion: { [weak self] _ in
        // 회전 완료 후 visible cells 재구성 (중복 문제 해결)
        self?.refreshVisibleCellsAfterRotation()
    })
}

/// 회전 완료 후 visible cells 재구성
private func refreshVisibleCellsAfterRotation() {
    let targetSize = thumbnailSize()

    for indexPath in collectionView.indexPathsForVisibleItems {
        // padding 셀 제외
        guard indexPath.item >= paddingCellCount else { continue }

        // 실제 에셋 인덱스 (새 padding 기준)
        let assetIndex = indexPath.item - paddingCellCount
        guard let asset = gridDataSource.asset(at: assetIndex),
              let cell = collectionView.cellForItem(at: indexPath) as? PhotoCell else {
            continue
        }

        // 셀 재구성 (assetID가 다르면 이미지 교체)
        cell.refreshImageIfNeeded(asset: asset, targetSize: targetSize)
    }
}
```

### 5단계: performZoom 수정

**파일:** `BaseGridViewController.swift` (line 559-608)

```swift
func performZoom(to columns: GridColumnCount) {
    lastPinchZoomTime = Date()

    // 1. 앵커 assetID 저장
    let anchorAssetID: String? = {
        if let id = pinchAnchorAssetID { return id }
        let centerPoint = CGPoint(
            x: collectionView.bounds.midX,
            y: collectionView.bounds.midY + collectionView.contentOffset.y
        )
        if let centerIndexPath = collectionView.indexPathForItem(at: centerPoint) {
            return assetIDForCollectionIndexPath(centerIndexPath)
        }
        return nil
    }()

    // 2. 열 수 업데이트
    currentGridColumnCount = columns
    updateCellSize()

    // 3. 레이아웃 애니메이션 (invalidateLayout 사용)
    UIView.animate(withDuration: 0.25) { [weak self] in
        guard let self = self else { return }

        // invalidateLayout + layoutIfNeeded (즉시 전환)
        self.collectionView.collectionViewLayout.invalidateLayout()
        self.collectionView.layoutIfNeeded()

        // 앵커 위치로 스크롤 (새 padding 기준)
        if let assetID = anchorAssetID,
           let indexPath = self.collectionIndexPath(for: assetID) {
            self.collectionView.scrollToItem(at: indexPath, at: .centeredVertically, animated: false)
        }
    } completion: { [weak self] _ in
        // 줌 완료 후 visible cells 재구성
        self?.refreshVisibleCellsAfterZoom()
        self?.didPerformZoom(to: columns)
    }

    print("[BaseGridViewController] Zoom to \(columns.rawValue) columns")
}
```

**참고:** `refreshVisibleCellsAfterZoom()`은 GridViewController에 이미 존재. BaseGridViewController에서도 호출하도록 수정하거나, `refreshVisibleCellsAfterRotation()`을 공용으로 사용.

## 검증 방법

1. **회전 테스트:**
   - 세로 ↔ 가로 회전 시 셀이 개별 이동하지 않고 전체가 자연스럽게 전환
   - 회전 후 동일한 사진이 화면 중앙에 유지
   - 이미지 중복 없음

2. **핀치 줌 테스트:**
   - 3열 → 5열 → 3열 전환 시 즉시 전환 (셀 이동 애니메이션 없음)
   - 줌 후 동일한 사진이 화면 중앙에 유지
   - 이미지 중복 없음

3. **초기 로드:** 앱 시작 시 3열로 정상 표시

4. **스크롤 테스트:** 스크롤 후 회전/줌 시 위치 유지
