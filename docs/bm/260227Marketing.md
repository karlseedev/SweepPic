## 1. ASO & 앱스토어 최적화 (E12) 📝

> 초안 — 출시 전 Apple Search Ads 키워드 플래너로 검색 인기도 확인 후 최종 조정 예정

### 타이틀 (30자 제한)

```
SweepPic - 목록에서 밀어서 사진 정리
```
→ 23자. "SweepPic", "목록", "밀어서", "사진 정리" 핵심 키워드 포함

### 서브타이틀 (30자 제한)

```
비슷한 사진 얼굴보며 정리, 빠른 자동 정리, 사진첩 정리
```
→ 27자. 유사사진+얼굴인식, 자동정리, 사진첩 정리 — 핵심 기능 3개

### 키워드 필드 — 한국 로케일 (100자)

```
사진정리,사진삭제,중복사진,갤러리정리,저장공간,비슷한사진,사진관리,폰정리,클리너,대청소,스와이프,사진앱
```
- 타이틀/서브타이틀에 이미 있는 단어 중복 금지
- 쉼표 뒤 공백 넣지 않기 (1자라도 절약)
- 단수형만 사용 (Apple이 자동 복수형 매칭)

### 키워드 필드 — 영어(UK) 로케일 (100자)

> 한국 로케일 + 영어(UK) 로케일 동시 설정 시 한국 스토어에서 모두 인덱싱 → 200자 확보

```
photo cleaner,gallery cleanup,duplicate photos,storage,organizer,similar photos,swipe delete,declutter
```

### 키워드 선정 근거

| 키워드 | 선정 이유 |
|--------|----------|
| 사진정리/사진관리 | 카테고리 대표 검색어 |
| 목록/밀어서 | SweepPic 차별화 (그리드 스와이프) |
| 유사사진/중복사진/비슷한사진 | 같은 기능의 다른 표현 (검색 커버리지) |
| 용량확보/저장공간 | 사용자 pain point 기반 검색 |
| 갤러리정리/폰정리 | "사진정리"의 유사 표현 |
| 클리너/대청소 | 경쟁앱 사용자가 쓰는 용어 |

### 프로모션 텍스트 (170자 제한)

**위치**: App Store 제품 페이지에서 앱 설명(Description) 바로 위에 표시되는 별도 텍스트 영역. 사용자가 앱 페이지를 열면 가장 먼저 눈에 띄는 텍스트.

**설정**: App Store Connect > 앱 > 앱 정보 > 프로모션 텍스트

**특징**:
- **심사 없이 언제든 변경 가능** — 앱 업데이트/심사 제출 없이 즉시 반영
- 170자 제한 (한글 기준)
- 앱 설명(Description)과 달리 앱 버전 제출 없이 수시로 교체 가능

**활용 시나리오**:
- 런칭 프로모션: "출시 기념 연간 구독 33% 할인 — 기간 한정"
- 신기능 안내: "새로운 유사 사진 자동 정리 기능 추가!"
- 시즌 이벤트: "새해맞이 사진 정리 — 지금 시작하세요"
- 수상/추천: "Apple이 선정한 오늘의 앱"

### ASO 도구

- **Apple Search Ads 키워드 플래너** (무료): 검색 인기도 확인
- **AppFollow** (무료 플랜): 경쟁 앱 키워드 역분석
- 2주 단위 키워드 교체하며 A/B 테스트

## 2. 스크린샷 & 앱 프리뷰 (E12) 📝

> 초안 — 추후 상세 확인 예정

### 스크린샷 (최소 5장, 이상적 8~10장)

검색 결과에는 첫 3장만 노출.

| 순서 | 장면 | 텍스트 오버레이 |
|-----|------|----------------|
| 1 | 그리드에서 밀어서 삭제 동작 | "목록에서 밀어서 바로 정리" |
| 2 | 유사사진 자동 분류 결과 | "비슷한 사진, 자동으로 찾아줘요" |
| 3 | 얼굴 인식 확대 비교 | "얼굴 보며 골라요" |
| 4 | 자동정리 실행 화면 | "한 번에 자동 정리" |
| 5 | 삭제대기함 + 용량 확보 결과 | "실수해도 복구 가능" |

### 앱 프리뷰 영상 (30초)

- 전환율 15~30% 향상 기대
- 직접 제작: 화면 녹화 + iMovie/CapCut
- 첫 5초에 그리드 스와이프 삭제 시연
- 무음 자동재생이므로 텍스트 오버레이 필수

### 텍스트 오버레이 규칙

- 폰트 크기: 최소 40pt
- 한 스크린샷당 10~15자 이내
- 위치: 상단 또는 하단 1/3
- 핵심 단어 1~2개 볼드/컬러 처리
- **OCR 심사 주의**: Apple 심사팀이 스크린샷 텍스트를 OCR로 읽어 검증함. 가격/혜택 표현이 실제와 다르면 리젝 (예: "무료"라고 적었는데 구독 필요한 경우)

## 3. Apple Search Ads (E12) 📝

### 단계별 전략

| 단계 | 시기 | 방식 | 월 예산 |
|------|------|------|--------|
| 1 | 출시 후 1~2개월 | **Basic** (Apple 자동 최적화) | 10~20만원 |
| 2 | 데이터 축적 후 | **Advanced** (키워드 직접 설정) | 30~50만원 |
| 3 | 최적화 | 성과 낮은 키워드 제거, 롱테일 확대 | 데이터 기반 |

### 한국 시장 CPI 참고

| 키워드 유형 | 예상 CPI |
|------------|---------|
| 브랜드 (SweepPic) | 200~500원 |
| 일반 (사진정리) | 800~2,000원 |
| 경쟁 (경쟁앱명) | 1,500~4,000원 |
| 롱테일 | 300~800원 |

### 캠페인 구조 (Advanced 전환 후)

```
캠페인 1: 브랜드 방어
  - 키워드: "sweeppic", "SweepPic"
  - 입찰: 낮게
  - 목적: 경쟁사가 자사 브랜드로 광고하는 것 방어

캠페인 2: 카테고리 일반
  - 키워드: "사진정리", "사진삭제", "갤러리정리"
  - 입찰: 중간
  - 목적: 핵심 사용자 획득

캠페인 3: 디스커버리
  - Search Match ON (자동)
  - 목적: 새로운 키워드 발굴 → 캠페인 2로 이동
```

## 5. 유닛 이코노믹스 — CAC/LTV (E12) 📝

### CAC 계산

```
CAC = 총 마케팅 비용 / 신규 사용자 수
유료 CAC = 총 마케팅 비용 / Plus 전환 사용자 수
```

### 적정 비율

| 비율 | 판단 |
|------|------|
| LTV:CAC = 2:1 | 최소 기준 |
| **LTV:CAC = 3:1** | **목표** |
| LTV:CAC = 5:1+ | 마케팅 투자 확대 가능 |

### Blended CAC 목표: 300~500원

| 채널 | 현실적 CAC |
|------|-----------|
| 오가닉 (ASO) | 0원 |
| SNS 바이럴 | 0~100원 |
| 커뮤니티 | 0~200원 |
| Apple Search Ads | 500~1,500원 |

> 초기에는 무료 채널 철저히 활용, 데이터 쌓인 후 유료 채널 확장

## 7. 고객 지원 — FAQ & CS 매뉴얼 (E22) 📝

> 초안 — 추후 실제 문의 데이터 기반 보완 예정
> 코드 구현(인앱 문의 버튼, FAQ 화면)은 BM 명세 §1(메뉴 구조), §14(고객센터 화면 구현) 참조
> FAQ 구현 방식: 인앱 정적 리스트 (아코디언/펼침), 오프라인 지원 (하드코딩), 앱 업데이트로 갱신

### FAQ (인앱 + 지원 웹페이지 공통)

> 인앱 FAQ와 지원 웹페이지(BusinessInfra E3) FAQ는 동일 내용 사용. 인앱은 하드코딩, 웹은 HTML.

#### 사진/기능

| # | 질문 | 답변 |
|---|------|------|
| F1 | 삭제한 사진을 복구할 수 있나요? | 삭제대기함에 있는 사진은 언제든 복구할 수 있습니다. 삭제대기함을 비운 후에는 최근 삭제된 항목(iOS 기본 사진 앱)에서 30일 이내에 복구 가능합니다. |
| F2 | 사진이 서버로 전송되나요? | 아니요. 모든 사진 처리(유사 사진 분석, 얼굴 감지 포함)는 기기 내에서만 이루어집니다. 사진 데이터는 외부 서버로 전송되지 않습니다. |
| F3 | 지원하는 iOS 버전은? | iOS 16 이상을 지원합니다. |
| F4 | 유사 사진 분석이 정확하지 않아요 | 유사 사진 분석은 AI 기반이라 일부 오분류가 있을 수 있습니다. 삭제 전 반드시 확인하시고, 실수로 삭제해도 삭제대기함에서 복구할 수 있습니다. |
| F5 | 자동 정리는 어떤 기준으로 사진을 선택하나요? | 유사 사진 그룹에서 화질, 초점, 구도 등을 분석하여 가장 좋은 사진을 남기고 나머지를 삭제대기함으로 이동합니다. 바로 삭제되지 않으니 안심하세요. |

#### 구독/결제

| # | 질문 | 답변 |
|---|------|------|
| F6 | 무료로 사용할 수 있나요? | 네. 사진 정리(스와이프 삭제, 유사 사진 분석, 자동 정리, 복구)는 모두 무료입니다. 삭제대기함 비우기에만 일일 한도(10장)가 있으며, 광고를 보면 추가 삭제가 가능합니다. |
| F7 | 구독했는데 프리미엄이 활성화되지 않아요 | 전체 메뉴 > 프리미엄 > "구독 복원"을 탭해주세요. 네트워크 연결 상태를 확인하고, 구독에 사용한 Apple ID로 로그인되어 있는지 확인해주세요. |
| F8 | 구독을 해지하고 싶어요 | 설정 > [내 이름] > 구독 > SweepPic Plus > 구독 취소를 탭하세요. 앱을 삭제해도 구독은 자동으로 해지되지 않으니 반드시 위 경로에서 취소해주세요. |
| F9 | 환불받을 수 있나요? | 환불은 Apple을 통해 처리됩니다. reportaproblem.apple.com에서 신청해주세요. |
| F10 | 삭제 한도가 뭔가요? | 무료 사용자는 하루 10장까지 삭제대기함 비우기가 가능합니다. 광고를 보면 하루 최대 30장까지 늘릴 수 있고, Plus 구독 시 무제한입니다. 한도는 매일 자정에 초기화됩니다. |

#### 개인정보/보안

| # | 질문 | 답변 |
|---|------|------|
| F11 | 얼굴 인식 데이터는 어떻게 처리되나요? | 얼굴 감지는 기기의 Vision 프레임워크를 사용하며, 기기 내에서만 처리됩니다. 얼굴 데이터는 서버에 전송되거나 저장되지 않습니다. |

### Apple 구독 오퍼 4종

| 오퍼 | 대상 | 설명 | SweepPic 활용 |
|------|------|------|-------------|
| Introductory Offer | 신규 구독자 | 첫 구독 시 할인/무료 체험 | **미사용** (Grace Period가 대체) |

| Offer Codes | 누구나 | 코드 입력 시 할인 (배포용) | 향후 검토 |
| Win-back Offer | 해지된 구독자 | ASC 설정, iOS 18+ 자동 노출 | **BM 명세 §10에 반영 완료** |

### 향후 검토 항목

- Promotional Offer: 해지/만료 구독자 대상 특별 가격
- Offer Codes (리딤 코드): 마케팅 배포용 할인 코드
  - 50% 할인 코드: 인플루언서, 커뮤니티 이벤트, 리뷰어 배포용
  - 100% 할인 코드: 베타 테스터, 초기 기여자 보상용
  - 배포 채널: 이메일(딥링크 URL에 코드 포함), SNS, 커뮤니티
  - 코드 구현은 BM 명세 §6 리딤 코드 참조
- 시즌 세일 계획 (블랙프라이데이, 새해 등)
