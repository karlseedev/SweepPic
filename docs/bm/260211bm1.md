## 1. 핵심 컨셉

### "정리는 무료, 비우기는 광고 한 편"

PickPhoto의 2단계 삭제 구조를 비즈니스 모델의 핵심으로 활용한다.

```
[무료 영역 — 가치 체험]                    [유료/광고 게이트]
━━━━━━━━━━━━━━━━━━━━━                    ━━━━━━━━━━━━━━━━━
스와이프 삭제    ─┐
Select 모드 삭제 ─┤→ 앱 내 휴지통 ──────→ 🔒 휴지통 비우기
유사 사진 정리   ─┤   (TrashStore)          (PHAssetChangeRequest)
자동 정리       ─┘                          ↓
                                        기본 사진앱에서도 삭제됨
                                        = 실제 용량 확보
                                        = 진짜 정리 완료
```

**설계 원칙**:
- 사진 정리 "작업"은 100% 무료 — 사용자가 충분히 가치를 체험
- 실제 삭제(휴지통 비우기)에서만 게이트 — 가장 동기가 높은 순간
- 광고는 "벌칙"이 아닌 "선택적 구제 수단" (Duolingo 패턴)

## 3. 3-Tier 수익 모델

### 가격 전략: 정가 높게 설정 + 런칭 할인

가격 인상의 부담을 피하기 위해 **정가를 높게 설정**하고, 초기에는 **런칭 할인**으로 판매한다.
이후 할인폭을 단계적으로 줄여 자연스럽게 실질 가격을 인상한다.
"가격을 올린다"가 아니라 **"할인이 종료된다"**로 프레이밍.

```
┌───────────────────┬──────────────────────────┬──────────────────────────┐
│    무료 (광고)      │   Plus                    │  Pro                      │
│                   │   정가 $2.99/월 | $19.99/년 │  정가 $5.99/월 | $49.99/년  │
│                   │  → 런칭 $1.99/월 | $12.99/년│ → 런칭 $3.99/월 | $29.99/년 │
│                   │                          │  라이프타임 정가 $59.99      │
│                   │                          │  → 런칭 $39.99             │
├───────────────────┼──────────────────────────┼──────────────────────────┤
│ 정리 기능 전부 무료  │ 정리 기능 전부              │ 모든 기능 무제한            │
│ 휴지통 비우기 제한   │ 휴지통 비우기 무제한          │ 휴지통 비우기 무제한          │
│ 광고 있음          │ 광고 제거                   │ 광고 제거                  │
│                   │                          │ + 프리미엄 기능             │
└───────────────────┴──────────────────────────┴──────────────────────────┘
```

- 정가 기준: Slidebox와 동일 수준($49.99/년)으로 카테고리 **중간 포지셔닝**
- 런칭 할인이 끝나도 카테고리 평균($42.4/년) 범위 안에 있어 가격 저항 최소화

### 3.3 가격 운영 전략 (할인 단계적 축소)

| 단계 | 시기 | 할인율 | Pro 연간 실판매가 | 프레이밍 |
|---|---|---|---|---|
| **런칭** | 출시 ~ +3개월 | 40% | **$29.99** | "런칭 기념 특별 할인" |
| **Stage 1** | +3 ~ +6개월 | 30% | **$34.99** | "얼리버드 할인" |
| **Stage 2** | +6 ~ +12개월 | 20% | **$39.99** | "기간 한정 할인" |
| **정가** | +12개월 이후 | 0% | **$49.99** | 정가 (시즌 할인으로 간헐적 30% off) |

**운영 원칙**:
- 기존 구독자는 **갱신 시점에도 가입 당시 가격 유지** (신규만 단계적 인상)
- 각 단계 전환 시 기존 할인가 안내: "현재 가격으로 가입할 수 있는 마지막 기회"
- 정가 도달 후에도 **시즌 할인** (연말, 새해, 봄맞이 정리 등)으로 간헐적 30% off 진행
- 앱 내 페이월에 ~~$49.99~~ **$29.99** 형태의 취소선 정가 표시 → 앵커링 효과

**앵커링 효과**:
```
┌──────────────────────────────────┐
│  Pro 연간                         │
│                                  │
│  ~~$49.99/년~~ → $29.99/년        │
│  런칭 기념 40% 할인               │
│                                  │
│  하루 약 ₩110 (커피 한 잔의 1/5)   │
│                                  │
│  [7일 무료 체험 시작]              │
└──────────────────────────────────┘
```
- 정가 $49.99가 보이면 $29.99가 훨씬 합리적으로 느껴짐
- Mojo 앱은 이 프레이밍만으로 ARPU 60% 성장 달성

## 4. 휴지통 게이트 설계

### 4.1 무료 사용자 일일 한도

| 행동 | 일일 무료 한도 | 한도 초과 시 |
|---|---|---|
| 개별 영구 삭제 | 5장/일 | 리워드 광고 1회 = 5장 추가 |
| 전체 비우기 | 10장 이하일 때만 무료 | 11장 이상이면 리워드 광고 필요 |
| 리워드 광고 시청 | 일 3회까지 | 3회 초과 → 구독 유도 |
| 복구 | 무제한 | - |

### 4.2 리워드 광고 보상 구조

```
[리워드 광고 1회 시청] (30초)
   ↓
선택지:
├── (A) 개별 삭제 5장 추가    ← 소량 정리 사용자용
├── (B) 전체 비우기 1회 해방   ← 대량 정리 사용자용
└── 일 3회 제한               ← 3회 초과 시 자연스러운 구독 전환점
```

### 4.3 휴지통 비우기 UI 흐름

**일반 케이스 (10장 이하)**:
```
[비우기] 버튼 탭
   ↓
iOS 시스템 팝업 "Delete N Photos?"
   ↓
삭제 완료 + 축하 화면
```

**게이트 케이스 (11장 이상)**:
```
[비우기] 버튼 탭
   ↓
┌────────────────────────────────────────┐
│  휴지통에 18장이 있습니다                 │
│                                        │
│  무료 비우기: 5장 (오늘 남은 횟수)         │
│                                        │
│  [5장 비우기]  [광고 보고 전체 비우기]     │
│               [Plus로 업그레이드]         │
└────────────────────────────────────────┘
   ↓
[광고 보고 전체 비우기] 선택 → 리워드 광고 → 전체 삭제
```

### 4.4 자동 정리 + 휴지통 연계

현재 앱의 "휴지통을 먼저 비워주세요" 흐름을 BM과 연결:

```
[자동 정리 버튼 탭]
   ↓
휴지통 체크: 이미 30장 있음
   ↓
┌──────────────────────────────────────────┐
│  휴지통에 이미 30장이 있어요                 │
│  새로운 정리를 시작하려면 먼저 비워주세요       │
│                                          │
│  [광고 보고 비우기]  [Plus로 비우기]  [취소] │
└──────────────────────────────────────────┘
```

자동 정리 파워 유저 → 휴지통 비우기 빈도 증가 → 리워드 한도 초과 → 구독 전환

### 4.5 자동 정리 결과 후 게이트

```
"38장의 저품질 사진을 발견했습니다"
   ↓
┌──────────────────────────────────────────┐
│  📊 정리 결과                              │
│                                          │
│  발견: 38장 (약 152MB)                     │
│  휴지통으로 이동 완료                        │
│                                          │
│  💡 실제로 용량을 확보하려면                  │
│     휴지통을 비워주세요                      │
│                                          │
│  [휴지통으로 이동]                           │
└──────────────────────────────────────────┘
   ↓
[휴지통 탭] → 동일한 비우기 게이트 적용
```

### 4.6 게이트 설계의 심리적 근거

| 설계 | 심리적 효과 |
|---|---|
| 정리 작업은 무료 → 휴지통에 사진 쌓임 | **매몰 비용**: "이미 시간 들여 정리했는데..." |
| 용량 확보 예상치 표시 (약 200MB) | **구체적 가치**: 눈에 보이는 혜택 |
| 리워드 일 3회 한도 | **적절한 불편함**: "매일 광고 보느니 차라리..." |
| 50장 자동 정리 결과 → 비우기 게이트 | **즉각적 전환 동기**: 한 번에 비우고 싶음 |
| "내일 다시 올게요" 옵션 항상 제공 | **강제 없음**: 존중 → 신뢰 → 나중에 전환 |

```
[권한 화면]                   ✕ 없음                 첫인상 보호
```

### 5.2 사용자 여정별 광고 타이밍

#### 시나리오 1: 빠른 사진 정리 (스와이프 삭제)

```
앱 실행 ──(A)──→ [보관함 그리드] → [뷰어] → 스와이프 삭제 10~20장
                                            (광고 전혀 없음)
   ↓
[휴지통 탭] → 🔒 비우기 게이트 ──(R)──→ 리워드 광고 → 삭제 완료 ──(I)──→ 축하+전면
```
- (A) 앱 오픈 광고 (1일 2회)
- (R) 리워드 비디오 (핵심 게이트)
- (I) 전면 광고 (세션당 1회, 완료 후)

#### 시나리오 2: 유사 사진 정리

```
[보관함] → 테두리 표시 → [뷰어] → + 버튼 → [얼굴 비교] → Delete
                        (광고 전혀 없음)
   ↓
반복 → [휴지통 탭] → 🔒 비우기 게이트 ──(R)──→ 리워드 → 완료 ──(I)──→ 축하+전면
```

#### 시나리오 3: 자동 정리

```
[보관함] → "정리" 버튼 → 탐색 → 38장 발견 → 휴지통 이동 ──(N)──→ 결과+네이티브
                        (광고 없음)
   ↓
[휴지통 탭] → 🔒 비우기 게이트 ──(R)──→ 리워드 → 완료 ──(I)──→ 축하+전면
```
- (N) 네이티브 배너 (결과 화면 하단)

### 5.3 광고 빈도 제한

| 광고 유형 | 최대 빈도 | 최소 간격 |
|---|---|---|
| 앱 오픈 광고 | 1일 2회 | - |
| 리워드 비디오 | 1일 3회 | - |
| 전면 (인터스티셜) | 세션당 1회 | 광고 간 5분+ |
| 네이티브 배너 | 결과 화면에만 | - |

### 5.4 광고 없는 기간 (Grace Period)

- **최초 설치 후 3일**: 모든 광고 비표시 (첫인상 보호)
- **트라이얼 기간 (7일)**: 구독 체험 중 광고 없음

## 6. 구독 전환 유도 설계

### 6.1 골든 모먼트: 리워드 3회 소진

```
[사용자] 오늘 리워드 광고 3회 모두 시청 완료
[휴지통 탭] 아직 25장 남아있음
   ↓
┌──────────────────────────────────────────┐
│  오늘의 무료 비우기 한도를 모두 사용했어요    │
│                                          │
│  휴지통: 25장 (약 98MB)                    │
│                                          │
│  내일 다시 비울 수 있어요                    │
│  또는...                                  │
│                                          │
│  [Plus 시작하기 — 광고 없이 무제한 비우기]   │
│  7일 무료 체험 → ~~$2.99~~ $1.99/월       │
│  런칭 할인 33% OFF                        │
│                                          │
│  [내일 다시 올게요]                         │
└──────────────────────────────────────────┘
```

**전환 동기가 최대인 이유**:
- 정리 작업을 이미 다 했음 (시간 투자 = 매몰 비용)
- 비우지 않으면 그 노력이 무의미
- 내일까지 기다리기 vs $1.99/월

### 6.2 골든 모먼트: 자동 정리 50장 후

```
자동 정리로 50장이 휴지통에 들어감
   ↓
┌──────────────────────────────────────────┐
│  📊 자동 정리 완료!                         │
│                                          │
│  50장의 저품질 사진을 발견했어요              │
│  약 200MB를 확보할 수 있어요                 │
│                                          │
│  [광고 보고 비우기 (1/3)]                   │
│  [Plus로 바로 비우기 — ~~$2.99~~ $1.99/월]  │
│  7일 무료 체험 + 런칭 할인                   │
│  [나중에]                                  │
└──────────────────────────────────────────┘
```

50장을 리워드로만 비우려면:
- 리워드 1회 = 전체 비우기 1회이므로 1회면 가능
- 하지만 일 3회 한도는 다른 비우기에도 공유됨
- 매일 정리하는 파워 유저 → 한도 초과 빈번 → 구독 전환

### 6.3 비우기 완료 후 축하 화면

```
┌──────────────────────────────────────────┐
│  ✨ 정리 완료!                              │
│                                          │
│  삭제: 18장                               │
│  확보 용량: 약 72MB                        │
│                                          │
│  총 누적 정리: 342장 (1.2GB)               │
│                                          │
│  [확인]                                   │
└──────────────────────────────────────────┘
   ↓
[확인] 탭 후 → 전면 광고 (세션당 1회)
   ↓
빈 휴지통 화면
```

**여기가 전면 광고 적합 시점인 이유**:
- 사용자가 성취감을 느끼는 순간 = 광고 거부감 최저
- 작업이 완전히 끝난 후 = 흐름 방해 없음

### 6.4 페이월 설계

**타입**: 소프트 페이월 (핵심 기능 체험 후 전환)
**트라이얼**: 7일 무료 체험
**위치**: 휴지통 비우기 게이트 시점 + 설정 화면

**구성 순서**:
1. 가치 헤드라인 ("광고 없이 무제한 정리")
2. 기능 비교 (무료 vs Plus vs Pro)
3. 가격 표시
4. CTA + 시각적 트라이얼 타임라인

**개인화 메시지 예시**:
- "지난주에 47장을 정리하셨어요. Plus로 더 빠르게 비울 수 있어요"
- "자동 정리로 X MB 확보 가능 — Pro로 무제한 실행"

## 7. 예상 수익 시뮬레이션

### 7.1 전제 조건

- 한국 시장 중심
- 리워드 eCPM: $29 (한국 iOS)
- 전면 eCPM: $10 (한국 iOS)
- 앱 오픈 eCPM: $8
- 무료 사용자 비율: 80%
- 구독 전환율: 5% (Plus + Pro 합산)
- Plus:Pro 비율: 7:3
- 가중평균 구독 단가: **런칭기 $2.60/월** → **정가기 $4.20/월**
- 런칭 할인 효과로 초기 전환율이 정가 대비 높을 것으로 예상

### 7.2 DAU별 예상 수익

| DAU | 리워드 | 전면 | 앱 오픈 | 구독 | **일 합계** | **월 합계** |
|---|---|---|---|---|---|---|
| 1,000 | $5.8 | $1.0 | $0.6 | $2.6 | **$10** | **$300** |
| 5,000 | $29 | $5.0 | $3.2 | $13 | **$50** | **$1,510** |
| 10,000 | $58 | $10 | $6.4 | $26 | **$100** | **$3,020** |
| 50,000 | $290 | $50 | $32 | $130 | **$502** | **$15,060** |

> 산출 근거:
> - 리워드: 무료 DAU의 25%가 평균 1회 시청
> - 전면: 무료 DAU의 50%에 1회 노출
> - 앱 오픈: 무료 DAU의 40%에 1회 노출
> - 구독: DAU × 5% × 가중평균 단가 / 30일

### 7.3 수익 비중 예상

| 수익원 | 비중 (런칭기) | 비중 (정가기) |
|---|---|---|
| 리워드 광고 | 58% | 25% |
| 전면 광고 | 10% | 6% |
| 앱 오픈 광고 | 6% | 4% |
| 구독 | 26% | **65%** |

- 런칭기: 낮은 할인가로 많은 전환 유도 → 광고 수익 비중이 높음
- 정가기: 구독 단가 상승($2.60→$4.20/월) + 구독자 누적 → 구독이 매출의 2/3
- 할인 축소 과정에서 구독 단가가 올라가므로, 전환율이 다소 하락해도 **ARPU는 증가**

## 9. 금지 사항

### 절대 하지 말 것

| 금지 사항 | 이유 |
|---|---|
| 스와이프 삭제 자체를 제한 | 핵심 가치 체험 차단 → 앱 이탈 |
| 그리드/뷰어에 배너 광고 | 사진 앱의 본질(감상) 훼손 |
| 얼굴 비교 기능 자체를 잠금 | 차별점 체험 불가 → 전환 동기 없음 |
| 유사 사진 테두리를 유료로 | 무료 가치 체험이 전환의 전제 |
| 휴지통 복구를 유료로 | 안전망 제거 → 삭제 공포 → 앱 사용 감소 |
| 최초 설치 3일 내 광고 | 첫인상 파괴 |
| 비우기 완전 차단 (광고도 불가) | 사용자 분노 → 1점 리뷰 |
| 기존 무료 기능을 나중에 유료 전환 | CapCut 실패 사례 — 사용자 신뢰 치명적 손상 |
| 편집/탐색 중 전면 광고 | 작업 흐름 파괴 → 이탈률 46% 증가 |
| 유료 구독자에게 광고 표시 | 약탈적 Cleaner 앱 패턴 → 최악의 리뷰 |
| 주간 구독 옵션 | 약탈적으로 인식됨 (연 환산 시 과도한 가격) |

### 핵심 원칙

1. **"감상은 성역, 비우기가 기회"** — 사진 보기/탐색 중 광고 절대 금지, 휴지통 비우기에서만
2. **"벌칙이 아닌 선택"** — 리워드 광고 중심, 사용자가 능동적으로 시청
3. **"적절한 불편함"** — 리워드 한도 소진이 구독 전환의 자연스러운 동기
4. **"항상 탈출구 제공"** — "내일 다시 올게요" 옵션으로 강제 없는 설계
5. **"가격은 내리지, 올리지 않는다"** — 정가를 높게 잡고 할인으로 운영, 기존 구독자 가격 보장
