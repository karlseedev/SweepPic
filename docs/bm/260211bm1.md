## 1. 핵심 컨셉

### "정리는 무료, 비우기는 광고 한 편"

PIClear의 2단계 삭제 구조를 비즈니스 모델의 핵심으로 활용한다.

```
[무료 영역 — 가치 체험]                    [유료/광고 게이트]
━━━━━━━━━━━━━━━━━━━━━                    ━━━━━━━━━━━━━━━━━
스와이프 삭제    ─┐
Select 모드 삭제 ─┤→ 앱 내 휴지통 ──────→ 🔒 휴지통 비우기
유사 사진 정리   ─┤   (TrashStore)          (PHAssetChangeRequest)
자동 정리       ─┘                          ↓
                                        기본 사진앱에서도 삭제됨
                                        = 실제 용량 확보
                                        = 진짜 정리 완료
```

**설계 원칙**:
- 사진 정리 "작업"은 100% 무료 — 사용자가 충분히 가치를 체험
- 실제 삭제(휴지통 비우기)에서만 게이트 — 가장 동기가 높은 순간
- 광고는 "벌칙"이 아닌 "선택적 구제 수단" (Duolingo 패턴)

## 3. 2-Tier 수익 모델

### 가격 전략: 정가 높게 설정 + 런칭 할인

가격 인상의 부담을 피하기 위해 **정가를 높게 설정**하고, 초기에는 **런칭 할인**으로 판매한다.
이후 할인폭을 단계적으로 줄여 자연스럽게 실질 가격을 인상한다.
"가격을 올린다"가 아니라 **"할인이 종료된다"**로 프레이밍.

```
┌───────────────────┬──────────────────────────────────┐
│    무료 (광고)      │   Plus                            │
│                   │   정가 $4.99/월 | $39.99/년         │
│                   │  → 런칭 $2.99/월 | $24.99/년        │
│                   │   라이프타임 정가 $59.99             │
│                   │  → 런칭 $39.99                     │
├───────────────────┼──────────────────────────────────┤
│ 정리 기능 전부 무료  │ 모든 기능 무제한                     │
│ 자동 정리 월 2회    │ 자동 정리 무제한                     │
│ 휴지통 비우기 제한   │ 휴지통 비우기 무제한                  │
│ 광고 있음          │ 광고 제거                           │
└───────────────────┴──────────────────────────────────┘
```

### 3.1 기능별 비교

| 기능 | 무료 (광고) | Plus |
|---|---|---|
| 그리드 브라우징 | O | O |
| 뷰어 (상세 보기) | O | O |
| 스와이프 삭제 → 휴지통 | 무제한 | 무제한 |
| Select 모드 삭제 → 휴지통 | 무제한 | 무제한 |
| 유사 사진 감지 (테두리) | 무제한 | 무제한 |
| 얼굴 비교 화면 | 무제한 | 무제한 |
| 유사 사진 삭제 → 휴지통 | 무제한 | 무제한 |
| 복구 (휴지통 → 보관함) | 무제한 | 무제한 |
| 앨범 탐색 | O | O |
| **자동 정리 (Precision) 실행** | **월 2회** | **무제한** |
| **휴지통 개별 영구삭제** | **일 5장** | **무제한** |
| **휴지통 전체 비우기** | **10장 이하만** | **무제한** |
| **광고** | 있음 | **없음** |

### 3.2 가격표 (정가 / 런칭 할인가)

| 플랜 | 정가 | 런칭가 (할인율) | 포지셔닝 |
|---|---|---|---|
| Plus 월간 | $4.99/월 | **$2.99/월** (40% off) | 체험 진입 |
| Plus 연간 | $39.99/년 | **$24.99/년** (37% off) | 메인 전환 타겟 |
| Plus 라이프타임 | $59.99 | **$39.99** (33% off) | 구독 피로 사용자 확보 |

**포지셔닝 비교**:
- 런칭가 기준: Gemini($11.99/년)와 Slidebox($49.99/년) 사이의 **중간 진입**
- 정가 기준: 카테고리 평균($42.4/년) 바로 아래인 $39.99/년 → 저항 최소화
- 라이프타임 $39.99(런칭) = 연간 정가와 동일 → 런칭기에 특히 매력적

### 3.3 가격 운영 전략 (할인 단계적 축소)

| 단계 | 시기 | 연간 판매가 | 월간 판매가 | 프레이밍 |
|---|---|---|---|---|
| **런칭** | 출시 ~ +3개월 | **$24.99** (37% off) | **$2.99** (40% off) | "런칭 기념 특별 할인" |
| **Stage 1** | +3 ~ +6개월 | **$29.99** (25% off) | **$3.99** (20% off) | "얼리버드 할인" |
| **Stage 2** | +6 ~ +12개월 | **$34.99** (12% off) | **$3.99** (20% off) | "기간 한정 할인" |
| **정가** | +12개월 이후 | **$39.99** (정가) | **$4.99** (정가) | 시즌 할인으로 간헐적 30% off |

**운영 원칙**:
- 기존 구독자는 **갱신 시점에도 가입 당시 가격 유지** (신규만 단계적 인상)
- 각 단계 전환 시 기존 할인가 안내: "현재 가격으로 가입할 수 있는 마지막 기회"
- 정가 도달 후에도 **시즌 할인** (연말, 새해, 봄맞이 정리 등)으로 간헐적 30% off 진행
- 앱 내 페이월에 ~~$39.99~~ **$24.99** 형태의 취소선 정가 표시 → 앵커링 효과
- **Apple 정책 준수**: 취소선 정가는 실제 IAP에 등록된 가격과 일치해야 함. 정가($39.99/년)는 +12개월 후 실제 판매가로 전환되므로 허위 가격에 해당하지 않음. 단, 할인 표기가 상시화되지 않도록 시즌별 운영으로 전환

**앵커링 효과**:
```
┌──────────────────────────────────┐
│  Plus 연간                        │
│                                  │
│  ~~$39.99/년~~ → $24.99/년        │
│  런칭 기념 37% 할인               │
│                                  │
│  하루 약 ₩95 (커피 한 잔의 1/5)    │
│                                  │
│  [7일 무료 체험 시작]              │
└──────────────────────────────────┘
```
- 정가 $39.99가 보이면 $24.99가 훨씬 합리적으로 느껴짐
- Mojo 앱은 이 프레이밍만으로 ARPU 60% 성장 달성

## 4. 휴지통 게이트 설계

### 4.1 무료 사용자 일일 한도

| 행동 | 일일 무료 한도 | 한도 초과 시 |
|---|---|---|
| 개별 영구 삭제 | 5장/일 | 리워드 광고 1회 = 5장 추가 |
| 전체 비우기 | 10장 이하일 때만 무료 | 11장 이상이면 리워드 광고 필요 |
| 리워드 광고 시청 | 일 3회까지 | 3회 초과 → 구독 유도 |
| 자동 정리 실행 | 월 2회 | 추가 실행 시 리워드 광고 또는 구독 |
| 복구 | 무제한 | - |

### 4.2 리워드 광고 보상 구조

```
[리워드 광고 1회 시청] (30초)
   ↓
선택지:
├── (A) 개별 삭제 5장 추가         ← 소량 정리 사용자용
├── (B) 전체 비우기 (50장 이하)     ← 일반 사용자용
├── (C) 50장 비우기 (50장 초과 시)  ← 대량 정리 → 추가 리워드 필요
└── 일 3회 제한                    ← 3회 초과 시 자연스러운 구독 전환점
```

> **대량 비우기 차등**: 휴지통 50장 초과 시 리워드 1회 = 50장만 비움.
> 예: 120장이면 리워드 3회 필요 → 일일 한도(3회) 즉시 소진 → 구독 전환 압력 극대화

### 4.3 휴지통 비우기 UI 흐름

**일반 케이스 (10장 이하)**:
```
[비우기] 버튼 탭
   ↓
iOS 시스템 팝업 "Delete N Photos?"
   ↓
삭제 완료 + 축하 화면
```

**게이트 케이스 (11~50장)**:
```
[비우기] 버튼 탭
   ↓
┌────────────────────────────────────────┐
│  휴지통에 18장이 있습니다                 │
│                                        │
│  오늘 개별 삭제 가능: 5장 (일일 한도)      │
│                                        │
│  [5장만 삭제]  [광고 보고 전체 비우기]     │
│               [Plus로 업그레이드]         │
└────────────────────────────────────────┘
   ↓
[광고 보고 전체 비우기] 선택 → 리워드 광고 → 18장 전체 삭제
```

**게이트 케이스 (51장 이상 — 대량)**:
```
[비우기] 버튼 탭
   ↓
┌────────────────────────────────────────┐
│  휴지통에 120장이 있습니다                │
│                                        │
│  오늘 개별 삭제 가능: 5장 (일일 한도)      │
│  광고 1회 시청 시: 50장 삭제              │
│                                        │
│  [5장만 삭제]  [광고 보고 50장 비우기]     │
│               [Plus로 전체 비우기]        │
└────────────────────────────────────────┘
   ↓
120장 전체를 광고로 비우려면 리워드 3회 필요
→ 일일 한도 즉시 소진 → 나머지는 내일 또는 Plus
```

### 4.4 자동 정리 + 휴지통 연계

현재 앱의 "휴지통을 먼저 비워주세요" 흐름을 BM과 연결:

```
[자동 정리 버튼 탭]
   ↓
휴지통 체크: 이미 30장 있음
   ↓
┌──────────────────────────────────────────┐
│  휴지통에 이미 30장이 있어요                 │
│  새로운 정리를 시작하려면 먼저 비워주세요       │
│                                          │
│  [광고 보고 비우기]  [Plus로 비우기]  [취소] │
└──────────────────────────────────────────┘
```

자동 정리 파워 유저 → 휴지통 비우기 빈도 증가 → 리워드 한도 초과 → 구독 전환

```
   ↓
┌──────────────────────────────────────────┐
│  이번 달 무료 정리 횟수를 모두 사용했어요     │
│  (2/2회 사용)                              │
│                                          │
│  [광고 보고 1회 추가 정리]                  │
│  [Plus로 무제한 정리 — ~~$39.99~~ $24.99/년]│
│  7일 무료 체험                              │
│  [다음 달에]                               │
└──────────────────────────────────────────┘
```

### 4.6 자동 정리 결과 후 게이트

```
"38장의 저품질 사진을 발견했습니다"
   ↓
┌──────────────────────────────────────────┐
│  📊 정리 결과                              │
│                                          │
│  발견: 38장 (약 152MB)                     │
│  휴지통으로 이동 완료                        │
│                                          │
│  💡 실제로 용량을 확보하려면                  │
│     휴지통을 비워주세요                      │
│                                          │
│  [휴지통으로 이동]                           │
└──────────────────────────────────────────┘
   ↓
[휴지통 탭] → 동일한 비우기 게이트 적용
```

### 4.7 게이트 설계의 심리적 근거

| 설계 | 심리적 효과 |
|---|---|
| 정리 작업은 무료 → 휴지통에 사진 쌓임 | **매몰 비용**: "이미 시간 들여 정리했는데..." |
| 용량 확보 예상치 표시 (약 200MB) | **구체적 가치**: 눈에 보이는 혜택 |
| 리워드 일 3회 한도 | **적절한 불편함**: "매일 광고 보느니 차라리..." |
| 50장 자동 정리 결과 → 비우기 게이트 | **즉각적 전환 동기**: 한 번에 비우고 싶음 |
| "내일 다시 올게요" / "다음 달에" 옵션 | **강제 없음**: 존중 → 신뢰 → 나중에 전환 |

### 5.1 앱 화면별 광고 맵

### 5.2 사용자 여정별 광고 타이밍

#### 시나리오 1: 빠른 사진 정리 (스와이프 삭제)

```
앱 실행 → [보관함 그리드] → [뷰어] → 스와이프 삭제 10~20장
                                            (광고 전혀 없음)
   ↓
[휴지통 탭] → 🔒 비우기 게이트 ──(R)──→ 리워드 광고 → 삭제 완료 ──(I)──→ 축하+전면
```
- (R) 리워드 비디오 (핵심 게이트)
- (I) 전면 광고 (세션당 1회, 완료 후)

#### 시나리오 2: 유사 사진 정리

```
[보관함] → 테두리 표시 → [뷰어] → + 버튼 → [얼굴 비교] → Delete
                        (광고 전혀 없음)
   ↓
반복 → [휴지통 탭] → 🔒 비우기 게이트 ──(R)──→ 리워드 → 완료 ──(I)──→ 축하+전면
```

#### 시나리오 3: 자동 정리

```
[보관함] → "정리" 버튼 → 탐색 → 38장 발견 → 휴지통 이동 ──(N)──→ 결과+네이티브
                        (광고 없음)
   ↓
[휴지통 탭] → 🔒 비우기 게이트 ──(R)──→ 리워드 → 완료 ──(I)──→ 축하+전면
```
- (N) 네이티브 배너 (결과 화면 하단)

### 5.3 광고 빈도 제한

| 광고 유형 | 최대 빈도 | 최소 간격 |
|---|---|---|
| 리워드 비디오 | 1일 3회 | - |
| 전면 (인터스티셜) | 세션당 1회 | 광고 간 5분+ |
| 네이티브 배너 | 결과 화면에만 | - |

### 5.4 광고 없는 기간 (Grace Period)

- **최초 설치 후 3일**: 모든 광고 비표시 (첫인상 보호)
- **트라이얼 기간 (7일)**: 구독 체험 중 광고 없음

## 6. 구독 전환 유도 설계

### 6.1 골든 모먼트: 리워드 3회 소진

```
[사용자] 오늘 리워드 광고 3회 모두 시청 완료
[휴지통 탭] 아직 25장 남아있음
   ↓
┌──────────────────────────────────────────┐
│  오늘의 무료 비우기 한도를 모두 사용했어요    │
│                                          │
│  휴지통: 25장 (약 98MB)                    │
│                                          │
│  내일 다시 비울 수 있어요                    │
│  또는...                                  │
│                                          │
│  [Plus 시작하기 — 광고 없이 무제한]          │
│  7일 무료 체험 → ~~$4.99~~ $2.99/월        │
│  런칭 할인 40% OFF                         │
│                                          │
│  [내일 다시 올게요]                         │
└──────────────────────────────────────────┘
```

**전환 동기가 최대인 이유**:
- 정리 작업을 이미 다 했음 (시간 투자 = 매몰 비용)
- 비우지 않으면 그 노력이 무의미
- 내일까지 기다리기 vs $2.99/월

### 6.2 골든 모먼트: 자동 정리 50장 후

```
자동 정리로 50장이 휴지통에 들어감
   ↓
┌──────────────────────────────────────────┐
│  📊 자동 정리 완료!                         │
│                                          │
│  50장의 저품질 사진을 발견했어요              │
│  약 200MB를 확보할 수 있어요                 │
│                                          │
│  [광고 보고 비우기 (1/3)]                   │
│  [Plus로 바로 비우기 — ~~$39.99~~ $24.99/년]│
│  7일 무료 체험 + 런칭 할인                   │
│  [나중에]                                  │
└──────────────────────────────────────────┘
```

50장을 리워드로만 비우려면:
- 리워드 1회 = 전체 비우기 1회이므로 1회면 가능
- 하지만 일 3회 한도는 다른 비우기에도 공유됨
- 매일 정리하는 파워 유저 → 한도 초과 빈번 → 구독 전환

### 6.3 비우기 완료 후 축하 화면

```
┌──────────────────────────────────────────┐
│  ✨ 정리 완료!                              │
│                                          │
│  삭제: 18장                               │
│  확보 용량: 약 72MB                        │
│                                          │
│  총 누적 정리: 342장 (1.2GB)               │
│                                          │
│  [확인]                                   │
└──────────────────────────────────────────┘
   ↓
[확인] 탭 후 → 전면 광고 (세션당 1회)
   ↓
빈 휴지통 화면
```

**여기가 전면 광고 적합 시점인 이유**:
- 사용자가 성취감을 느끼는 순간 = 광고 거부감 최저
- 작업이 완전히 끝난 후 = 흐름 방해 없음

### 6.4 페이월 설계

**타입**: 소프트 페이월 (핵심 기능 체험 후 전환)
**트라이얼**: 7일 무료 체험
**위치**: 휴지통 비우기 게이트 시점 + 자동 정리 횟수 초과 시점 + 설정 화면

**구성 순서**:
1. 가치 헤드라인 ("광고 없이 무제한 정리")
2. 기능 비교 (무료 vs Plus)
3. 가격 표시 (정가 취소선 + 할인가)
4. CTA + 시각적 트라이얼 타임라인

**개인화 메시지 예시**:
- "지난주에 47장을 정리하셨어요. Plus로 더 빠르게 비울 수 있어요"
- "자동 정리로 X MB 확보 가능 — Plus로 무제한 실행"

## 7. 예상 수익 시뮬레이션

### 7.1 전제 조건

- 한국 시장 중심
- 무료 사용자 비율: 80% (DAU 기준)
- 활성 구독자 비율: 3% (DAU 기준, 전환율 5%에서 이탈/해지 반영)
- 가중평균 구독 단가: **런칭기 $2.30/월** → **정가기 $3.80/월**
  - 런칭기: 월간 $2.99 × 30% + 연간 $24.99/12=$2.08 × 70% = $2.30
  - 정가기: 월간 $4.99 × 30% + 연간 $39.99/12=$3.33 × 70% = $3.80

**광고 eCPM 및 effective reach rate**:

| 광고 유형 | eCPM | 도달율 (무료 DAU 대비) | 근거 |
|---|---|---|---|
| 리워드 비디오 | $29 | 25% | 휴지통 비우기/자동정리 사용자 중 광고 선택 비율 |
| 전면 (인터스티셜) | $10 | 12.5% | 세션당 1회, 비우기 완료자만 → 전체의 ~1/8 |

> ※ 도달율은 광고 빈도 제한, 사용 패턴, fill rate를 종합한 effective rate

### 7.2 DAU별 예상 수익 (런칭기 기준)

| DAU | 리워드 | 전면 | 구독 | **일 합계** | **월 합계** |
|---|---|---|---|---|---|
| 1,000 | $5.8 | $1.0 | $2.3 | **$9.1** | **$273** |
| 5,000 | $29 | $5.0 | $11.5 | **$45.5** | **$1,365** |
| 10,000 | $58 | $10 | $23 | **$91** | **$2,730** |
| 50,000 | $290 | $50 | $115 | **$455** | **$13,650** |

> 산출 근거 (DAU 1,000 예시):
> - 리워드: 800(무료) × 25% × $29/1000 = **$5.8**
> - 전면: 800 × 12.5% × $10/1000 = **$1.0**
> - 구독: 1,000 × 3%(활성 구독자) × $2.30/월 / 30일 = **$2.3**

### 7.3 수익 비중 예상

| 수익원 | 비중 (런칭기) | 비중 (정가기) |
|---|---|---|
| 리워드 광고 | 64% | 29% |
| 전면 광고 | 11% | 6% |
| 구독 | 25% | **65%** |

- 런칭기: 낮은 할인가로 많은 전환 유도 → 리워드 광고 수익 비중이 높음
- 정가기: 구독 단가 상승($2.30→$3.80/월) + 구독자 누적 → 구독이 매출의 약 2/3
- 할인 축소 과정에서 구독 단가가 올라가므로, 전환율이 다소 하락해도 **ARPU는 증가**

| 결제 | RevenueCat | 구독 관리, 분석, A/B 테스트 통합 |

### ATT 프리프롬프트 설계

```
┌──────────────────────────────────┐
│                                  │
│   PIClear를 무료로 유지하기 위해   │
│   맞춤형 광고를 제공하고 있어요      │
│                                  │
│   - 개인 사진은 절대 수집하지 않아요  │
│   - 광고 경험만 개선됩니다          │
│                                  │
│   [계속]          [나중에]         │
└──────────────────────────────────┘
         ↓ [계속] 선택 시
   시스템 ATT 다이얼로그 표시
```

> **ATT 주의사항**: 시스템 ATT 다이얼로그는 **앱 설치 후 1회만 표시** 가능 (재설치 전까지 재표시 불가).
> 따라서 프리프롬프트의 목적은 "1회 기회의 opt-in 비율을 극대화"하는 것이며, 표시 타이밍이 중요.
> **권장 타이밍**: 설치 후 첫 휴지통 비우기 시도 시 (사용자가 앱 가치를 체험한 직후)

## 10. 금지 사항

### 절대 하지 말 것

| 금지 사항 | 이유 |
|---|---|
| 스와이프 삭제 자체를 제한 | 핵심 가치 체험 차단 → 앱 이탈 |

### 핵심 원칙

1. **"감상은 성역, 비우기가 기회"** — 사진 보기/탐색 중 광고 절대 금지, 휴지통 비우기에서만
2. **"벌칙이 아닌 선택"** — 리워드 광고 중심, 사용자가 능동적으로 시청
3. **"적절한 불편함"** — 리워드 한도 소진이 구독 전환의 자연스러운 동기
4. **"항상 탈출구 제공"** — "내일 다시 올게요" 옵션으로 강제 없는 설계
5. **"가격은 내리지, 올리지 않는다"** — 정가를 높게 잡고 할인으로 운영, 기존 구독자 가격 보장
