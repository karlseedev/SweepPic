## 1. 사용자 페르소나

### P1. 캐주얼 정리러 (전체 사용자의 ~50%)

```
프로필: 20~30대, 사진 3,000~10,000장
행동: 주 1~2회 앱 실행, 스와이프 10~30장 삭제
동기: "사진이 너무 많아서 좀 정리해야지"
지불 의향: 낮음 (무료로 충분하면 그냥 사용)
예상 경로: 무료 → 가끔 리워드 광고 → 유지 or 이탈
```

### P2. 적극 정리러 (전체 사용자의 ~30%)

```
프로필: 20~40대, 사진 10,000~50,000장
행동: 주 3~5회 앱 실행, 자동 정리 + 스와이프 병행
동기: "저장 공간 부족, 체계적으로 정리하고 싶다"
지불 의향: 중간 (가치가 명확하면 지불)
예상 경로: 무료 → 리워드 한도 반복 도달 → 구독 전환 (핵심 타겟)
```

### P3. 파워 유저 (전체 사용자의 ~15%)

```
프로필: 사진 50,000장+, 용량 부족 심각
행동: 매일 앱 실행, 자동 정리 + 대량 삭제
동기: "용량 확보가 시급, 효율적으로 정리해야 함"
지불 의향: 높음 (시간 vs 돈 trade-off 인지)
예상 경로: 무료 → Day 1~3에 리워드 3회 즉시 소진 → 빠른 구독 전환
```

### P4. 일회성 방문자 (전체 사용자의 ~5%)

```
프로필: 다양, 앱스토어에서 검색으로 유입
행동: 설치 → 1~2번 사용 → 삭제
동기: "한번 써볼까?"
지불 의향: 없음
예상 경로: D1 이탈 → 푸시 알림으로 재방문 유도
```

## 3. Day 0: 첫 경험 시뮬레이션

### 3.1 시간순 여정 (P2 적극 정리러 기준)

```
[0:00] 앱 실행
  │
  ├─ 사진 라이브러리 권한 요청
  │   └─ "사진을 정리하려면 접근이 필요해요" (1스텝)
  │
  ├─ ⚠️ 광고 없음 (Grace Period)
  │
[0:30] 보관함 그리드 표시
  │   └─ 18,000장의 사진 로딩
  │   └─ 유사 사진 그룹에 테두리 표시 시작
  │
[1:00] 🌟 Aha Moment #1: "유사 사진이 자동으로 표시되네?"
  │   └─ 테두리가 있는 사진 탭 → 뷰어 → + 버튼 → 얼굴 비교 화면
  │   └─ 비교 후 삭제 → 휴지통으로 이동 (무제한, 무료)
  │
[3:00] 스와이프 삭제 체험
  │   └─ 뷰어에서 위로 스와이프 → 삭제 (원터치)
  │   └─ 10~15장 연속 삭제 → 속도감에 만족
  │
[5:00] 🌟 Aha Moment #2: "이거 진짜 빠르다, 기본 사진앱보다 낫네"
  │
[8:00] 자동 정리 발견
  │   └─ "정리" 버튼 탭 → 최신순 정리 시작
  │   └─ 38장 저품질 사진 발견 → 휴지통 이동
  │
[10:00] 🌟 Aha Moment #3: "38장이나 자동으로 찾아주네!"
  │   └─ 결과 화면: "38장 발견 (약 152MB)"
  │   └─ ⚠️ 네이티브 배너 미표시 (Grace Period)
  │
[11:00] 휴지통 비우기 (Grace Period — 게이트 미적용, 방안 A)
  │
  ├─ 휴지통 탭 이동 → 53장 누적 (스와이프 15 + 자동 38)
  │
  ├─ [비우기] 탭 → ✅ Grace Period 중이므로 게이트 없이 바로 iOS 시스템 팝업
  │   └─ "Delete 53 Photos?" → [Delete] → 53장 전체 영구삭제
  │
  ├─ 🌟 Aha Moment #4: "한 번에 다 비워지네! 600MB 확보!"
  │   └─ 만족도: ★★★★★ (마찰 없음 = 최상의 첫인상)
  │   └─ Loss Aversion 씨앗: Day 4 이후 "예전처럼 쓰려면" 동기 형성
  │
  └─ 💡 이 경험이 핵심: Day 0에 마찰 없이 비울 수 있어야
      Day 4 게이트 만남 시 "다시 자유롭게 쓰고 싶다"는 전환 동기가 됨
```

### 3.2 Grace Period 확정 정책: 방안 A (게이트 미적용)

**확정**: Grace Period(설치 후 3일) 동안 **광고 없음 + 게이트 없음** = 비우기 무제한

**배경**: 시뮬레이션에서 Grace Period + 게이트 조합 시 Dead Button 문제를 발견하여 3가지 방안을 검토한 결과, 방안 A를 확정함.

| 방안 | 설명 | 채택 여부 |
|-----|------|---------|
| **A. Grace Period 중 게이트 미적용** | 3일간 비우기 무제한 | ✅ **확정** |
| B. Grace Period 중 리워드만 허용 | 게이트 적용 + 광고 허용 | ❌ "광고 없음" 약속 위반 |
| C. Grace Period 중 한도 상향 | 일일 20장 무료 | ❌ 구현 복잡도 과다 |

**방안 A 확정 근거:**
- 업계 베스트 프랙티스: "첫 가치 체험에 마찰 없어야 함" (Appcues: 온보딩 마찰 1개 추가 시 완료율 10% 하락)
- CleanMyPhone 사례: 3일 트라이얼 중 모든 기능 무제한 → 트라이얼 종료 후 전환
- Duolingo: 첫 레슨은 하트 소모 없음 → 가치 체험 보장
- **Loss Aversion 극대화**: Day 0에 53장을 무료로 비운 경험 → Day 4 게이트 만남 시 "예전처럼 쓰려면" 동기

**Day 4 소프트 랜딩 (하이브리드 보완):**

방안 A의 단점("비우기가 원래 무료"로 인식)을 보완하기 위해 3가지 장치를 함께 적용:

1. **Grace Period 종료 예고** (Day 3 마지막 비우기 후 알림 — 4.2절 참조)
2. **Day 4 첫 게이트에서 1회 무료 비우기 쿠폰** (8.3절 참조)
3. **"5장만 삭제" 무료 옵션** 항상 제공 (BM 4.1 기존 설계)

```
Day 0~3 (GP)     Day 4 (전환일)          Day 5+ (정상 운영)
─────────────── ─────────────────────── ──────────────────
비우기 무제한    → 1회 무료 쿠폰 +         → 5장 무료 / 리워드 /
광고 없음          GP 종료 예고 済           Plus 구독
                   "소프트 랜딩"
```

> **3문서 통일**: 이 정책은 BM 전략(260211bm1.md 5.4절), 구현 계획(260212bm-plan.md 검증 #8)에도 동일하게 반영해야 함.

## 4. Day 1~3: Grace Period 여정

### 4.1 일별 예상 행동 (P2 기준)

```
Day 1:
  앱 실행 → 스와이프 20장 삭제 → 비우기 무료 → 성공 체험
  └─ 만족도: ★★★★★ (마찰 없음)

Day 2:
  앱 실행 → 자동 정리 실행 (1/2회) → 42장 발견 → 비우기 무료 → 성공
  └─ 만족도: ★★★★★
  └─ 누적 정리: ~100장

Day 3 (Grace Period 마지막 날):
  앱 실행 → 스와이프 + 자동 정리 → 비우기 무료 → 성공
  └─ 만족도: ★★★★★
  └─ 누적 정리: ~150장, 약 600MB 확보
  └─ 💡 "이 앱 없으면 안 되겠다" — 습관 형성 시작
```

### 4.2 Grace Period 종료 인지

Grace Period 종료를 사용자에게 어떻게 알릴 것인가?

**현재 기획: 별도 알림 없음** (다음 게이트 만남 시 자연스럽게 인지)

**대안: 소프트 전환 알림 고려**

```
Day 3 마지막 비우기 완료 후:
┌────────────────────────────────────────┐
│  🎉 3일간 150장을 정리했어요!            │
│                                        │
│  내일부터 비우기에 일일 한도가 적용돼요     │
│  광고를 보거나 Plus를 시작하면             │
│  지금처럼 자유롭게 정리할 수 있어요         │
│                                        │
│  [확인]                                 │
└────────────────────────────────────────┘
```

**장점**: 갑작스러운 게이트 → 배신감 방지
**단점**: "무료로 잘 쓰고 있었는데 갑자기 돈 내라고?" 느낌 가능

**권장**: 위 알림 **표시** (예고 없는 제한은 이탈의 주요 원인 — RevenueCat 2025: "이탈의 대부분은 페이월 이전에 발생")

### 5.2 ATT 프롬프트 타이밍

**현재 기획**: "설치 후 첫 휴지통 비우기 시도 시"

**시뮬레이션 결과 수정 제안**: Grace Period 중에는 게이트가 없으므로(방안 A 채택 시), ATT 프롬프트 타이밍 재검토 필요.

**권장 타이밍**: **Day 4, 첫 리워드 광고 시청 직전**

```
[광고 보고 비우기] 버튼 탭
    ↓
ATT 프리프롬프트 표시 (1회만)
┌────────────────────────────────────────┐
│  PIClear를 무료로 유지하기 위해         │
│  맞춤형 광고를 제공하고 있어요             │
│                                        │
│  • 개인 사진은 절대 수집하지 않아요         │
│  • 광고 경험만 개선됩니다                 │
│                                        │
│  [계속]           [나중에]               │
└────────────────────────────────────────┘
    ↓ [계속]
시스템 ATT 다이얼로그
    ↓
리워드 광고 재생
```

**근거**: 사용자가 이미 "광고를 보겠다"는 의사결정을 한 순간 → ATT opt-in 비율 극대화 (앱 가치 인지 + 광고 수용 의향이 모두 높은 시점)

### 5.3 10장 이하 무료 비우기 시나리오 (P1 캐주얼)

> BM 4.1 핵심 규칙: "전체 비우기 10장 이하일 때만 무료" — P1 캐주얼이 가장 흔하게 겪는 시나리오

```
[Day 5, P1 캐주얼 사용자]
  │
[앱 실행]
  │
[보관함] → 스와이프 삭제 7장 → 휴지통 이동
  │
[휴지통 탭] → 7장 → [비우기] 탭
  │
  ├─ ✅ 10장 이하 → 게이트 없이 바로 iOS 시스템 팝업
  │   └─ "Delete 7 Photos?" → [Delete] → 7장 삭제 완료
  │
  └─ 💭 "광고 안 봐도 되네, 편하다"
      └─ P1은 이 패턴으로 매일 5~10장씩 무료 정리 가능
      └─ 수익: 게이트 미도달로 광고 수익 거의 없음
```

**이 시나리오가 중요한 이유:**
- P1 캐주얼(50%)은 대부분 일 10장 이하 정리 → **게이트를 거의 만나지 않음**
- 게이트를 만나지 않으면 리워드도, 페이월도 노출되지 않음
- P1의 직접 수익은 매우 낮음 → 그러나 DAU 기여 + 앱스토어 평점 기여로 간접 가치 높음

```
P1 캐주얼 수익 구조:
  리워드: 0회 (게이트 미도달)
  구독: 0% (전환 동기 없음)
  → ARPU 극히 낮음, 그러나 DAU 기여 + 앱스토어 평점 기여
```

> **설계 의도 확인**: P1이 무료로 편하게 쓰는 것은 **의도된 동작**. P1은 구독 전환 타겟이 아니라 **긍정 리뷰 + 바이럴**의 기반 사용자층.

```
  ↓

  ↓

  ↓

  ↓

  ↓

→ Day 7 리텐션 대비 전환: 1.0%
→ 게이트 만남 대비 전환: 1.1%
```

**업계 비교**: 프리미엄 모델 중위값 2.2% (RevenueCat 2025) → 현재 설계는 **상당히 낮음**

> **전환율 근거**: 트라이얼→유료 38%는 업계 7일 트라이얼 중위값 (RevenueCat 2025).

**원인 분석**:
- 게이트 → 페이월 도달까지 너무 많은 단계 (게이트 → 리워드 시청 → 리워드 한도 → 페이월)

- 설치→전환 0.2%는 업계 중위 2.2% 대비 1/10 수준 → **페이월 노출 경로 다양화 필수**

→ **10장 "개선 제안"에서 상세 분석**

## 9. BM 타당성 종합 검증

### 9.1 현재 설계의 강점

| 항목 | 평가 | 근거 |
|-----|------|------|
| **핵심 가치 무료** | ✅ 우수 | 스와이프 삭제, 유사 사진, 자동 정리 모두 무료 → 가치 체험 보장 |
| **게이트 위치** | ✅ 우수 | "비우기"는 정리 작업 완료 후 → 매몰 비용 극대화 (Duolingo 하트 패턴과 동일 구조) |
| **리워드 중심** | ✅ 우수 | 강제 광고 아닌 선택적 리워드 → 사용자 65%가 리워드 선호 (Emarketer) |
| **탈출구 제공** | ✅ 우수 | "내일 다시 올게요", "5장만 삭제" → 강제 없는 설계 |
| **가격 전략** | ✅ 우수 | 정가 높게 + 런칭 할인 → CapCut 실패 사례 회피 |

### 9.2 현재 설계의 위험 & 보완 필요

| 항목 | 위험도 | 문제 | 제안 |
|-----|-------|------|------|
| **Grace Period + 게이트 충돌** | 🔴 높음 | 3.2절 참조 — Dead Button 문제 | 방안 A 채택 (GP 중 게이트 미적용) |
| **페이월 노출 지연** | 🟡 중간 | 리워드 3회 소진 후에야 페이월 → 전환 퍼널 길다 | 골든 모먼트 다양화 (아래 10절) |

## 10. 개선 제안

### 10.1 Grace Period 하이브리드 설계 (P0) — 확정

**문제**: Grace Period(광고 없음) + 게이트(작동) = Dead Button (3.2절에서 발견)
**확정**: 방안 A + 소프트 랜딩 3단 장치

| 단계 | 시점 | 내용 | 구현 |
|-----|------|------|------|
| 1 | Day 0~3 | 게이트 미적용, 비우기 무제한 | `GracePeriodService.isInGracePeriod → onAllowed()` |
| 2 | Day 3 마지막 비우기 후 | GP 종료 예고 알림 | 4.2절 알림 UI |
| 3 | Day 4 첫 게이트 | 1회 무료 비우기 쿠폰 | `UsageLimit.gracePeriodBonusUsed` |

```swift
// TrashGateCoordinator.evaluateGate()
if GracePeriodService.shared.isInGracePeriod {
    onAllowed()  // 단계 1: GP 중 게이트 없음
    return
}

if !usageLimitStore.current.gracePeriodBonusUsed {
    usageLimitStore.useGracePeriodBonus()
    onAllowed()  // 단계 3: Day 4 첫 게이트에서 1회 무료
    return
}

// 정상 게이트 로직
presentGateSheet(...)
```

> **이 설계는 3.2절, 4.2절, 8.3절에 분산되어 있던 내용을 통합한 것입니다.**
> 구현 계획(260212bm-plan.md)의 GracePeriodService, UsageLimitStore에 반영 필요.

### 10.2 페이월 노출 시점 다양화 (P1)

**문제**: 리워드 3회 소진 후에만 페이월 → 전환 퍼널이 너무 길다
**제안**: 골든 모먼트를 3곳으로 확장

| 골든 모먼트 | 현재 | 제안 |
|-----------|------|------|
| 리워드 3회 소진 | ✅ 있음 | 유지 |
| 자동 정리 결과 직후 | ❌ 없음 | **추가** — "Plus로 바로 비우기" 소프트 제안 |
| 주간 정리 리포트 | ❌ 없음 | **추가** — 푸시 알림 + 인앱 리포트에 Plus 배너 |

자동 정리 결과 직후 소프트 제안:
```
┌────────────────────────────────────────┐
│  📊 정리 결과                           │
│  발견: 45장 (약 180MB)                  │
│  휴지통으로 이동 완료                     │
│                                        │
│  [휴지통으로 이동]                        │
│                                        │
│  ───────────────────────────            │
│  💎 Plus 회원이라면 바로 비울 수 있어요     │
│     7일 무료 체험 →                      │
│  ───────────────────────────            │
└────────────────────────────────────────┘
```

→ 강제 아닌 **소프트 배너** (전체 화면 페이월이 아님) → 가치 인지 직후 제안
