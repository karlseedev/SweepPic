## 1. 제품 철학 & 수익화 원칙 ✅

### 구조

```
[사진 정리]        →  [휴지통]       →  [비우기]
항상 무료, 무제한      무료 보관         여기만 게이트
```

- **무료 영역**: 스와이프 삭제, 유사 사진 분석, 자동 정리, 복구 — 전부 무제한
- **게이트**: "비우기"(영구 삭제) 한 곳에만 존재

### 광고 배치 방향

- 정리 행위 **도중**에는 광고 없음 (비교 화면, 뷰어, 그리드)
- 정리 행위 **완료 시점**에 전면 광고 (짝수 회차만, 스킵 가능)
- **대기 화면**에 배너 (분석 중 하단)

| 시점 | 타입 | 빈도 |
|------|------|------|
| 휴지통 비우기 (한도 초과) | 리워드 | 일 2회 |
| 유사사진 삭제 완료 | 전면 | 짝수 회차 |
| 자동정리 휴지통 이동 완료 | 전면 | 짝수 회차 |
| 사진 분석 대기 화면 | 배너 | 매회 |

### 운영 방향

- 사진 정리 행위는 게이트로 막지 않는다 (비우기만 게이트)
- 그리드, 뷰어, 비교 화면 등 탐색 중에는 광고를 넣지 않는다
- 휴지통 복구는 유료화하지 않는다
- Plus 구독자에게는 광고를 보여주지 않는다
- 기존 무료 기능을 나중에 유료로 전환하지 않는다

## 2. 사용자 세그먼트 & 라이프사이클 ✅

### 세그먼트 (가정 — 출시 후 데이터로 검증)

| 유형 | 사진 수 | 일일 정리량 | 게이트 빈도 |
|------|:---:|:---:|:---:|
| 라이트 | ~5,000장 | 1~10장 | 거의 안 걸림 |
| 미디엄 | 5,000~20,000장 | 10~30장 | 가끔 |
| 헤비 | 20,000장+ | 30장+ | 자주~매일 |

### 라이프사이클

```
설치 → Grace Period(3일) → 무료 활성 → 한도 도달 → 전환 or 이탈
                                         ↓
                                    Plus 구독자 → 갱신 or 해지
```

| 단계 | 기간 | 경험 |
|------|------|------|
| 신규 | Day 0~3 | 무제한, 광고 없음 (Grace Period) |
| 무료 활성 | Day 4~ | 일일 10장 무료, 카운터 보임, 전면 광고 노출 |
| 한도 도달 | 수시 | 게이트 시트 → 리워드/Plus 선택지 |
| Plus 전환 | - | 무제한, 광고 없음 |
| 이탈 위험 | 사용 감소 시 | 윈백 대상 |

## 3. 유료 모델 핵심 가치 제안 ✅

> **"제한/광고 없이 바로 정리"**

| 가치 | 무료 | Plus |
|------|------|------|
| 일일 삭제 | 최대 30장 (무료 10 + 리워드 2회) | 무제한 |
| 광고 | 전면(짝수 회차) + 배너(분석 중) + 게이트 시트 | 없음 |
| 대청소 (200장) | 7일 걸림 | 바로 |

- **용량 해방**: 일 30장 캡 해제 → 헤비 유저/대청소 시 결정적
- **귀찮음 제거**: 게이트, 전면 광고, 배너, 카운터 전부 없음

## 4. Free Tier 설계 ✅

### 일일 삭제 한도 (총량 캡 방식)

```
일일 총 삭제 한도 = 무료 N장 + 리워드 M회 × K장
```

| 항목 | 시작값 | 조정 범위 | 비고 |
|------|:---:|:---:|------|
| 무료 | 10장 | 5~15 | 방법 불문 |
| 리워드 1회당 | +10장 | 5~20 | "전체 비우기"가 아닌 추가 장수 |
| 리워드 횟수 | 2회/일 | 1~3 | |
| **일일 최대** | **30장** | **15~45** | 초과 시 Plus만 가능 |

- 매일 자정 리셋
- 모든 숫자는 출시 후 A/B 테스트로 조정
- 자동 정리 실행 횟수: **무제한** (게이트는 비우기에만)
- 한도 초과 시 관용 없음 — 12장이어도 무료 비우기 불가, 게이트 표시
- 리워드 소진 시 → 골든 모먼트 (Plus 전환 유도)

### 게이트 판단

```
비우기 탭
  ↓
휴지통 ≤ 남은 무료 → 게이트 없이 바로 iOS 팝업 → 삭제
휴지통 > 남은 무료 → 게이트 시트 표시
```

### Grace Period

- 설치 후 3일: 광고 없음, 게이트 없음, 카운터 없음 → 무제한
- Day 4부터: 정상 운영 (게이지 + 게이트 + 광고)

### Grace Period 전환 UX

휴지통 화면 상단, 게이지와 동일한 위치·높이로 사전 안내 표시:

**Day 0~2:**
```
┌────────────────────────────────────────┐
│  무료 체험 중 — N일 남음                 │
│  체험 종료 후 일일 무료 삭제 한도가        │
│  적용됩니다                              │
└────────────────────────────────────────┘
```

**Day 3 (마지막 날):**
```
┌────────────────────────────────────────┐
│  무료 체험 마지막 날                      │
│  내일부터 일일 무료 삭제 한도가            │
│  적용됩니다                              │
└────────────────────────────────────────┘
```

**Day 4~ (정상 운영):**
```
██████████ 10 / 10장 남음   ← 같은 위치가 게이지로 전환
```
- 게이지 첫 표시 시 1회 툴팁: `"오늘의 무료 삭제 한도예요. 탭해서 자세히 볼 수 있어요"`

## 5. 광고 모델 ✅

| 광고 | 트리거 | 빈도 |
|-----|--------|------|
| **리워드** | 게이트에서 사용자 선택 | 일 2회 |
| **전면** | 유사사진 삭제 완료 / 자동정리 휴지통 이동 완료 | 짝수 회차 |
| **배너** | 사진 분석 대기 화면 | 매회 |

### 리워드 광고 보상
- 1회 시청 = 일일 삭제 한도 +10장 추가 (전체 비우기가 아님)
- 무료 10 + 리워드 2회 = 일일 최대 30장
- 리워드 횟수는 삭제 성공 후에만 차감 (iOS 팝업 취소 시 소모 안 됨)
- 2회 소진 시 → 골든 모먼트 (Plus 전환 유도)

### 광고 배치 기준
- 정리 행위 **도중**: 광고 없음 (그리드, 뷰어, 비교 화면)
- 정리 행위 **완료 시점**: 전면 광고 (짝수 회차, 스킵 가능)
- **대기 화면**: 배너 (분석 중 하단)
- Grace Period (3일): 모든 광고 없음
- Plus 구독자: 모든 광고 없음

### 광고 로딩 & no-fill 처리
- **사전 로드**: 앱 시작 시 + 만료 전 자동 갱신 + 시청 완료 후 즉시 다음 광고 로드
- **재시도**: 지수 백오프 (2→4→8초), 즉시 재로드 금지
- **no-fill 시**: 평생 1회 무료 제공 (+10장) → 이후 스피너 10초 대기 → 실패 시 비활성
- **미디에이션**: 출시 시 AdMob 단일, 이후 fill rate 확인 후 네트워크 추가

## 6. 구독 상품 & 가격 전략 ✅

### Plus 혜택
- 광고 없음, 게이트 없음, 무제한 삭제 및 정리

### 가격 전략 (USD 기준)

| 플랜 | 정가 | 런칭 할인가 | 비고 |
|------|------|--------|------|
| **월간** | $3.99 | **$2.99** | 가벼운 진입 |
| **연간** | $29.99 | **$19.99** | 메인 전환 타겟 (월간 정가 대비 37%↓) |

- **할인율 정책**: 연간 정가($29.99)는 월간 정가($47.88) 대비 약 37% 할인된 가격으로 설정하여 장기 구독 유도.
- **라이프타임(Lifetime) 플랜**: 출시 초기에는 운영하지 않음. 서비스 안정화 이후 특정 마일스톤이나 명절 등 특별 이벤트 형식으로 한정 판매 검토.

### 무료 체험 정책
- **별도 무료 체험 없음**: Grace Period(3일)가 사실상 강력한 무료 체험 역할을 수행함.
- 사용자는 결제 수단 등록 없이도 3일간 앱의 모든 가치를 경험할 수 있음.
- 3일 종료 시점에 자연스럽게 Plus 전환 넛지 노출.

### 할인 축소 계획
- 런칭 후 초기 일정 기간(또는 목표 사용자 달성 시)까지 할인가 유지.
- 데이터 분석을 통해 전환율이 안정화되면 단계적으로 정가로 복구.

## 8. 게이트 & 카운터 UI ✅

### 게이트 판단 로직

| 조건 | 결과 |
|---|---|
| 휴지통 ≤ 남은 한도 | 게이트 없이 바로 삭제 |
| 휴지통 ≤ 남은 한도 + (남은 광고 × 10) | 광고 N회 보고 전체 비우기 |
| 휴지통 > 남은 한도 + (남은 광고 × 10) | Plus만 가능 |

- **남은 한도** = 오늘 일일 한도 - 이미 삭제한 수
- **남은 광고** = 2 - 오늘 시청한 광고 수
- 비우기는 항상 **전체 비우기** (부분 삭제 없음)

### 게이트 시트 — 광고로 가능한 경우

```
┌────────────────────────────────────────┐
│  휴지통: X장                            │
│  오늘 남은 삭제: N장                     │
│                                        │
│  광고 시청 시 삭제 한도 +10장             │
│                                        │
│  [광고 N회 보고 X장 삭제]                │
│  [무제한 삭제 (구독하기)]                 │
│  [닫기]                                │
└────────────────────────────────────────┘
```

**광고 버튼 3단계 상태:**

| 상태 | 버튼 표시 |
|---|---|
| Ready | "광고 N회 보고 X장 삭제" (활성) |
| Loading | "광고 준비 중..." (비활성, 스피너) |
| Failed | 탭 시 토스트 "잠시 후 다시 시도해주세요" |

### 게이트 시트 — Plus만 가능한 경우

```
┌────────────────────────────────────────┐
│  휴지통: X장                            │
│  오늘 남은 삭제: N장                     │
│                                        │
│  광고 시청 시 삭제 한도 +10장 (M회 가능)  │
│                                        │
│  최대 무료 삭제 한도보다 K장 많아요.       │
│  K장을 보관함으로 되돌리거나              │
│  구독하면 비울 수 있어요                  │
│                                        │
│  [무제한 삭제 (구독하기)]                 │
│  [닫기]                                │
└────────────────────────────────────────┘
```

### 카운터 게이지

**휴지통 화면 상단**: 프로그레스 바로 남은 삭제 한도 표시

```
██░░░░░░░░ 5 / 20장 남음
```

탭 시 팝업:

```
┌────────────────────────────────────────┐
│  오늘의 삭제 한도                        │
│                                        │
│  ██░░░░░░░░ 5 / 20장 남음               │
│  광고 시청 시 +10장 (M회 가능)            │
│                                        │
│  [광고 보기]  [닫기]                     │
└────────────────────────────────────────┘
```

- M=0일 때 `(0회 가능)` 표시, 숨기지 않음
- Plus 구독자 / Grace Period: 게이지 미표시 (무제한)

### 비우기 버튼

- 버튼 텍스트는 항상 `[비우기]` — 상태와 무관하게 통일
- 탭 시 게이트 판단 → 게이트 필요 시 시트 표시, 불필요 시 바로 iOS 삭제 팝업

## 13. 유닛 이코노믹스 & 수익 시뮬레이션 ✅

> 기존 초안(260211bm1.md 섹션 7) 기반, 확정 가격으로 재계산

### 전제 조건

- 한국 시장 중심
- 무료 사용자 비율: 80% (DAU 기준)
- 활성 구독자 비율: 3% (DAU 기준)
- 가중평균 구독 단가 (월간 30% : 연간 70%):
  - **런칭기**: $2.99 × 30% + $19.99/12 × 70% = **$2.06/월**
  - **정가기**: $3.99 × 30% + $29.99/12 × 70% = **$2.95/월**

### 광고 eCPM (한국 iOS)

| 광고 유형 | eCPM | 도달율 (무료 DAU 대비) | 근거 |
|---|---|---|---|
| 리워드 비디오 | $29 | 25% | 한도 초과 사용자 중 광고 선택 비율 |
| 전면 (인터스티셜) | $10 | 12.5% | 짝수 회차만, 완료자 중 일부 |
| 배너 | $0.5 | 15% | 분석 중에만 노출, 짧은 시간 |

### DAU별 예상 수익 (런칭기)

| DAU | 리워드 | 전면 | 배너 | 구독 | **일 합계** | **월 합계** |
|---|---|---|---|---|---|---|
| 1,000 | $5.8 | $1.0 | $0.06 | $2.1 | **$9.0** | **$269** |
| 5,000 | $29 | $5.0 | $0.3 | $10.3 | **$44.6** | **$1,338** |
| 10,000 | $58 | $10 | $0.6 | $20.6 | **$89.2** | **$2,676** |
| 50,000 | $290 | $50 | $3.0 | $103 | **$446** | **$13,380** |

> 산출 (DAU 1,000): 리워드 800×25%×$29/1000=$5.8 / 전면 800×12.5%×$10/1000=$1.0 / 구독 1,000×3%×$2.06/30=$2.1

### 수익 비중

| 수익원 | 런칭기 | 정가기 |
|---|---|---|
| 리워드 광고 | 65% | 55% |
| 전면 광고 | 11% | 9% |
| 배너 광고 | ~1% | ~1% |
| 구독 | 23% | **35%** |

- 런칭기: 낮은 구독가 → 광고 수익 비중이 높음
- 정가기: 구독 단가 상승($2.06→$2.95/월) + 구독자 누적 → 구독 비중 증가
- 배너 수익은 미미하나 사용자 경험 비용도 낮음

## 16. 리스크 관리 & 의사결정 로그 ⬜

### 16.1 리스크 관리

**리스크 1: 광고 no-fill** ✅

> 리워드 광고 요청 시 광고가 없는 경우 (fill rate ~95%, 5%는 발생)

- **사전 로드**: 앱 시작 시 + 만료 전 자동 갱신 + 시청 완료 후 즉시 다음 광고 로드
- **재시도**: 지수 백오프 (2→4→8초), 즉시 재로드 금지 (Google 공식 경고)
- **버튼 3단계 상태**:

| 상태 | 버튼 표시 |
|---|---|
| Ready | "광고 보고 +10장" (활성) |
| Loading | "광고 준비 중..." (비활성, 스피너) |
| Failed | 탭 시 토스트 "잠시 후 다시 시도해주세요" |

- **no-fill 시 대응 흐름**:
  1. 평생 1회 무료 제공: "광고를 불러오지 못했어요. 이번은 무료!" (+10장)
  2. 1회 소진 후: 스피너 표시 (최대 10초 대기) → 성공 시 재생, 실패 시 버튼 비활성
- **미디에이션**: 출시 후 fill rate 확인 후 네트워크 추가 (AdMob 단일로 시작)

**리스크 2: ATT 거부율 높음** ✅

> 사용자 대부분이 ATT를 거부하면 광고 단가 ~35% 하락

- **대응**: 프리프롬프트로 옵트인율 ~50% 목표 (섹션 15.2에서 확정)
- 거부해도 비개인화 광고 정상 동작, 기능 제한 없음
- **프라이버시 인식 리스크**: 사진 앱은 프라이버시 민감도가 높은 카테고리 — ATT 프롬프트 자체가 "이 앱이 나를 추적한다"는 부정적 인식을 줄 수 있음. 프리프롬프트 문구에서 "데이터는 외부에 판매하지 않습니다"로 해소
- **보완**: 출시 후 프리프롬프트 문구/디자인 A/B 테스트로 옵트인율 개선
- **구조적 방어**: 구독 + 광고 하이브리드 모델이라 광고 단가 하락의 전체 수익 영향 제한적
