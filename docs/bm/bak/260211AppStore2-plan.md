  Gate 1~3의 나머지 작업 (아이콘, 스크린샷, #if DEBUG 등)은 기존 문서 그대로.

  ---
  의존성 요약

  DB ──→ BM ──→ 온보딩 ──→ Gate 문서 수정 ──→ Gate 실행 ──→ 제출
   │      │                        ↑
   │      └── IAP 상품 등록 (ASC) ──┘
   └── Firebase 프로젝트 생성 (BM도 사용)

  - DB → BM: Firebase를 공유하고, BM의 KPI 측정에 분석 인프라 필요
  - BM → 온보딩: 온보딩 자체는 독립적이나, BM의 UI가 확정되어야 스크린샷 촬영 가능
  - 전부 → Gate 문서 수정: 프라이버시 전제가 바뀌므로 Gate 1~3 문서 업데이트
  - Gate 문서 → Gate 실행: 수정된 문서 기준으로 코드/에셋/포털 작업
