# 추천 리워드 프로그램

## 개요

공유 링크 기반 추천 프로그램을 통해 바이럴 성장을 유도하고, Apple 구독 Offer Code로 양쪽에 보상을 지급하는 구조. 외부 서비스(Airbridge 등) 없이 Supabase 자체 구현.

## 유저 플로우

```
1. 유저가 앱 내 공유 버튼 탭
2. 고유 추천 링크 자동 생성 (예: sweeppic.link/r/abc123)
3. SNS에 링크 공유 (카카오톡, 인스타그램 등)
4. 친구가 링크 클릭 → 웹 랜딩 페이지
5. 랜딩 페이지 JS가 추천 코드를 클립보드에 자동 복사
6. App Store로 리다이렉트 → 앱 설치 → 앱 실행
7. 앱이 추천인 자동 감지 (아래 이중 트래킹 참조)
8. 양쪽에 Offer Code 자동 지급
```

## 추천인 감지: 이중 트래킹

App Store를 거치면 링크 정보가 사라지므로, 두 가지 방식을 병행하여 빈틈을 없앤다.

### 플로우

```
앱 첫 실행
  ↓
1차: 클립보드 자동 확인 → 추천 코드 발견?
  ├─ YES → 자동 매칭 완료 ✓
  └─ NO (허용 거부 or 다른 걸 복사함)
        ↓
2차: "초대 코드가 있으신가요?" 입력 UI 표시
        ├─ 코드 입력 → 매칭 완료 ✓
        └─ 건너뛰기 → 일반 가입
```

### 1차: 클립보드 자동 감지 (메인)

| 항목 | 내용 |
|------|------|
| 원리 | 랜딩 페이지 JS가 클립보드에 추천 코드 복사 → 앱이 `UIPasteboard` 읽기 |
| 정확도 | 허용 시 100% (deterministic) |
| Apple 정책 | **위반 아님** (공식 API) |
| 제약 | iOS 16+에서 "붙여넣기 허용" 시스템 팝업 표시 |
| 팝업 회피 | `UIPasteControl` 버튼 사용 시 팝업 없이 접근 가능 |

> Branch/Airbridge도 동일한 클립보드 방식(NativeLink)을 사용. 특별한 기술이 아님.

### 2차: 수동 코드 입력 (Fallback)

| 항목 | 내용 |
|------|------|
| 원리 | 유저가 추천인 코드를 직접 입력 |
| 정확도 | 100% |
| 제약 | UX 마찰 (코드를 기억하거나 전달받아야 함) |

> 랜딩 페이지에 "초대 코드: ABC123" 을 함께 표시하여, 클립보드 실패 시 수동 입력 유도

### 왜 외부 서비스(Airbridge 등)가 필요 없는가?

- Airbridge/Branch의 핵심 차별점은 **IP 핑거프린팅** 기반 매칭
- 그런데 Apple이 핑거프린팅을 **명시적으로 금지** (WWDC 2023, Privacy Manifest 정책)
- → Airbridge를 써도 핑거프린팅은 정책상 사용 불가
- → 결국 작동하는 건 클립보드 방식뿐이고, 이건 직접 구현 가능
- 수동 코드 입력 Fallback까지 더하면 Airbridge보다 오히려 확실함

## 보상 구조

| 대상 | 상태 | 보상 |
|------|------|------|
| **피추천인** (신규) | 구독 없음 | 1개월 무료 구독 (이후 자동 유료 전환) |
| **추천인** (기존 구독자) | 구독 중 | 2주 구독 연장 (다음 결제일 연기) |
| **추천인** (비구독자) | 구독 없음 | 1개월 무료 구독 |

## Apple Offer Code 활용

### Offer Code란?
- App Store Connect에서 생성하는 구독 할인/무료 코드
- 신규/기존/만료 구독자 모두 대상 설정 가능
- 기존 구독자가 입력 시 → 구독 기간 연장 (결제일 연기)
- 신규 유저가 입력 시 → 무료 기간 후 자동 정기결제 시작

### 선택 가능한 무료 기간
3일 / 1주 / 2주 / 1개월 / 2개월 / 3개월 / 6개월 / 1년

### 발급 한도
- 1회 생성: 최대 25,000개
- 분기당 한도: 앱당 1,000,000개
- 유효 기간: 최대 6개월
- 동시 활성 오퍼: 구독 SKU당 최대 10개

### 코드 지급 자동화 방식
1. Offer Code를 미리 대량 생성 (25,000개씩)
2. Supabase DB에 코드 저장
3. 추천 매칭 완료 시 → DB에서 코드 하나 꺼내서 자동 지급
4. 소진 시 재생성하여 보충

## 기술 인프라

| 요소 | 사용 기술 |
|------|----------|
| 추천 링크 추적 | 자체 구현 (클립보드 + 수동 입력) |
| 랜딩 페이지 | Supabase Edge Function 또는 별도 호스팅 |
| 추천 데이터 관리 | Supabase (기존 인프라 활용) |
| 코드 발급 | App Store Connect Offer Code |
| 앱 내 코드 리딤 UI | StoreKit `presentOfferCodeRedeemSheet` |

### Supabase 테이블 구조

```sql
-- 추천 링크
referral_links (user_id, code, created_at)

-- 추천 기록
referrals (referrer_id, referred_id, rewarded, created_at)

-- Offer Code 풀 (미리 생성해둔 코드)
offer_codes (code, type, is_used, assigned_to, created_at)
```

## 조사한 다른 방법들 (사용 불가 / 비권장)

| 방법 | 상태 | 사유 |
|------|------|------|
| IP 핑거프린팅 | **Apple 정책 위반** | WWDC 2023에서 명시적 금지 |
| Universal Links 단독 | **불가** | 앱 미설치 시 데이터 전달 안 됨 |
| Safari 쿠키 | **불가** | iOS 11(2017)부터 완전 차단 |
| SKAdNetwork | **불가** | 광고 캠페인 전용, 오가닉 추적 불가 |
| Firebase Dynamic Links | **서비스 종료** | 2025년 8월 완전 종료 |
| App Clips | 가능하지만 과도 | 별도 타겟 개발 필요, 10MB 제한 |
