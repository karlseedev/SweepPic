# PIClear BM 구현 계획

## Context

BM 전략 문서(`docs/bm/260211bm1.md` v3)에 정의된 2-Tier 모델(무료+광고 / Plus 구독)을 앱에 반영한다.
현재 앱에는 결제/광고 인프라가 전혀 없으므로 신규 구축이 필요하다.

핵심 설계: **기존 삭제 로직은 변경하지 않고, 앞에 게이트 레이어만 삽입**한다.

### 게이트 삽입 패턴 (핵심)

```swift
// 변경 전:
private func performEmptyTrash() {
    Task { try await trashStore.emptyTrash() }
}

// 변경 후:
private func performEmptyTrash() {
    let count = trashStore.trashedCount
    TrashGateCoordinator.shared.presentGateIfNeeded(
        from: self, action: .emptyAll, count: count
    ) { [weak self] in
        Task {
            do {
                try await self?.trashStore.emptyTrash()
                // ⚠️ 카운트 기록은 실제 삭제 성공 후에만 (iOS 팝업 취소 시 미소모)
                UsageLimitStore.shared.recordDelete(count: count)
            } catch {
                // iOS 시스템 팝업 취소 또는 오류 → 카운트 미소모
            }
        }
    }
}
```

> **이중 확인 참고**: 게이트 시트 통과 후에도 `trashStore.permanentlyDelete()`가 iOS 시스템 팝업("Delete N Photos?")을 표시함.
> 게이트 = "광고/구독 선택", 시스템 팝업 = "최종 삭제 확인"으로 목적이 다르므로 의도된 동작.
>
> **카운트 소모 규칙**: 삭제/정리 카운트는 **실제 성공 커밋 후에만 증가**한다. `permanentlyDelete()`는 async throws이므로, iOS 시스템 팝업에서 취소하면 throw → 카운트 미소모. 이 규칙은 recordDelete(), recordRewardView(), recordCleanup() 모두에 동일 적용.

### 게이트 판단 흐름

```
삭제 시도 → TrashGateCoordinator.evaluateGate()
  ├─ Plus 구독자? → 바로 실행
  ├─ 개별삭제 5장/일 이내? → 바로 실행
  ├─ 전체 비우기 10장 이하? → 바로 실행
  └─ 한도 초과 → TrashGateSheet 표시
       ├─ [N장만 삭제] → 일일 한도 내 삭제 (성공 후 recordDelete())
       ├─ [광고 보고 비우기] → ⚠️ 리워드 일일 3회 미소진 시만 활성화
       │    ├─ RewardedAdPresenter → 시청 완료 시 recordRewardView() + 삭제 실행
       │    │    └─ 50장 초과 시: permanentlyDelete(oldest 50장) — emptyTrash() 아님
       │    │    └─ 50장 이하 시: emptyTrash() — 전체 삭제
       │    │    └─ 삭제 대상: trashStore.oldestTrashedAssetIDs(limit: 50) (FIFO)
       │    └─ 3회 소진 시: 버튼 비활성화 + "Plus로 무제한 삭제" 텍스트 전환
       ├─ [Plus 업그레이드] → Phase 1: 비활성화(feature flag) / Phase 2: PaywallVC
       └─ [나중에] → dismiss
```

## 검토에서 발견된 이슈 및 보완

### 보완 1: 누락된 게이트 삽입 지점 추가
- `GridViewController.swift:966`, `AlbumGridViewController.swift:416`에도 `viewerDidRequestPermanentDelete()` 존재
- 현재 ViewerMode.normal에서는 permanentDelete 버튼이 표시되지 않아 실제 호출 가능성 낮음
- 방어적으로 게이트 삽입 (기존 파일 수정 표에 추가 완료)

### 보완 2: 부분 삭제(50장 차등) 시 기술적 처리
- 120장 중 50장만 삭제 시 `emptyTrash()`가 아닌 `permanentlyDelete(assetIDs:)`를 직접 호출
- **TrashStore.state가 private** → 외부에서 trashDates 직접 접근 불가
- **해결**: `TrashStore+FIFO.swift` extension으로 `oldestTrashedAssetIDs(limit:)` public API 추가 (Phase 1 구현 순서 1-1에 포함)
- TrashGateCoordinator에서 `trashStore.oldestTrashedAssetIDs(limit: 50)` 호출 → `permanentlyDelete(assetIDs:)` 실행

### 보완 3: 카운트 소모 통일 규칙
- **모든 카운트(삭제/리워드/정리)는 실제 성공 후에만 증가**
- `recordDelete()`: `permanentlyDelete()` async throws 성공 후에만 호출 (iOS 시스템 팝업 취소 시 미소모)
- `recordRewardView()`: 리워드 광고 시청 완료 콜백 수신 후에만 호출
- `recordCleanup()`: `CleanupResult`가 `.cancelled`가 아닌 경우에만 호출
- 사용자 취소/오류 시 어떤 카운트도 소모하지 않음

### 보완 4: 일일 리셋 방어적 처리
- `sceneWillEnterForeground` 리셋 외에도, `canDeleteIndividually()` 등 한도 체크 메서드 내부에서 자동 리셋 체크
- 앱이 포그라운드에서 자정을 넘기는 경우 대비

### 보완 5: 네이티브 배너 광고 (Phase 1 추가)
- BM 5.1에 따라 자동 정리 결과 화면 하단에 네이티브 배너 추가
- 신규 파일: `Features/Monetization/Ad/NativeBannerAdView.swift`
- 수정 파일: `PreviewGridViewController.swift` (결과 화면 하단에 배너 삽입)

### 보완 6: 게이트 시트 표시 체인
- TrashGateSheet → 사용자 "광고 보고 비우기" 선택 → sheet dismiss → 리워드 광고 present
- 표시 체인에서 presenting VC 관리 주의: sheet dismiss completion 블록에서 ad present

### 보완 7: 리워드 일일 3회 제한 (Phase 1 필수)

- TrashGateSheet UI: 3회 미소진 시 "광고 보고 비우기" 활성화, 소진 시 비활성화 + Plus 유도 텍스트

### 보완 8: Phase 1 Plus 버튼 임시 처리

## 주의사항

- **TrashAlbumVC 786줄** → 게이트 로직은 `TrashAlbumViewController+Gate.swift` Extension으로 분리
- **AppCore에 UIKit 불가** → 광고/UI 코드는 반드시 PIClear 앱 레이어에
- **50줄 이상 수정 전 커밋** (CLAUDE.md 규칙)
- **1,000줄 제한** — 파일별 분할 유지
- **iOS 16~25/26+ 분기** — 게이트 시트가 FloatingOverlay 위에 올바르게 표시되는지 확인 필요
- **이중 확인 UX** — 게이트 시트 + iOS 시스템 팝업은 목적이 다르므로 의도된 동작

## 검증 방법

1. **게이트 전수 테스트**: 5개 삽입 지점 각각에서 한도 초과 시 게이트 시트 표시 확인
2. **10장 이하 무료 비우기**: 10장 → 게이트 없이 iOS 시스템 팝업만 → 삭제 완료
3. **11장 이상 게이트**: 18장 → 게이트 시트 → "5장만 삭제" → 5장 삭제 + 13장 남음
4. **50장 차등**: 120장 → 광고 보고 → 50장만 삭제 (FIFO) → 70장 남음
5. **리워드 연동**: AdMob 테스트 광고 ID로 시청 → 비우기 해방 확인
6. **일일 한도 리셋**: 5장 삭제 후 → 날짜 변경 → 한도 초기화 확인
7. **자동 정리 한도**: 월 2회 실행 → 3회째 CleanupGateSheet 표시 → 취소 시 횟수 미소모
8. **Grace Period**: 설치 직후 3일간 → 광고 미표시, 게이트는 작동
9. **Phase 2 검증**: RevenueCat sandbox 구독 구매 → 모든 게이트 스킵 + 광고 미표시
10. **Phase 3 검증**: 비우기 완료 → 축하 화면 → 전면 광고 순서 확인
11. **iOS 팝업 취소 검증**: 게이트 통과 → iOS "Delete N Photos?" 팝업에서 취소 → 카운트 미소모 확인
12. **리워드 3회 소진 검증**: 리워드 3회 시청 → 4회째 "광고 보고 비우기" 버튼 비활성화 + Plus 유도 텍스트 표시
13. **광고 no-fill 검증**: 리워드 광고 로드 실패 시 → 사용자에게 안내 + 재시도/대기 처리 확인
14. **Phase 1 Plus 버튼 검증**: Plus 버튼 feature flag 비활성화 상태에서 표시 안 됨 (또는 비활성화 UI) 확인
