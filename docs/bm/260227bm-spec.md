## 1. 제품 철학 & 수익화 원칙 ✅

### 구조

```
[사진 정리]        →  [휴지통]       →  [비우기]
항상 무료, 무제한      무료 보관         여기만 게이트
```

- **무료 영역**: 스와이프 삭제, 유사 사진 분석, 자동 정리, 복구 — 전부 무제한
- **게이트**: "비우기"(영구 삭제) 한 곳에만 존재

### 광고 배치 방향

- 정리 행위 **도중**에는 광고 없음 (비교 화면, 뷰어, 그리드)
- 정리 행위 **완료 시점**에 전면 광고 (짝수 회차만, 스킵 가능)
- **대기 화면**에 배너 (분석 중 하단)

| 시점 | 타입 | 빈도 |
|------|------|------|
| 휴지통 비우기 (한도 초과) | 리워드 | 일 2회 |
| 유사사진 삭제 완료 | 전면 | 짝수 회차 |
| 자동정리 휴지통 이동 완료 | 전면 | 짝수 회차 |
| 사진 분석 대기 화면 | 배너 | 매회 |

### 운영 방향

- 사진 정리 행위는 게이트로 막지 않는다 (비우기만 게이트)
- 그리드, 뷰어, 비교 화면 등 탐색 중에는 광고를 넣지 않는다
- 휴지통 복구는 유료화하지 않는다
- Plus 구독자에게는 광고를 보여주지 않는다
- 기존 무료 기능을 나중에 유료로 전환하지 않는다

### 전체 메뉴 구조

> 우측 상단 ellipsis 버튼 풀다운 메뉴. BM 관련 항목 포함.

```
전체 메뉴 (ellipsis 버튼)
├─ 자동정리
├─ 사용자
├─ 프리미엄
│   ├─ 구독 관리          → showManageSubscriptions()
│   └─ 구독 복원          → AppStore.sync()
├─ 기타
└─ 고객센터
    ├─ 피드백 보내기       → MFMailComposeViewController (기기 정보 자동 첨부)
    ├─ 자주 묻는 질문      → FAQ 화면 또는 웹뷰
    ├─ 이용약관            → 웹 링크
    ├─ 개인정보처리방침     → 웹 링크
    └─ 사업자 정보         → 상호, 대표자, 사업자번호, 연락처 (전자상거래법 제10조)
```

- **프리미엄**: 무료 사용자 탭 시 → 페이월, 구독자 탭 시 → 하위 메뉴
- **고객센터**: 하위 화면으로 이동
- **구독 복원**: §8 페이월에도 있지만 여기에도 배치 (접근성)
- **FAQ**: 콘텐츠는 `260227Marketing.md` §7에서 정의 예정

## 2. 사용자 세그먼트 & 라이프사이클 ✅

### 세그먼트 (가정 — 출시 후 데이터로 검증)

| 유형 | 사진 수 | 일일 정리량 | 게이트 빈도 |
|------|:---:|:---:|:---:|
| 라이트 | ~5,000장 | 1~10장 | 거의 안 걸림 |
| 미디엄 | 5,000~20,000장 | 10~30장 | 가끔 |
| 헤비 | 20,000장+ | 30장+ | 자주~매일 |

### 라이프사이클

```
설치 → Grace Period(3일) → 무료 활성 → 한도 도달 → 전환 or 이탈
                                         ↓
                                    Plus 구독자 → 갱신 or 해지
```

| 단계 | 기간 | 경험 |
|------|------|------|
| 신규 | Day 0~3 | 무제한, 광고 없음 (Grace Period) |
| 무료 활성 | Day 4~ | 일일 10장 무료, 카운터 보임, 전면 광고 노출 |
| 한도 도달 | 수시 | 게이트 시트 → 리워드/Plus 선택지 |
| Plus 전환 | - | 무제한, 광고 없음 |
| 이탈 위험 | 사용 감소 시 | 윈백 대상 |

## 3. 유료 모델 핵심 가치 제안 ✅

> **"제한/광고 없이 바로 정리"**

| 가치 | 무료 | Plus |
|------|------|------|
| 일일 삭제 | 최대 30장 (무료 10 + 리워드 2회) | 무제한 |

- **용량 해방**: 일 30장 캡 해제 → 헤비 유저/대청소 시 결정적
- **귀찮음 제거**: 게이트, 전면 광고, 배너, 카운터 전부 없음

## 4. 한도 & Grace Period ✅

### 일일 삭제 한도 (총량 캡 방식)

```
일일 총 삭제 한도 = 무료 N장 + 리워드 M회 × K장
```

| 항목 | 시작값 | 조정 범위 | 비고 |
|------|:---:|:---:|------|
| 무료 | 10장 | 5~15 | 방법 불문 |
| 리워드 1회당 | +10장 | 5~20 | "전체 비우기"가 아닌 추가 장수 |
| 리워드 횟수 | 2회/일 | 1~3 | |
| **일일 최대** | **30장** | **15~45** | 초과 시 Plus만 가능 |

- **일일 리셋 방식**:
  - 로컬 자정 기준 (사용자 기기 타임존)
  - 리셋 체크 시점 (이중 체크): ① 앱 Foreground 진입 시 (UI 즉시 갱신) ② `significantTimeChangeNotification` 수신 시 (앱 열린 상태에서 자정 넘길 때 iOS가 자동 발송)
  - 날짜 비교: `Calendar.isDate(_:inSameDayAs:)` 사용 (DST/타임존 자동 처리). 24시간(86400초) 비교 금지
- 모든 숫자는 출시 후 A/B 테스트로 조정
- 자동 정리 실행 횟수: **무제한** (게이트는 비우기에만)
- 한도 초과 시 관용 없음 — 12장이어도 무료 비우기 불가, 게이트 표시
- 리워드 소진 시 → 골든 모먼트 (Plus 전환 유도)

### Grace Period

- 설치 후 3일: 광고 없음, 게이트 없음, 카운터 없음 → 무제한
- Day 4부터: 정상 운영 (게이지 + 게이트 + 광고)

### Grace Period 전환 UX

휴지통 화면 상단, 게이지와 동일한 위치·높이로 사전 안내 표시:

**Day 0~2:**
```
┌────────────────────────────────────────┐
│  무료 체험 중 — N일 남음                 │
│  체험 종료 후 일일 무료 삭제 한도가        │
│  적용됩니다                              │
└────────────────────────────────────────┘
```

**Day 3 (마지막 날):**
```
┌────────────────────────────────────────┐
│  무료 체험 마지막 날                      │
│  내일부터 일일 무료 삭제 한도가            │
│  적용됩니다                              │
└────────────────────────────────────────┘
```

**Day 4~ (정상 운영):**
```
██████████ 10 / 10장 남음   ← 같은 위치가 게이지로 전환
```
- 게이지 첫 표시 시 1회 툴팁: `"오늘의 무료 삭제 한도예요. 탭해서 자세히 볼 수 있어요"`

## 5. 광고 ✅

| 광고 | 트리거 | 빈도 |
|-----|--------|------|
| **리워드** | 게이트에서 사용자 선택 | 일 2회 |
| **전면** | 유사사진 삭제 완료 / 자동정리 휴지통 이동 완료 | 짝수 회차 |
| **배너** | 사진 분석 대기 화면 | 매회 |

### 리워드 광고 보상
- 1회 시청 = 일일 삭제 한도 +10장 추가 (전체 비우기가 아님)
- 무료 10 + 리워드 2회 = 일일 최대 30장
- 리워드 횟수는 삭제 성공 후에만 차감 (iOS 팝업 취소 시 소모 안 됨)
- 2회 소진 시 → 골든 모먼트 (Plus 전환 유도)

### 광고 배치 기준
- 정리 행위 **도중**: 광고 없음 (그리드, 뷰어, 비교 화면)
- 정리 행위 **완료 시점**: 전면 광고 (짝수 회차, 스킵 가능)
- **대기 화면**: 배너 (분석 중 하단)
- Grace Period (3일): 모든 광고 없음
- Plus 구독자: 모든 광고 없음

### 광고 로딩 & no-fill 처리
- **사전 로드**: 앱 시작 시 + 만료 전 자동 갱신 + 시청 완료 후 즉시 다음 광고 로드
- **재시도**: 지수 백오프 (2→4→8초), 즉시 재로드 금지
- **no-fill 시**: 평생 1회 무료 제공 (+10장) → 이후 스피너 10초 대기 → 실패 시 재시도/취소 팝업 (횟수 제한 없음)
- **미디에이션**: 출시 시 AdMob 단일, 이후 fill rate 확인 후 네트워크 추가

### 광고 버튼 3단계 상태

| 상태 | 버튼 표시 |
|---|---|
| Ready | "광고 N회 보고 X장 삭제" (활성) |
| Loading | "광고 준비 중..." (비활성, 스피너) |
| Failed | 탭 시 스피너 10초 대기 → 실패 시 재시도/취소 팝업 |

## 6. 구독 ✅

### Plus 혜택
- 광고 없음, 게이트 없음, 무제한 삭제 및 정리

### 가격 전략 (USD 기준)

| 플랜 | 정가 | 런칭 할인가 | 비고 |
|------|------|--------|------|
| **월간** | $3.99 | **$2.99** | 가벼운 진입 |
| **연간** | $29.99 | **$19.99** | 메인 전환 타겟 (월간 정가 대비 37%↓) |

- **할인율 정책**: 연간 정가($29.99)는 월간 정가($47.88) 대비 약 37% 할인된 가격으로 설정하여 장기 구독 유도.
- **라이프타임(Lifetime) 플랜**: 출시 초기에는 운영하지 않음. 서비스 안정화 이후 특정 마일스톤이나 명절 등 특별 이벤트 형식으로 한정 판매 검토.

### 무료 체험 정책
- **별도 무료 체험 없음**: Grace Period(3일)가 사실상 강력한 무료 체험 역할을 수행함.
- 사용자는 결제 수단 등록 없이도 3일간 앱의 모든 가치를 경험할 수 있음.
- 3일 종료 시점에 자연스럽게 Plus 전환 넛지 노출.

### 할인 축소 계획
- 런칭 후 초기 일정 기간(또는 목표 사용자 달성 시)까지 할인가 유지.
- 데이터 분석을 통해 전환율이 안정화되면 단계적으로 정가로 복구.

### 환불 처리 프로세스 (E9)

**방침**: 서버 없는 클라이언트 전용 처리 (StoreKit 2 네이티브)

| 구성 요소 | 구현 방식 |
|----------|----------|
| 환불 감지 | `Transaction.currentEntitlements` — 앱 실행 시 유효 구독 확인 |
| 실시간 감지 | `Transaction.updates` — 포그라운드 중 환불 즉시 반영 |
| 권한 회수 | 환불 감지 시 Plus 기능 잠금 (게이트 재활성화, 광고 복원) |
| 영수증 검증 | StoreKit 2 자체 서명 검증 (JWS) — 서버 불필요 |

**처리 흐름**:
```
앱 실행/포그라운드 복귀
  → Transaction.currentEntitlements 순회
  → Plus 구독 트랜잭션 존재?
     ├─ YES → Plus 유지
     └─ NO  → Plus 해제 (Free tier 복귀)
         ├─ 게이트 재활성화
         ├─ 광고 표시 복원
         └─ 조용히 처리 (사용자 알림 없음)
```

**향후 확장 가능성** (서버 도입 시):
- App Store Server Notifications V2 → 실시간 환불 웹훅
- Consumption API → 환불 결정에 사용 데이터 제출 (환불 거부율 향상)
- 현 단계에서는 ROI 대비 과투자로 판단, 클라이언트 처리로 충분

## 7. 게이트 ✅

### 게이트 판단 로직

| 조건 | 결과 |
|---|---|
| 휴지통 ≤ 남은 한도 | 게이트 없이 바로 삭제 |
| 휴지통 ≤ 남은 한도 + (남은 광고 × 10) | 광고 N회 보고 전체 비우기 |
| 휴지통 > 남은 한도 + (남은 광고 × 10) | Plus만 가능 |

- **남은 한도** = 오늘 일일 한도 - 이미 삭제한 수
- **남은 광고** = 2 - 오늘 시청한 광고 수
- 비우기는 항상 **전체 비우기** (부분 삭제 없음)

### 게이트 시트 — 광고로 가능한 경우

```
┌────────────────────────────────────────┐
│  휴지통: X장                            │
│  오늘 남은 삭제: N장                     │
│                                        │
│  광고 시청 시 삭제 한도 +10장             │
│                                        │
│  [광고 N회 보고 X장 삭제]                │
│  [무제한 삭제 (구독하기)]                 │
│  [닫기]                                │
└────────────────────────────────────────┘
```

> 광고 버튼 상태 및 no-fill 처리 → 섹션 5 참고

### 게이트 시트 — Plus만 가능한 경우

```
┌────────────────────────────────────────┐
│  휴지통: X장                            │
│  오늘 남은 삭제: N장                     │
│                                        │
```

## 10. 리텐션 & 윈백 ✅

> 리텐션(푸시/뱃지), 윈백(재활성화)은 1차 오픈 범위 밖. 이탈 방지 설정만 출시 시 적용.

### 이탈 방지 (설정만으로 적용)

| 항목 | 설정 위치 | 내용 |
|---|---|---|
| **Retention Messaging** | App Store Connect → 구독 → 이탈 방지 메시지 | 사용자가 Apple 설정에서 해지 시도 시 표시할 메시지 작성 (예: "해지하면 삭제 한도가 일 10장으로 돌아갑니다") |
| **Billing Grace Period** | App Store Connect → 구독 → 결제 유예 기간 활성화 | 결제 실패 시 일정 기간 Plus 유지하며 Apple이 자동 재시도. 전체 해지의 ~20%가 결제 실패 원인 |
| **Win-back Offer** | App Store Connect → 구독 → Win-back Offer 설정 | 해지된 구독자에게 할인 오퍼 자동 표시 (iOS 18+). ASC 설정만으로 동작, 앱 코드 불필요 |

### 리뷰/평점 관리 (E11)

#### 자동 리뷰 요청 (`requestReview()`)

**기본 조건 (AND — 전부 충족 시 트리거 이벤트에서 호출):**

| 조건 | 값 | 이유 |
|------|---|------|
| 설치 경과 | **3일 이내** (Grace Period) | 광고/게이트 시작 전, 깨끗한 경험 상태에서 받기 |
| 세션 수 | **3회 이상** | 일회성 사용자 제외 |
| 누적 삭제대기함 이동 | **30장 이상** | 핵심 기능 반복 사용 확인 |
| 버전당 | **1회** | 연간 3회 제한 아끼기 |
| 쿨다운 | **90일** | 마지막 프롬프트 이후 |

- 삭제대기함 이동 카운트: 스와이프 삭제, 유사사진 정리, 자동정리 **전부 포함**
- Review Gating(사전 질문) 없음 — 조건 충족 시 바로 `requestReview()` 호출

**트리거 이벤트 (조건 충족 상태에서 아래 중 하나 발생 시):**

| 이벤트 | 시점 |
|--------|------|
| 유사사진 정리 | 삭제대기함 이동 버튼 탭 시 |
| 자동정리 | 삭제대기함 이동 버튼 탭 시 |
| 삭제대기함 비우기 | 비우기 완료 시 |
| 누적 30장 달성 | 삭제대기함 이동 완료 시 |

**금지 타이밍 (조건 충족되어도 요청 안 함):**

| 상황 | 이유 |
|------|------|
| 광고 시청 직후 (최소 1세션 간격) | 1점 리뷰 유발 |
| 구독 결제 직후 | 반감 |
| 에러/크래시 발생 세션 | 부정 경험 |
| 게이트 시트 노출 직후 | 제한 느낌 직후 |

> Grace Period 3일 이내를 주 타겟으로 하므로, 금지 타이밍 대부분이 자동으로 회피됨 (광고/게이트 없는 기간)

#### Grace Period 이후 사용자

Grace Period(3일) 안에 조건을 못 채운 사용자도 리뷰 요청을 받을 수 있음:
- 기본 조건에서 "3일 이내" 제약은 **우선 타겟**이지, 이후를 차단하지 않음
- Day 4 이후에도 조건(3세션+, 30장+, 금지 타이밍 아님) 충족 시 트리거
- 단, 광고/게이트 **직후**가 아닌 타이밍에서만 호출

> 산출 (DAU 1,000): 리워드 800×25%×$29/1000=$5.8 / 전면 800×12.5%×$10/1000=$1.0 / 구독 1,000×3%×$2.06/30=$2.1

### 수익 비중

| 수익원 | 런칭기 | 정가기 |
|---|---|---|
| 리워드 광고 | 65% | 55% |
| 전면 광고 | 11% | 9% |
| 배너 광고 | ~1% | ~1% |
| 구독 | 23% | **35%** |

- 런칭기: 낮은 구독가 → 광고 수익 비중이 높음
- 정가기: 구독 단가 상승($2.06→$2.95/월) + 구독자 누적 → 구독 비중 증가
- 배너 수익은 미미하나 사용자 경험 비용도 낮음

## 15. 리스크 관리 & 의사결정 로그 ✅

### 15.1 리스크 관리

**리스크 1: 광고 no-fill** ✅

> 리워드 광고 요청 시 광고가 없는 경우 (fill rate ~95%, 5%는 발생)

- **대응**: 섹션 5 "광고 로딩 & no-fill 처리"에 전체 반영 완료
  - 사전 로드 + 지수 백오프 재시도 + 버튼 3단계 상태
  - no-fill 시: 평생 1회 무료 → 이후 재시도/취소 팝업
  - 미디에이션: 출시 후 추가

**리스크 2: ATT 거부율 높음** ✅

> 사용자 대부분이 ATT를 거부하면 광고 단가 ~35% 하락

- **대응**: 프리프롬프트로 옵트인율 ~50% 목표 (섹션 14.2에서 확정)
- 거부해도 비개인화 광고 정상 동작, 기능 제한 없음
- **프라이버시 인식 리스크**: 사진 앱은 프라이버시 민감도가 높은 카테고리 — ATT 프롬프트 자체가 "이 앱이 나를 추적한다"는 부정적 인식을 줄 수 있음. 프리프롬프트 문구에서 "데이터는 외부에 판매하지 않습니다"로 해소
- **보완**: 출시 후 프리프롬프트 문구/디자인 A/B 테스트로 옵트인율 개선
- **구조적 방어**: 구독 + 광고 하이브리드 모델이라 광고 단가 하락의 전체 수익 영향 제한적

**리스크 3: 게이트 과도 → 이탈** ✅

> 무료 사용자가 게이트에 막혀 앱을 이탈할 가능성

- **현황**: 게이트가 비우기 1곳에만 있고, 정리 행위 자체는 무제한 → 업계 대비 관대한 설계
- **대응**: 한도 값은 출시 후 데이터 보고 조정

**리스크 4: App Store 리젝** ✅

> 페이월·광고·게이팅 관련 가이드라인 위반으로 리젝

- **주요 리젝 사유** (리서치 기준):
  1. 페이월에 가격/갱신조건/취소방법 미표시 (가이드라인 3.1.2)
  2. 구매 복원(Restore Purchases) 기능 누락 (가이드라인 3.1.1)
  3. 리뷰어가 페이월 뒤 기능을 테스트할 수 없음 (가이드라인 2.1)
  4. 광고 버튼 무반응 — no-fill 시 처리 미비
- **대응**:
  - a, b → 섹션 8 페이월에 반영 완료 (자동 갱신 고지, 취소 경로, 구매 복원 버튼)
  - c → 섹션 14.6 Review Notes에 반영 완료
  - d → 섹션 5 광고에 반영 완료 (광고 버튼 3단계 상태)

**리스크 5: StoreKit 2 결제 실패** ✅

> 구독 구매 시 네트워크 오류, Ask to Buy 대기, 자동 갱신 실패, 검증 실패

- **대응**: 섹션 8 "구독 결제 처리"에 전체 반영 완료
  - 결제 실패 시나리오별 처리 (네트워크/Ask to Buy/검증 실패)
  - 자동 갱신 실패: Billing Grace Period 16일 + 전체메뉴 ⚠️ 뱃지
  - 앱 시작 시 구독 상태 동기화

**리스크 6: 일일 한도 리셋 누락** ✅

> 자정을 넘겼는데 한도가 리셋되지 않아 무료 사용자가 삭제를 못 하는 경우

- **대응**: 섹션 4 "일일 리셋 방식"에 반영 완료
  - 이중 체크 (Foreground 진입 + `significantTimeChangeNotification`)
  - `Calendar.isDate` 기반 날짜 비교 (DST/타임존 안전)
