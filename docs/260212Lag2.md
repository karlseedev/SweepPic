# 앨범 탭 랙 근본 해결 계획

## Context

앨범 탭 진입 시 심한 랙이 발생합니다. 원인은 `AlbumsViewController.loadData()`가 메인 스레드에서 **50회 이상의 동기 PhotoKit 호출**을 수행하기 때문입니다:
- `fetchSmartAlbums()`: 10종 × 2회 = 20회 동기 호출 (각 앨범 전체 에셋 fetch)
- `fetchUserAlbums()`: N개 앨범 × 1회 = N회 추가 호출
- `AlbumCell.loadThumbnail()`: 셀마다 `PHAsset.fetchAssets` 동기 호출
- `viewWillAppear`에서 매번 무조건 전체 재로드
- iOS 18/26 모두 동일한 코드 경로 → 동일한 문제

---

## 수정 파일 및 변경 내용

### 1. `Sources/AppCore/Services/AlbumService.swift`

**변경 내용:**

**(A) 기존 동기 메서드 내부 최적화 — fetchLimit = 1**

`fetchSmartAlbum(type:)` 및 `fetchUserAlbums()` 내부에서:
- keyAsset용 fetch와 count용 fetch를 분리
- keyAsset: `fetchLimit = 1` + `sortDescriptors = [creationDate descending]` → 1개만 로드
- count: 기존 `PHAsset.fetchAssets(in:options:).count` 유지 (PHFetchResult.count는 내부 최적화됨)
- `estimatedAssetCount == 0`인 스마트 앨범은 fetch 없이 건너뛰기 (빠른 필터)
  - 주의: `NSNotFound` 반환 시에는 스킵하지 않고 fetch로 확인

**(B) 비동기 일괄 fetch 메서드 추가**

```swift
/// 비동기 앨범 전체 로드
/// - 내부: DispatchQueue.global(qos: .userInitiated)에서 PhotoKit fetch 실행
/// - completion: 메인 스레드에서 호출됨
/// - keyAssets: 앨범ID → PHAsset 딕셔너리 (셀에서 재 fetch 방지)
public func fetchAllAlbumsAsync(
    completion: @escaping (
        _ smartAlbums: [SmartAlbum],
        _ userAlbums: [Album],
        _ keyAssets: [String: PHAsset]
    ) -> Void
)
```

- 내부에서 `DispatchQueue.global(qos: .userInitiated).async`로 fetch 실행
- fetch 완료 후 `DispatchQueue.main.async`로 completion 호출
- fetchLimit=1로 가져온 PHAsset을 `keyAssets` 딕셔너리에 수집
- TrashAlbumViewController의 `loadTrashedAssets()` 패턴과 동일한 구조

**(C) AlbumServiceProtocol에 비동기 메서드 선언 추가**

### 2. `PickPhoto/PickPhoto/Features/Albums/AlbumCell.swift`

**변경 내용:**

**(A) PHAsset 직접 전달 configure 오버로드 추가**

```swift
func configure(album: Album, keyAsset: PHAsset?, targetSize: CGSize)
func configure(smartAlbum: SmartAlbum, keyAsset: PHAsset?, targetSize: CGSize)
```

**(B) loadThumbnail을 PHAsset 기반으로 변경**

- 기존: `loadThumbnail(assetID:)` → `PHAsset.fetchAssets(withLocalIdentifiers:)` 동기 호출 후 `PHImageManager.default().requestImage()`
- 변경: `loadThumbnail(asset:)` → `ImagePipeline.shared.requestImage(for:targetSize:)` 직접 호출
  - `ImagePipeline`은 `PHCachingImageManager` + 백그라운드 OperationQueue 사용 (이미 구현됨)
  - `Cancellable` 리턴 → `prepareForReuse`에서 취소
  - **iCloud 주의**: 현재 AlbumCell은 `isNetworkAccessAllowed = true`이나, ImagePipeline 기본값은 `false`. 앨범 셀 전용으로 `isNetworkAccessAllowed = true` 옵션을 적용하거나, ImagePipeline에 quality 파라미터로 분기 필요

**(C) 기존 assetID 기반 configure는 하위 호환을 위해 유지** (TrashAlbum 등에서 사용)

**(D) keyAsset cache miss 시 fallback**

- `keyAsset == nil && keyAssetIdentifier != nil`인 경우, 기존 assetID 기반 `loadThumbnail(assetID:)` 경로로 fallback
- 정상 흐름에서는 발생하지 않으나 edge case(iCloud 동기화 지연 등) 대비

### 3. `PickPhoto/PickPhoto/Features/Albums/AlbumsViewController.swift`

**변경 내용:**

**(A) loadData() 비동기화 + 응답 역전 방지**

```swift
private var loadGeneration: UInt = 0

private func loadData() {
    guard !isLoading else {
        // 로딩 중 요청 → 완료 후 재로드 필요
        needsReload = true
        return
    }
    isLoading = true
    loadGeneration += 1
    let currentGeneration = loadGeneration

    albumService.fetchAllAlbumsAsync { [weak self] smartAlbums, userAlbums, keyAssets in
        guard let self = self else { return }
        // stale 응답 무시 (더 새로운 요청이 발행된 경우)
        guard currentGeneration == self.loadGeneration else { return }

        self.isLoading = false
        self.lastLoadTime = CFAbsoluteTimeGetCurrent()
        self.smartAlbums = smartAlbums
        self.userAlbums = userAlbums
        self.keyAssetCache = keyAssets
        self.updateEmptyState()
        self.collectionView.reloadData()

        // 로딩 중 추가 변경이 있었으면 재로드
        if self.needsReload {
            self.needsReload = false
            self.loadData()
        }
    }
}
```

새 프로퍼티:
- `isLoading: Bool` — 중복 로딩 방지
- `loadGeneration: UInt` — 비동기 응답 역전(stale overwrite) 방지
- `lastLoadTime: CFAbsoluteTime` — 재로딩 간격 제어
- `needsReload: Bool` — 로딩 중 또는 백그라운드 변경 플래그
- `keyAssetCache: [String: PHAsset]` — 셀에 전달할 PHAsset 캐시

**(B) viewWillAppear 조건부 로딩**

```swift
override func viewWillAppear(_ animated: Bool) {
    super.viewWillAppear(animated)
    let now = CFAbsoluteTimeGetCurrent()
    if needsReload || (now - lastLoadTime > 1.0) {
        loadData()
        // needsReload는 loadData() completion에서 처리
    }
    configureFloatingOverlayForAlbums()
}
```

**(C) photoLibraryDidChange 개선**

- 화면 보이는 중: `loadData()` (비동기이므로 블로킹 없음)
- 화면 안 보이는 중: `needsReload = true` (viewWillAppear에서 처리)

**(D) cellForItemAt에서 keyAssetCache 활용 + fallback**

```swift
case .smartAlbums:
    let smartAlbum = smartAlbums[indexPath.item]
    let keyAsset = keyAssetCache[smartAlbum.id]
    cell.configure(smartAlbum: smartAlbum, keyAsset: keyAsset, targetSize: targetSize)
    // keyAsset == nil이면 셀 내부에서 assetID fallback 처리
```

## 데이터 플로우 (수정 후)

```
loadData()
  ├→ loadGeneration += 1  (응답 역전 방지 토큰)
  └→ albumService.fetchAllAlbumsAsync {
       // 백그라운드에서 실행:
       //   fetchSmartAlbums (fetchLimit=1로 keyAsset)
       //   fetchUserAlbums (fetchLimit=1로 keyAsset)
       //   keyAssets 딕셔너리 구축
       //
       // 완료 후 메인 스레드에서 completion 호출:
       └→ guard currentGeneration == self.loadGeneration  // stale 체크
          self.smartAlbums = smartAlbums
          self.keyAssetCache = keyAssets
          self.collectionView.reloadData()
          if self.needsReload { self.loadData() }  // 대기 변경 처리
     }

cellForItemAt
  └→ cell.configure(album:, keyAsset: keyAssetCache[id], targetSize:)
       ├→ keyAsset != nil → ImagePipeline.requestImage(for: asset, ...)
       └→ keyAsset == nil → fallback: loadThumbnail(assetID:)  // 기존 경로
```

## 성능 개선 예상

| 항목 | Before (메인 스레드) | After |
|------|---------------------|-------|
| AlbumService fetch | 동기 50회+ PhotoKit 호출 | 백그라운드, fetchLimit=1 |
| AlbumCell 썸네일 | PHAsset.fetchAssets 동기 호출 | PHAsset 직접 전달 + ImagePipeline |
| viewWillAppear | 매번 전체 재로드 | 조건부 (needsReload/시간 검증) |
| photoLibraryDidChange | 동기 전체 재로드 | 비동기 + 비가시 시 지연 |

**PhotoKit 동기 호출에 의한 메인 스레드 블로킹 대폭 감소** (fetch 작업이 백그라운드로 이동, reloadData/레이아웃만 메인 스레드)

---

## 구현 순서

1. AlbumService — fetchLimit=1 최적화 + 비동기 API 추가
2. AlbumCell — PHAsset 직접 전달 configure + ImagePipeline 사용 (iCloud 옵션 반영)
3. AlbumsViewController — 비동기 loadData, 응답 역전 방지, 조건부 viewWillAppear, photoLibraryDidChange 개선

## 검증

- 빌드: `xcodebuild -project PickPhoto/PickPhoto.xcodeproj -scheme PickPhoto -destination 'platform=iOS Simulator,name=iPhone 17'`
- 앨범 탭 진입 시 랙 없이 데이터 로딩되는지 확인
- 앨범 상세 → 뒤로가기 → 불필요한 재로딩 없는지 확인
- 사진 추가/삭제 후 앨범 탭에 반영되는지 확인
- iCloud 최적화 기기에서 앨범 썸네일 정상 표시 확인
