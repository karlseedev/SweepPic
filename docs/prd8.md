# PRD 8: 유사 사진 정리 기능

**버전**: 1.5
**작성일**: 2025-01-01
**상태**: Draft

---

## 1. 개요

### 1.1 배경

사진을 촬영할 때 "혹시 모르니까" 여러 장을 연속으로 찍는 경우가 많다. 특히 단체 사진은 눈 깜빡임 등을 대비해 여러 장을 찍게 된다. 결과적으로 거의 같은 사진이 여러 장 쌓이고, 이를 정리하고 싶은 니즈가 있다.

### 1.2 목표

- 불필요한 사진 삭제
- 손쉽고 간편하게
- 좋은 사진은 확실히 남기기

### 1.3 핵심 가치

단체사진에서 **특정 인물의 표정/눈 상태**를 확대 비교하여, 눈 감은 사진이나 표정이 이상한 사진을 쉽게 발견하고 삭제할 수 있다.

## 2. 기능 요구사항

### FR-201: 유사 사진 감지

#### 2.1.1 유사 사진 범위

**그리드 (범위 기반 분석):**
1. 화면에 보이는 사진 범위 파악 (index N ~ M)
2. 확장 범위: **max(0, N-7) ~ min(총개수-1, M+7)** (경계 클램핑)
3. 확장 범위 내에서 유사 그룹 탐색
4. 화면에 보이는 셀 중 그룹에 속하면 테두리 표시

**뷰어:**
- 현재 표시된 사진 기준 **앞 7장 + 뒤 7장** (총 14장 범위) 비교

**공통:**
- 정렬 순서: 현재 그리드/뷰어 표시 순서 기준
- **보관 중인 사진만** 대상 (앱 내 휴지통에 있는 사진 제외)

#### 2.1.2 유사 판정 기준

- **현재 사진 포함하여 3장 이상** 유사한 사진이 있을 때 기능 활성화
- 배지 숫자도 동일 기준 (현재 사진 포함 개수 표시)
- 유사도 임계값: Vision 피처 벡터 거리 **10.0** (MVP 고정, 추후 조정 가능)

| 거리 | 의미 |
|------|------|
| 0 | 완전 동일 |
| 0~5 | 거의 같은 사진 (연속 촬영) |
| 5~10 | 유사한 사진 |
| 10~15 | 어느 정도 비슷 |
| 15+ | 다른 사진 |

#### 2.1.3 얼굴 필터링

- **얼굴이 있는 사진만** 유사사진 분류 대상
- 풍경, 음식 등 얼굴 없는 사진은 유사사진정리 기능 제외
- 분류 순서: 유사도 비교 먼저 → 얼굴 필터 (성능 최적화)

**처리 규칙:**
```
1. 유사도 비교 → 유사 그룹 생성 (예: 5장)
2. 그룹 내 얼굴 필터 → 얼굴 없는 사진 제외
3. 남은 사진이 3장 이상이면 유효
4. 3장 미만이면 그룹 무효 (테두리/버튼 미표시)
```

예시:
- 유사 5장 중 얼굴 있는 사진 4장 → 유효 (4장으로 진행)
- 유사 5장 중 얼굴 있는 사진 2장 → 무효

#### 2.1.4 기술 스택

iOS Vision 프레임워크 통일:

| 기능 | API |
|------|-----|
| 이미지 유사도 | `VNGenerateImageFeaturePrintRequest` |
| 얼굴 감지 | `VNDetectFaceRectanglesRequest` |
| 얼굴 랜드마크 | `VNDetectFaceLandmarksRequest` (필요 시) |

성능 이슈 발생 시 이미지 유사도를 pHash로 최적화 가능.

#### 2.1.5 분석 이미지 해상도

| 화면 | 분석 대상 | 이미지 해상도 |
|------|----------|--------------|
| 그리드 | 유사도 + 얼굴 필터 | 썸네일 (중간 해상도) |
| 뷰어 | 유사도 + 얼굴 감지 (+버튼용) | 원본 이미지 |

**결과 불일치 처리:**
- 그리드에서 테두리 표시 → 뷰어 진입 → 뷰어에서 **재분석**
- 뷰어에서 조건 미충족 시 버튼 미표시 (정상 동작)
- 그리드 테두리는 "힌트" 역할, 최종 판정은 뷰어에서

### FR-202: 그리드 트리거 (스크롤 멈춤)

#### 2.2.1 동작 조건

- 썸네일 그리드에서 스크롤이 멈춤
- 현재 화면에 보이는 사진들 기준으로 유사 사진 분석
- **뷰어와 동일한 조건** 적용 (유사도 + 얼굴 필터)
- 3장 이상 유사 사진 그룹 발견 시 테두리 애니메이션 표시

**테두리 표시 대상:**
- **얼굴 필터 통과 + 유사 그룹에 포함된 셀**만 테두리 표시
- 얼굴 없는 사진 → 테두리 미표시
- 여러 그룹이 있으면 여러 테두리 동시 표시 가능

#### 2.2.2 테두리 애니메이션

**빛이 사각형 테두리를 빙글빙글 도는 효과**

```
┌──✨────────┐
│             │  → 빛(흰색 그라데이션)이
│             │    테두리를 시계방향으로
│             │    계속 회전
└─────────────┘
```

- 구현: `CAShapeLayer` (사각형 path) + `CAKeyframeAnimation` (strokeStart/strokeEnd)
- 빛 색상: 흰색 그라데이션

#### 2.2.3 테두리 탭 동작

- **탭한 사진**의 뷰어 화면으로 이동 (그룹 첫 번째가 아님)
- 뷰어에서 유사사진정리버튼이 자동으로 표시됨

#### 2.2.4 분석 타이밍

- 스크롤 멈춤 후 디바운싱 (0.3초)
- 다시 스크롤 시작하면 분석 취소 및 테두리 해제
- 사전 캐싱 없이 실시간 분석 (MVP)
- 애니메이션이 늦게 나타나도 UX상 문제없음 (사진은 이미 보이는 상태)

#### 2.2.5 테두리 애니메이션 최적화

```
1. didEndDisplaying → 애니메이션 제거
2. prepareForReuse → 테두리 레이어 제거
3. willDisplay → 조건 확인 후 테두리 재적용
4. 스크롤 중 → 새 테두리 추가 안 함 (스크롤 멈춘 후 분석)
```

#### 2.2.6 다중 그룹 처리

- 화면에 여러 유사 그룹이 있으면 **모두 테두리 표시**
- 한 셀이 여러 그룹에 속하면 → 동일한 테두리 표시 (중복 허용)
- 탭 시 뷰어로 이동 → 해당 사진 기준으로 재분석

### FR-203: 뷰어 트리거 (유사사진정리버튼)

#### 2.3.1 버튼 표시 조건

- 현재 보고 있는 사진 기준 앞뒤 **7장씩** 분석 (FR-201과 동일)
- 3장 이상 유사 사진이 있으면 버튼 표시
- 얼굴이 없으면 버튼 표시 안 함

#### 2.3.2 버튼 위치 및 디자인

- 위치: 뷰어 우측 상단
- 아이콘: `square.stack.3d.up` (SF Symbols)
- 유사 사진 개수 배지 표시 (예: 빨간 원에 "5")
- 반투명 원형 배경 (기존 뷰어 버튼과 통일)

#### 2.3.3 버튼 애니메이션

- 조건 충족 시 페이드인으로 버튼 생성
- 다른 사진으로 스와이프하면 조건 재평가

---

### FR-204: 얼굴 + 버튼 표시

#### 2.4.1 동작 흐름

1. 유사사진정리버튼 탭
2. 현재 사진의 얼굴 분석 (Vision)
3. 각 얼굴 위에 + 버튼(반투명 원형) 생성

#### 2.4.2 + 버튼 제한

- **최대 5개**
- 얼굴 크기 필터: **뷰어에 표시되는 이미지** 기준 화면 너비의 **5% 이상**인 얼굴만
- 5% 미만 얼굴 → **미검출로 취급** (비교 그리드에서도 제외)
- 우선순위: 얼굴 크기 큰 순

#### 2.4.3 + 버튼 오버레이 닫기

- **유사사진정리버튼 재탭**: 토글로 닫기
- **스와이프로 다른 사진 이동**: 자동 해제 (다시 돌아와도 + 버튼 안 보임)

#### 2.4.4 인물 번호 부여 순서

**+버튼 선택 및 인물 번호 매핑:**
```
1. 크기 필터: 화면 너비 5% 이상인 얼굴만 추출
2. 크기순 정렬: 상위 5개 선택
3. 위치순 재정렬: 선택된 얼굴을 좌→우, 위→아래로 인물 번호 부여
4. +버튼 표시: 인물 1, 2, 3... 순서로 표시
```

→ **+버튼 표시 순서 = 인물 번호 = 위치 기준**

#### 2.4.5 인물 클러스터링 기준 (MVP)

**위치 기반 매칭:**
- 사진 내 얼굴 순서로 인물 구분 (왼→오, 위→아래)
- "인물 1" = 유사 사진들에서 왼쪽 첫 번째 얼굴끼리 비교
- 연속 촬영 사진은 구도가 거의 동일하므로 대부분 맞음
- 추후 얼굴 크롭 → `VNGenerateImageFeaturePrintRequest` → feature vector 비교로 업그레이드 가능

**얼굴 개수 불일치 처리:**
- 기준 사진: 현재 뷰어에 표시된 사진 (얼굴 N개)
- 유사 사진의 얼굴 개수가 N개 미만이면 → **해당 인물 비교에서 제외**
- 예: 기준 사진 3명, 사진 B 2명 → "인물 3" 선택 시 사진 B 제외

**인물 미검출 시:**
- 유사 사진 중 특정 사진에서 해당 인물 얼굴이 감지 안 되면 → **그 사진 제외**
- 5% 미만 크기 얼굴도 미검출로 취급
- 예: 인물 2 선택 시 5장 중 1장에서 미검출 → 4장만 비교 그리드에 표시

**동일 인물 중복 감지:**
- Vision이 같은 인물을 여러 얼굴로 감지할 수 있음 (정면 + 옆모습 등)
- MVP에서는 **감지된 얼굴 그대로** 처리 (별도 클러스터링 없음)
- "인물 1" = 왼쪽 첫 번째 **감지된** 얼굴

#### 2.4.6 + 버튼 위치

- 기본 위치: 얼굴 위 중앙

**겹침 처리 알고리즘:**
```
1. 기본 위치 설정
2. 다른 버튼과 겹침 감지 (CGRect intersects)
3. 겹침 시 이동: 버튼 지름의 1.2배씩
4. 이동 순서: 좌 → 우 → 아래
5. 화면 밖이면 반대 방향 시도
6. 최대 시도: 4회 (이후 현재 위치 유지)
```

### FR-205: 얼굴 비교 화면 (2열 그리드)

#### 2.5.1 진입

- 특정 얼굴의 + 버튼 탭
- 해당 얼굴만 크롭한 2열 그리드 화면으로 전환

#### 2.5.2 화면 구성

```
┌─────────────────────────────────────────────────┐
│  ← 뒤로        인물 1 (5장)              [↻]   │  ← 헤더
├─────────────────────────────────────────────────┤
│                                                 │
│  ┌──────────┐  ┌──────────┐                    │
│  │  얼굴1   │  │  얼굴2   │                    │
│  │          │  │    ✓     │  ← 선택됨          │
│  └──────────┘  └──────────┘                    │
│                                                 │
│  ┌──────────┐  ┌──────────┐                    │
│  │  얼굴3   │  │  얼굴4   │                    │
│  │    ✓     │  │          │                    │
│  └──────────┘  └──────────┘                    │
│                                                 │
├─────────────────────────────────────────────────┤
│  [Cancel]      N개 선택됨      [Delete]         │  ← 하단바
└─────────────────────────────────────────────────┘
```

#### 2.5.3 헤더 요소

| 요소 | 위치 | 동작 |
|------|------|------|
| ← 뒤로 | 좌측 | 뷰어 화면으로 복귀 |
| 인물 N (M장) | 중앙 | 현재 인물 번호 + 사진 수 |
| [↻] 순환 버튼 | 우측 | 인물 1→2→3→1... 순환 |

- 인물 명칭: "인물 1", "인물 2", "인물 3"... (실제 이름 알 수 없음)

#### 2.5.4 그리드 요소

- 2열, 정사각형 비율
- 5장 이상이면 세로 스크롤
- 선택 표시: 체크마크 오버레이 (기존 선택 모드와 동일)

**얼굴 크롭 규칙:**
- 여백: 얼굴 bounding box 너비/높이 각각 **30% 추가**
- 비율: **정사각형 (1:1)**
- 회전: MVP는 **수평 유지** (얼굴 각도 무시, 추후 개선)

#### 2.5.5 하단바

기존 `FloatingTabBar`의 Select 모드 UI 재사용:

| 요소 | 위치 | 스타일 |
|------|------|--------|
| Cancel | 좌측 | 흰색 텍스트 |
| N개 선택됨 | 중앙 | **전체 선택된 사진** 개수 (인물 무관) |
| Delete | 우측 | 빨간색, 0개면 비활성화 |

### FR-206: 얼굴 비교 화면 인터랙션

#### 2.6.1 선택

- 탭: 선택/해제 토글
- 선택 시 체크마크 오버레이 표시

#### 2.6.2 인물 순환

- 우측 상단 [↻] 버튼 탭
- 인물 1 → 인물 2 → 인물 3 → 인물 1... 순환
- 애니메이션: 크로스페이드
- **선택 상태 유지**: 인물 변경해도 선택한 **사진**은 그대로 유지됨

#### 2.6.3 Cancel 버튼

- 선택 체크만 해제 (전체 선택 취소)
- 얼굴 비교 화면 유지 (다시 선택 가능)

#### 2.6.4 Delete 버튼

- 선택된 사진들 → 앱 내 휴지통 이동
- 얼굴 비교 화면 닫힘 → 기본 뷰어 화면으로 복귀

**현재 사진이 삭제된 경우:**
- **이전 사진**으로 자동 이동 (기존 로직과 동일)
- 이전 사진 없으면 → 다음 사진으로 이동
- 모든 사진 삭제 시 → 뷰어 닫힘 → 그리드로 복귀

**TrashStore 동기화:**
- `TrashStore.moveToTrash(assetIDs:)` 일괄 호출
- 기존 Undo 기능과 동일하게 동작
- PHPhotoLibraryChangeRequest는 TrashStore 내부에서 처리

#### 2.6.5 뒤로 버튼

- 얼굴 비교 화면 닫힘 → 뷰어 화면으로 복귀
- + 버튼 오버레이는 해제된 상태 (유사사진정리버튼 재탭 필요)

## 3. 비기능 요구사항

### NFR-201: 성능

| 항목 | 목표 |
|------|------|
| 유사도 분석 | 스크롤 멈춤 후 1초 이내 테두리 표시 |
| 얼굴 감지 | 버튼 탭 후 0.5초 이내 + 버튼 표시 |
| 그리드 스크롤 | **60fps 유지** (ProMotion 기기는 **120fps**) |
| 테두리 동시 표시 | 10개 이상 테두리 표시해도 스크롤 성능 유지 |

### NFR-202: 메모리

- Vision 분석 결과 캐싱 (선택적, 성능 이슈 시)
- 얼굴 크롭 이미지는 필요 시에만 생성

### NFR-203: 접근성

**VoiceOver 라벨:**

| 요소 | 라벨 |
|------|------|
| 테두리 애니메이션 | "유사 사진 있음" 음성 알림 |
| 유사사진정리버튼 | "유사 사진 N장, 정리하기, 버튼" |
| + 버튼 | "인물 N 얼굴 비교, 버튼" |
| 얼굴 크롭 셀 | "사진 N, 선택됨/선택 안됨" |
| 순환 버튼 | "다음 인물, 버튼" |

**모션 감소 설정 시:**
- 테두리 애니메이션 → 정적 테두리로 대체
- 화면 전환 애니메이션 최소화

**VoiceOver 활성화 시:**
- **기능 전체는 동일하게 작동** (버튼, 비교 화면, 삭제 등 모두 사용 가능)
- 테두리만 정적으로 대체 (애니메이션 불필요)

### NFR-204: iOS 버전별 UI

| iOS 버전 | UI 구현 방식 |
|----------|-------------|
| iOS 16~25 | 커스텀 플로팅 UI (FloatingTitleBar/TabBar) |
| iOS 26+ | 시스템 네비바/툴바 (Liquid Glass 자동 적용) |

**화면 전환 방식:**
- 뷰어: Navigation Push 사용 → 시스템 뒤로가기 버튼 자동 생성
- 얼굴 비교 화면: Navigation Push 사용 → 시스템 뒤로가기 버튼 자동 생성

**유사사진정리버튼 (FR-203):**
- iOS 26+: `navigationItem.rightBarButtonItem`으로 표시
- 배지(숫자): `UIBarButtonItem(customView:)` 사용 또는 타이틀로 대체 ("유사 5장")

**얼굴 비교 화면 (FR-205):**
- 헤더: 시스템 네비바 사용
  - 뒤로: Navigation Push로 시스템 자동 생성
  - 타이틀: `navigationItem.title = "인물 1 (5장)"`
  - 순환: `UIBarButtonItem(image: UIImage(systemName: "arrow.trianglehead.2.clockwise.rotate.90"))`
- 하단바: 시스템 툴바 사용
  - Cancel: `UIBarButtonItem(title: "Cancel")`
  - Delete: `UIBarButtonItem(title: "Delete")`
  - 선택 개수: `navigationItem.title`에 표시 또는 `UIBarButtonItem(customView: UILabel)`

**iOS 26+ 분기 코드 패턴:**
```swift
if #available(iOS 26.0, *) {
    // 시스템 네비바/툴바 사용
} else {
    // 커스텀 플로팅 UI 사용
}
```
