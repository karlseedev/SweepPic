# L1 일상 스크롤 Hitch 개선 계획 (2025-12-23)

## 문제 정의

### 목표
- **1차 목표**: Warning (< 10 ms/s)
- **최종 목표**: Good (< 5 ms/s)
- **기준 시나리오**: 일상 스크롤 (사진 찾는 정도, 15~25초 연속)

### Apple 공식 기준 (WWDC 2020)

| 등급 | 임계값 | 사용자 체감 |
|------|--------|-------------|
| Good | < 5 ms/s | 거의 인지 못함 |
| Warning | 5-10 ms/s | 인지하기 시작함 |
| Critical | > 10 ms/s | 크게 불편함 |

### 현재 상태 (P0 적용 전)

| 지표 | 수치 | 판정 |
|------|------|------|
| L1 hitch | 25.4~43.1 ms/s | Critical (목표의 2.5~4.3배) |
| dropped | 31~79 | 체감 버벅임 |
| preheat | 54~86회, 207~423개 에셋 | 스크롤 중 과도한 호출 |

## 근본 원인 분석

### 코드 분석 결과

**1차 용의자: prefetchItemsAt에서 스크롤 중 preheat** 🔴

```swift
// GridViewController.swift line 1254
imagePipeline.preheat(assetIDs: assetIDs, ...)

// ImagePipeline.swift - 내부에서 동기 호출!
public func preheat(assetIDs: [String], targetSize: CGSize) {
    let assets = assetIDs.compactMap { fetchAsset(for: $0) }  // 동기!
}

private func fetchAsset(for assetID: String) -> PHAsset? {
    // 캐시 미스 시 메인 스레드에서 동기 호출!
    let result = PHAsset.fetchAssets(withLocalIdentifiers: [assetID], options: nil)
}
```

**문제점:**
- prefetchItemsAt → preheat(assetIDs:) → fetchAsset() → PHAsset.fetchAssets() 동기 호출
- **메인 스레드에서** 실행되어 hitch 유발
- 스크롤 중 54~86회 호출 (200ms 스로틀에도 불구)

**기타 용의자:**

| 원인 | 영향도 | 비고 |
|------|--------|------|
| 콜백 폭발 (~156/s) | 중간 | 기여 요인 후보 |
| statsLock 경합 | 낮음 | P0 후 필요시 검토 |
| memMiss 100% | 구조적 | 요청 수 증가의 기반 |

## 개선 계획 (저위험 순)

| 순서 | 작업 | 설명 | 상태 |
|------|------|------|------|
| P0 | prefetchItemsAt preheat 완전 제거 | 동기 fetchAsset 비용 제거 | ✅ 적용 |
| P1 | assetIDs → PHAsset 직접 preheat | 동기 fetch 제거 (필요시) | ⏳ 대기 |
| P2 | degraded 처리 정책 실험 | 콜백 수 감소 (필요시) | ⏳ 대기 |

### 적용 내용

// GridViewController.swift
func collectionView(_ collectionView: UICollectionView, prefetchItemsAt indexPaths: [IndexPath]) {
    // [P0 실험] 스크롤 중 preheat 완전 제거

func collectionView(_ collectionView: UICollectionView, cancelPrefetchingForItemsAt indexPaths: [IndexPath]) {
    // [P0 실험] prefetch preheat 제거에 따라 cancel도 불필요
    return
}

### 테스트 결과

#### 테스트 1 (14:48)

| 구간 | 시간 | hitch | dropped | preheat |
|------|------|-------|---------|---------|
| L1 First | 3.2초 | 4.0 ms/s [Good] | 0 | 0회 |
| L2 Steady #1 | ~4초 | 11.2 ms/s [Critical] | 2 | 1회 |
| L2 Steady #2 | ~10초 | 16.5 ms/s [Critical] | 16 | 1회 |

**주의:** L1이 3.2초로 짧아서 이전 테스트(16~23초)와 직접 비교 어려움.
**비교 대상:** L2 Steady (11.2~16.5 ms/s) vs 이전 L1 (25.4~43.1 ms/s)

#### 테스트 2 (14:56)

|------|------|-------|---------|---------|

### 비교 분석

| 지표 | P0 전 (16~23초) | P0 후 (14~20초) | 변화 |
|------|-----------------|-----------------|------|
| hitch | 25.4~43.1 ms/s | 11.2~16.5 ms/s | 🟡 **50~65% 개선** |
| dropped | 31~79 | 2~21 | 🟡 **개선** |
| preheat | 54~86회 | 0~1회 | 🟢 **제거됨** |
| req/s | 39~82/s | 35~55/s | 감소 |

### 결론

- **preheat 제거 효과 확인**: 54~86회 → 0~1회
- **hitch 50~65% 개선**: 43 → 14~16 ms/s
- **하지만 여전히 Critical**: 14~16 > 10 ms/s
- **목표 미달**: Warning (< 10 ms/s) 아직 못 미침
