# 원인 분석 및 수정 계획 (유사사진 +버튼 미표시)

## 문제 요약

- 그리드에서 유사사진썸네일그룹 테두리가 표시된 사진을 클릭해 뷰어로 진입하면,
  해당 사진에서 +버튼이 표시되지 않는 현상이 발생함.
- 문서 기준상 그리드 테두리 표시 사진은 뷰어에서도 +버튼이 자동 표시되어야 함.

## 문서 기준 (PRD9/PRD9Algorithm)

- **그리드 테두리 표시된 사진 = 뷰어 +버튼 자동 표시** (그리드 분석 결과 재사용)
- **테두리 표시 조건**: 유효한 유사사진썸네일그룹 소속 + 해당 사진에 유효 슬롯 얼굴 존재
- **뷰어 동작**: analyzed 상태면 캐시된 CachedFace로 즉시 +버튼 표시

## 원인 분석 (코드 기준)

1. **그리드 테두리 표시 기준이 과도하게 넓음**
   - 현재 그리드는 "그룹 멤버 전체"를 테두리 대상으로 취급함.
   - 유효 슬롯 얼굴이 없는 사진도 테두리가 표시됨.

2. **분석 파이프라인에서 얼굴 없는 사진을 그룹에서 제외하지 않음**
   - 그룹 유효성은 전체 그룹 단위로만 판단되고,
     개별 사진의 유효 얼굴 여부에 따른 제외 로직이 없음.
   - 결과적으로 그룹 소속이지만 유효 슬롯 얼굴이 없는 사진이 존재 가능.

3. **뷰어 +버튼은 유효 슬롯 얼굴이 있는 경우에만 표시됨**
   - 뷰어는 getValidSlotFaces 결과가 비어 있으면 +버튼을 숨김.
   - 따라서 "그리드 테두리 있음"과 "뷰어 +버튼 표시"가 불일치함.

## 필요한 로직 (문서 기준 반영)

### A. 분석 단계에서 사진 단위 유효성 필터링

- 유사사진썸네일그룹 생성 시, **5% 이상 유효 얼굴이 없는 사진은 그룹 멤버에서 제외**해야 함.
- 제외된 사진은 `.analyzed(inGroup: false)`로 상태 저장.
- 이 로직이 있어야 "테두리 표시 사진 = 유효 슬롯 얼굴 있음" 조건을 만족.

### B. 그리드 테두리 표시 기준 강화

- 테두리 대상은 "그룹 멤버 전체"가 아니라 **유효 슬롯 얼굴을 가진 사진만**이어야 함.
- 캐시의 `getValidSlotFaces` 결과가 비어 있으면 테두리 미표시.

## 수정 계획 (요약)

1. **SimilarityAnalysisQueue**
   - 그룹 분석 후 `photoFacesMap`에서 유효 얼굴 없는 사진을 제외한 리스트로 그룹 멤버를 구성.
   - 제외된 사진은 상태를 `.analyzed(inGroup: false)`로 저장.

2. **GridViewController+SimilarPhoto**
   - 테두리 표시 대상을 `getValidSlotFaces` 기반으로 필터링.
   - 유효 슬롯 얼굴이 없는 사진은 테두리 미표시.

3. **ViewerViewController+SimilarPhoto**
   - 현 로직 유지 (유효 슬롯 얼굴이 있는 경우에만 +버튼 표시).
   - 그리드/뷰어 일치성은 위의 A/B 수정으로 확보.

## 기대 결과

- 그리드에서 테두리가 보이는 사진은 항상 뷰어에서도 +버튼이 표시됨.
- 유효 슬롯 얼굴이 없는 사진은 그리드에서도 테두리가 표시되지 않음.
- 문서(PRD9/PRD9Algorithm) 규칙과 구현 로직이 일치함.
