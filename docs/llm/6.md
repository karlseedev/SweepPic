# 유사 사진 기능 오류 분석 및 수정 계획 (2026-01-05)

## 1. 개요
*   **증상**:
    1.  iOS 26 환경에서 유사 사진 그룹핑이 되지 않음 (모두 개별 사진으로 처리).
    2.  iOS 18 환경에서 휴지통 사진 조회 시 불필요한 분석 로딩 표시 발생.
    3.  그리드에서 테두리가 생긴(유사 그룹에 속한) 사진을 뷰어에서 열었을 때 +버튼이 표시되지 않음.
    4.  iOS 16~25 환경에서 FaceComparisonViewController의 상단 UI가 중복 표시되고 하단 버튼이 눌리지 않음.

이 문서는 위 증상들의 원인을 분석하고 구체적인 수정 계획을 정리한 것이다.

---

## 2. 원인 분석 및 해결 방안

### 이슈 1: iOS 26에서 그룹핑 실패
*   **원인**: `SimilarityAnalyzer.swift`에서 Vision Framework의 `VNGenerateImageFeaturePrintRequest` 생성 시 `revision`을 명시하지 않아, iOS 버전에 따라 기본 리비전이 달라짐. iOS 26의 기본 리비전이 기존 임계값(`0.5`)과 호환되지 않는 거리 스케일을 가질 가능성이 높음.
*   **해결**: OS 버전에 따라 리비전을 명시적으로 고정.
    *   iOS 17+: `VNGenerateImageFeaturePrintRequestRevision2` (임계값 0.5 대응)
    *   iOS 16-: `VNGenerateImageFeaturePrintRequestRevision1` (임계값 10.0 대응)

### 이슈 2: 휴지통 사진 분석 시도
*   **원인**: `ViewerViewController+SimilarPhoto.swift`의 `shouldEnableSimilarPhoto` 프로퍼티에서 휴지통 모드(`viewerMode == .trash`) 여부를 확인하는 로직이 누락됨 (주석 처리됨).
*   **해결**:
    *   `ViewerViewController.swift`의 `viewerMode` 접근 제어를 `private`에서 `internal`로 변경.
    *   `shouldEnableSimilarPhoto`에서 `viewerMode == .trash`일 경우 `false` 반환.

### 이슈 3: 뷰어에서 +버튼 미표시
*   **원인**: `SimilarityAnalysisQueue.swift`의 `assignPersonIndicesForGroup` 메서드 구현 오류.
    *   문서(`prd9algorithm.md`) 의도와 달리, **"첫 번째 사진의 얼굴 위치"를 절대 기준**으로 삼고, 위치가 `0.15` 이상 차이나는 얼굴 데이터를 **삭제(`continue`)**하는 과도하게 엄격한 로직이 적용됨.
    *   이로 인해 사람이 조금만 움직여도 얼굴 정보가 `CachedFace`에 저장되지 않아, 뷰어에서 "얼굴 없는 사진"으로 인식됨.
*   **해결**:
    *   매칭 실패 시 얼굴 데이터를 버리지 않고, **새로운 슬롯(새 인물 번호)을 생성**하여 보존하는 유연한 로직으로 수정.

### 이슈 4: FaceComparison UI 문제 (상단 중복, 하단 터치 불가)
*   **원인**:
    *   **상단 중복**: 커스텀 타이틀바(`FaceComparisonTitleBar`)를 추가하면서 컬렉션 뷰 헤더(`FaceComparisonHeaderView`)도 동시에 표시함.
    *   **하단 터치 불가**: `viewDidLoad` 시점의 `safeAreaInsets`(0)을 사용하여 고정 높이를 설정, 버튼이 홈 바 영역과 겹침.
*   **해결**:
    *   상단: 컬렉션 뷰 헤더 높이를 `.zero`로 설정하여 제거.
    *   하단/레이아웃: 모든 제약 조건을 `view.safeAreaLayoutGuide` 앵커 기준으로 변경하여 반응형으로 수정.

## 3. 상세 수정 계획 (코드 레벨)

### 3.1. `SimilarityAnalyzer.swift` (이슈 1)
```swift
func generateFeaturePrint(for image: CGImage) throws -> VNFeaturePrintObservation {
    let request = VNGenerateImageFeaturePrintRequest()
    // 리비전 명시적 설정
    if #available(iOS 17.0, *) {
        request.revision = VNGenerateImageFeaturePrintRequestRevision2
    } else {
        request.revision = VNGenerateImageFeaturePrintRequestRevision1
    }
    // ...
}
```

### 3.2. `ViewerViewController.swift` & Extension (이슈 2)
1.  `ViewerViewController.swift`: `viewerMode`를 `internal`로 변경.
2.  `ViewerViewController+SimilarPhoto.swift`:
    ```swift
    private var shouldEnableSimilarPhoto: Bool {
        // ... 기존 체크 ...
        // 휴지통 모드 체크 추가
        guard viewerMode != .trash else { return false }
        return true
    }
    ```

### 3.3. `SimilarityAnalysisQueue.swift` (이슈 3)
`assignPersonIndicesForGroup` 메서드 로직 전면 수정:
*   `referenceSlots` 고정 방식 폐기.
*   `slots` 배열을 동적으로 관리.
*   각 얼굴에 대해:
    1.  기존 `slots` 중 가장 가까운 것 탐색.
    2.  임계값(`0.15`) 이내면 해당 슬롯 번호 사용.
    3.  아니면 **새 슬롯 생성 및 추가**, 새 번호 부여.
*   모든 얼굴에 대해 `CachedFace` 생성 (누락 방지).

### 3.4. `FaceComparisonViewController.swift` (이슈 4)
*   `referenceSizeForHeaderInSection` -> `return .zero`
*   `setupBottomBar`, `setupCustomTitleBar` 등의 제약 조건을 `safeAreaLayoutGuide` 앵커 기준으로 재작성.
    *   예: `bottomBarContainer.topAnchor.constraint(equalTo: view.safeAreaLayoutGuide.bottomAnchor, constant: -56)`

---

## 4. 검증 계획
1.  **iOS 26 시뮬레이터**: 그리드 스크롤 시 그룹핑(테두리) 정상 작동 확인.
2.  **휴지통**: 휴지통 사진 진입 시 로딩 인디케이터 뜨지 않는지 확인.
3.  **뷰어 +버튼**: 그리드 테두리 있는 사진 진입 시 +버튼 정상 표시 확인. (사람이 움직인 연속 사진 테스트)
4.  **UI**: iOS 16~25 시뮬레이터에서 상단 중복 없음 및 하단 버튼 터치 정상 작동 확인.
