## 1. 기술적 의사결정 및 정당성

### 왜 `UICollectionViewTransitionLayout`이 아닌가?
*   **표준 방식의 한계**: `TransitionLayout`은 `Layout A`에서 `Layout B`로의 이산적인 전환을 전제로 합니다. 5열에서 3열로 가다가 멈추지 않고 바로 1열로 가는 **"Continuous Multi-stage Pinch"**를 구현하려면, 중간에 Transition을 취소하고 새로운 Transition을 시작해야 하는데, 이 과정에서 필연적으로 프레임 드랍이나 위치 튀김(Glitch)이 발생합니다.
*   **Custom Layout의 우위**: 단일 Layout 내에서 `currentColumns` 속성(Float)만 변경하고 `invalidateLayout()`을 호출하는 방식은 **무한히 연속적인 줌**과 **즉각적인 방향 전환**을 완벽하게 지원합니다. Apple Photos의 경험은 이 방식으로만 구현 가능합니다.

## 2. 상세 아키텍처

### 2.1 핵심 컴포넌트 구조
1.  **`DynamicZoomLayout` (Subclass of UICollectionViewFlowLayout)**
    *   **State**: `var activeColCount: CGFloat` (예: 3.0 -> 2.45 -> 1.1)
    *   **Behavior**: `activeColCount` 변경 시 셀의 Frame과 Alpha를 즉시 재계산.
    *   **Optimization**: `shouldInvalidateLayout`을 최적화하여 불필요한 연산 방지.
2.  **`ZoomTransitionAttributes` (Subclass of UICollectionViewLayoutAttributes)**
    *   **Data**: `var transitionProgress: CGFloat`, `var isAnchor: Bool`, `var targetScale: CGFloat`
    *   **Purpose**: 셀에게 현재 줌 상태를 전달하여 내부 이미지의 Cross-fade나 Transform을 제어.
3.  **`PinchController` (Logic Controller)**
    *   `UIPinchGestureRecognizer` 핸들링.
    *   **Anchoring Math**: 핀치 중심점을 유지하기 위한 `ContentOffset` 정밀 보정 계산 담당.

## 3. 핵심 구현 로직 (Deep Dive)

### 3.1 앵커링 & ContentOffset 보정 공식
핀치 줌 시 화면이 울렁거리지 않고 손가락 아래 사진이 딱 붙어있으려면 `ContentOffset`을 매 프레임 보정해야 합니다.

```swift
// Gesture Changed 단계
let scale = gesture.scale
let newColCount = startColCount / scale

// 1. 레이아웃 갱신
layout.activeColCount = newColCount
layout.invalidateLayout()
collectionView.layoutIfNeeded() // 즉시 반영하여 프레임 갱신

// 2. 앵커 위치 보정 (핵심)
// anchorCellFrame: 레이아웃 갱신 후 변경된 앵커 셀의 Frame
// pinchCenterInCell: 핀치 시작 시 셀 내부에서의 터치 상대 좌표 (0.0 ~ 1.0)
let newAnchorPointX = anchorCellFrame.origin.x + (anchorCellFrame.width * pinchCenterInCell.x)
let newAnchorPointY = anchorCellFrame.origin.y + (anchorCellFrame.height * pinchCenterInCell.y)

// 3. Offset 적용
// gestureLocation: 현재 화면상(Bounds 기준)의 터치 좌표
let targetOffsetX = newAnchorPointX - gestureLocation.x
let targetOffsetY = newAnchorPointY - gestureLocation.y

collectionView.contentOffset = CGPoint(x: targetOffsetX, y: targetOffsetY)
```

### 3.2 Fade-In 및 이미지 전환 전략 (LayoutAttributes 활용)

요구사항 "앵커 제외 Fade-In"은 셀 내부에서 처리합니다.

1.  **Layout**:
    *   `layoutAttributesForItem`에서 `attributes.isAnchor = (indexPath == anchorIndexPath)` 설정.
    *   `attributes.zoomLevel = activeColCount` 전달.
2.  **Cell (`apply(_:)`)**:
    ```swift
    override func apply(_ layoutAttributes: UICollectionViewLayoutAttributes) {
        guard let attr = layoutAttributes as? ZoomTransitionAttributes else { return }
        
        if attr.isAnchor {
            // 앵커 셀: 기존 고해상도 이미지 유지, 변형 없음
            imageView.alpha = 1.0
            overlayView.alpha = 0.0
        } else {
            // 주변 셀: 줌 레벨에 따라 목표 이미지(또는 오버레이) Fade-In
            // 예: 3열 -> 1열로 가는 중이라면(3.0 -> 1.0), 1열용 이미지를 서서히 보여줌
            // 여기서는 단순 Alpha 조정 예시
            let progress = ... // calculate based on attr.zoomLevel
            imageView.alpha = progress 
            // 또는 targetImageView.alpha = progress (Cross-fade)
        }
    }
    ```

### 3.3 종료 시 스냅 로직 (10% / 50% Rule)

```swift
// Gesture Ended
let velocity = gesture.velocity
let progress = calculateProgress() // 0.0 ~ 1.0

var targetCols: Int
if isUnidirectional && progress > 0.1 {
    targetCols = endCols // 10% 이상 진행 시 무조건 이동
} else if progress > 0.5 {
    targetCols = endCols // 방향 전환 시 50% 이상이면 이동
} else {
    targetCols = startCols // 복귀
}

// 부드러운 마무리 애니메이션 (Spring Animation 권장)
UIView.animate(withDuration: 0.5, delay: 0, usingSpringWithDamping: 0.8, ...) {
    layout.activeColCount = CGFloat(targetCols)
    // ContentOffset 보정도 애니메이션 블록 안에서 수행해야 자연스러움
}
```
