# 자동 삭제 기능 보완의견 (통합/검증본)

## 확정 이슈 (명세 불일치/누락)
- Precision 모드에서 제외된 항목이 Strong으로 포함되어 모드별 동작이 모순됩니다. 모드별 활성 조건을 명시하거나 Conditional 분류가 필요합니다. `docs/260114AutoDel.md:41` `docs/260114AutoDel.md:89` `docs/260114AutoDel.md:126`
- Safe Guard 적용 시점/근거가 Stage 1과 Stage 4 설명에서 불일치합니다. 조기 필터 vs 후처리 필터를 명확히 구분해야 합니다. `docs/260114AutoDel.md:119` `docs/260114AutoDel.md:154`
- iOS 18 AestheticsScore가 기존 4단계 파이프라인을 대체/보완 중 무엇인지 정의돼 있지 않습니다. `docs/260114AutoDel.md:287`
- Recall 모드에서 Weak 조합 규칙이 모호합니다(유효 조합/가중치). `docs/260114AutoDel.md:98` `docs/260114AutoDel.md:47`
- 50장 미만/0장/중단 시 UX 및 진행 표기 규칙이 없습니다. `docs/260114AutoDel.md:184` `docs/260114AutoDel.md:209` `docs/260114AutoDel.md:234`
- `비네팅<0.05` 등 지표의 계산 정의가 없어 구현 편차가 큽니다. `docs/260114AutoDel.md:44`
- 앱 내 휴지통 정책(보관 기간, 재시작 후 복구, 용량/삭제 방식)이 명시돼 있지 않습니다. `docs/260114AutoDel.md:268`

## 보완 아이디어 (검증됨)
- iCloud 제외 정책을 UX에 명시하고, 로컬만 스캔했음을 안내하는 문구 추가. 후보군에 한해 선택적 다운로드 옵션도 고려. `docs/260114AutoDel.md:198`
- 비디오 판정은 최악 프레임 기준이 과탐 위험이 있으므로, 2/3 실패 기준 또는 대표 프레임 가중 평균 같은 완화 옵션을 문서에 명시. `docs/260114AutoDel.md:219`
- 비디오 프레임 추출 비용을 고려한 제한(타임아웃/길이 제한/동시 처리 수)을 성능 제약에 추가. `docs/260114AutoDel.md:189`
- Burst/유사 사진이 50장을 독점하지 않도록 그룹화 후 대표 1장 우선 후보화 정책 추가. `docs/260114AutoDel.md:184`
- 스캔 방향(과거 우선)의 근거를 한 줄로 명시해 정책 설명 보강. `docs/260114AutoDel.md:185`
- 검증 계획 섹션 신설: 라벨링 데이터셋, 목표 Precision/Recall, 사용자 피드백(Undo/Keep 비율) 지표. `docs/260114AutoDel.md:359`
- iOS 18 API 시뮬레이터 미지원 대비 테스트용 샘플/Mock 데이터 준비. `docs/260114AutoDel.md:310`
