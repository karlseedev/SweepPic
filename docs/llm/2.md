**Findings**
- High: `TrashAlbumViewController`는 `pendingScrollAssetID`를 저장하지만 이를 적용하는 `applyPendingViewerReturn()` 호출이 없습니다. 뷰어 복귀 후 스크롤 복원이 작동하지 않는 상태로 보입니다. `PickPhoto/PickPhoto/Features/Albums/TrashAlbumViewController.swift:591`
- High: Grid의 padding 보정 누락이 아직 남아 있습니다. 뷰어 삭제/복구/복귀에서 `dataSourceDriver.indexPath(for:)`를 그대로 사용해 상단 패딩이 있는 경우 잘못된 셀이 갱신/스크롤됩니다. `PickPhoto/PickPhoto/Features/Grid/GridViewController.swift:956`, `PickPhoto/PickPhoto/Features/Grid/GridViewController.swift:981`, `PickPhoto/PickPhoto/Features/Grid/GridViewController.swift:1028`
- Medium: `TrashDataSource.assetIndex(for:)`가 선형 탐색입니다. 줌 전환 시 sourceViewProvider에서 호출되고 있어 휴지통 사진이 많으면 성능 저하가 발생할 수 있습니다. 인덱스 캐시 도입을 고려하세요. `PickPhoto/PickPhoto/Features/Grid/GridDataSource.swift:145`
- Medium: `TrashAlbumViewController.configureCell`에서 다시 `cell.configure(...)`를 호출해 Base에서 이미 수행한 이미지 요청을 중복 실행합니다. 휴지통에서 dim 상태만 바꾸는 방식으로 경량화하는 편이 좋습니다. `PickPhoto/PickPhoto/Features/Albums/TrashAlbumViewController.swift:535`

**Questions/Assumptions**
