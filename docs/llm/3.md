**Findings**
- High: `TrashDataSource.assets`가 공개 `var`라 외부에서 직접 변경 시 index/orderedAssetIDs가 stale되거나 중간 상태가 될 수 있습니다. Phase 5에서 캐시/갱신 경로를 강제하려던 의도와 어긋나므로 `private(set)` + `updateAssets()` 경로로 묶는 편이 안전합니다. `PickPhoto/PickPhoto/Features/Grid/GridDataSource.swift:120`
- Medium: `TrashDataSource.assetIndex(for:)`가 선형 탐색입니다. 휴지통 뷰어의 zoom transition에서 매번 호출되므로 사진이 많으면 렉이 날 수 있습니다. 인덱스 캐시 도입이 필요합니다. `PickPhoto/PickPhoto/Features/Grid/GridDataSource.swift:145`, `PickPhoto/PickPhoto/Features/Albums/TrashAlbumViewController.swift:505`
- Medium: Grid/Album/Trash 간에 뷰어용 `PHFetchResult` 정렬 기준이 코드로 강제돼 있습니다(creationDate ascending). 실제 그리드 정렬과 불일치하면 복귀/zoom 인덱스가 틀어질 수 있으니, “그리드 정렬 = fetchResult 정렬” 계약을 문서에 명시하거나 공통 헬퍼로 통일하는 게 안전합니다. `PickPhoto/PickPhoto/Features/Albums/TrashAlbumViewController.swift:452`
- Low: Base의 `contentInsetAdjustmentBehavior`가 `.never`인데, iOS 26에서 시스템 바를 쓰는 경우에만 각 VC가 수동으로 `.automatic` 복구를 하고 있습니다. Album/Grid/Trash 간 정책이 분산되어 있으니 Base에서 일괄 처리하거나 문서화하는 편이 유지보수에 유리합니다. `PickPhoto/PickPhoto/Features/Grid/BaseGridViewController.swift:29`

**Questions/Assumptions**
