# 핀치줌/회전 구현 방안 조사

**조사일**: 2026-01-21
**목적**: 요구사항을 충족하는 핀치줌/회전 구현 방안 탐색

## 1. 조사 배경

### 요구사항 요약
- 셀 개별 이동 애니메이션 금지
- 새로고침/끊김 금지
- 핀치 진행도에 따라 실제 레이아웃이 점진적으로 변화
- fadeIn으로 부드러운 이미지 전환

### 폐기된 방안: transform 방식
- 축소 시 외곽 검정색 (렌더링된 셀 없음)
- 손 떼면 끊김 (transform → layout 전환)
- fadeIn 효과 무효화 (레이아웃 재구성)

## 2. 방안 A: startInteractiveTransition (UICollectionViewTransitionLayout)

### 개요
Apple 공식 API로, 두 레이아웃 간 인터랙티브 전환 지원.

```swift
// 전환 시작
let transitionLayout = collectionView.startInteractiveTransition(
    to: newLayout,
    completion: { completed, finished in ... }
)

// 진행도 업데이트 (핀치 changed에서)
transitionLayout.transitionProgress = progress  // 0.0 ~ 1.0

// 완료/취소
if progress > 0.5 {
    collectionView.finishInteractiveTransition()
} else {
    collectionView.cancelInteractiveTransition()
}
```

### 장점
- Apple 공식 API
- `transitionProgress` (0~1)로 제어 가능
- 레이아웃 간 선형 보간 지원

### 단점
- **기본 UICollectionViewTransitionLayout은 contentOffset 처리 미흡**
- **셀 개별 이동 애니메이션 발생** → 요구사항 위반 가능
- 새 레이아웃 객체로 "전환"하는 개념이라 끊김 발생 가능

### 보완책: TLLayoutTransitioning 라이브러리
- [TLLayoutTransitioning](https://github.com/SwiftKickMobile/TLLayoutTransitioning)
- `TLTransitionLayout`: contentOffset 제어 개선
- Minimal/Center/Visible 등 배치 옵션 제공
- 핀치 예제 포함

### 요구사항 충족도
| 요구사항 | 충족 |
|---------|:----:|
| 셀 개별 이동 애니메이션 금지 | ⚠️ 부분적 |
| 새로고침/끊김 금지 | ✅ |
| 점진적 레이아웃 변화 | ✅ |
| fadeIn 이미지 전환 | ➕ 별도 구현 |

## 3. 방안 B: 커스텀 ScalingLayout (ZoomCollectionView 스타일)

### 개요
레이아웃에 `scale` factor를 전달하고, 레이아웃이 scale에 따라 셀 크기/위치 실시간 계산.

```swift
protocol ScalingLayoutProtocol {
    func getScale() -> CGFloat
    func setScale(_ scale: CGFloat)
    func contentSizeForScale(_ scale: CGFloat) -> CGSize
}

// 핀치 changed에서
layout.setScale(gesture.scale)
collectionView.collectionViewLayout.invalidateLayout()
```

### 구조 (ZoomCollectionView 참고)
```
┌─────────────────────────────────┐
│ UIScrollView (줌/스크롤 감지)     │
│  ┌───────────────────────────┐  │
│  │ UICollectionView          │  │
│  │  └─ ScalingLayout         │  │
│  │      (scale에 따라 셀 계산) │  │
│  └───────────────────────────┘  │
│  └─ DummyView (줌 대상)         │
└─────────────────────────────────┘
```

### 장점
- **핀치 진행도에 따라 실제 레이아웃이 점진적으로 변화**
- **셀 이동 애니메이션 없음** (레이아웃이 직접 위치 계산)
- 연속적인 스케일링 가능

### 단점
- **복잡한 구현** (UIScrollView + UICollectionView 조합)
- **바운스 효과 문제** (`scrollViewDidZoom` 미호출)
- **외곽 셀 렌더링 문제** (prefetch 범위 밖)
- 기존 Compositional Layout과 호환성 문제

### 요구사항 충족도
| 요구사항 | 충족 |
|---------|:----:|
| 셀 개별 이동 애니메이션 금지 | ✅ |
| 새로고침/끊김 금지 | ✅ |
| 점진적 레이아웃 변화 | ✅ |
| fadeIn 이미지 전환 | ➕ 별도 구현 |
| 외곽 새 셀 표시 (축소 시) | ⚠️ 추가 작업 필요 |

## 4. 방안 C: 스냅샷 오버레이 방식

### 개요
현재 화면을 스냅샷으로 캡처 → 오버레이로 transform → 뒤에서 새 레이아웃 준비 → crossfade

```swift
// 핀치 시작
let snapshot = collectionView.snapshotView(afterScreenUpdates: false)
view.addSubview(snapshot)

// 핀치 중
snapshot.transform = CGAffineTransform(scaleX: scale, y: scale)

// 핀치 종료
UIView.animate(withDuration: 0.3) {
    snapshot.alpha = 0
} completion: { _ in
    snapshot.removeFromSuperview()
    // 새 레이아웃 표시
}
```

### 장점
- 셀 이동 애니메이션 없음
- 구현 비교적 단순

### 단점
- **화면에 보이는 셀만 스냅샷 가능**
- **축소 시 외곽에 새 셀이 나타나지 않음** (transform과 동일 문제)
- fadeIn이 "점진적"이 아닌 "최종 크로스페이드"
- 핀치 중 레이아웃 변화 없음 (transform만)

### 요구사항 충족도
| 요구사항 | 충족 |
|---------|:----:|
| 셀 개별 이동 애니메이션 금지 | ✅ |
| 새로고침/끊김 금지 | ⚠️ 크로스페이드 시 끊김 |
| 점진적 레이아웃 변화 | ❌ |
| fadeIn 이미지 전환 | ⚠️ 크로스페이드만 |
| 외곽 새 셀 표시 | ❌ |

## 5. 방안 D: Compositional Layout + 동적 열 수 + invalidateLayout

### 개요
기존 Compositional Layout 유지, `currentGridColumnCount`를 동적으로 변경하고 `invalidateLayout()` 호출.

```swift
// createLayout에서 동적 참조
func createLayout() -> UICollectionViewLayout {
    UICollectionViewCompositionalLayout { [weak self] _, env in
        let columns = self?.currentGridColumnCount.rawValue ?? 3
        // columns 기반 레이아웃 생성
    }
}

// 핀치/회전 시
currentGridColumnCount = newColumnCount
collectionView.collectionViewLayout.invalidateLayout()
```

### 장점
- **기존 코드 최소 변경**
- **셀 이동 애니메이션 없음** (동일 레이아웃 객체 유지)
- 회전도 동일 방식으로 처리

### 단점
- **"점진적" 변화가 아닌 "discrete" 변화** (1열 → 3열 → 5열 점프)
- 중간 상태 (2열, 4열) 표현 불가

### 요구사항 충족도
| 요구사항 | 충족 |
|---------|:----:|
| 셀 개별 이동 애니메이션 금지 | ✅ |
| 새로고침/끊김 금지 | ✅ |
| 점진적 레이아웃 변화 | ❌ discrete |
| fadeIn 이미지 전환 | ➕ 별도 구현 |

## 7. 권장 방안

### 1순위: 방안 A (InteractiveTransition) + fadeIn 커스텀

**이유:**
- Apple 공식 API 기반으로 안정성 확보
- `transitionProgress`로 점진적 변화 구현 가능
- TLLayoutTransitioning 참고하여 셀 이동 문제 완화
- fadeIn은 별도 레이어로 구현

**구현 방향:**
```swift
// 1. 커스텀 TransitionLayout 생성
class PinchTransitionLayout: UICollectionViewTransitionLayout {
    // transitionProgress에 따라 셀 크기 보간
    // contentOffset 적절히 계산
}

// 2. 핀치 시작 시 fadeIn 트리거
// 3. transitionProgress 업데이트
// 4. 완료/취소 처리
```

### 2순위: 방안 D (invalidateLayout) + fadeIn 강화

**이유:**
- 기존 코드 최소 변경
- discrete 변화를 fadeIn으로 시각적 보완
- 빠른 구현 가능
