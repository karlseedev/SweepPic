## 1. 논리 및 판정 기준 정합성 보완 (Critical)

`1.md`와 `2.md`에서 공통적으로 지적된 **Precision 모드와 Strong 신호 간의 모순**을 해결하기 위해 판정 등급을 재정의해야 합니다.

*   **현황**: '주머니 샷', '렌즈 가림'이 Strong으로 분류되지만 Precision 모드에서는 제외됨 (논리적 모순).
*   **보완 제안**: **[Conditional (조건부)]** 등급 신설.

| 등급 | 정의 | 해당 항목 | 적용 모드 |
|:---:|---|---|:---:|
| **Strong** | 단일 항목으로 즉시 저품질 확정 (기술적 결함 명백) | 극단 노출(밝음/어두움), 심각 블러 | All Modes |
| **Conditional** | 기술적 실패지만, 의도적일 수 있어 사용자 확인 필요 | 주머니 샷, 렌즈 가림, 극단 단색 | Recall Mode |
| **Weak** | 단독 판정 불가, 2개 이상 조합 시 유효 | 일반 블러, 일반 노출, 저해상도 | Recall Mode |

## 2. 알고리즘 및 기술 로직 고도화

### 2.1 비디오 판별 로직 개선 (from `3.md`, `1.md`)
*   **문제**: 3프레임 중 '최악값' 사용 시 과탐지 위험 높음. 프레임 추출 비용 문제.
*   **개선안**:
    1.  **가중 평균 도입**: `(Thumbnail × 0.6) + (Start × 0.2) + (End × 0.2)` 공식 적용.
    2.  **성능 제약**: 비디오 분석 타임아웃(5초) 및 10분 초과 대용량 영상 SKIP 조건 추가.

### 2.2 주머니 샷/렌즈 가림 정교화 (from `3.md`)
*   **문제**: 비네팅 수치만으로는 예술 사진과 구분이 모호함.
*   **개선안**: **히스토그램 분석** 추가. 0~20 구간(어두운 영역)에 픽셀의 95% 이상이 몰려있는지 확인.

### 2.3 iOS 18 AestheticsScore 전략 (from `1.md`)
*   **문제**: 기존 파이프라인과 대체/보완 관계 불명확.
*   **개선안**: **Hybrid 전략** 명시.
    *   `Score < -0.5`: 즉시 저품질 (Strong)
    *   `Score > 0.5`: 즉시 정상 (Pass)
    *   `-0.5 <= Score <= 0.5`: 애매한 구간이므로 기존 파이프라인(Blur/Exposure) 수행.

### 2.4 Safe Guard 적용 시점 명확화 (from `2.md`)
*   **문제**: 문서 내에서 Stage 1과 Stage 4의 역할이 혼재됨.
*   **개선안**:
    *   **Pre-Check (Stage 1)**: 분석 비용 절감 목적 (즐겨찾기, 편집됨, 스크린샷).
    *   **Post-Check (Stage 4)**: 분석 후 구제 목적 (얼굴 인식 품질, 심도 효과).

## 3. UX 및 정책 시나리오 보완

### 3.1 iCloud 및 네트워크 정책 (from `2.md`, `3.md`)
*   **문제**: `NetworkAccessAllowed = false`로 인해 로컬 사진이 없으면 결과가 0장이 됨.
*   **개선안**:
    *   분석 시작 전 로컬 자산 비율 체크.
    *   로컬 자산 부족 시: "iCloud 사진은 기기 용량 확보를 위해 분석에서 제외됩니다." 안내 문구 노출.

### 3.2 엣지 케이스 및 결과 처리 (from `1.md`, `3.md`)
*   **50장 미만 발견 시**: 스캔 종료 후 "N장의 저품질 사진을 찾았습니다" 결과 페이지로 이동 (오류 아님).
*   **Burst (연속 촬영)**: 동일 시간대(±1초) 그룹 내에서는 **가장 품질이 낮은 1장**만 후보로 선정 (유사 사진 도배 방지).
*   **0장 발견 시**: "완벽하게 관리되고 있습니다!" 축하 메시지 제공.

### 3.3 휴지통 정책 (from `2.md`)
*   **문제**: 앱 내 휴지통 이동이 용량 확보로 직결되지 않음.
*   **개선안**:
    *   삭제 완료 화면에 "시스템 앨범에서 삭제해야 실제 용량이 확보됩니다" 안내.
    *   앱 내 휴지통 보관 기간(예: 30일) 명시.
