# 자동 정리 기능: 저품질 사진 정의 및 구현 로직

본 문서는 '정리' 버튼 클릭 시 수행되는 저품질 사진 판별 기준과 이를 실현하기 위한 기술적/기능적 로직을 정의한다.

---

## 1. [확정] 저품질 사진의 정량적 정의 (Criteria)

사용자의 소중한 추억을 보호하기 위해 **'확실한 기술적 실패'**만을 저품질로 규정한다.

### A. 판정 원칙
1. **강한 실패 신호 (Strong Signal)**: 단일 항목만으로도 즉시 저품질 확정.
2. **약한 실패 신호 (Weak Signal)**: 2개 이상의 항목이 조합될 때 저품질 확정.
3. **스크린샷 처리**: 기술적 실패가 아니므로 '저품질' 로직에서 제외(Pass)하고 별도 카테고리로 관리.

### B. 정량적 기준
| 구분 | 항목 | 상세 기준 (Threshold) |
| :--- | :--- | :--- |
| **강한 실패** | **극단 노출** | 평균 휘도 < 0.10 또는 > 0.90 |
| | **심각 블러** | Laplacian Variance < 50 |
| | **주머니 샷** | (휘도 < 0.10) AND (RGB Std < 15) AND (Laplacian < 50) AND (Vignetting < 0.05) |
| | **극단 단색** | (RGB Std < 10) AND (휘도 < 0.15 또는 > 0.85) |
| | **렌즈 가림** | 모서리 휘도 < 중앙 휘도 * 0.4 또는 모서리 휘도 < 0.15 |
| **약한 실패** | **일반 지표** | Laplacian Var < 100, 휘도 < 0.15 또는 > 0.85, RGB Std < 15, 저해상도(< 1MP) |

### C. 안전장치 (Safe Guard) - 오분류 방지
저품질 판정 시에도 아래 조건 충족 시 즉시 **정상 사진으로 복구**.
- **전체 무효**: 즐겨찾기(`isFavorite`), 편집됨(`hasAdjustments`)
- **블러 무효**: 심도 효과(`depthEffect`), 인물 선명도(`FaceQuality >= 0.4`)

---

## 2. 판별 로직 및 기능 기획 (Logic & Functionality)

확정된 정의를 바탕으로 **"오래된 사진 50장을 찾아 휴지통에 넣는"** 동작을 구현한다.

### A. 탐색 및 분석 로직 (Step a, b)
1. **시작점 및 방향**: 
   - 현재 화면 최상단 사진(`anchorIndex`)부터 **과거 방향(역순)**으로 탐색.
2. **고성능 배치 스캔**:
   - 100~500장 단위로 **Batch Fetch**를 수행하여 메모리 점유율 최소화.
   - **Swift Concurrency(`TaskGroup`)**를 사용하여 배치 내 사진들을 병렬 분석.
3. **리소스 최적화**:
   - 분석용 이미지는 **긴 변 256px 이하**로 다운샘플링하여 로드.
   - 분석 루프 내 `autoreleasepool` 적용으로 임시 메모리 즉시 해제.
4. **조기 종료**: 50장의 후보가 모이는 즉시 스캔 중단.

### B. 기능 실행 및 UX (Step c)
1. **스캔 피드백**: '정리' 클릭 시 화면에 "저품질 사진 찾는 중... (n/50)" 진행 상태 표시.
2. **검토 단계 (Review)**: 
   - 50장을 찾으면 즉시 삭제하지 않고, **후보 사진들을 그리드 형태로 제시**.
   - 사용자는 삭제를 원치 않는 사진을 **선택 해제(Keep)** 가능.
3. **일괄 삭제**: 
   - 최종 확인된 사진들을 `TrashStore`를 통해 한 번의 트랜잭션으로 앱 내 휴지통 이동.
4. **사후 처리**: 삭제 완료 토스트 메시지 및 '실행 취소(Undo)' 옵션 제공.
