# AutoDel 문서 검토 (docs/AutoDel)

## docs/AutoDel/260120AutoDel.md

### 1) 임계값 검증
- 다수 임계값이 "설계값"으로 표기되어 있어 실제 검증 결과/근거가 문서 본문에 남아 있지 않음 (v10에서 "검증 결과 반영"이라고만 언급).
- `Weak` 가중치 합산 기준(>=3)과 일반 블러 가중치(2점)의 검증 근거가 없음.
- 주머니 샷 조건(휘도<0.10, RGB Std<15, Lap<50, 비네팅<0.05)의 조합이 강한 기준이라 탐지율/오탐 여부 확인 필요.

### 2) 논리 일관성
- Precision 모드는 Strong만 사용한다고 정의되지만, Stage 4의 Weak 합산 로직이 모드별로 명시적으로 차단되는지 서술이 부족함.
- 주머니 샷 조건의 `휘도<0.10`은 Stage 2의 "극단 노출(어두움)" Strong 기준과 완전히 겹쳐 조건상 의미가 축소됨(대부분 Strong에서 조기 종료).
- iOS 18+에서 AestheticsScore 성공 시 Stage 2~4를 완전히 건너뛰는지(혹은 병행 평가인지) 문구가 다소 모호함.

### 3) 기술적 정확성
- 얼굴 품질(0.4)과 블러 Safe Guard의 상호작용(얼굴이 선명하면 블러 무효)은 개념적으로 타당하나, 실제 Vision API 값의 분포가 기기/조명에 따라 달라질 수 있어 튜닝 필요.
- Laplacian 임계값은 이미지 리사이즈(256x256) 방식에 민감하므로, 구현에서 다운샘플 방식이 달라지면 오탐/미탐이 변할 수 있음.

### 4) 누락 항목
- "심도 효과"와 "공유 앨범" 판별을 어떤 Photos API로 확인할지 구체 언급이 없음.
- "유틸리티 이미지(isUtility)" 처리 정책이 iOS 18 설명에 포함되어 있지 않음(impl 문서에는 있음).
- 연도별 정리에서 연도 기준(현지 타임존 vs 메타데이터 기준) 정의가 없음.

## docs/AutoDel/260120AutoDel-impl.md

### 1) 임계값 검증
- RGB Std, Face Quality, AestheticsScore, 비네팅 등 설계값이 많고 "테스트 필요"로만 표기되어 있어 실제 검증 결과/조정 기준이 없음.
- Laplacian 기준이 PyImageSearch 기준을 인용하지만, 리사이즈/색공간 차이를 반영한 재튜닝 필요성이 큼.

### 2) 논리 일관성
- iOS 18에서 `isUtility == true`를 스크린샷과 동일 취급하는 정책이 핵심 기획 문서에 명시되지 않아 문서 간 불일치가 있음.
- Stage 1 안전 필터는 AestheticsScore에도 선행 적용된다고 강조되는데, 핵심 기획서와 표현 방식이 달라 실제 구현 의도 확인 필요.

### 3) 기술적 정확성
- `CalculateImageAestheticsScoresRequest` 등 Vision API 명칭/호출 방식이 실제 SDK와 일치하는지 확인 필요(일반적으로 `VN...Request` 계열).
- `CIImage` 변환, 이미지 방향(orientation), 색공간 처리 등이 예제 코드에 생략되어 있어 실제 구현 시 품질 산출에 영향을 줄 수 있음.

### 4) 누락 항목
- iCloud 썸네일 요청 시 `PHImageManager`의 `targetSize`/`contentMode`/`resizeMode` 구체 전략이 없음.
- GPU 작업 실패 시(예: MPS unsupported) 대체 경로/폴백 처리 설명이 없음.

## docs/AutoDel/260120AutoDel-test.md

### 1) 임계값 검증
- 주머니 샷의 비네팅(0.05)은 포함되어 있으나 렌즈 가림(모서리 휘도 < 중앙 x 0.4) 검증 항목이 빠져 있음.
- 저해상도(<1MP)와 Weak 가중치 합산 기준(>=3)의 검증 항목이 없음.
- 모드별(Precision/Recall) 임계값 적용 확인에 대한 테스트 케이스가 분리되어 있지 않음.

### 2) 논리 일관성
- AestheticsScore 실패 시 Stage 2~4로 폴백되는 경로에 대한 테스트가 명시되지 않음.
- Precision 모드에서 Conditional/Weak가 배제되는지 확인하는 테스트 항목이 없음.

### 3) 기술적 정확성
- iCloud 썸네일 "342x256"은 환경에 따라 달라질 수 있어, 테스트 기대값을 "범위/허용오차"로 정의하는 편이 안전함.

### 4) 누락 항목
- 메타데이터 필터(즐겨찾기/편집됨/숨김/공유 앨범) 동작 검증 항목이 없음.
- 스크린샷 분리, Live Photo/Burst/RAW+JPEG 처리, 비디오 10분 초과 SKIP 등 특수 미디어 처리에 대한 테스트 항목이 없음.
- 백그라운드 전환/복귀 및 취소 처리에 대한 UX/상태 유지 테스트 항목이 없음.
