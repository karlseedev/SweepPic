## 1. iOS 네이티브 API (Vision Framework)

### 1.1 CalculateImageAestheticsScoresRequest (iOS 18+)

iOS 18에서 새롭게 도입된 이미지 미학 점수 평가 API.

| 속성 | 범위 | 설명 |
|-----|------|------|
| **overallScore** | -1 ~ 1 | 높을수록 미학적으로 우수한 이미지 |
| **isUtility** | Bool | 스크린샷, 영수증, 문서 등 비기억용 이미지 여부 |

**코드 예시:**
```swift
import Vision

func calculateAestheticsScore(image: UIImage) async throws -> ImageAestheticsScoresObservation? {
    guard let ciimage = CIImage(image: image) else { return nil }
    let request = CalculateImageAestheticsScoresRequest()
    return try await request.perform(on: ciimage)
}
```

**제한사항:**
- 시뮬레이터 미지원 (실제 기기에서만 동작)
- 에러: `Failed to create espresso context` (시뮬레이터에서 발생)

### 1.3 VNDetectFaceCaptureQualityRequest

얼굴 캡처 품질을 평가하는 API.

| 속성 | 범위 | 설명 |
|-----|------|------|
| **faceCaptureQuality** | 0 ~ 1 | 1에 가까울수록 고품질 얼굴 캡처 |

**평가 요소:**
- 노출 (Exposure)
- 포즈 (Pose)
- 흐림 (Blurriness)
- 표정 (Facial Expression)
- 중앙 배치 (Centrality)

**코드 예시:**
```swift
import Vision

func detectFaceQuality(cgImage: CGImage) throws -> Float? {
    let request = VNDetectFaceCaptureQualityRequest()
    let handler = VNImageRequestHandler(cgImage: cgImage)
    try handler.perform([request])

    return request.results?.first?.faceCaptureQuality
}
// quality >= 0.4 → 선명한 얼굴로 판정
```

**성능 팁:**
- `.accurate` 레벨은 정확도 우선
- `.fast` 레벨은 FPS 우선
- 프레임 스로틀링 권장 (3~5프레임당 1회 처리)

**출처:**
- [Apple Documentation - VNDetectFaceCaptureQualityRequest](https://developer.apple.com/documentation/vision/vndetectfacecapturequalityrequest)

## 2. Blur Detection (Laplacian Variance)

### 2.1 알고리즘 원리

Laplacian 연산자의 분산(Variance)을 이용한 흐림 감지 방법.

| 단계 | 설명 |
|-----|------|
| 1 | 이미지를 Grayscale로 변환 |
| 2 | 3×3 Laplacian 커널로 2차 미분 계산 |
| 3 | 결과 픽셀값의 **분산(Variance)** 계산 |
| 4 | 분산이 낮으면 흐림, 높으면 선명 |

**원리:**
- Laplacian 연산자는 "급속한 명도 변화 영역"을 강조
- 선명한 이미지: 엣지가 많음 → 높은 분산값
- 흐린 이미지: 엣지가 적음 → 낮은 분산값

---

### 2.2 Threshold 값

| 상태 | Variance 값 |
|-----|------------|
| **흐린 이미지** | 64 ~ 83 범위 |
| **선명한 이미지** | 1000+ 범위 |
| **경험적 임계값** | **100** |

> "threshold is a critical parameter to tune correctly and you'll often need to tune it on a per-dataset basis"
> — PyImageSearch

**100은 경험적 구분점**이며, 데이터셋에 따라 조정 필요.

### 2.3 Metal Performance Shaders 구현 (GPU 가속)

Metal을 사용한 고성능 blur detection 구현.

**사용하는 MPS 클래스:**
- `MPSImageLaplacian`: Laplacian 필터 적용
- `MPSImageStatisticsMeanAndVariance`: 평균/분산 계산

**코드 예시:**
```swift
import Metal
import MetalPerformanceShaders

class BlurDetector {
    let device: MTLDevice
    let commandQueue: MTLCommandQueue
    let laplacian: MPSImageLaplacian
    let meanAndVariance: MPSImageStatisticsMeanAndVariance

    init() {
        device = MTLCreateSystemDefaultDevice()!
        commandQueue = device.makeCommandQueue()!
        laplacian = MPSImageLaplacian(device: device)
        meanAndVariance = MPSImageStatisticsMeanAndVariance(device: device)
    }

    func detectBlur(texture: MTLTexture) -> Float {
        let commandBuffer = commandQueue.makeCommandBuffer()!

        // 1. Laplacian 적용
        let lapDescriptor = MTLTextureDescriptor.texture2DDescriptor(
            pixelFormat: texture.pixelFormat,
            width: texture.width,
            height: texture.height,
            mipmapped: false
        )
        lapDescriptor.usage = [.shaderWrite, .shaderRead]
        let lapTexture = device.makeTexture(descriptor: lapDescriptor)!

        laplacian.encode(
            commandBuffer: commandBuffer,
            sourceTexture: texture,
            destinationTexture: lapTexture
        )

        // 2. Mean & Variance 계산
        let varDescriptor = MTLTextureDescriptor.texture2DDescriptor(
            pixelFormat: .rg32Float,
            width: 2,
            height: 1,
            mipmapped: false
        )
        varDescriptor.usage = [.shaderWrite, .shaderRead]
        let varTexture = device.makeTexture(descriptor: varDescriptor)!

        meanAndVariance.encode(
            commandBuffer: commandBuffer,
            sourceTexture: lapTexture,
            destinationTexture: varTexture
        )

        commandBuffer.commit()
        commandBuffer.waitUntilCompleted()

        // 3. Variance 값 읽기
        var result = [Float](repeating: 0, count: 2)
        varTexture.getBytes(
            &result,
            bytesPerRow: 8,
            from: MTLRegion(
                origin: MTLOrigin(x: 0, y: 0, z: 0),
                size: MTLSize(width: 2, height: 1, depth: 1)
            )
        )

        return result[1]  // index 0: mean, index 1: variance
    }
}
```

**성능:**
- iPhone 8 기준: 1000×600 이미지에서 **3~9ms**
- GPU 가속으로 CPU 대비 월등히 빠름

- [LinkedIn - Blur Detection via GPU-Accelerated Metal Performance Shaders](https://www.linkedin.com/pulse/blur-detection-via-gpu-accelerated-metal-performance-shaders-qadri)

## 3. 휘도(Luminance) 계산

### 3.1 픽셀 기반 직접 계산

ITU-R BT.601 표준 공식 사용:
```
Y = 0.299 × R + 0.587 × G + 0.114 × B
```

**코드 예시:**
```swift
func calculateMeanLuminance(cgImage: CGImage) -> Double {
    guard let data = cgImage.dataProvider?.data,
          let bytes = CFDataGetBytePtr(data) else { return 0 }

    let width = cgImage.width
    let height = cgImage.height
    let bytesPerPixel = cgImage.bitsPerPixel / 8
    var totalLuminance: Double = 0

    for i in stride(from: 0, to: width * height * bytesPerPixel, by: bytesPerPixel) {
        let r = Double(bytes[i])
        let g = Double(bytes[i + 1])
        let b = Double(bytes[i + 2])
        totalLuminance += 0.299 * r + 0.587 * g + 0.114 * b
    }

    return totalLuminance / Double(width * height)
}
```

---

### 3.2 판정 기준

| 상태 | 평균 휘도 (0~255) |
|-----|------------------|
| **너무 어두움** | < 30 (또는 < 34) |
| **정상** | 30 ~ 230 |
| **너무 밝음** | > 230 (또는 > 227) |

### 4.3 TaskGroup 병렬 처리

Swift Concurrency를 활용한 병렬 이미지 분석.

```swift
func scanPhotos(assets: [PHAsset]) async -> [PHAsset] {
    await withTaskGroup(of: (PHAsset, Bool).self) { group in
        for asset in assets {
            group.addTask {
                let isLowQuality = await self.analyzeQuality(asset)
                return (asset, isLowQuality)
            }
        }

        var results: [PHAsset] = []
        for await (asset, isLowQuality) in group {
            if isLowQuality {
                results.append(asset)
            }
        }
        return results
    }
}
```

**성능 비교:**

| 방식 | 전체 시간 | 사진당 시간 |
|-----|----------|-----------|
| 동기 처리 | 40.4초 | 6.19ms |
| 비동기 처리 | 16.28초 | 2.49ms |

**개선율:** 약 **60% 성능 향상**

### 5.2 Polyblur 알고리즘

Google Research에서 개발한 고속 blur 추정/제거 방법.

> "the maximal value of the image gradient across all directions at any point in a sharp image follows a particular distribution"

- 메가픽셀 이미지도 **밀리초 단위**로 처리 가능
