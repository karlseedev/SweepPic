**작성일**: 2026-01-21
**목표**: iOS Photos 앱과 동일한 핀치줌/회전 전환 구현
**상태**: 설계 중

---

## 1. 목표 동작

### iOS Photos 앱 동작 분석

```
핀치 시작
    ↓
1. 전체 화면이 transform으로 확대/축소 (스크린샷처럼)
2. 기준점(1/3/5열) 통과 시 → 해당 열 기준 이미지로 fade in
   - 앵커(핀치 중심점): 이미지 유지
   - 나머지 셀: 새 이미지로 fade in
    ↓
핀치 종료
    ↓
transform 제거 + 새 레이아웃 즉시 적용
```

### 핵심 특징

| 항목 | 현재 구현 | 목표 구현 |
|-----|----------|----------|
| 레이아웃 전환 | 셀 개별 이동 애니메이션 | 전체가 스크린샷처럼 스케일 |
| 이미지 전환 | 없음 (레이아웃만 변경) | fade in으로 부드럽게 전환 |
| 앵커 처리 | 위치만 유지 | 이미지도 유지 |
| 다단계 전환 | 1단계씩만 | 연속 전환 가능 (5→3→1) |

### 핀치줌 vs 회전 공통 원리

| | 핀치줌 | 회전 |
|---|--------|------|
| 앵커 | 핀치 중심점 셀 | 화면 중앙 셀 |
| 앵커 이미지 | 유지 | 유지 |
| 나머지 셀 | fade in | fade in |
| 트리거 | 기준점(1/3/5열) 통과 시 | 회전 시작 시 |
| 레이아웃 | transform 스케일링 | 시스템 회전 애니메이션 |

## 2. 핵심 개념

### 2.1 스케일 → 열 수 매핑

핀치 스케일을 가상 열 수로 변환:

```swift
/// 핀치 스케일 → 가상 열 수 계산
/// - 스케일 1.0 = 현재 열 수
/// - 스케일 > 1.0 = 열 수 감소 (확대)
/// - 스케일 < 1.0 = 열 수 증가 (축소)
func virtualColumnCount(scale: CGFloat, baseColumns: Int) -> CGFloat {
    // 예: 3열에서 scale 2.0 → 1.5열
    // 예: 3열에서 scale 0.5 → 6열
    return CGFloat(baseColumns) / scale
}
```

### 2.2 기준점 통과 감지

```swift
/// 열 수 기준점: 1, 3, 5
/// 이전 가상 열 수와 현재 가상 열 수를 비교하여 기준점 통과 감지
func detectThresholdCrossing(
    previousColumns: CGFloat,
    currentColumns: CGFloat
) -> GridColumnCount? {
    let thresholds: [CGFloat] = [1, 3, 5]

    for threshold in thresholds {
        let crossedForward = previousColumns > threshold && currentColumns <= threshold
        let crossedBackward = previousColumns < threshold && currentColumns >= threshold

        if crossedForward || crossedBackward {
            return GridColumnCount(rawValue: Int(threshold))
        }
    }
    return nil
}
```

### 2.3 Fade In 애니메이션

```swift
/// 셀 이미지 fade in 전환
/// - 이전 이미지 위에 새 이미지를 올리고 fade in
/// - 앵커 셀은 제외
func fadeInNewImage(cell: PhotoCell, newImage: UIImage, duration: TimeInterval = 0.25) {
    let overlayImageView = UIImageView(image: newImage)
    overlayImageView.frame = cell.imageView.bounds
    overlayImageView.contentMode = .scaleAspectFill
    overlayImageView.clipsToBounds = true
    overlayImageView.alpha = 0
    cell.contentView.addSubview(overlayImageView)

    UIView.animate(withDuration: duration) {
        overlayImageView.alpha = 1.0
    } completion: { _ in
        cell.imageView.image = newImage
        overlayImageView.removeFromSuperview()
    }
}
```

## 3. 핀치줌 구현 설계

### 3.1 상태 관리

```swift
struct PinchZoomState {
    var isActive: Bool = false
    var initialScale: CGFloat = 1.0
    var baseColumnCount: GridColumnCount = .three  // 핀치 시작 시 열 수
    var previousVirtualColumns: CGFloat = 3.0      // 이전 프레임의 가상 열 수
    var currentTargetColumns: GridColumnCount?     // 현재 목표 열 수 (fade in 대상)
    var anchorAssetID: String?                     // 앵커 에셋 ID (이미지 유지)
    var anchorPoint: CGPoint = .zero               // 핀치 중심점 (화면 좌표)
}

private var pinchState = PinchZoomState()
```

### 3.2 Phase: began

```swift
case .began:
    // 1. 상태 초기화
    pinchState = PinchZoomState()
    pinchState.isActive = true
    pinchState.initialScale = gesture.scale  // 보통 1.0
    pinchState.baseColumnCount = currentGridColumnCount
    pinchState.previousVirtualColumns = CGFloat(currentGridColumnCount.rawValue)

    // 2. 앵커 결정 (핀치 중심점의 셀)
    let location = gesture.location(in: collectionView)
    pinchState.anchorPoint = CGPoint(
        x: location.x - collectionView.contentOffset.x,
        y: location.y - collectionView.contentOffset.y
    )

    if let indexPath = collectionView.indexPathForItem(at: location),
       indexPath.item >= paddingCellCount {
        pinchState.anchorAssetID = assetIDForCollectionIndexPath(indexPath)
    } else {
        // 빈 공간 → 화면 중앙 셀을 앵커로
        let centerPoint = CGPoint(
            x: collectionView.contentOffset.x + collectionView.bounds.width / 2,
            y: collectionView.contentOffset.y + collectionView.bounds.height / 2
        )
        if let centerIndexPath = collectionView.indexPathForItem(at: centerPoint) {
            pinchState.anchorAssetID = assetIDForCollectionIndexPath(centerIndexPath)
        }
        pinchState.anchorPoint = CGPoint(
            x: collectionView.bounds.width / 2,
            y: collectionView.bounds.height / 2
        )
    }

    // 3. 스크롤/제스처 비활성화
    collectionView.isScrollEnabled = false
```

### 3.3 Phase: changed

```swift
case .changed:
    guard pinchState.isActive else { return }

    let scale = gesture.scale

    // 1. transform 적용 (전체 화면 스케일링)
    // 앵커 포인트 기준으로 스케일
    let anchorInView = CGPoint(
        x: pinchState.anchorPoint.x / collectionView.bounds.width,
        y: pinchState.anchorPoint.y / collectionView.bounds.height
    )
    collectionView.layer.anchorPoint = anchorInView
    collectionView.transform = CGAffineTransform(scaleX: scale, y: scale)

    // 2. 가상 열 수 계산
    let baseColumns = CGFloat(pinchState.baseColumnCount.rawValue)
    let currentVirtualColumns = baseColumns / scale

    // 3. 기준점 통과 감지 → fade in 트리거
    if let crossedThreshold = detectThresholdCrossing(
        previousColumns: pinchState.previousVirtualColumns,
        currentColumns: currentVirtualColumns
    ) {
        // 새 목표 열 수로 fade in 시작
        pinchState.currentTargetColumns = crossedThreshold
        fadeInVisibleCells(
            targetColumns: crossedThreshold,
            anchorAssetID: pinchState.anchorAssetID
        )
    }

    // 4. 이전 열 수 업데이트
    pinchState.previousVirtualColumns = currentVirtualColumns
```

### 3.4 Phase: ended

```swift
case .ended, .cancelled:
    guard pinchState.isActive else { return }

    // 1. 최종 열 수 결정
    let scale = gesture.scale
    let baseColumns = CGFloat(pinchState.baseColumnCount.rawValue)
    let finalVirtualColumns = baseColumns / scale

    // 가장 가까운 기준점으로 스냅
    let finalColumns = snapToNearestColumnCount(finalVirtualColumns)

    // 2. transform 제거 + 레이아웃 적용
    UIView.animate(withDuration: 0.2) { [weak self] in
        self?.collectionView.transform = .identity
    } completion: { [weak self] _ in
        guard let self = self else { return }

        // 레이아웃 변경
        self.currentGridColumnCount = finalColumns
        self.collectionView.collectionViewLayout.invalidateLayout()
        self.updateCellSize()

        // 앵커 위치로 스크롤
        if let anchorID = self.pinchState.anchorAssetID,
           let indexPath = self.collectionIndexPath(for: anchorID) {
            self.collectionView.scrollToItem(at: indexPath, at: .centeredVertically, animated: false)
        }

        // 고해상도 썸네일 재요청
        self.didPerformZoom(to: finalColumns)
    }

    // 3. 상태 초기화
    pinchState = PinchZoomState()
    collectionView.isScrollEnabled = true
    updateSwipeDeleteGestureEnabled()

default:
    break
```

### 3.5 Visible Cells Fade In

```swift
/// visible cells에 새 이미지 fade in (앵커 제외)
private func fadeInVisibleCells(
    targetColumns: GridColumnCount,
    anchorAssetID: String?
) {
    // 새 열 수 기준 썸네일 크기 계산
    let targetSize = thumbnailSize(for: targetColumns)

    for indexPath in collectionView.indexPathsForVisibleItems {
        guard indexPath.item >= paddingCellCount else { continue }

        // 앵커 셀은 스킵
        let assetID = assetIDForCollectionIndexPath(indexPath)
        if assetID == anchorAssetID { continue }

        // 새 열 수 기준으로 이 위치에 올 에셋 계산
        // (paddingCellCount가 달라지므로 다른 에셋이 올 수 있음)
        let newPaddingCount = paddingCellCount(for: targetColumns)
        let newAssetIndex = indexPath.item - newPaddingCount

        guard newAssetIndex >= 0,
              let asset = gridDataSource.asset(at: newAssetIndex),
              let cell = collectionView.cellForItem(at: indexPath) as? PhotoCell else {
            continue
        }

        // 새 이미지 로드 및 fade in
        imagePipeline.loadThumbnail(for: asset, targetSize: targetSize) { [weak cell] image in
            guard let cell = cell, let image = image else { return }
            self.fadeInNewImage(cell: cell, newImage: image)
        }
    }
}
```

## 4. 회전 구현 설계

### 4.1 viewWillTransition

```swift
override func viewWillTransition(
    to size: CGSize,
    with coordinator: UIViewControllerTransitionCoordinator
) {
    super.viewWillTransition(to: size, with: coordinator)

    // 1. 앵커 저장 (화면 중앙 셀)
    saveScrollAnchorAssetID()
    let anchorAssetID = scrollAnchorAssetID

    // 2. 새 열 수 결정
    let isLandscape = size.width > size.height
    let newColumnCount = isLandscape
        ? currentGridColumnCount.landscapeColumnCount
        : currentGridColumnCount.portraitColumnCount

    // 3. 회전과 동시에 fade in 시작
    fadeInVisibleCells(targetColumns: newColumnCount, anchorAssetID: anchorAssetID)

    coordinator.animate(alongsideTransition: { [weak self] _ in
        guard let self = self else { return }

        // 열 수 업데이트
        self.currentGridColumnCount = newColumnCount

        // 레이아웃 즉시 변경 (셀 이동 애니메이션 없음)
        self.collectionView.collectionViewLayout.invalidateLayout()
        self.updateCellSize()
        self.updateContentInset()

    }, completion: { [weak self] _ in
        // 앵커 위치로 스크롤
        self?.restoreScrollAnchorAssetID()

        // 고해상도 썸네일 재요청
        self?.didPerformZoom(to: newColumnCount)
    })
}
```

## 5. 헬퍼 함수

### 5.1 가장 가까운 열 수로 스냅

```swift
/// 가상 열 수를 가장 가까운 기준점(1/3/5)으로 스냅
func snapToNearestColumnCount(_ virtualColumns: CGFloat) -> GridColumnCount {
    let thresholds: [(range: ClosedRange<CGFloat>, columns: GridColumnCount)] = [
        (0...2, .one),      // 0~2 → 1열
        (2...4, .three),    // 2~4 → 3열
        (4...CGFloat.infinity, .five)  // 4~ → 5열
    ]

    for (range, columns) in thresholds {
        if range.contains(virtualColumns) {
            return columns
        }
    }
    return .three
}
```

### 5.2 열 수별 paddingCellCount

```swift
/// 열 수에 따른 paddingCellCount 계산
func paddingCellCount(for columns: GridColumnCount) -> Int {
    // 현재 구현에 맞게 수정 필요
    switch columns {
    case .one: return 0
    case .three: return 2
    case .five: return 4
    }
}
```

### 5.3 열 수별 썸네일 크기

```swift
/// 열 수에 따른 썸네일 크기 계산
func thumbnailSize(for columns: GridColumnCount) -> CGSize {
    let spacing = Self.cellSpacing
    let columnCount = CGFloat(columns.rawValue)
    let totalSpacing = spacing * (columnCount - 1)
    let availableWidth = view.bounds.width - totalSpacing
    let cellWidth = floor(availableWidth / columnCount)

    let scale = UIScreen.main.scale
    let pixelSize = cellWidth * scale

    return CGSize(width: pixelSize, height: pixelSize)
}
```

## 7. 테스트 계획

### 7.1 핀치줌 테스트

| 테스트 | 기대 결과 |
|--------|----------|
| 천천히 핀치 | 기준점 통과 시 fade in, 핀치 끝나기 전에 이미지 전환 완료 |
| 빠르게 2단계 핀치 | 5→3→1열 연속 전환, fade in 2번 발생 |
| 핀치 중 방향 전환 | 새 목표 열 수로 fade in 재시작 |
| 핀치 후 원래대로 | 스냅백, 이미지도 원래대로 |
| 앵커 셀 확인 | 핀치 중심점 셀은 이미지 변경 없음 |

### 7.2 회전 테스트

| 테스트 | 기대 결과 |
|--------|----------|
| 세로 → 가로 | 셀 이동 없이 전체가 자연스럽게 전환, 앵커 외 fade in |
| 가로 → 세로 | 동일 |
| 회전 중 스크롤 위치 | 앵커 셀이 화면에 유지됨 |

### 7.3 성능 테스트

| 지표 | 기준 |
|------|------|
| 60fps 유지 | hitch 0 ms/s |
| fade in 끊김 없음 | 부드러운 전환 |
| 메모리 증가 | 10MB 이내 |

## 8. 주의사항

### 8.1 transform과 layout 분리

- 핀치 **중**: transform만 적용 (실제 레이아웃 변경 없음)
- 핀치 **후**: transform 제거 + 레이아웃 변경

### 8.2 fade in 중복 방지

같은 셀에 fade in이 여러 번 트리거될 수 있음 (빠른 핀치 시):
- 이전 fade in 애니메이션 취소
- 또는 현재 진행 중이면 스킵

### 8.3 paddingCellCount 변경

열 수 변경 시 paddingCellCount도 변경됨:
- 같은 indexPath에 다른 에셋이 매핑됨
- fade in 시 새 열 수 기준 paddingCellCount로 계산 필요
