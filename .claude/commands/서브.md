## 사용자 입력

```text
$ARGUMENTS
```

## 목표

사용자가 입력한 조사 주제들을 파싱하여, 각 주제별로 독립적인 서브에이전트를 **백그라운드**로 동시에 실행한다. 모든 에이전트가 완료되면 결과를 종합하여 보고한다.

## 입력 형식

사용자는 `$ARGUMENTS`에 조사할 주제들을 자유롭게 입력한다. 각 주제는 줄바꿈, 쉼표, 또는 번호로 구분할 수 있다.

예시:
```
/서브 Swift Concurrency의 TaskGroup 성능 특성, CoreML 배치 추론 최적화 방법, PhotoKit의 PHCachingImageManager 캐시 전략
```

```
/서브
1. Swift Concurrency의 TaskGroup 성능 특성
2. CoreML 배치 추론 최적화 방법
3. PhotoKit의 PHCachingImageManager 캐시 전략
```

## 실행 절차

### 1. 주제 파싱

`$ARGUMENTS`에서 조사 주제들을 추출한다. 구분자는 다음 중 하나:
- 줄바꿈
- 쉼표(,)
- 번호 매기기 (1. 2. 3. 또는 1) 2) 3))

주제가 1개뿐이면 에이전트 1개만 실행한다.

### 2. 사용자에게 실행 계획 보고

파싱된 주제 목록과 에이전트 수를 사용자에게 간략히 보여준다. 형식:

```
🔍 N개 에이전트 백그라운드 실행:
  1. [주제1 요약]
  2. [주제2 요약]
  ...
```

### 3. 서브에이전트 동시 실행

각 주제별로 Agent 도구를 사용하여 **모든 에이전트를 하나의 메시지에서 동시에** 실행한다.

각 에이전트 설정:
- `subagent_type`: "general-purpose"
- `run_in_background`: true
- `description`: 주제를 3-5단어로 요약
- `prompt`: 아래 템플릿 사용

```
다음 주제에 대해 철저하게 조사해줘. 웹 검색과 코드베이스 탐색을 모두 활용해.

## 조사 주제
{주제 내용}

## 조사 방법
1. WebSearch로 최신 자료, 공식 문서, 기술 블로그를 검색
2. 필요시 WebFetch로 핵심 문서의 상세 내용을 가져옴
3. 현재 프로젝트 코드베이스에서 관련 코드가 있으면 함께 분석

## 결과 형식
다음 구조로 정리해서 반환:

### 핵심 요약 (3줄 이내)
### 상세 내용
### 실용적 권장사항 (현재 프로젝트에 적용 가능한 것 위주)
### 참고 자료 (URL 포함)
```

### 4. 결과 종합 보고

모든 에이전트가 완료되면, 각 에이전트의 결과를 다음 형식으로 종합하여 사용자에게 보고한다:

```markdown
# 조사 결과 종합

## 1. [주제1]
[에이전트1 결과 요약]

## 2. [주제2]
[에이전트2 결과 요약]

...

## 종합 인사이트
[주제들 간의 연관성이나 종합적인 시사점이 있으면 정리]
```

## 주의사항

- 모든 에이전트는 반드시 `run_in_background: true`로 실행하여 사용자가 대기하지 않도록 한다
- 에이전트가 완료될 때까지 기다리지 말고, 실행 후 바로 "백그라운드에서 조사 중입니다" 메시지를 보낸다
- 에이전트 완료 알림이 오면 그때 결과를 종합 보고한다
- 한글로 결과를 보고한다
